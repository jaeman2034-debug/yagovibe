# ⏱️ 개발 일정별 체크포인트

> 3일차/5일차에 확인할 것

---

## Day 1 체크포인트 (오늘 저녁)

### Frontend

- [ ] 음성 버튼 클릭 가능
- [ ] 음성 인식 시작/종료 상태 표시
- [ ] 인식 결과 → BE API 호출
- [ ] 지도에 핀 하나라도 표시

### Backend

- [ ] `/api/search/voice` 엔드포인트 작동
- [ ] 키워드 → 카테고리 매핑 작동
- [ ] Mock 데이터 응답 반환
- [ ] FE와 통신 성공

### 통합

- [ ] 버튼 → 음성 → API → 핀 표시 **흐름 완성**

**성공 기준:**
> 버튼 누르고 "근처 축구장" 말하면
> 지도에 핀 하나라도 찍힌다

---

## Day 2 체크포인트

### Frontend

- [ ] 핀 여러 개 표시 (3-5개)
- [ ] 가장 가까운 핀 강조 (크기/색상)
- [ ] 지도 자동 줌 (모든 핀 보이도록)
- [ ] 하단 카드 1개 표시

### Backend

- [ ] 장소 데이터 조회 (실제/Mock)
- [ ] 거리순 정렬
- [ ] 응답 포맷 정리 (primary/others)

**성공 기준:**
> 검색 결과가 지도에 여러 핀으로 표시되고
> 가장 가까운 것이 강조된다

---

## Day 3 체크포인트

### Frontend

- [ ] 결과 카드 정보 완성 (장소명, 거리, 태그)
- [ ] 카드 클릭 가능
- [ ] "여기로 가기" 버튼 작동

### Backend

- [ ] 조건 필터 (지금 여는, 무료 등)
- [ ] 정렬 로직 완성

**성공 기준:**
> 하단에 결과 카드 1개가 표시되고
> "여기로 가기" 버튼이 작동한다

---

## Day 4 체크포인트

### Frontend

- [ ] 길찾기 연결 (외부 지도 앱)
- [ ] 음성 실패 UX 처리
- [ ] 통합 테스트

### Backend

- [ ] 에러 핸들링
- [ ] 로그 수집 설정

**성공 기준:**
> "여기로 가기" 클릭 시 외부 지도 앱이 열린다

---

## Day 5 체크포인트

### 전체

- [ ] 사용자 테스트 (3-5명)
- [ ] 버그 수정
- [ ] 배포 준비

**성공 기준:**
> 사용자 테스트에서 "편하다" 피드백이 나온다
