# ✅ QR 로그인 알림 - 추천 액션 자동화 완료

## 🎯 완성된 기능

### ✅ 추천 액션 자동 생성
- 룰 기반 액션 제안 (ML 불필요)
- 각 액션에 근거(지표 값) 포함
- 우선순위별 정렬 (high → medium → low)

### ✅ 액션 룰 (6가지)

| 감지 조건 | 추천 액션 | 우선순위 |
|---------|---------|---------|
| SMS 실패율 > 5% | 🔧 SMS 인증 UX 점검<br>📞 통신사 상태 확인 | 🔴 High |
| 평균 로그인 시간 > 60초 | ⏱️ 로딩/네트워크 구간 점검 | 🟡 Medium |
| 만료율 > 10% | ⏰ QR 만료 시간 임시 +1분 검토 | 🟡 Medium |
| 모바일 실패 비중 높음 | 📱 모바일 입력 UX/키보드/포커스 확인<br>📱 모바일 환경 테스트 권장 | 🔴 High / 🟡 Medium |
| 중복 토큰 방지 증가 | 🔒 QR 노출/공유 방식 점검 | 🔴 High |
| 성공률 전반 저하 | 🔍 시스템 리소스 및 로그 확인<br>☁️ Firebase 서비스 상태 확인 | 🔴 High / 🟡 Medium |

## 📊 최종 Slack 메시지 예시

### 🚨 Critical 알림

```
🚨 @here *QR 로그인 장애 감지* (CRITICAL)

기간: 최근 10분
전체 세션: 142건

지표:
- 성공률: 82.4% ❌
- SMS 실패율: 9.1% ❌
- 평균 로그인 시간: 78초 ❌
- 세션 만료율: 13.3% ❌

플랫폼별:
- 모바일: 75.2% (98건)
- 데스크톱: 95.5% (44건)

문제:
- 성공률 82.4% (< 90%)
- SMS 실패율 9.1% (> 5%)
- 평균 로그인 시간 78초 (> 60초)
- 세션 만료율 13.3% (> 10%)

🧠 추정 원인:
SMS 인증 구간 문제 가능성 높음 / 모바일 UX/네트워크 이슈 가능성
- 모바일 성공률: 75.2% vs 데스크톱: 95.5%

🛠 추천 액션:
1. 🔧 SMS 인증 UX 점검 🔴
   └ SMS 실패율 9.1% (기준: 5%)
2. 📞 통신사 상태 확인 🔴
   └ SMS 실패율 9.1%
3. 📱 모바일 입력 UX/키보드/포커스 확인 🔴
   └ 모바일 성공률 75.2% (데스크톱 대비 20.3%p 낮음)
4. 📱 모바일 환경 테스트 권장 🟡
   └ 모바일 실패 비중 72.0%
5. ⏱️ 로딩/네트워크 구간 점검 🟡
   └ 평균 로그인 시간 78초 (기준: 60초)
6. ⏰ QR 만료 시간 임시 +1분 검토 🟡
   └ 만료율 13.3% (기준: 10%)

👉 상태: 즉시 조치 필요
```

## 🔜 다음 단계 (선택)

### 추천 액션 → 자동 실험 플래그

예:
- SMS 실패율 ↑ → `useNewSMSUX = true` 자동 제안
- 만료율 ↑ → `QR_EXPIRE_TIME = +60s` 임시 조정 제안
- 모바일 실패 ↑ → 모바일 전용 UX 실험 ON 제안

⚠️ 이건 **완전 자동 운영** 영역입니다.

원하면 바로 이어서 진행 가능합니다.
