# 🚨 DNS 설정 문제점 및 해결 방법

## 📊 현재 DNS 설정 분석

### ✅ 정상:
- `www.yagovibe.com` - CNAME 설정 올바름 (연결됨)
- TXT 레코드 - Firebase 인증용 (올바름)

### ⚠️ 문제점:

1. **A 레코드가 4개임**
   - Firebase Hosting은 보통 **2개의 A 레코드**만 필요
   - 현재 4개의 IP 주소:
     - 199.36.158.100
     - 199.36.158.101
     - 199.36.158.102
     - 199.36.158.103

2. **IP 주소가 Firebase Hosting IP인지 확인 필요**
   - Firebase Hosting은 특정 IP 주소를 사용
   - 현재 IP 주소들이 Firebase가 요청하는 IP와 일치하는지 확인 필요

---

## 🔍 확인 결과

**nslookup 결과:**
```
yagovibe.com의 IP 주소:
- 199.36.158.100
- 199.36.158.101
- 199.36.158.102
- 199.36.158.103
```

→ DNS 전파는 정상적으로 되어 있습니다.

---

## ✅ 해결 방법

### **1단계: Firebase 콘솔에서 정확한 IP 주소 확인**

1. **Firebase 콘솔** → Hosting → 커스텀 도메인
2. `yagovibe.com` 클릭
3. **"설정 필요"** 섹션에서 요청하는 DNS 레코드 확인:
   - 정확히 몇 개의 A 레코드가 필요한가?
   - 정확한 IP 주소는 무엇인가?

---

### **2단계: Gabia DNS에서 A 레코드 수정**

#### **방법 A: Firebase가 요청하는 IP 주소가 다른 경우**

1. **Firebase 콘솔에서 요청한 IP 주소 확인**
2. **Gabia DNS에서 기존 A 레코드 4개 모두 삭제**
3. **Firebase가 요청한 IP 주소로 A 레코드 2개 추가**

#### **방법 B: 현재 IP 주소가 맞는 경우**

Firebase가 요청하는 IP 주소가 `199.36.158.100-103` 중 일부라면:
1. **Firebase 콘솔에서 정확히 어떤 IP를 요청하는지 확인**
2. **요청하지 않은 IP 주소는 삭제**
3. **요청한 IP 주소 2개만 유지**

---

### **3단계: DNS 전파 후 Firebase 검증**

1. DNS 레코드 수정 후 **5-15분 대기**
2. Firebase 콘솔에서 **"검증" 또는 "다시 확인"** 클릭
3. SSL 인증서 발급 대기 (최대 1시간)

---

## 🎯 즉시 확인해야 할 사항

### **가장 중요: Firebase 콘솔 확인**

Firebase 콘솔에서 `yagovibe.com`을 클릭하면:
- ✅ 어떤 DNS 레코드를 요청하는지
- ✅ 정확한 IP 주소가 무엇인지
- ✅ A 레코드가 몇 개 필요한지

**확인하신 후 알려주시면 정확한 수정 방법을 안내하겠습니다!**

---

## 💡 일반적인 Firebase Hosting A 레코드 IP 주소

Firebase Hosting은 보통 다음 IP 주소를 사용합니다 (참고용):
- `151.101.1.195`
- `151.101.65.195`

하지만 **프로젝트마다 다를 수 있으므로**, Firebase 콘솔에서 확인한 정확한 IP 주소를 사용해야 합니다.

## 🚨 예상되는 문제

현재 `199.36.158.100-103` IP 주소들이:
- ✅ Firebase Hosting IP가 맞다면 → 레코드 개수만 조정
- ❌ Firebase Hosting IP가 아니라면 → 모든 A 레코드를 Firebase가 요청한 IP로 교체

**Firebase 콘솔에서 정확한 IP 주소를 확인해야 합니다!**
