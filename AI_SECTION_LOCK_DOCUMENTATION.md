# 🔒 AI 분석 섹션 최종 잠금 문서 (FINAL LOCK)

**목적**: AI 분석 섹션의 역할, 구조, 행동 규칙을 명확히 정의하고 동결  
**원칙**: 상품 등록 성공률 > AI 완성도  
**대상**: 개발자 / 기획자 / 모든 이해관계자

---

## 0️⃣ 최종 역할 정의 (한 줄)

**AI 분석 섹션은 "조언자"지 "심사관"이 아니다.**

---

## 1️⃣ 하지 않는 것 (절대 금지)

### ❌ 기능적 금지 사항
- [ ] 상품 등록의 필수 단계로 만들기
- [ ] 자동 실행 (페이지 진입 시, 이미지 선택 시 등)
- [ ] 페이지 진입 시 기본 노출 (항상 접힘 상태로 시작)
- [ ] 실패 시 화면 차단 (등록 불가 상태로 만들기)
- [ ] 성공 시 자동으로 필드 채우기

### ❌ UX 금지 사항
- [ ] "AI 분석 필요" 같은 강제 문구
- [ ] AI 섹션을 시각적으로 강조 (배너, 하이라이트)
- [ ] 등록 흐름 중간에 AI 섹션 배치
- [ ] AI 결과를 자동으로 적용

---

## 2️⃣ 하는 것 (허용 범위)

### ✅ 기능적 허용 사항
- [ ] 등록을 도와주는 선택 기능으로 존재
- [ ] 사용자가 원할 때만 열고 실행
- [ ] 실패해도 아무 일 없던 것처럼 돌아감
- [ ] 성공 시 사용자 승인 후 결과 적용

### ✅ UX 허용 사항
- [ ] 접힌 상태로 시작 ("▸ AI로 상품 설명 추천받기 (선택)")
- [ ] 명시적 "선택" 단어 사용
- [ ] 사용자가 직접 "AI 분석 시작" 버튼 클릭
- [ ] 결과 미리보기 + "적용하기" 버튼
- [ ] 실패 시 toast 1줄 메시지

---

## 3️⃣ 최종 UX 구조 (이 구조로 잠금)

### 3-1. 위치
- **상품 등록 CTA 아래** (권장)
- 또는 CTA 위지만 **접힘 상태**로 시작
- 👉 등록 흐름을 절대 끊지 않는 위치

### 3-2. 기본 상태 (페이지 진입 시)
```
▸ AI로 상품 설명 추천받기 (선택)
```
- **접힘 상태**
- 시각적 강조 없음
- "선택"이라는 단어 명시

### 3-3. 열었을 때 상태
```
AI 분석
[ 이미지 기반으로 상품 설명을 추천해드려요 ]
[ AI 분석 시작 버튼 ]
```
- 실행 버튼 명시적 클릭 필요
- 로딩 중 → 스피너 + "분석 중..."
- 성공 시 → 미리보기 + "적용하기" 버튼

### 3-4. 성공 시 동작
- ❌ 설명/제목 자동 채우기 금지
- ✅ "적용하기" 버튼으로만 반영
- ✅ 사용자가 통제권을 가짐

```
추천 설명 미리보기
[ 적용하기 ]
```

### 3-5. 실패 시 동작 (중요)
- ❌ alert 사용 금지
- ❌ 전체 에러 화면 금지
- ✅ toast 1줄만 표시

**실패 메시지**:
```
AI 분석에 실패했어요. 다시 시도할 수 있어요.
```

그리고 아무 일도 없었던 것처럼 유지 (등록 흐름 계속 가능)

## 4️⃣ 기술·구조 원칙 (이미 합의된 것 재확인)

### 4-1. Lazy Import
```typescript
const AISection = lazy(() => 
  import("./AISection").catch(() => ({
    default: () => <div>AI 기능을 불러올 수 없습니다.</div>
  }))
);
```

### 4-2. Error Boundary + Try/Catch
- AI 컴포넌트 단독 실패 가능
- 상품 등록 form state와 완전 분리
- 👉 AI가 죽어도 상품 등록은 살아 있어야 함

### 4-3. State 분리
- AI 섹션의 state는 자체적으로 관리
- 상품 등록 form state와 독립
- AI 결과는 사용자 승인 후에만 form에 반영

---

## 5️⃣ 잠금 규칙 (이건 문서로 남겨라)

### 🔒 규칙 1: 항상 접힌 상태로 시작
- 페이지 진입 시 AI 섹션은 기본적으로 접힘
- 사용자가 명시적으로 열기 전에는 노출되지 않음

### 🔒 규칙 2: AI 실패는 UX 영향 0
- AI 분석 실패 시 등록 흐름에 전혀 영향 없음
- 에러 메시지는 toast 1줄만, 그 외 변경 없음

### 🔒 규칙 3: AI 결과는 사용자 승인 후 적용
- AI 분석 성공 시에도 자동으로 필드에 채우지 않음
- "적용하기" 버튼을 통해 사용자가 명시적으로 승인해야 함

### 🔒 규칙 4: AI는 베타 느낌 유지 (과신 금지)
- AI 기능은 "추천" 수준으로 표현
- "필수" 또는 "강력 추천" 같은 표현 금지
- 베타/실험적 기능임을 암시하는 표현 가능

---

## 6️⃣ 왜 이게 정답이냐 (천재 모드 논리)

### 6-1. KPI 우선순위
**상품 등록의 KPI = 완료율**

AI는:
- 성공률을 조금 올릴 수는 있어도
- 실패율을 조금이라도 올리면 바로 독

### 6-2. 영향력 vs 비용
그래서:
- **영향력은 작게**: 등록 흐름을 가로채지 않음
- **통제권은 사용자에게**: 언제 열고 적용할지 사용자가 결정
- **실패 비용은 0으로**: AI 실패해도 등록은 정상 진행

### 6-3. 최종 한 줄 결론
**AI 분석 섹션은 "있으면 좋은 버튼"이지 "있어야만 하는 단계"가 아니다.**

## 8️⃣ 절대 변경 금지 사항

### ❌ 레이아웃 변경 금지
- AI 섹션을 등록 흐름 중간에 배치
- AI 섹션을 시각적으로 강조

### ❌ 기능 변경 금지
- AI 자동 실행 추가
- AI 결과 자동 적용 추가
- AI를 필수 단계로 만들기

### ❌ UX 변경 금지
- "AI 분석 필요" 같은 강제 문구
- AI 실패 시 등록 차단
- AI 성공 시 자동 채우기

---

## 9️⃣ 변경 허용 범위 (미세 조정만)

### ✅ 허용되는 미세 조정
- AI 버튼 문구 개선 (의미 유지)
- AI 결과 적용 UX 미세 조정 (승인 방식 유지)
- "AI 베타" 배지 추가 (선택적)
- Toast 메시지 문구 개선
- 로딩 애니메이션 개선

### ❌ 허용되지 않는 변경
- 구조 변경 (위치, 접힘/펼침 동작)
- 자동 실행 추가
- 필수 단계화
- 실패 시 차단 로직 추가
