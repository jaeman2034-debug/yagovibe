# 회비 납부 기준·대회 연동 규칙 구현 체크리스트

**목표: 회비 납부 기한 = 대회 참가 의사 표시 전까지**
**연회비 기준 확인 = 사무국 고시 사항**

---

## ✅ 필수 구현 항목

### 1️⃣ 회비 납부 기한 기준 고정

#### 공식 기준
- [ ] **대회 참가 신청 전까지 회비 납부 완료**

#### 시스템 반영 규칙
- [ ] 회비 미납 상태: 참가 신청 버튼 ❌
- [ ] 회비 납부 완료: 참가 신청 버튼 ⭕ 즉시 활성

#### 제거되는 것
- [ ] 마감일 계산 ❌
- [ ] 연체 기준 ❌
- [ ] 예외 판단 ❌

**목표: "참가하려면 먼저 납부"**

---

### 2️⃣ 연회비 기준 = 사무국 공지

#### 기준
- [ ] 연회비 기준은 사무국이 공지한 내용을 따른다.

#### 이유
- [ ] 금액 변경 가능
- [ ] 시기 조정 가능
- [ ] 고정 UI에 박아두면 분쟁 발생

---

### 3️⃣ 시스템 역할 분리

#### 시스템이 하는 일
- [ ] 회비 납부 여부 판단
- [ ] 참가 가능 / 불가 제어

#### 시스템이 하지 않는 일
- [ ] 연회비 정책 설명
- [ ] 금액 결정 이유 설명
- [ ] 시기 논쟁

**👉 이건 사람의 영역으로 남기는 게 맞습니다.**

---

### 4️⃣ 화면에 들어가야 할 기준 문구 (고정)

#### 위치
- [ ] 참가 버튼 근처

#### 문구 (고정)
```
대회 참가 안내

대회 참가를 위해서는
참가 신청 전까지 회비 납부가 완료되어야 합니다.
연회비 기준은 사무국 공지를 따릅니다.
```

#### 사용처
- [ ] 공지
- [ ] 대회
- [ ] 규정

**어디에 써도 동일해야 합니다.**

---

### 5️⃣ 사무국 효과

이제 사무국은 더 이상:

- [ ] ❌ "언제까지 내면 돼요?" 설명
- [ ] ❌ "이번 대회는 예외로…" 판단
- [ ] ❌ 연체 기준 계산

대신 이 말만 반복:

- [ ] **"참가 전 납부 기준입니다.**
- [ ] **연회비는 사무국 공지 확인하세요."**

---

## 📋 구현 체크 (짧게)

1. [ ] 참가 신청 버튼 조건 = 회비 납부 여부
2. [ ] 미납 시 버튼 비활성
3. [ ] 안내 문구 고정
4. [ ] 연회비 상세 설명 UI ❌ (의도적 미구현)

**이게 끝나면**
**대회·회비·참가 정책은 완전히 닫힙니다.**
