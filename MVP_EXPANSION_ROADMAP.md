# 🧭 3️⃣ 1번 MVP → 2번(AI/상태) 확장 로드맵

## 🎯 전제 (이거 안 지키면 망함)

> **2번 기능은
> 1번 MVP가 '이미 잘 쓰이고 있을 때만' 얹는다.**

조건은 딱 하나다:

* 사용자가
  👉 **말로 장소 찾는 걸 습관처럼 쓰고 있다**

이거 아니면 **2번은 독이다.**

---

# 🪜 확장은 "3단계"로만 간다

한 번에 AI ❌
→ **조금씩, 조용히**

---

## 🟢 Phase 1 — "기억" (가장 안전한 AI)

### 언제?

* 1번 MVP 사용 2~3주 후
* 재방문 사용자 생겼을 때
* 같은 카테고리를 3회 이상 검색했을 때

### 추가 기능 (UX 거의 안 변함)

#### 기능

* 사용자가 **자주 찾는 장소 타입 기억**

예:

* 축구장 자주 찾음 (5회 이상)
* 헬스장 자주 찾음 (3회 이상)

#### UI 변화 (아주 작게)

상단 문구만 살짝 바뀜

**기존:**
```
주변 장소를 말로 찾아보세요
```

**Phase 1:**
```
자주 찾던 축구장도 말해보세요
```

또는 (더 자연스럽게)

```
근처 축구장·헬스장·카페를 말로 찾아보세요
```

❗ 자동 추천 ❌
❗ 먼저 보여주기 ❌
❗ 버튼 추가 ❌

👉 **"말했을 때만" 반응**

---

### 구현 범위

**Backend:**
```typescript
// 사용자 검색 이력 저장
interface SearchHistory {
  userId: string;
  category: string;
  count: number;
  lastSearched: Timestamp;
}

// 자주 찾는 카테고리 조회
GET /api/user/frequent-categories
```

**Frontend:**
```typescript
// 상단 문구만 동적 변경
const frequentCategories = useFrequentCategories();
const headerText = frequentCategories.length > 0
  ? `자주 찾던 ${frequentCategories[0]}도 말해보세요`
  : "주변 장소를 말로 찾아보세요";
```

**UX 원칙:**
- 문구만 바뀜
- 기능 추가 없음
- 자동 동작 없음

---

## 🟡 Phase 2 — "상황 힌트" (조심 단계)

### 언제?

* 음성 검색 성공률 높음 (80% 이상)
* 사용자가 "편하다"라고 말할 때
* 재방문 사용자 비율 높아질 때

### 추가 기능

#### 예시

* 저녁 시간 + "축구장" 자주 검색

사용자가 앱 켰을 때 **한 번만**

**상단에 1줄만:**
```
지금 시간엔 축구장 찾는 분들이 많아요
```

또는

```
오늘 저녁엔 헬스장이 한적할 거예요
```

👉 질문 ❌
👉 강요 ❌
👉 버튼 ❌
👉 자동 검색 ❌

**정보 힌트 1줄만**

---

### 구현 범위

**Backend:**
```typescript
// 시간대별 인기 카테고리 분석
interface TimeBasedInsight {
  timeOfDay: "morning" | "afternoon" | "evening" | "night";
  popularCategory: string;
  message: string;
}

GET /api/insights/time-based
```

**Frontend:**
```typescript
// 상단에 힌트 1줄만 표시
const insight = useTimeBasedInsight();
{insight && (
  <div className="text-xs text-gray-500 mb-2">
    {insight.message}
  </div>
)}
```

**UX 원칙:**
- 힌트만 표시
- 클릭 불가
- 5초 후 자동 사라짐
- 하루 1회만 표시

---

## 🔵 Phase 3 — "이동 인식" (2번의 시작)

### 언제?

* 길찾기 클릭률이 높아짐 (30% 이상)
* 사용자가 실제로 이동하는 패턴이 보일 때
* "여기로 가기" 버튼을 자주 누를 때

### 이때부터 2번 시작이다

#### 기능

* 길찾기 시작 시 상태 인식

**기존:**
```
[길찾기 시작]
```

**Phase 3:**
```
○○축구장 가는 중이에요
도보 · 12분 남음
```

#### 여기서 처음으로

* 네비 톤
* 이동 UX
* 상태 개념 등장

👉 **하지만 이때도 자동 시작 ❌**
👉 **사용자가 "여기로 가기" 눌렀을 때만**

---

### 구현 범위

**Backend:**
```typescript
// 이동 세션 시작
POST /api/movement/start
{
  destination: { lat, lng, name },
  userId: string
}

// 이동 상태 조회
GET /api/movement/status
{
  status: "navigating",
  destination: {...},
  timeRemaining: "12분"
}
```

**Frontend:**
```typescript
// 길찾기 시작 시 상태 표시
const [movementStatus, setMovementStatus] = useState(null);

function startNavigation(destination) {
  // 외부 지도 앱 열기
  openExternalMap(destination);
  
  // 상태 표시 (선택적)
  setMovementStatus({
    type: "navigating",
    destination: destination.name,
    timeRemaining: "12분"
  });
}
```

**UX 원칙:**
- 길찾기 시작 시에만 표시
- 자동 시작 없음
- 사용자가 원하면 숨김 가능

---

# ❌ 절대 먼저 하면 안 되는 것 (경고)

이건 **1번 성공 전엔 절대 금지**다.

* ❌ "조금 있다가 가고 싶어"
* ❌ 자동 목적지 추론
* ❌ 말 안 해도 추천
* ❌ AI 동행 멘트
* ❌ 상태 기반 홈 화면
* ❌ "쉬는 중이에요" 같은 상태 표시
* ❌ 자동 네비게이션 시작
* ❌ 이동 예측

👉 이거 하면
**"말로 찾는 지도" 정체성 깨진다**

---

# 🧠 한 문장으로 정리한 확장 철학

> **AI는
> 먼저 도와주지 않는다.
>
> 사용자가 계속 쓰고 있을 때
> '조금 덜 귀찮게' 만들 뿐이다.**

---

# 📍 로드맵 요약 (한 장)

```
1번 MVP (음성 검색 지도)
  ↓
  [조건: 재방문 사용자 생김]
  ↓
Phase 1: "기억" (2-3주 후)
  - 자주 찾는 카테고리 기억
  - 상단 문구만 살짝 변경
  - 자동 동작 없음
  ↓
  [조건: 검색 성공률 80% 이상]
  ↓
Phase 2: "상황 힌트" (1-2개월 후)
  - 시간대별 힌트 1줄
  - 하루 1회만 표시
  - 자동 검색 없음
  ↓
  [조건: 길찾기 클릭률 30% 이상]
  ↓
Phase 3: "이동 인식" (2-3개월 후)
  - 길찾기 시작 시 상태 표시
  - 네비 톤 등장
  - 2번(AI/상태) 시작
```

# ⏱️ 타임라인 (현실적)

## Month 1: 1번 MVP
- 음성 검색 지도 출시
- 사용자 테스트
- 피드백 수집

## Month 2-3: Phase 1 준비
- 재방문 사용자 모니터링
- 검색 이력 수집
- "기억" 기능 개발

## Month 3-4: Phase 1 출시
- 자주 찾는 카테고리 기억
- 상단 문구 동적 변경

## Month 4-5: Phase 2 준비
- 검색 성공률 모니터링
- 시간대별 인사이트 분석
- "상황 힌트" 기능 개발

## Month 5-6: Phase 2 출시
- 시간대별 힌트 표시
- 하루 1회 제한

## Month 6-7: Phase 3 준비
- 길찾기 클릭률 모니터링
- 이동 패턴 분석
- "이동 인식" 기능 개발

## Month 7-8: Phase 3 출시
- 길찾기 시작 시 상태 표시
- 2번(AI/상태) 본격 시작

# ✅ 각 Phase 성공 기준

## Phase 1 성공 기준

- [ ] 재방문 사용자 비율 증가
- [ ] 자주 찾는 카테고리 문구가 자연스러움
- [ ] 사용자가 "편해졌다" 느끼지 않음 (자연스러움)

## Phase 2 성공 기준

- [ ] 힌트 표시 후 검색률 증가
- [ ] 사용자가 "방해되지 않는다" 느낌
- [ ] 힌트 무시해도 문제없음

## Phase 3 성공 기준

- [ ] 길찾기 사용률 증가
- [ ] 상태 표시가 자연스러움
- [ ] "이동 중" 개념 이해

# 🧠 확장 원칙 (절대 어기면 안 됨)

## 1. 사용자가 먼저 움직여야 함

* 자동 동작 ❌
* 먼저 제안 ❌
* 강요 ❌

## 2. 조금씩만

* 한 번에 여러 기능 ❌
* 큰 변화 ❌
* 작은 변화 ⭕

## 3. 실패해도 1번은 안전

* Phase 실패해도 1번 MVP는 그대로
* 롤백 가능
* 선택적 기능

---

# 📊 확장 판단 기준

## Phase 1 시작 조건

- [ ] 재방문 사용자 비율 ≥ 30%
- [ ] 같은 카테고리 3회 이상 검색한 사용자 ≥ 20%
- [ ] 1번 MVP 안정적 사용 중

## Phase 2 시작 조건

- [ ] 음성 검색 성공률 ≥ 80%
- [ ] 재방문 사용자 비율 ≥ 50%
- [ ] Phase 1 성공

## Phase 3 시작 조건

- [ ] 길찾기 클릭률 ≥ 30%
- [ ] Phase 2 성공
- [ ] 사용자 이동 패턴 명확
