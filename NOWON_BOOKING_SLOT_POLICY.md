# 노원구 축구협회 대관 슬롯 우선권 정책 (확정)

## 🎯 핵심 원칙

**회원 팀은 "먼저 선택할 권리"를 갖고, 비회원 팀은 "남은 시간"을 쓴다.**

- 복잡 ❌
- 운영 가능 ⭕
- 분쟁 최소화 ⭕

---

## ⏱️ 슬롯 모델

대관은 **"슬롯"** 기준으로 관리합니다.

```
날짜 × 시간대 × 운동장 = slot
```

### 슬롯 데이터 구조

```typescript
slots/{slotId} {
  facilityId: string,
  date: string,              // YYYY-MM-DD
  timeRange: string,         // 예: "18:00-20:00"
  status: "open" | "reserved",
  reservedBy?: string,       // teamId
  createdAt?: Timestamp,
  reservedAt?: Timestamp,
}
```

### 슬롯 ID 생성 규칙

```typescript
slotId = `${facilityId}_${date}_${timeRange}`
// 예: "facility-001_2026-01-15_18:00-20:00"
```

## 🥇 우선권 규칙

### Rule 1️⃣ 오픈 시점 차등

| 팀 유형 | 예약 가능 시점 | 설명 |
|---------|---------------|------|
| **회원 팀** | D-7일부터 | 선점 기간 4일 |
| **비회원 팀** | D-3일부터 | 회원 선택 후 잔여 시간 |

**예시:**
- 오늘이 2026-01-10인 경우
- 회원 팀: 2026-01-17부터 예약 가능
- 비회원 팀: 2026-01-13부터 예약 가능

### Rule 2️⃣ 슬롯 가시성

#### 비회원 팀의 경우

- **D-7 ~ D-4**: 회색(잠김) 표시
  - 메시지: "회원 우선 기간"
- **D-3 이후**: 선택 가능
  - 메시지: "잔여 시간대 이용 가능"

#### 회원 팀의 경우

- **D-7부터**: 선택 가능
  - 메시지: "우선 배정 대상입니다"

### Rule 3️⃣ 동시성 처리

예약 클릭 시 **트랜잭션**으로 처리:

1. 슬롯 상태 확인 (`status === "open"`)
2. 슬롯 상태 업데이트 (`status = "reserved"`, `reservedBy = teamId`)
3. Booking 문서 생성

**실패 시:**
- 메시지: "이미 다른 팀이 예약했습니다"
- 사용자에게 다른 시간대 선택 안내

## 🧩 팀 상태별 UX 문구

### 회원 팀

```
✅ 우선 배정 대상입니다
   D-7부터 예약 가능
```

### 비회원 팀

```
⚠️ 잔여 시간대 이용 가능
   회원 전환 시 우선 예약 가능
   
[ 회원 전환 신청 ]
```

**UX 플로우:**
1. 비회원 팀이 D-7~D-4 슬롯 클릭
2. 메시지: "회원 우선 기간입니다. D-3부터 예약 가능합니다."
3. CTA: "회원 전환 신청" → STEP 2로 자연스럽게 회귀

## 🔔 관리자 화면 보너스

관리자에게는 이렇게 표시:

```
[운동장 A - 2026-01-17]

18:00–20:00  🔒 회원 우선 (D-7~D-4)
20:00–22:00  🔓 전체 가능 (D-3 이후)
```

**설명 필요 없음** - 시각적으로 즉시 이해 가능

## 🧠 왜 이 로직이 "천재"인가

### 협회 관점
- ✅ 운영 단순
- ✅ 규정 설명 쉬움 ("회원은 7일 전부터, 비회원은 3일 전부터")

### 사용자 관점
- ✅ 차별은 느끼되, 불공정은 아님
- ✅ 명확한 규칙 (D-7 vs D-3)

### 플랫폼 관점
- ✅ 코드 단순 (날짜 차이 계산만)
- ✅ 슬롯 충돌 거의 없음 (시간대 분리)
- ✅ 트랜잭션으로 동시성 해결

**"우선권 = 시간"** 이 공식은 현장에서도 먹힌다.
