# 🔍 DNS 설정 분석 및 문제점 확인

## 📊 현재 DNS 설정 (Gabia)

### ✅ 정상 설정:

1. **www 서브도메인 (CNAME)**
   ```
   타입: CNAME
   호스트: www
   값: yago-vibe-spt.web.app. (끝에 점 있음)
   ```
   - ✅ 올바른 설정 (www는 연결됨)

### ⚠️ 확인 필요한 설정:

2. **루트 도메인 A 레코드들 (4개)**
   ```
   타입: A
   호스트: @
   값: 
     - 199.36.158.100
     - 199.36.158.101
     - 199.36.158.102
     - 199.36.158.103
   ```
   - ⚠️ **문제**: Firebase Hosting은 보통 1-2개의 A 레코드만 필요
   - ⚠️ **문제**: IP 주소가 Firebase Hosting IP인지 확인 필요
   - ⚠️ **문제**: 4개의 A 레코드는 불필요할 수 있음

3. **TXT 레코드**
   ```
   타입: TXT
   호스트: @
   값: "hosting-site=yago-vibe-spt"
   ```
   - ✅ Firebase 사이트 인증용 (올바름)

---

## 🔧 문제점 및 해결 방법

### **문제 1: A 레코드가 너무 많음**

Firebase Hosting은 일반적으로 **2개의 A 레코드**만 필요합니다. 4개의 A 레코드가 있으면:
- 오래된 IP 주소가 남아있을 수 있음
- 잘못된 IP 주소가 포함되어 있을 수 있음

**해결 방법:**
1. Firebase 콘솔에서 `yagovibe.com` 클릭
2. Firebase가 요청하는 **정확한 IP 주소** 확인
3. Gabia DNS에서 **모든 A 레코드 삭제**
4. Firebase가 요청한 IP 주소로 **새로운 A 레코드 2개만 추가**

---

### **문제 2: CNAME 레코드 끝에 점(.)**

`www` CNAME 레코드 값이 `yago-vibe-spt.web.app.` (끝에 점 있음)

**일반적으로:**
- 끝에 점(.)이 있으면: 절대 도메인 (FQDN - Fully Qualified Domain Name)
- 끝에 점(.)이 없으면: 상대 도메인

**확인 필요:**
- Firebase Hosting은 보통 끝에 점 없이 설정
- 하지만 현재 `www.yagovibe.com`이 연결되어 있다면 문제없을 수 있음

**권장:**
- Firebase 콘솔에서 확인한 정확한 값 사용
- 일반적으로는 끝에 점 없이 설정

---

### **문제 3: DNS 전파 및 Firebase 검증**

설정은 올바른 것처럼 보이지만 Firebase가 인식하지 못하는 경우:

1. **DNS 전파 시간** (보통 5-15분)
2. **Firebase 검증 대기 시간** (최대 1시간)
3. **잘못된 IP 주소** (Firebase가 요청한 IP와 다름)

---

## ✅ 해결 단계

### **1단계: Firebase 콘솔에서 정확한 설정 확인**

1. **Firebase 콘솔** → Hosting → 커스텀 도메인
2. `yagovibe.com` 클릭
3. **"설정 필요"** 메시지 확인:
   - 어떤 DNS 레코드를 요청하는지?
   - 정확한 IP 주소는 무엇인지?

---

### **2단계: Gabia DNS에서 A 레코드 정리**

1. **기존 A 레코드 4개 모두 삭제**
2. **Firebase 콘솔에서 요청한 IP 주소 확인**
3. **새로운 A 레코드 2개만 추가** (Firebase가 요청한 IP)

---

### **3단계: DNS 전파 대기**

1. DNS 레코드 수정 후 **5-15분 대기**
2. Firebase 콘솔에서 **"검증" 또는 "다시 확인"** 클릭

---

### **4단계: SSL 인증서 발급 대기**

1. DNS 인증 완료 후 **최대 1시간 대기**
2. 상태가 "연결됨"으로 변경될 때까지 대기

---

## 🎯 즉시 확인해야 할 사항

1. **Firebase 콘솔에서 `yagovibe.com` 클릭**
   - 요청하는 정확한 IP 주소 확인
   - A 레코드가 몇 개 필요한지 확인

2. **현재 설정과 Firebase 요청 사항 비교**
   - IP 주소가 일치하는지?
   - A 레코드 개수가 맞는지?

3. **불필요한 A 레코드 삭제**
   - Firebase가 요청하지 않은 IP 주소 제거

---

## 💡 참고 사항

- **www는 이미 연결됨**: CNAME 설정이 올바름
- **루트 도메인만 문제**: A 레코드 설정 확인 필요
- **DNS 제공자**: Gabia (Cloudflare가 아님)
- **TXT 레코드**: 이미 올바르게 설정됨

## 🚨 가장 가능성 높은 문제

**Firebase Hosting은 보통 2개의 특정 IP 주소를 요청합니다.**
현재 4개의 A 레코드가 있는데, 이 중 일부가:
- 오래된 IP 주소일 수 있음
- 잘못된 IP 주소일 수 있음
- Firebase가 현재 요청하는 IP 주소와 다를 수 있음

**해결:**
Firebase 콘솔에서 요청하는 **정확한 IP 주소 2개**로 A 레코드를 **재설정**해야 합니다.
