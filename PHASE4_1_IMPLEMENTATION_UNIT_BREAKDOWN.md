# Phase 4-1 첫 구현 단위 분해 (회계·정산)

**목적**: 사무국의 손을 가장 많이 묶는 일을 클릭 몇 번으로 끝내는 상태로 만든다 (아직 구현 ❌, 무엇을 만들지만 명확히 함)

---

## 1️⃣ Phase 4-1 목표 (한 줄)

**Phase 4-1은 '회계 시스템'이 아니라 '사무국을 밤에 집에 보내는 기능'이다.**

---

## 2️⃣ Phase 4-1 구현 범위 (고정)

### A. 대관료 수납/정산 보드 (읽기 → 기록)

#### 기능
- 협회 선지급 금액 표시
- 팀/비회원 수납 상태 표시

#### 상태 (3종만)
- **미수**: 아직 수납 안 됨
- **수납완료**: 수납 완료
- **환불완료**: 환불 완료

#### 제한 사항
- ❌ 결제 시스템 없음
- ⭕ 상태 기록만

#### UI 구조
```
┌─────────────────────────────────────┐
│ 대관료 수납/정산 보드                │
│                                     │
│ [기간 선택]                          │
│                                     │
│ [수납 상태 목록]                     │
│ - 팀 A: 미수                         │
│ - 팀 B: 수납완료                     │
│ - 팀 C: 환불완료                     │
│                                     │
│ [협회 선지급 금액]                   │
│ 총액: XXX원                          │
└─────────────────────────────────────┘
```

---

### B. 영수증 디지털화 (촬영 → 자동 분류)

#### 기능
- 모바일 사진 업로드
- 항목 자동 분류:
  - 시설 사용료
  - 심판 수당
  - 운영비
- 날짜·대회 자동 매칭

#### 성공 기준
**풀칠·스캔·폴더 관리 사라짐**

#### UI 구조
```
┌─────────────────────────────────────┐
│ 영수증 업로드                        │
│                                     │
│ [사진 업로드 버튼]                   │
│ 또는 드래그 앤 드롭                  │
│                                     │
│ [업로드된 영수증 목록]               │
│ - 영수증1.jpg                        │
│   항목: 시설 사용료                  │
│   날짜: 2025-07-21                   │
│   대회: 제36회 노원구청장기          │
│   상태: ✅ 자동 분류 완료            │
└─────────────────────────────────────┘
```

#### 자동 분류 로직 (개념)
- OCR 또는 AI 기반 텍스트 추출
- 항목 키워드 매칭 (시설/심판/운영비)
- 날짜 추출 및 대회 매칭
- 수동 수정 가능 (Admin만)

---

### C. 보고서 자동 생성 (구청/체육회용)

#### 기능
- 기간 선택 → PDF/엑셀 자동 생성

#### 항목 고정
- **수입**: 협회 수입 내역
- **지출**: 협회 지출 내역
- **잔액**: 수입 - 지출

#### 제한 사항
- 수정 ❌ (읽기 전용)
- 보고서 생성만 가능

#### UI 구조
```
┌─────────────────────────────────────┐
│ 보고서 생성                          │
│                                     │
│ [기간 선택]                          │
│ 시작일: [YYYY-MM-DD]                 │
│ 종료일: [YYYY-MM-DD]                 │
│                                     │
│ [보고서 항목]                        │
│ - 수입: XXX원                        │
│ - 지출: XXX원                        │
│ - 잔액: XXX원                        │
│                                     │
│ [생성 버튼]                          │
│ PDF 생성 | 엑셀 생성                 │
└─────────────────────────────────────┘
```

---

## 3️⃣ 권한 규칙 (단순)

### 협회 관리자
- 기록 ⭕ (수납 상태 변경, 영수증 업로드, 보고서 생성)
- 수정 ⭕ (영수증 분류 수정)

### 그 외 전원
- 열람 ❌ (모든 기능 접근 불가)

---

## 4️⃣ 명시적 제외 (Phase 4-1)

### ❌ Phase 4-1에서 하지 않는 것
1. **온라인 결제**
   - 결제 시스템 연동 없음
   - 수납 상태만 기록

2. **자동 송금**
   - 은행 연동 없음
   - 수동 처리

3. **회계 기준 변경**
   - 항목 추가/삭제 불가
   - 고정 항목만 사용

4. **외부 시스템 연동**
   - 회계 소프트웨어 연동 없음
   - 독립 시스템

---

## 5️⃣ 착수 선언 조건 (최종)

### Phase 4-1 IMPLEMENTATION START 조건

다음 조건이 모두 충족되어야 Phase 4-1 착수:

1. ✅ **Phase 4 트리거 2개 이상 충족**
   - 직접 수정 요구 발생
   - 동일 수기 작업 주 3회 이상

2. ✅ **사무국장 "이거 하나만 먼저 만들자" 선언**
   - 명확한 우선순위 선언
   - Phase 4-1 범위에 대한 합의

3. ✅ **개발 2~3주 이내 가능**
   - 기술적 난이도 확인
   - 개발 리소스 확보

## 6️⃣ 구현 우선순위 (Phase 4-1 내부)

### 1순위: 대관료 수납/정산 보드
- 가장 빈번한 작업
- 즉시 효과 체감 가능

### 2순위: 영수증 디지털화
- 시간 절감 효과 큼
- OCR/AI 기술 필요 (난이도 높을 수 있음)

### 3순위: 보고서 자동 생성
- 구청 보고용
- 월간/연간 보고서 생성

---

## 7️⃣ 기술 스택 고려사항 (참고용)

### 대관료 수납/정산 보드
- Firestore: 수납 상태 저장
- React: 상태 관리 UI
- Admin 전용 페이지

### 영수증 디지털화
- Firebase Storage: 영수증 이미지 저장
- OCR API: 텍스트 추출 (Google Cloud Vision 또는 유사 서비스)
- AI 분류: 키워드 매칭 또는 간단한 ML 모델

### 보고서 자동 생성
- PDF 생성: jsPDF 또는 유사 라이브러리
- 엑셀 생성: ExcelJS 또는 유사 라이브러리
- Firestore 쿼리: 기간별 데이터 집계

---

## 8️⃣ 성공 기준

### Phase 4-1 완료 기준
1. ✅ 대관료 수납/정산 보드 정상 작동
   - 수납 상태 변경 가능
   - 협회 선지급 금액 표시

2. ✅ 영수증 디지털화 정상 작동
   - 사진 업로드 가능

   - 수동 수정 가능

3. ✅ 보고서 자동 생성 정상 작동
   - PDF/엑셀 생성 가능
   - 수입/지출/잔액 정확히 계산

4. ✅ 사무국장 피드백
   - "풀칠·스캔·폴더 관리 사라짐" 확인
