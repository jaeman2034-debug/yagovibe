# FacilitySection 회원/비회원 팀 UI 구분 규칙 (최종 확정)

**목적**: FacilitySection에서 회원 팀과 비회원 팀의 대관 가능 시간을 안내 문구로 구분하여 표시 (신청/결제/문의 기능 없음, 표시만)

---

## 1️⃣ FacilitySection의 역할 (재정의)

### 역할 정의
- ❌ 예약 시스템 아님
- ❌ 신청 접수 창구 아님
- ❌ 문의 유도 페이지 아님
- ⭕ **"공식 대관 현황판"**

**핵심**: 이 섹션은 판단을 대신해 주는 표지판이다.

### 현재 상태 (Phase 4-1 기준)
- 슬롯 상태 표시: `available` / `blocked` / `event`
- 관리자 토글: `available` ↔ `blocked` (confirm 필수)
- 자동 알림: 상태 변경 시 Kakao Alimtalk (Stub)

---

## 2️⃣ 회원/비회원 구분 기준 (데이터 개념)

### 회원 팀 (Member Teams)
```
조건: team.associationRelation?.status === 'official'
의미: 노원구 축구협회 공식 산하 팀
```

### 비회원 팀 (Non-Member Teams)
```
조건: team.associationRelation?.status !== 'official' OR 없음
의미: 협회 산하가 아닌 팀 (연계/독립 포함)
```

---

## 3️⃣ 핵심 원칙: 슬롯은 공정하게, 문구로 우선순위 표현

### 슬롯 카드 구조 (기존 유지, 변경 없음)

```
┌─────────────────────────────────────┐
│ [날짜] [시간]                        │
│                                     │
│ 상태: 🟢 사용 가능                   │
│                                     │
└─────────────────────────────────────┘
```

**중요**: 슬롯 자체는 회원/비회원 구분 표시 ❌
- 모든 사용자가 동일한 슬롯 상태를 본다
- `available` / `blocked` / `event` 3종만 표시
- 회원/비회원 구분은 **안내 문구**로만 표현

### 상태별 표시 방식 (슬롯 자체는 동일)

#### 상태: `available` (사용 가능)
```
상태: 🟢 사용 가능
```
- 회원/비회원 표시 없음
- 모든 사용자에게 동일하게 표시

#### 상태: `blocked` (협회 사용)
```
상태: 🔴 협회 사용
```
- 회원/비회원 표시 없음

#### 상태: `event` (대회/행사)
```
상태: 🟣 대회/행사: [대회명]
```
- 회원/비회원 표시 없음

---

## 4️⃣ 상단 안내 문구 (슬롯 그리드 위, 필수)

### 안내 박스 위치
- 슬롯 그리드 **위**에 딱 한 블록 추가
- 배경: 밝은 회색 또는 흰색 (`bg-gray-50` 또는 `bg-white`)
- 테두리: 얇은 테두리 (`border border-gray-200`)
- 패딩: 적절한 여백 (`p-4` 또는 `p-6`)

### 공식 안내 문구 (고정, 그대로 사용)

```
운동장 사용 안내

협회 선대관 및 자체 관리 구장은
협회 운영 기준에 따라 배정됩니다.
잔여 시간은 상황에 따라 조정될 수 있습니다.
```

**문구 특징**:
- 중립적 톤 (법적 분쟁 유발 단어 없음)
- "신청하세요" 같은 행동 유도 ❌
- "회원/비회원" 직접 언급 ❌ (간접적 표현만)

---

## 5️⃣ 하단 보조 문구 (슬롯 그리드 아래, 필수)

### 보조 문구 위치
- 슬롯 그리드 **아래**에 아주 작게 표시
- 텍스트 크기: 작게 (`text-xs` 또는 `text-sm`)
- 색상: 회색 (`text-gray-500`)

### 보조 문구 (고정, 그대로 사용)

```
※ 협회 회원 팀 우선 운영
```

**문구 특징**:
- 한 줄로 간결
- 회원/비회원 구분 명확하게 표현
- "차별"이 아닌 "운영 기준"으로 표현
- 전화 질문 대부분 차단

## 7️⃣ Admin / Public 동일 규칙

### Public 사용자
- 안내 문구: 동일하게 표시
- 슬롯: 동일하게 표시 (회원/비회원 구분 없음)
- 클릭/인터랙션: ❌ 없음

### Admin 사용자 (Edit Mode)
- 안내 문구: **동일하게 표시** (수정 불가)
- 슬롯: **동일하게 표시** (회원/비회원 구분 없음)
- 상태 토글: 기존과 동일 (`available` ↔ `blocked`)
- 회원/비회원 선택 UI: ❌ 없음

**중요**: Admin도 문구 수정 불가 (공식 안내이므로)

---

## 8️⃣ 절대 하지 않는 것 (강조)

### 금지 사항
- ❌ "회원 전용" 버튼
- ❌ "비회원 문의" 링크
- ❌ 가격 표시
- ❌ 잔여 시간 강조 애니메이션
- ❌ 슬롯 카드에 회원/비회원 배지
- ❌ 클릭/인터랙션 기능

**이유**: 보여주는 순간 문의가 생긴다.

---

## 9️⃣ 클릭/인터랙션 규칙

### Public 사용자 (일반 사용자)
- **클릭 동작**: ❌ 없음 (표시만)
- **신청 버튼**: ❌ 없음
- **문의 버튼**: ❌ 없음
- **상세 정보**: ❌ 없음

### 회원 팀 사용자 (로그인 + 협회 산하 팀 소속)
- **클릭 동작**: ❌ 없음 (표시만, Phase 4-1에서는 기능 없음)
- **예약 버튼**: ❌ 없음 (Phase 4 이후)

### 관리자 (Edit Mode)
- **기존 토글 기능**: `available` ↔ `blocked` (변경 없음)
- **안내 문구 수정**: ❌ 불가 (공식 문구이므로)

## 🔟 모바일 반응형

### 모바일 화면
- 안내 박스: 패딩 줄이기 (`p-4`)
- 문구 크기: 적절하게 조정 (`text-sm`)
- 하단 보조 문구: `text-xs` 유지

### 데스크탑 화면
- 안내 박스: 패딩 넓게 (`p-6`)
- 문구 크기: 기본 크기 (`text-base`)

## 1️⃣1️⃣ 이 설계의 실제 효과

### 효과 1: 사무국장님의 부담 감소
- "왜 우리는 안 돼요?" 질문을 화면으로 대신 설명
- 캡처해서 보내도 설명이 필요 없음

### 효과 2: 운영 관행과 일치
- 실제 운영 방식과 100% 일치
- 법적 분쟁 유발 요소 없음

### 효과 3: Phase 5 확장 가능성
- 신청/결제 붙여도 구조 유지 가능
- 안내 문구만 업데이트하면 됨

---

## 1️⃣2️⃣ 성공 기준 (완료 판정 질문)

다음 질문에 모두 **YES**면 설계 완료:

1. **비회원이 봐도 "문의해야 하나?" 생각 안 드는가?**
   - 안내 문구가 명확해서 추가 문의 불필요

2. **회원이 봐도 "우리가 우선이구나" 자연스럽게 느끼는가?**
   - 하단 보조 문구가 자연스럽게 인지됨

3. **사무국장이 이 화면을 캡처해서 보내도 설명이 필요 없는가?**
   - 화면만으로 모든 정보 전달 가능
