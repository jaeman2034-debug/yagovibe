# 🔒 상품 등록 페이지 UI 잠금 체크리스트 (FINAL LOCK)

**목적**: 상품 등록 UX를 기준선(baseline)으로 고정하고 더 이상 변경하지 않음

## 0️⃣ Baseline 고정 선언 (절대 변경 금지)

### 현재 상태 = 최종 기준선
- **파일**: `src/pages/MarketAddPage.tsx`
- **고정일**: 현재
- **핵심 원칙**: 등록 성공률 > 완성도

### Baseline 구성 요소 (변경 금지)
1. ✅ 페이지 타이틀 + 서브 문구
2. ✅ 이미지 업로드 영역 (드래그 앤 드롭 스타일)
3. ✅ 필수 필드 (상품명, 가격) + 선택 필드 (설명)
4. ✅ 필드별 inline validation
5. ✅ CTA 버튼 + 안내 문구
6. ✅ AI 섹션 (격리됨, lazy import)

## 5️⃣ 절대 금지 사항 (이 단계에서)

### ❌ 하면 안 되는 것
1. **레이아웃 변경**: 필드 순서, 위치 변경
2. **필드 추가**: 카테고리, 배송, 상태 등 추가 필드
3. **AI 자동화**: 이미지 선택 시 AI 자동 실행
4. **복잡한 옵션**: 배송 방법, 거래 방식 등
5. **애니메이션 추가**: 불필요한 애니메이션
6. **튜토리얼/온보딩**: 설명 페이지 추가

**이유**: 등록 성공률이 완성도보다 중요

## 🔒 Baseline 고정 선언 (최종)

**현재 상태(`src/pages/MarketAddPage.tsx`)는 상품 등록 UX의 기준선으로 고정됩니다.**

- 이 상태에서 절대 후퇴하지 않습니다.
- 기능 추가는 등록 성공률을 해치지 않는 범위에서만 가능합니다.

**이 선언이 있으면:**
- ✅ 디자이너가 뭐라 해도 흔들릴 이유 없음
- ✅ "완성도" 강박에서 벗어남
- ✅ 등록 성공률에 집중 가능
