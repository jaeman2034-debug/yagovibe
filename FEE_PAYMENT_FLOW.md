# 💰 회비 납부 처리 흐름 (초간단)

## 🎯 목표

관리자가 버튼 한 번 누르면  
**"홍길동 · 2025년 7월 · 20,000원 · 완납"** 이 DB에 남는다.

## 1️⃣ 버튼 클릭 시 전송 데이터

프론트 → 서버로 보내는 값 (딱 이것만):

```json
{
  "teamId": "team1",
  "memberId": "hong",
  "month": "2025-07",
  "amount": 20000
}
```

**핵심**: 이름, 상태, 미납 개수 필요 없음  
**필요한 것**: 누구 / 언제 / 얼마

## 2️⃣ 서버가 하는 일 (3단계)

### STEP 1: 회비 기록 찾기

```
teamId + memberId + month
```

- 있으면 → 업데이트
- 없으면 → 새로 생성

### STEP 2: 납부 완료로 저장

```typescript
fee {
  teamId: "team1",
  memberId: "hong",
  month: "2025-07",
  amount: 20000,
  paid: true,
  paidAt: serverTimestamp()
}
```

**🔥 여기서 끝**  
👉 회원(member) 테이블은 건드리지 않는다 (계산 결과만 업데이트)

### STEP 3: 응답

```json
{ "success": true }
```

## 3️⃣ 화면 자동 반영

DB에 이 한 줄이 들어가면:

- ✅ 회비 / 회계 화면 → 완납 표시
- ✅ 회원 관리 화면 → 미납 0 (계산 결과)
- ✅ 관리보드 요약 → 미납 인원 감소

**👉 따로 계산 로직 안 짜도 됨**  
모두 "fee 기록 기준"으로 다시 그리면 된다.

## 4️⃣ 왜 이게 천재 구조냐

- ❌ "미납 숫자"를 저장하지 않는다
- ❌ "완납 상태"를 사람에 저장하지 않는다
- ✅ **기록만 저장한다**
- ✅ **상태는 항상 계산 결과**

**결과**:
- 실수 없음
- 되돌릴 일 없음
- 자동화 붙이기 쉬움

## 5️⃣ 실전 예시

**홍길동이 계좌로 20,000원 보냄**

관리자:
```
우리 팀 관리
 → 회비 / 회계
   → 2025년 7월
     → 홍길동 [납부 완료]
```

확인 누름 → **끝**

## 🔑 한 문장 요약

**버튼은 화면에 있고, 진실은 DB에 남고, 상태는 계산으로 나온다.**
