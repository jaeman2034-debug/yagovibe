# 🔧 QR 코드 수정 지시문 (실전용) - 업그레이드 버전

> **현장에 그대로 던질 수 있는 버전**
> (디자이너 / 프론트 / 운영 중 누구한테 줘도 통하는 버전)

---

## ❗ 요청 요약

> **이 QR 코드는 전화번호 로그인 진입용이다.**
> **홈/마켓 분기용이 아니다.**
> **중간 페이지와 트래킹 파라미터를 제거하고**
> **로그인 시작 지점으로 바로 연결해야 한다.**

---

## 1️⃣ QR 코드의 원래 역할 (팩트 정리)

이 QR은 성격상 **100% 이거였어야 한다** 👇

> **전화번호 로그인(Phone Auth)을 바로 시작시키는 고정 진입점**

즉,

* 앱/웹 어디서 스캔하든
* **즉시 전화번호 로그인 UX로 진입**
* 중간 페이지 ❌
* 마케팅 분기 ❌
* 의미 없는 파라미터 ❌

---

## 2️⃣ 수정 대상

* **메인 홈 화면 QR 코드**
* 현재 연결된 URL:

  ```
  https://www.yagovibe.com/qr?market=home
  ```

### ❌ 현재 상태가 왜 문제냐면

이건 논리적으로 보면:

```
QR
 → 중간 /qr 페이지
   → market=home 해석
     → 다시 home or map
       → (그 다음에야 로그인)
```

❌ 문제 포인트 3개

1. **전화번호 로그인 전용 진입이 아님**
2. QR 스캔 → 동작이 예측 불가
3. 로그인 퍼널 측정도 망가짐

👉 **QR의 존재 이유가 흐려짐**

---

## 3️⃣ 수정 지시 (중요)

### ✅ 변경 요구

* QR 코드 URL을 **전화번호 로그인 시작 지점으로 변경**

  * **권장 최종 구조**:
    ```
    https://www.yagovibe.com/login/phone
    ```
    
    또는
    
    ```
    https://www.yagovibe.com/login
    ```
    
    (로그인 페이지에서 전화번호 로그인으로 자동 분기)

* **불필요한 쿼리 파라미터(`?market=home`) 제거**
* **중간 리다이렉트 페이지(`/qr`) 제거**

### ✅ 로그인 완료 후 자동 분기

로그인 성공 후에는 자동으로:
* **모바일**: `/app/map` (지도 바로 진입)
* **PC/데스크탑**: `/home` (홈 화면)

이 분기는 **로그인 완료 후**에만 적용됩니다.

### ❌ 금지 사항

* 마케팅/트래킹 목적의 파라미터 **QR에 직접 삽입 금지**
* QR 코드 목적지를 **중간 리다이렉트 페이지로 연결 ❌**
* `?market=home`, `?utm_source=qr` 같은 파라미터 **절대 사용 금지**
* 홈/마켓으로 직접 연결 (로그인 없이) ❌

## 4️⃣ 권장 구조 (표준)

### ✅ 가장 안전한 1안 (추천)

```
QR
 → https://www.yagovibe.com/login/phone
   (전화번호 로그인 바로 시작)
   
로그인 성공 후 자동 분기:
   ├ 모바일 → /app/map
   └ PC     → /home
```

**장점:**
* QR의 목적이 명확함 (전화번호 로그인)
* 중간 리다이렉트 없음
* 로그인 퍼널 측정 정확
* 사용자 경험 예측 가능

### 🔹 차선책 (이미 정해진 경우)

```
QR
 → https://www.yagovibe.com/login
   (로그인 페이지에서 전화번호 로그인으로 자동 분기)
```

하지만 이건 **"로그인 전용 QR" 관점에서는 반쪽짜리**다.

### ❌ 사용 금지

```text
❌ https://www.yagovibe.com/qr?market=home
   (중간 리다이렉트 + 홈/마켓 분기용)

❌ https://www.yagovibe.com/app/map
   (로그인 없이 직접 진입)

❌ https://www.yagovibe.com/home
   (로그인 없이 직접 진입)
```

## 8️⃣ 프론트엔드 개발자용 지시문

### 현재 코드 위치

- `src/lib/qrRouter.ts` - `generateQRUrl` 함수
- QR 생성 시 `market=home` 파라미터 사용 중
- `src/pages/qr/QRPage.tsx` - QR 진입 처리

### 수정 방향

**권장 수정 (전화번호 로그인 진입용으로 변경):**

```typescript
// src/lib/qrRouter.ts
export function generateQRUrl(type: QRType, id: string, utm?: string): string {
  // 🔥 전화번호 로그인 진입용 QR
  if (type === "market" && id === "home") {
    // 기존: getPublicUrl(`/qr?market=home`)
    // 변경: 전화번호 로그인 페이지로 직접 연결
    return getPublicUrl("/login/phone");
  }
  
  // 기타 타입은 기존 로직 유지
  // ...
}
```

**또는 QR 생성 시점에서 직접 지정:**

```typescript
// QR 코드 생성 시
const qrUrl = "https://www.yagovibe.com/login/phone";
```

### 로그인 완료 후 분기 처리

로그인 성공 후 자동 분기는 이미 구현되어 있을 가능성이 높습니다.
`PhoneLoginPage.tsx` 또는 `QRPhoneLoginPage.tsx`에서 확인:

```typescript
// 로그인 성공 후
const isMobile = isMobileLikeDevice();
navigate(isMobile ? "/app/map" : "/home");
```

## 🔟 QR 변경 관리 가이드 (사내용)

### 변경 절차

1. **변경 요청**
   - 변경 사유 명시
   - 목적지 URL 확인
   - 트래킹 필요 여부 확인

2. **개발팀 검토**
   - 기술적 타당성 검토
   - 기존 QR 코드와의 호환성 확인

3. **디자인팀 확인**
   - QR 코드 재생성
   - 인쇄물/배포물 확인

4. **운영팀 공지**
   - 변경 사항 공지
   - 기존 QR 코드 사용처 확인

5. **변경 이력 기록**
   - 변경 날짜
   - 변경 사유
   - 담당자
   - 영향 범위
