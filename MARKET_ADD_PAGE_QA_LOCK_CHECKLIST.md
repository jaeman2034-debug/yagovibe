# 🔒 상품 등록 페이지 QA 잠금 체크리스트 (FINAL)

**목적**: 상품 등록 UX의 최종 검증 및 잠금 확정  
**원칙**: 구조 변경 금지, 성공률 중심 검증  
**대상**: 개발자 / QA / 배포 전 최종 확인

---

## 0️⃣ 잠금 선언 (Lock Declaration)

### 현재 상태 = 최종 기준선 (동결)
- ✅ **상품 등록 UI**: 구조/레이아웃 동결
- ✅ **헤더**: 풀폭 유지 확정
- ✅ **그리드/콘텐츠**: 기존 max-width 유지 확정
- ✅ **AI**: 보조 기능 위치 유지 확정

### 행동 원칙
- ❌ 새 기능 추가 금지
- ❌ 구조 변경 금지
- ✅ 성공률만 올리는 미세 조정만 허용

### 허용 범위
- 문구 1줄 수정
- 입력 검증 개선
- 로딩/에러 피드백 개선
- 접근성(키패드/포커스) 개선

## 1️⃣ 모바일 입력 흐름 검증

### 1-1. 기본 입력 흐름 (모바일)
- [ ] 페이지 로드 시 첫 필드(이미지)에 자동 포커스 안 가도록 확인
- [ ] 이미지 선택 영역 터치 → 카메라/갤러리 선택 가능
- [ ] 이미지 선택 후 미리보기 즉시 표시
- [ ] 상품명 입력 시 키보드 정상 표시
- [ ] 가격 입력 시 숫자 키패드 표시 (`inputMode="numeric"`)
- [ ] 설명 입력 시 일반 키보드 표시
- [ ] 입력 필드 간 이동이 자연스러움

**통과 기준**: 7개 모두 YES  
**실패 시**: 접근성/키보드 이슈 확인

---

### 1-2. 모바일 스크롤/레이아웃
- [ ] 키보드 올라와도 입력 필드 가려지지 않음
- [ ] 스크롤이 자연스럽고 부드러움
- [ ] textarea 높이가 적절함 (`rows={3}`)
- [ ] 저장 버튼이 항상 접근 가능한 위치
- [ ] 화면 회전 시 레이아웃 깨지지 않음

**통과 기준**: 5개 모두 YES  
**실패 시**: 모바일 반응형 확인

---

### 1-3. 모바일 터치 영역
- [ ] 이미지 선택 영역 터치하기 쉬움 (최소 44px)
- [ ] 저장 버튼 터치하기 쉬움
- [ ] 입력 필드 터치하기 쉬움
- [ ] 이미지 변경 버튼(✕) 터치하기 쉬움
- [ ] 실수로 다른 영역 터치해도 문제 없음

**통과 기준**: 5개 모두 YES  
**실패 시**: 터치 영역 크기 확인

---

## 2️⃣ 데스크탑 입력 흐름 검증

### 2-1. 기본 입력 흐름 (데스크탑)
- [ ] 마우스로 이미지 선택 영역 클릭 가능
- [ ] 이미지 드래그 앤 드롭 동작 확인 (없어도 OK)
- [ ] 키보드 Tab으로 필드 간 이동 가능
- [ ] Enter 키로 저장 가능 (있는 경우)
- [ ] 입력 필드 포커스 시각적 피드백 명확

**통과 기준**: 4개 이상 YES  
**실패 시**: 키보드 네비게이션 확인

---

### 2-2. 데스크탑 레이아웃
- [ ] 페이지가 중앙 정렬됨 (`max-width: 600px`)
- [ ] 헤더가 풀폭으로 표시됨
- [ ] 입력 필드 너비가 적절함 (너무 넓거나 좁지 않음)
- [ ] 이미지 미리보기 크기가 적절함

**통과 기준**: 4개 모두 YES  
**실패 시**: 레이아웃 구조 확인

---

## 3️⃣ 이미지 업로드 엣지 케이스

### 3-1. 이미지 선택 엣지 케이스
- [ ] 이미지 선택 취소 시 → 에러 없음
- [ ] 매우 큰 이미지(>10MB) 선택 시 → 에러 메시지 표시
- [ ] 이미지가 아닌 파일 선택 시 → 에러 메시지 표시
- [ ] 동일 이미지 재선택 시 → 정상 동작
- [ ] 이미지 변경 버튼(✕) 클릭 시 → 초기 상태로 복귀

**통과 기준**: 5개 모두 YES  
**실패 시**: 이미지 검증 로직 확인

---

### 3-2. 이미지 미리보기
- [ ] 세로형 이미지 → 비율 유지하며 표시
- [ ] 가로형 이미지 → 비율 유지하며 표시
- [ ] 정사각형 이미지 → 정상 표시
- [ ] 매우 긴 이미지 → 스크롤 가능하거나 크기 제한
- [ ] 이미지 테두리/그림자가 명확히 보임

**통과 기준**: 5개 모두 YES  
**실패 시**: 이미지 미리보기 스타일 확인

---

### 3-3. 네트워크/업로드 엣지 케이스
- [ ] 업로드 중 네트워크 끊김 → 에러 메시지 표시
- [ ] 업로드 실패 시 → 재시도 가능한 상태
- [ ] 업로드 중 저장 버튼 클릭 차단됨
- [ ] 업로드 성공 후 저장 진행

**통과 기준**: 4개 모두 YES  
**실패 시**: 에러 처리 로직 확인

---

## 4️⃣ 입력 검증 검증

### 4-1. 필수 필드 검증
- [ ] 상품명 미입력 → 필드 아래 에러 표시 ("상품명을 입력해주세요.")
- [ ] 가격 미입력 → 필드 아래 에러 표시 ("가격을 입력해주세요.")
- [ ] 이미지 미선택 → 이미지 영역 아래 에러 표시
- [ ] `alert()` 사용하지 않음
- [ ] 입력 시작 시 에러 자동 해제

**통과 기준**: 5개 모두 YES  
**실패 시**: validation 로직 확인

---

### 4-2. 가격 검증
- [ ] 가격 0 입력 → 에러 표시 ("올바른 가격을 입력해주세요.")
- [ ] 가격 음수 입력 → 에러 표시
- [ ] 가격에 문자 입력 시도 → 숫자만 입력 가능
- [ ] 가격에 소수점 입력 → 허용/불허용 정책 확인
- [ ] 매우 큰 숫자 입력 → 정상 처리 (있는 경우)

**통과 기준**: 4개 이상 YES  
**실패 시**: 가격 검증 로직 확인

---

### 4-3. 설명 필드 검증
- [ ] 설명은 선택 필드 (비워도 OK)
- [ ] 설명에 긴 텍스트 입력 → 정상 처리
- [ ] 설명 줄바꿈 정상 동작

**통과 기준**: 3개 모두 YES  
**실패 시**: 설명 필드 검증 확인

---

## 5️⃣ 등록 성공/실패 피드백

### 5-1. 등록 성공 피드백
- [ ] 저장 성공 시 → 성공 메시지 표시 ("✅ 상품이 성공적으로 등록되었습니다!")
- [ ] 성공 메시지 색상/배경 명확함 (초록색 계열)
- [ ] 성공 후 1.5초 후 마켓 홈으로 자동 이동
- [ ] 저장 중 버튼에 스피너 표시
- [ ] 저장 중 버튼 disabled 상태

**통과 기준**: 5개 모두 YES  
**실패 시**: 성공 피드백 로직 확인

---

### 5-2. 등록 실패 피드백
- [ ] 네트워크 에러 → 에러 메시지 표시
- [ ] 로그인 필요 → 에러 메시지 표시
- [ ] 서버 에러 → 에러 메시지 표시
- [ ] 에러 메시지 색상/배경 명확함 (빨간색 계열)
- [ ] 에러 후 재시도 가능 (필드 값 유지)
- [ ] `alert()` 사용하지 않음

**통과 기준**: 6개 모두 YES  
**실패 시**: 에러 처리 로직 확인

---

### 5-3. 로딩 상태
- [ ] 저장 중 버튼 텍스트: "저장 중..."
- [ ] 저장 중 버튼에 스피너 아이콘 표시
- [ ] 저장 중 다른 필드 입력 차단되지 않음 (선택적)
- [ ] 저장 중 이미지 변경 불가능

**통과 기준**: 3개 이상 YES  
**실패 시**: 로딩 상태 UI 확인

## 6️⃣ 접근성 검증

- [ ] Tab으로 모든 입력 필드 접근 가능
- [ ] 포커스 시각적 피드백 명확

## 9️⃣ 사용자 흐름 검증 (통합)

### 9-1. 정상 흐름 (Happy Path)
- [ ] 이미지 선택 → 상품명 입력 → 가격 입력 → 저장 → 성공 → 마켓 홈 이동
- [ ] 전체 흐름이 자연스럽고 막히는 지점 없음
- [ ] 각 단계에서 명확한 피드백 제공
- [ ] 사용자가 다음 액션을 알 수 있음

**통과 기준**: 4개 모두 YES  
**실패 시**: 사용자 흐름 재검토

---

### 9-2. 에러 복구 흐름
- [ ] 필수 필드 에러 표시 → 입력 → 에러 해제 → 저장 성공
- [ ] 이미지 에러 표시 → 재선택 → 정상 진행
- [ ] 저장 실패 → 에러 메시지 확인 → 재시도 성공

**통과 기준**: 3개 모두 YES  
**실패 시**: 에러 복구 로직 확인

## 🔒 최종 잠금 선언

**이 체크리스트를 통과하면 상품 등록 페이지는 완전히 잠금됩니다.**

- 구조/레이아웃 변경 금지
- 새 기능 추가 금지
- 성공률 중심의 미세 조정만 허용
- 배포 준비 완료

**이 선언이 있으면:**
- ✅ "완성도" 강박에서 벗어남
- ✅ 등록 성공률에 집중 가능
- ✅ 안정적인 서비스 운영 가능
