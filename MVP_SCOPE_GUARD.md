# 🛡️ MVP 범위 가드 (개발 중 필수)

> 개발 중 "이거 넣어야 하지 않나?" 질문을 즉시 걸러내는 기준

---

## 🎯 3초 판단 기준

### 넣어도 되는 것

✅ **말로 주변 장소 찾기**에 직접 필요
✅ 없으면 음성 검색 작동 안 함
✅ 사용자가 "뭘 말해야 하지?" 질문하게 만듦

### 나중인 것

❌ 있으면 좋지만 없어도 됨
❌ AI/추천/상태 관련
❌ 사용자 요청 (테스트 후)
❌ "멋져 보이는" 기능

---

## ❓ 자주 나오는 질문 + 즉시 답변

### Q: "이건 지금 넣어야 하지 않나?"

**체크:**
1. 말로 주변 장소 찾기에 직접 필요함? → YES/NO
2. 없으면 음성 검색이 작동 안 함? → YES/NO
3. 사용자가 "뭘 말해야 하지?" 질문하게 만듦? → YES/NO

**결과:**
- 3개 모두 YES → 넣기
- 하나라도 NO → 나중

---

### Q: "이 기능 빠진 거 아니야?"

**체크:**
1. 말로 주변 장소 찾기가 작동 안 함? → YES/NO
2. 사용자가 "어떻게 하면 되죠?" 질문하게 만듦? → YES/NO
3. 기존 지도보다 불편함? → YES/NO

**결과:**
- 3개 모두 YES → 빠진 게 맞음
- 하나라도 NO → 나중

---

### Q: "이거 AI로 하면 더 멋질 텐데?"

**체크:**
1. 말로 주변 장소 찾기에 필수 아님? → YES/NO
2. AI 없어도 음성 검색 작동함? → YES/NO
3. AI 넣으면 복잡해짐? → YES/NO

**결과:**
- 3개 모두 YES → 나중 (Phase 2 이후)

---

### Q: "사용자가 이걸 원할 것 같은데?"

**체크:**
1. 말로 주변 장소 찾기에 직접 필요 없음? → YES/NO
2. 있으면 좋지만 없어도 됨? → YES/NO
3. 사용자 테스트에서 요청이 나왔음? → YES/NO

**결과:**
- 3개 모두 YES → Phase 1 이후
- 하나라도 NO → 지금은 NO

## ✅ MVP 완성 기준

### 기능
- [ ] 음성 버튼 클릭 → 인식 시작
- [ ] 음성 텍스트 → 카테고리 매핑
- [ ] 검색 결과 → 지도 핀 표시
- [ ] 가장 가까운 핀 강조
- [ ] 결과 카드 1개 표시
- [ ] "여기로 가기" 버튼 작동

### UX
- [ ] 키보드 없이 말만으로 검색 가능
- [ ] 결과가 바로 지도에 표시
- [ ] 3초 안에 결정 가능
