# 🔒 AI 분석 버튼 UX 최종 설계 (실패 포함)

**Baseline 고정**: 현재 `MarketAddPage.tsx` 구조는 절대 변경하지 않음  
**핵심 원칙**: AI는 보조 기능, 실패해도 상품 등록은 반드시 성공

---

## 1️⃣ AI 버튼 위치 및 시각적 계층

### 위치 규칙 (절대 변경 금지)
- ✅ **CTA 이후**: "상품 등록" 버튼 아래
- ✅ **물리적 분리**: AI 섹션은 별도 영역 (border, 배경색으로 구분)
- ✅ **시각적 약화**: 메인 CTA보다 작고 덜 강조

### 시각적 계층 구조
```
[상품 등록 버튼] ← 메인 CTA (큰 버튼, 강조)
    ↓
[AI 분석 섹션] ← 보조 기능 (작은 영역, 약한 강조)
```

---

## 2️⃣ AI 버튼 활성화 조건

### 활성화 조건 (모두 만족해야 함)
1. ✅ 이미지가 선택되어 있음 (`imageFile !== null`)
2. ✅ 상품 등록 중이 아님 (`!saving`)
3. ✅ AI 분석 중이 아님 (`!analyzing`)

### 비활성화 상태
- 이미지 없음: "이미지를 먼저 선택해주세요" (회색, 클릭 불가)
- 분석 중: "AI 분석 중..." (로딩 표시)
- 저장 중: 모든 AI 기능 비활성화

---

## 3️⃣ AI 분석 실행 흐름

### 사용자 액션
1. 이미지 선택
2. (선택) "AI 분석하기" 버튼 클릭
3. AI 분석 결과 표시
4. (선택) 결과를 입력 필드에 자동 채우기
5. 상품 등록 진행 (AI 결과 없어도 가능)

### 핵심 원칙
- ❌ **자동 실행 금지**: 이미지 선택 시 자동으로 AI 실행 안 함
- ✅ **수동 실행만**: 사용자가 버튼을 명시적으로 클릭해야 함
- ✅ **결과는 선택적**: AI 결과를 사용하지 않아도 상품 등록 가능

---

## 4️⃣ AI 분석 결과 표시

### 결과 표시 위치
- AI 섹션 내부 (접을 수 있게)
- 결과가 없으면 섹션 접힘
- 결과가 있으면 섹션 펼침

### 결과 내용
```
AI 분석 결과
├─ 카테고리: [자동 채우기 버튼]
├─ 추천 가격: [자동 채우기 버튼]
├─ 태그: [자동 채우기 버튼]
└─ 설명: [자동 채우기 버튼]
```

### 자동 채우기 규칙
- ✅ **버튼 클릭 시에만**: 각 필드 옆에 "적용" 버튼
- ❌ **자동 채우기 금지**: AI 결과가 자동으로 입력 필드에 들어가지 않음
- ✅ **사용자 선택**: 사용자가 원하는 것만 선택적으로 적용

---

## 5️⃣ AI 실패 처리 (가장 중요)

### 실패 시나리오
1. 네트워크 에러
2. AI API 타임아웃
3. 이미지 파싱 실패
4. AI 서버 오류

### 실패 처리 규칙
- ✅ **Toast 메시지만**: 에러는 toast로만 표시
- ✅ **페이지 유지**: 페이지 크래시 없음
- ✅ **상품 등록 계속 가능**: AI 실패해도 저장 버튼 정상 작동
- ✅ **재시도 가능**: "다시 시도" 버튼 제공

### 실패 메시지 예시
```
❌ AI 분석에 실패했습니다. 수동으로 입력해주세요.
[다시 시도] [닫기]
```

## 8️⃣ 절대 금지 사항

### ❌ 하면 안 되는 것
1. **AI 자동 실행**: 이미지 선택 시 자동으로 AI 실행
2. **AI 필수화**: "AI 분석 먼저 하세요" 같은 메시지
3. **AI 실패 시 차단**: AI 실패하면 상품 등록 불가
4. **AI 결과 강제**: AI 결과를 자동으로 입력 필드에 채우기
5. **AI 섹션 상단 이동**: CTA 위로 AI 섹션 이동
6. **AI 버튼 강조**: 메인 CTA보다 크거나 눈에 띄게 만들기
