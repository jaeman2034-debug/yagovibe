# 🧭 운영자 대시보드 – 최종 UI 설계 (실서비스 기준)

## 🎯 대시보드의 역할

"공지 몇 개 있지?" ❌

**지금 운영 상태가 어떤지 한눈에 판단** ⭕

즉, **관리자가 '결정'을 내리기 위한 화면**

## 1️⃣ 대시보드 전체 구조 (Top → Down)

```
[ 상단 요약 KPI ]
[ 승인 대기 / 예약 / 만료 예정 ]
[ 공지 리스트 (상태별) ]
[ 통계 / 트렌드 ]
[ 감사 리포트 / 다운로드 ]
```

## 2️⃣ 상단 KPI 카드 (한눈에 상태 파악)

### 📊 KPI 카드 4종 (필수)

| 카드 | 의미 |
|------|------|
| 📢 게시중 공지 | 현재 사용자에게 보이는 공지 수 |
| ⏳ 승인 대기 | SuperAdmin 액션 필요한 건 |
| 📌 고정 공지 | 상단 고정된 공지 수 |
| ⏱ 만료 예정 | 7일 이내 만료 |

**UI 예시:**
```
[📢 게시중 12] [⏳ 승인대기 3] [📌 고정 2] [⏱ 만료예정 1]
```

👉 **이것만 봐도 오늘 할 일 보임**

## 3️⃣ "즉시 액션" 영역 (가장 중요)

### 🔥 승인 / 반려 / 고정 관리

**🔔 승인 대기 공지**
```
- 노원구 구정장기 축구대회 안내
  [승인] [반려]
```

- SuperAdmin만 노출
- 클릭 시 → Drawer 열림 (이미 만든 UX 재사용)

## 4️⃣ 공지 리스트 (운영 중심 설계)

### 📋 기본 필터 (상단)

- **상태**: 전체 / draft / pending / published / scheduled / expired
- **고정**: 전체 / pinned만
- **기간**: 최근 7일 / 30일 / 전체

### 📄 리스트 컬럼 (실무 최적)

| 제목 | 상태 | 고정 | 게시일 | 만료 | 조회 | 액션 |
|------|------|------|--------|------|------|------|
| 노원구 구정장기… | 게시중 | 📌 | 2/5 | 6/30 | 324 | 수정 |

**상태 뱃지 색상:**
- 🟢 published
- 🟡 pending
- 🔵 scheduled
- ⚪ expired
- ⚫ draft

## 5️⃣ 공지 통계 요약 (9번 활용)

### 📈 미니 차트 (선택)

- 최근 7일 조회수
- pinned vs 일반 공지 비교
- 조회수 TOP 3
  ```
  1. 노원구 구정장기 축구대회 (324)
  2. ...
  ```

👉 **"이 공지 잘 나갔나?" 바로 판단**

## 6️⃣ 만료 관리 섹션 (운영 핵심)

### ⏱ 곧 만료되는 공지

```
- 노원구 구정장기 축구대회 안내
  만료: 6월 30일
  [연장] [그대로 만료]
```

- 연장 → expiresAt 수정
- 히스토리 자동 기록

## 7️⃣ 감사 / 리포트 영역 (10번 연계)

### 📥 버튼 2개면 충분

- 📊 공지 운영 리포트 다운로드 (엑셀)
- 🧾 히스토리 전체 보기

- SuperAdmin 전용

## 8️⃣ 권한별 화면 차이 (중요)

### 👤 Admin

- draft / pending / rejected만 관리
- 승인 / 고정 버튼 ❌
- 통계 조회 ⭕
- 엑셀 다운로드 ❌

### 👑 SuperAdmin

- 전체 기능 ⭕
- 승인 / 반려 / 고정 ⭕
- 엑셀 ⭕
- 만료 강제 처리 ⭕ (권장)

## 9️⃣ 기술적 구성 제안 (현실적인)

### 📦 페이지 구성

- `/admin/dashboard` – 메인
- `/admin/notices` – 리스트
- `/admin/notices/:id` – 상세 + 히스토리
- `/admin/reports` – 엑셀 / 감사

### 🧩 재사용 컴포넌트

- `NoticeStatusBadge`
- `NoticeKPI`
- `NoticeActionDrawer` (이미 만든 Drawer 재사용)
- `NoticeHistoryTimeline`

## 🔟 이 대시보드의 진짜 가치

이제 관리자는:

❌ "왜 안 됨?"
❌ "이거 누가 했음?"
❌ "언제 내려감?"

**이걸 묻지 않아도 됨**

👉 **화면이 답을 줌**

## 🧠 최종 한 줄 요약

운영자 대시보드는 '기능 모음'이 아니라
**'운영자의 판단을 대신해주는 화면'**이다.

지금 설계는 그 기준을 충족한다.
