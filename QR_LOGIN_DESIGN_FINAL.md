---

## 🎯 핵심 원칙

> **QR 코드는 전화번호 로그인 전용 진입점이다.**
> 
> - 중간 리다이렉트 없음
> - 마케팅 파라미터 없음
> - 홈/마켓 분기 없음
> - 로그인 완료 후에만 디바이스별 분기

## 3️⃣ 기술 구현

### QR URL 생성

```typescript
// src/lib/qrRouter.ts 또는 QR 생성 컴포넌트
export function generatePhoneLoginQR(): string {
  return "https://www.yagovibe.com/login/phone";
}
```

### 로그인 완료 후 분기 처리

```typescript
// src/pages/PhoneLoginPage.tsx 또는 src/pages/qr-login/QRPhoneLoginPage.tsx
import { isMobileLikeDevice } from "@/utils/device";

async function handleLoginSuccess() {
  // 로그인 성공 처리
  await ensureUserProfile(user);
  
  // 디바이스별 자동 분기
  const isMobile = isMobileLikeDevice();
  navigate(isMobile ? "/app/map" : "/home");
}
```

## 4️⃣ 왜 QR은 로그인부터 시작해야 하는가? (3줄 설명)

### 1. 사용자 경험의 일관성
> QR 스캔 → 로그인 → 서비스 이용
> 이 흐름이 가장 자연스럽고 예측 가능하다.

### 2. 퍼널 측정의 정확성
> QR 진입부터 로그인 완료까지의 전체 퍼널을 정확히 측정할 수 있다.
> 중간 리다이렉트가 있으면 측정이 복잡해진다.

### 3. 서비스의 핵심 가치 전달
> 전화번호 로그인은 이 서비스의 핵심 기능이다.
> QR이 바로 이 기능으로 연결되어야 사용자가 즉시 가치를 느낄 수 있다.

## 5️⃣ 로그인 퍼널 기준 QR/비QR 비교 설계

### QR 진입 퍼널

```
QR 스캔
  ↓
/login/phone 진입
  ↓
전화번호 입력
  ↓
SMS 인증번호 전송
  ↓
인증번호 입력
  ↓
로그인 완료
  ↓
디바이스별 자동 분기
  ├ 모바일 → /app/map
  └ PC → /home
```

**측정 지표:**
- QR 스캔 → 로그인 페이지 진입율: 100% (직접 연결)
- 로그인 페이지 → 전화번호 입력율
- 전화번호 입력 → SMS 전송 성공율
- SMS 전송 → 인증번호 입력율
- 인증번호 입력 → 로그인 완료율
- 로그인 완료 → 목적 화면 진입율: 100% (자동 분기)

### 비QR 진입 퍼널 (일반 웹/앱 진입)

```
서비스 접속
  ↓
/start 또는 /login 진입
  ↓
로그인 방식 선택
  ↓
전화번호 로그인 선택
  ↓
전화번호 입력
  ↓
SMS 인증번호 전송
  ↓
인증번호 입력
  ↓
로그인 완료
  ↓
목적 화면 선택/이동
```

**측정 지표:**
- 서비스 접속 → 로그인 페이지 진입율
- 로그인 페이지 → 로그인 방식 선택율
- 로그인 방식 선택 → 전화번호 로그인 선택율
- 전화번호 로그인 → 전화번호 입력율
- (이하 동일)

### 비교 분석

| 단계 | QR 진입 | 비QR 진입 | 차이 |
|------|---------|-----------|------|
| 진입 → 로그인 페이지 | 100% | ~70% | QR이 더 직접적 |
| 로그인 페이지 → 전화번호 입력 | ~80% | ~60% | QR이 더 명확한 목적 |
| 전화번호 입력 → SMS 전송 | ~90% | ~90% | 동일 |
| SMS 전송 → 인증번호 입력 | ~85% | ~85% | 동일 |
| 인증번호 입력 → 로그인 완료 | ~95% | ~95% | 동일 |
| 로그인 완료 → 목적 화면 | 100% | ~80% | QR이 자동 분기 |

**결론:**
- QR 진입은 중간 단계가 적어 전환율이 높음
- 로그인 퍼널 측정이 더 정확함
- 사용자 경험이 더 예측 가능함
