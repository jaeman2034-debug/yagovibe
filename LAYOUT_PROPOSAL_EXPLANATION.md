# 📐 레이아웃 제안 해석

## 🎯 제안 요약

### "MainLayout은 절대 중앙정렬하지 말고 w-full 유지"
```
❌ 현재: MainLayout에 max-w-7xl mx-auto (1280px 최대, 중앙 정렬)
✅ 제안: MainLayout에 w-full (전체 폭)
```

### "각 페이지 내부 섹션 단위로 max-w-4xl mx-auto 적용"
```
❌ 현재: 페이지 전체가 풀폭 (main 내부)
✅ 제안: 페이지 내부 섹션마다 max-w-4xl mx-auto
```

## 🔍 차이점 분석

### 현재
- MainLayout: **1280px 최대** 중앙 정렬
- 페이지 내용: **MainLayout 내부 풀폭** (실질적으로 1280px)
- 장점: 일관된 최대 너비, 깔끔한 대시보드
- 단점: 화면이 넓으면 좌우 여백 큼

### 제안
- MainLayout: **전체 화면 폭** (100%)
- 페이지 섹션: **896px(4xl) 중앙**으로 제한
- 장점: 헤더/푸터 풀폭, 섹션 가독성
- 단점: 대시보드 느낌 약함, 섹션 간 연결 약함

## 🎯 영향 분석

### 3열 위젯 그리드

#### 현재 (1280px 기준)
```
각 위젯: 약 427px 폭
여백: 16px (gap-4)
```

#### 제안 (896px 기준)
```
각 위젯: 약 299px 폭 (축소됨!)
여백: 16px (gap-4)
```

**위젯 크기 약 30% 감소**

---

## ⚠️ 문제점

1. **3열 그리드 축소**
   - 대시보드 느낌 약화
   - 정보 밀도 감소

2. **일관성 저하**
   - 섹션마다 개별 제한
   - 통합 디자인 어려움

3. **대시보드 성격 약화**
   - 대시보드는 넓은 폭 선호
   - 제안은 카드형 페이지에 적합

---

## ✅ 권장사항

**제안 반영하지 않음 (강력 권장)**

이유:
1. 현재 구조가 대시보드에 적합
2. 3열 그리드 축소되면 가독성 저하
3. MainLayout의 max-w-7xl이 적절
4. 제안은 인증 페이지나 단순 페이지에 적합

---

**결론: 제안은 좁은 카드형 페이지용. 대시보드는 현재 구조가 더 적합합니다.**
