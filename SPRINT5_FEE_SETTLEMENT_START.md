# Sprint 5: 회비/정산 실전 엔진 구현 시작

## ✅ 완료된 작업

### 1. 데이터 타입 정의
- `FeeType`: "annual" | "monthly" | "temporary" | "donation"
- `MembershipFee`: 회비 정보
- `Expense`: 지출(영수증) 정보
- `SettlementSummary`: 정산 요약

### 2. 컴포넌트 생성
- `MemberFeeStatus` - 회원 회비 상태 표시
- `useMemberFees` - 회비 조회 Hook

### 3. 페이지 생성
- `FeeManagementPage` - Admin 회비 관리 페이지
  - 회원별 회비 리스트
  - 필터: 미납 / 납부
  - 납부 처리 토글

## 📋 다음 단계

1. 정산 입력/관리 페이지
2. 정산 화면 (자동 합산)
3. 정산 보고서 출력 기능
4. 회원 화면에 회비 상태 통합
5. 대회 참가 조건에 회비 미납 체크 추가

## 🎯 핵심 원칙

- 회비는 "대회 참가 의사 전"까지 납부
- 기준·금액 설명은 사무국 화면에만 표시
- "혹시 회비 냈나요?" 전화 자동 제거
- 엑셀 ❌ / 메신저 사진 ❌
- 실시간 계산, 수정 즉시 반영
