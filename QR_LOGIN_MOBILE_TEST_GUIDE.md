# 📱 QR 로그인 모바일 테스트 가이드

## 🎯 테스트 목적

QR 코드 스캔 후 모바일에서 `/qr-login` 페이지가 정상적으로 열리는지 확인

---

## ✅ 권장 테스트 방법

### 방법 A: 기본 카메라 앱 사용 (가장 정확)

1. **기본 카메라 앱**으로 QR 코드 스캔
2. 상단에 뜨는 **"브라우저에서 열기"** 선택
3. Chrome / Safari로 열림 확인
4. `/qr-login?sessionId=xxx` 페이지가 표시되는지 확인

**예상 결과:**
- ✅ 전화번호 입력 화면 표시
- ✅ 에러 없음

---

### 방법 B: QR URL 직접 입력 (강제 분기 테스트)

1. PC에서 QR 코드 생성 후 개발자 도구로 URL 확인
2. QR 코드 URL을 **길게 눌러 복사** (또는 수동으로 복사)
3. 모바일 Chrome 주소창에 **직접 붙여넣기**
   ```
   https://yago-vibe-spt.web.app/qr-login?sessionId=xxx
   ```
4. 전화번호 입력 화면이 뜨는지 확인

**예상 결과:**
- ✅ 전화번호 입력 화면 표시
- ✅ 에러 없음

**이 방법에서 화면이 뜨면:**
- 👉 **QR 자체는 문제 없음**
- 👉 **스캐너 앱 문제 확정**

---

## ⚠️ 문제가 나는 경우

### 인앱 브라우저에서 스캔한 경우

다음 앱들의 QR 스캐너를 사용하면 문제가 발생할 수 있습니다:

- ❌ 카카오톡 QR 스캐너
- ❌ 네이버 앱 QR
- ❌ 인스타그램/페이스북 인앱 브라우저
- ❌ 삼성 인터넷 "QR 빠른 스캔"
- ❌ 라인 앱

**증상:**
- URL을 **외부 브라우저로 안 열고**
- **인앱 WebView**로 열어버림
- PWA / Firebase Auth / reCAPTCHA → **정상 분기 안 됨**

**해결:**
- 인앱 브라우저 감지 로직 추가 완료 ✅
- 사용자에게 외부 브라우저 사용 안내 표시

---

## 🔍 PWA 설치 상태 체크

만약 모바일에 **PWA로 설치된 상태**라면:

- QR 스캔 → PWA로 열릴 수 있음
- 이 경우 **reCAPTCHA / SMS 인증 막히는 케이스 존재**

**테스트 방법:**
- **PWA 삭제 후**
- 일반 브라우저에서 테스트 1회

### 체크리스트

#### PC에서 확인
- [ ] QR 코드 생성 후 개발자 도구에서 `value` 속성 확인
- [ ] URL이 `/qr-login?sessionId=xxx` 형식인지 확인
- [ ] `window.location.origin`이 올바른지 확인

#### 모바일에서 확인
- [ ] 기본 카메라 앱으로 QR 스캔 → 브라우저에서 열기
- [ ] QR URL 직접 모바일 브라우저 입력
- [ ] `/qr-login?sessionId=xxx`로 이동하는지 확인
- [ ] 전화번호 입력 화면이 표시되는지 확인
- [ ] 인앱 브라우저 안내 메시지 표시 확인 (인앱 브라우저 사용 시)

## 🎯 최종 판단 기준

### ✅ 정상 동작

1. 기본 카메라 앱 → 브라우저에서 열기 → 전화번호 화면 표시
2. QR URL 직접 입력 → 전화번호 화면 표시

### ⚠️ 부분 동작

1. 인앱 브라우저에서 스캔 → 안내 메시지 표시
2. PWA에서 스캔 → 안내 메시지 표시

### ❌ 문제 발생

1. 기본 카메라 앱 → 브라우저에서 열기 → 에러 또는 빈 화면
2. QR URL 직접 입력 → 에러 또는 빈 화면
