# 협회 확장 체크리스트 (도봉/강북 구 복제)

## 🎯 확장 목표

**노원구 축구협회에서 검증된 구조를 다른 구(도봉/강북)로 복제**

**원칙:** 설정만 변경, 코드 수정 없음

**검증 완료:** 협회 추가 시 코드 수정 불필요 ✅

---

## 📋 사전 준비 체크리스트

### 1. 기본 정보 수집
- [ ] 협회명 확인 (예: "도봉구 축구협회")
- [ ] 지역명 확인 (예: "서울시 도봉구")
- [ ] 협회 ID 생성 규칙 확인 (예: "assoc-dobong-football")
- [ ] 관리자 이메일 목록 수집

### 2. 시설 정보 수집
- [ ] 우선 대관 시설 목록 (3~5곳 권장)
  - 시설명
  - 위치
  - 접근 정책 (ASSOCIATION_PRIORITY)
- [ ] 일반 시설 목록 (선택사항)
  - 시설명
  - 위치
  - 접근 정책 (PUBLIC_OPEN)
- [ ] 시설별 운영 시간대 확인 (선택사항)

### 3. 정책 확인
- [ ] 회원비 금액 확인 (기본: 240만원/년)
- [ ] 아카데미 발전기금 금액 확인 (100~200만원)
- [ ] 대관 우선권 정책 확인 (기본: D-7 vs D-3)
- [ ] 특별 규정 여부 확인 (예: 지역 외 팀 제한 등)

---

## 🔧 시스템 설정 (5분)

### 1. 협회 생성 (Association Create Wizard)

**[화면: /admin/association/create]**

#### Step 1: 기본 정보
- [ ] 협회명 입력 (예: "도봉구 축구협회")
- [ ] 지역 선택 (예: "서울시 도봉구")
- [ ] 협회 ID 자동 생성 확인

#### Step 2: 템플릿 선택
- [ ] "노원구형 (표준)" 선택
- [ ] 기본 정책 확인
  - 회원: D-7일 예약
  - 비회원: D-3일 예약
  - 아카데미: 협회 배정

#### Step 3: 시설 등록
- [ ] 우선 대관 시설 추가 (최소 3개)
  - 시설명 입력
  - 위치 입력
  - 접근 정책: "협회 우선 대관" 선택
- [ ] 일반 시설 추가 (선택사항)
  - 접근 정책: "일반 공공" 선택

#### Step 4: 관리자 이메일
- [ ] 회장 이메일 입력
- [ ] 사무국 이메일 입력 (선택사항)
- [ ] 총무 이메일 입력 (선택사항)

#### Step 5: 생성 완료
- [ ] 협회 생성 확인
- [ ] 협회 홈 URL 확인 (예: `/association/assoc-dobong-football`)
- [ ] 관리자 계정 생성 안내

---

### 2. 관리자 계정 생성

- [ ] 협회 관리자 이메일로 계정 초대
- [ ] 관리자 권한 부여 확인
- [ ] 협회 홈 접근 테스트

---

## ✅ 검증 체크리스트 (10분)

### 1. 협회 홈 대시보드
- [ ] 협회명 표시 확인
- [ ] 운영 지표 카드 4개 표시
  - 이번 달 대관 요청
  - 우선 배정 사용률
  - 비회원 대기
  - 회원 전환 유도
- [ ] 리포트 설정 메뉴 접근 확인

### 2. 회원 관리 기능
- [ ] 회원 신청 목록 페이지 접근 (`/association/{id}/membership-approval`)
- [ ] 승인/거절 기능 테스트
- [ ] 승인 후 팀 상태 변경 확인

### 3. 대관 예약 시스템
- [ ] 시설 목록 표시 확인
- [ ] 예약 캘린더 접근 확인
- [ ] 우선권 정책 적용 확인
  - 회원 팀: D-7일부터 예약 가능
  - 비회원 팀: D-3일부터 예약 가능

### 4. 리포트 시스템
- [ ] 월간 리포트 PDF 생성 테스트
- [ ] 리포트 이메일 발송 테스트
- [ ] 리포트 수신자 설정 확인

---

## 🧪 시범 운영 체크리스트 (1주일)

### 1. 테스트 팀 등록
- [ ] 테스트 팀 2~3개 생성
  - 회원 팀 1개
  - 비회원 팀 1개
  - 아카데미 1개 (선택사항)
- [ ] 팀 상태별 권한 확인

### 2. 대관 예약 테스트
- [ ] 회원 팀 예약 테스트 (D-7일)
- [ ] 비회원 팀 예약 테스트 (D-3일)
- [ ] 충돌 상황 테스트 (동시 예약 시도)
- [ ] 예약 취소 테스트 (선택사항)

### 3. 회원 승인 테스트
- [ ] 회원 신청 테스트
- [ ] 승인 처리 테스트
- [ ] 승인 후 권한 변경 확인

### 4. 리포트 테스트
- [ ] 월간 리포트 생성 테스트
- [ ] 리포트 이메일 수신 확인
- [ ] 리포트 내용 검증

---

## 📝 커스터마이징 체크리스트 (필요 시)

### 정책 변경
- [ ] 회원 예약 기간 변경 (예: D-7 → D-10)
- [ ] 비회원 예약 기간 변경 (예: D-3 → D-5)
- [ ] 아카데미 정책 변경

### 시설 관리
- [ ] 시설 추가/삭제
- [ ] 시설별 운영 시간 설정
- [ ] 시설별 정책 설정

### 리포트 커스터마이징
- [ ] 리포트 수신자 변경
- [ ] 리포트 발송 주기 변경 (월간 → 주간)
- [ ] 리포트 내용 커스터마이징 (추가 지표)

---

## 🚀 전면 도입 체크리스트 (2주)

### 1. 기존 팀 등록
- [ ] 기존 회원 팀 목록 수집
- [ ] 일괄 등록 스크립트 실행 (또는 수동 등록)
- [ ] 팀 상태 확인 (MEMBER)

### 2. 사용자 교육
- [ ] 팀장 대상 온보딩 자료 제공
- [ ] 온라인 가이드 영상 제작 (선택사항)
- [ ] FAQ 문서 제공

### 3. 공지 및 안내
- [ ] 협회 공지 발송
  - 시스템 도입 안내
  - 사용자 가이드 링크
  - 문의 연락처
- [ ] 공식 홈페이지/카페 공지 (선택사항)

### 4. 정식 운영 시작
- [ ] 협회 홈 대시보드 공개
- [ ] 대관 예약 시스템 활성화
- [ ] 회원 신청 프로세스 활성화
- [ ] 월간 리포트 자동 발송 시작

---

## 🔍 문제 해결 체크리스트

### 일반적인 문제
- [ ] 시설 목록이 보이지 않음
  - 해결: 시설 등록 확인, 접근 정책 확인
- [ ] 회원 승인이 안 됨
  - 해결: 관리자 권한 확인, 팀 상태 확인
- [ ] 리포트 이메일 미수신
  - 해결: 리포트 설정 확인, 이메일 주소 확인, 스팸함 확인

### 정책 관련 문제
- [ ] 우선권 정책이 적용되지 않음
  - 해결: Policy Resolver 로그 확인, 팀 상태 확인
- [ ] 예약 날짜 제한이 잘못됨
  - 해결: 슬롯 정책 설정 확인, 날짜 계산 로직 확인

## 📊 확장 성공 기준

다음 중 3개 이상 달성 시 성공:

1. ✅ 협회 생성 완료 (5분 내)
2. ✅ 시설 등록 완료 (10개 이내)
3. ✅ 관리자 계정 생성 및 접근 확인
4. ✅ 테스트 팀 등록 및 승인 완료
5. ✅ 대관 예약 테스트 성공
6. ✅ 리포트 생성 및 발송 확인
7. ✅ 기존 회원 팀 80% 이상 등록 완료
8. ✅ 정식 운영 시작 (1개월 내)

- [ ] 노원구형 템플릿 선택
- [ ] 시설 정보만 변경
- [ ] 관리자 이메일만 변경
- **예상 시간:** 10분

### 시나리오 B: 정책 커스터마이징 (중간 시간)
- [ ] 기본 템플릿 선택
- [ ] 정책 설정 변경 (예약 기간 등)
- [ ] 시설 정보 입력
- [ ] 리포트 수신자 설정
- **예상 시간:** 30분

### 시나리오 C: 완전 커스터마이징 (최대 시간)
- [ ] 정책 설정 커스터마이징
- [ ] 시설별 정책 설정
- [ ] 리포트 커스터마이징
- [ ] 추가 기능 개발 (필요 시)
- **예상 시간:** 2시간 ~ 1일

## 🔑 핵심 원칙

1. **설정만 변경, 코드 수정 없음**
2. **템플릿 기반 빠른 복제**
3. **최소 커스터마이징 (표준 정책 권장)**
4. **단계별 검증 (설정 → 테스트 → 운영)**

---

## 💡 확장 팁

1. **표준 정책 유지**: 커스터마이징 최소화로 유지보수 부담 감소
2. **시설 정보 정확히**: 나중에 수정보다 처음부터 정확히
3. **관리자 교육 중요**: 시스템 이해도가 높을수록 운영 효율 상승
4. **단계별 도입**: 한 번에 전면 도입보다 단계적 도입 권장
