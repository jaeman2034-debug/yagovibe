# 🧠 Y 단계 — 채팅 기반 매칭 MVP (1-Page Pitch)

> **투자·팀 공유·의사결정에 바로 쓰는 ONE-PAGER**

**작성일**: 2024년  
**버전**: 1.0.0

---

## 1️⃣ 문제 (Why)

**대화는 많은데 결정이 남지 않는다.**

멘토링/컨설팅/협업에서:
- 합의가 구두로 끝나고
- 책임·신뢰·이력이 남지 않는다.

**결과:**
- 신뢰 구축 어려움
- 재협업 판단 불가
- 분쟁 시 증거 없음

---

## 2️⃣ 해결 (What)

**채팅 안에서 '제안 → 확정 → 평가'가 남는 구조**

- 대화 중 세션 제안 카드
- 멘토가 세션 확정
- 완료 후 선택적 평점

**핵심:**
- 채팅은 유지하고, 결정만 구조화한다.

---

## 3️⃣ 제품 핵심 (How)

### 도메인 구조

```
Chat: 관계
Message(세션 제안): 맥락
Deal(세션 확정): 결정
Review(평점): 신뢰
```

### 설계 원칙

- 결정은 항상 **국소적(카드 내부)**
- 플랫폼은 중재하지 않고 기록만 남긴다
- 사용자는 스스로 보호할 수 있다

---

## 4️⃣ MVP 범위

### KEEP

- ✅ 채팅 (텍스트/이미지)
- ✅ 세션 제안/확정/취소 (확정 전)
- ✅ 평점 (텍스트 없음)
- ✅ 차단/신고

### CUT

- ❌ 결제/환불
- ❌ 캘린더 연동
- ❌ 분쟁 버튼
- ❌ 자동 제재

---

## 5️⃣ 차별점

### 채팅을 닫지 않는다 → 관계 유지

**다른 서비스:**
- 거래 완료 후 채팅 종료
- 관계 단절

**우리:**
- 세션 완료 후에도 채팅 유지
- 새 세션 제안 가능
- 관계 지속

---

### 거래/세션을 분리 → 확장 용이

**다른 서비스:**
- 채팅과 거래 혼재
- 확장 어려움

**우리:**
- Chat (관계) ≠ Deal (결정)
- 한 채팅에서 여러 세션 가능
- 확장 용이

---

### 평점은 힌트 → 악용 최소화

**다른 서비스:**
- 단순 평점만 표시
- 악용 가능

**우리:**
- 평점 + 거래 수 함께 표시
- 3건 미만 숨김
- 악용 최소화

---

## 6️⃣ 초기 지표 (30일)

### 세션 확정률 ≥ 15%

**정의:**
```
확정률 = confirmed / proposed
```

**목표:** 15% 이상

---

### 평점 작성률 ≥ 30%

**정의:**
```
작성률 = review.created / sessionDeal.completed
```

**목표:** 30% 이상

---

### 7일 재방문 ≥ 20%

**정의:**
```
재방문율 = 첫 세션 후 7일 내 재방문 / 첫 세션 사용자
```

**목표:** 20% 이상

---

## 7️⃣ 수익 연결 (보류 가능)

### 1. 거래/세션 성사 후불 수수료 (표시부터)

**구조:**
- 세션 확정 후 수수료 안내
- 초기에는 표시만 (실결제 X)
- 전환율 관찰 후 결제 연결

**가격:** 세션 금액의 3~5%

---

### 2. 제안 카드 강조 옵션

**구조:**
- 세션 제안 카드에 강조 옵션
- 채팅 맥락 유지

**가격:** 강조 1회 1,000원

---

### 3. 신뢰 배지 구독

**구조:**
- 일정 조건 충족 시 신뢰 배지
- 구독형

**가격:** 월 5,000원
