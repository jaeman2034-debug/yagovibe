# 협회 공식 페이지 「산하 팀」 섹션 UX 구조 설계

**목적**: 협회 산하 팀을 읽기 전용으로 표시하여 관계를 공식적으로 드러내는 UX 구조

---

## 1️⃣ 섹션 위치 및 역할

### 위치
- **AssociationOfficialPage 6개 섹션 중 마지막 섹션**
- 현재 `ClubSummarySection` → `OfficialTeamsSection`으로 역할 명확화
- 섹션 ID: `#teams` (Hero의 Quick Links와 연결 가능)

### 역할
- ❌ 팀 관리/수정/승인 기능 없음
- ❌ 협회가 팀을 소유하는 개념 없음
- ⭕ **"협회와의 관계를 공식적으로 표시"**만 수행
- ⭕ 팀 페이지로의 링크 제공 (읽기 전용)

---

## 2️⃣ 데이터 기준 (개념)

### 표시 대상 팀 조건
```
associationRelation: {
  associationId: 'assoc-nowon-football',
  status: 'official' | 'related' | 'independent'
}
```

### 상태별 표시 규칙
- **OFFICIAL (공식 산하)**: ✅ 표시 (최상단)
- **RELATED (관계 있음)**: ✅ 표시 (중간)
- **INDEPENDENT (독립)**: ❌ 표시 안 함 (허브에만 일반 팀으로 존재)

---

## 3️⃣ UI 구조 (읽기 전용)

### 섹션 헤더
```
┌─────────────────────────────────────┐
│ 산하 팀 (Associated Teams)          │
│                                     │
│ 노원구 축구협회와 함께하는 팀들      │
└─────────────────────────────────────┘
```

### 팀 카드 구조 (각 카드)

```
┌─────────────────────────────────────┐
│ [팀 이름]                           │
│                                     │
│ 🟢 노원구 축구협회 산하              │  ← 상태 뱃지
│                                     │
│ [팀 정보 요약]                      │
│ • 멤버 수: XX명                      │
│ • 활동 지역: 노원구                  │

│ [팀 페이지 보기 →]                   │  ← 링크만 (클릭 → /teams/{teamId})
```

### 상태 뱃지 규칙

| 상태 | 뱃지 색상 | 뱃지 텍스트 |
|------|----------|------------|
| OFFICIAL | 🟢 초록 | 노원구 축구협회 산하 |
| RELATED | 🟡 노랑 | 노원구 축구협회 연계 |

## 4️⃣ 정렬 규칙

### 우선순위 (자동 정렬)
1. **OFFICIAL (공식 산하)** → 최상단
2. **RELATED (관계 있음)** → 하단

### 동일 상태 내 정렬
- 팀 이름 가나다순 (한글 우선)

---

## 5️⃣ 표시 제한

### 기본 노출 개수
- **최대 6개** (OFFICIAL + RELATED 합산)
- 더보기 클릭 시 → 전체 리스트 (같은 페이지 내 스크롤 확장)

### 빈 상태 처리
```
┌─────────────────────────────────────┐
│ 산하 팀                              │
│                                     │
│ 등록된 산하 팀이 없습니다.            │
└─────────────────────────────────────┘
```

---

## 6️⃣ 링크 동작

### 팀 카드 클릭
- **동작**: `/teams/{teamId}` 또는 `/sports/football/team/{teamId}`로 이동
- **권한**: Public 접근 가능 (팀 페이지는 팀 자체 권한 정책 따름)

### 새 창/탭 열기
- ❌ 새 창 열기 금지 (UX 일관성 유지)
- ⭕ 현재 창에서 이동

---

## 7️⃣ 관리자 Edit Mode 규칙

### Edit Mode (`?mode=admin`)에서
- **문구 변경**: ❌ (섹션 자체는 읽기 전용)
- **팀 추가/삭제**: ❌ (팀-협회 관계는 Phase 5 승인 시스템에서 처리)
- **팀 정보 수정**: ❌ (팀 페이지에서 처리)

### Edit Mode 표시
- ✏️ 아이콘 없음 (섹션 자체가 읽기 전용이므로)
- 헤더에 "관리자 편집 모드" 텍스트만 표시 (페이지 레벨)

## 🔟 데이터 모델 (참고용, 구현 아님)

### Team Document (개념)
```typescript
{
  id: string;
  name: string;
  associationRelation?: {
    associationId: 'assoc-nowon-football',
    status: 'official' | 'related',
    since?: Timestamp  // 관계 시작일
  }
}
```

### 쿼리 로직 (개념)
```
1. teams 컬렉션에서
2. associationRelation.associationId === 'assoc-nowon-football' AND
3. associationRelation.status IN ('official', 'related')
4. status별 정렬 (official → related)
5. 이름순 정렬
```

## ✅ 성공 기준

1. **공식성 증명**: "노원구 축구협회 산하 팀이 누구인가?" → 즉시 답변 가능
2. **분쟁 차단**: "우리 팀은 산하냐?" → 뱃지로 명확히 표시
3. **전화 문의 감소**: "산하 팀 목록이 어디에?" → 페이지에서 확인
4. **확장 가능성**: Phase 5 승인 시스템 추가 시 자연스럽게 연결
