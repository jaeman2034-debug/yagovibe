# 🎲 조 추첨 시스템 아키텍처 (구청/협회 제출용 1장 요약)

## 📋 개요

**비대면 조 추첨 시스템**은 사람의 개입 없이 시스템이 자동으로 조를 배정하고, 모든 과정을 로그로 기록하여 감사·민원·법무 대응을 완전히 자동화한 시스템입니다.

---

## 🎯 핵심 원칙

### 1. 시스템이 판결한다
- 사람이 조를 짜지 않음
- 랜덤 알고리즘이 공정하게 배정
- 시드 기반 재현 가능 (감사 시 재검증)

### 2. 로그가 모든 논쟁을 끝낸다
- 추첨 전·후 모든 과정 기록
- 입력 데이터 해시 저장 (사후 변경 불가 증명)
- 완전한 감사 로그 (PDF/CSV/JSON)

### 3. 단일 실행 원칙
- 추첨은 1회만 실행 가능
- 재실행 불가
- 수정 불가 (LOCKED 상태)

---

## 🏗️ 시스템 구조

### 1. 권한 설계

| 역할 | 조 추첨 실행 | 결과 조회 | 로그 다운로드 |
|------|-------------|----------|--------------|
| 사무국 관리자 | ✅ | ✅ | ✅ |
| 팀/일반 사용자 | ❌ | ✅ (공개 시) | ❌ |

### 2. 상태 다이어그램

```
조 추첨 예정
   ↓
   [검수 기간 종료]
   ↓
   [승인 팀 ≥ 2팀]
   ↓
   [조 추첨 실행] (1회만)
   ↓
조 추첨 완료
   ↓
LOCKED (불변)
```

### 3. 데이터 흐름

```
승인된 팀 목록
   ↓
랜덤 시드 생성 (SHA256)
   ↓
Fisher-Yates 셔플
   ↓
조별 균형 분배
   ↓
결과 저장 + 로그 기록
```

## 🔐 알고리즘 규칙

### 1. 랜덤성 보장
- **시드 생성**: `SHA256(대회ID + 실행타임스탬프 + 관리자ID)`
- **난수 생성기**: Seeded RNG (재현 가능)
- **로그 기록**: 시드 값 저장

### 2. 팀 수 ÷ 조 수 분배 규칙
- **나누어떨어질 때**: 각 조 동일 수 배정
- **홀수/나머지 발생 시**: 나머지는 앞 조부터 +1
  - 예: 14팀 / 4조 → 4·4·3·3
- **조 간 최대 편차 = 1**

### 3. 시드 규칙 (선택)
- **시드팀 없음 (기본)**: 완전 랜덤
- **시드팀 있음**: 시드팀을 각 조에 1팀씩 먼저 배치

### 4. 동일 클럽 분산 (선택)
- 가능한 한 서로 다른 조 배치 시도
- 불가능하면: "분산 불가" 로그 기록 후 진행

## 📊 감사 로그 포맷

### 필수 기록 항목

1. **로그 메타데이터**
   - 대회 ID / 대회명
   - 조 추첨 실행 일시 (KST)
   - 실행 관리자 ID / 이름
   - 실행 방식: SYSTEM_AUTO
   - 실행 횟수: 1

2. **입력 데이터 스냅샷**
   - 승인 팀 수
   - 승인 팀 목록 해시 (SHA-256)
   - 시드팀 적용 여부
   - 시드팀 목록 (있을 경우)

3. **랜덤성 증명**
   - 랜덤 시드 값
   - 난수 생성 방식: Seeded RNG
   - 시드 생성 규칙: SHA256(대회ID+실행시각+관리자ID)

4. **분배 규칙 기록**
   - 조 수
   - 조별 팀 수 분배 규칙
   - 동일 클럽 분산 규칙: 적용 / 미적용

5. **최종 결과**
   - 조 편성 결과 테이블
   - 결과 공개 방식: 즉시 / 예약

6. **무결성 잠금**
   - 재실행 가능 여부: 불가
   - 수정 가능 여부: 불가
   - 잠금 시각
   - 잠금 사유: "공식 기록 생성"

### 출력 포맷
- **PDF**: 구청/감사용 (완전한 증빙 자료)
- **CSV**: 내부 검증용
- **JSON**: 시스템 보관용

## 🛡️ 보안 및 무결성

### 1. 권한 체크
- Cloud Function에서 관리자 권한 검증
- Firestore Security Rules로 읽기 제어

### 2. 재실행 방지
- `drawExecuted` 플래그로 단일 실행 보장
- 이미 실행됨 시 즉시 차단

### 3. 데이터 무결성
- 입력 데이터 SHA-256 해시 저장
- 사후 변경 불가 증명

### 4. 로그 보존
- 모든 로그는 Firestore에 영구 보관
- PDF/CSV/JSON으로 다운로드 가능
