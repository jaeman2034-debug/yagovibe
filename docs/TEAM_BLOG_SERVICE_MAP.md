홈 → 블로그 → 가입 → 운영 → Pro까지의 전체 플로우를 한 눈에 보여주는 서비스 맵

## 📊 사용자 여정별 상세 플로우

### 1️⃣ 외부 사용자 (비로그인)

```
공개 블로그 페이지 방문
  ↓
히어로 영역 확인 (최근 활동)
  ↓
"함께 운동하기" 클릭
  ↓
로그인 페이지
  ↓
로그인 완료
  ↓
원래 블로그 페이지로 복귀 (?join=true)
  ↓
자동 가입 요청 생성
  ↓
"가입 요청이 전송되었습니다!"
  ↓
관리자 승인 대기
```

---

### 2️⃣ 관리자 (팀 소유자)

```

  ↓

  ↓
블로그 관리 페이지
  ↓
"✨ AI 블로그 생성하기" 클릭

  ↓

  ↓
Stripe 결제
  ↓
Pro 기능 활성화
  ↓
주간 자동 포스팅 시작
```

## 🎯 핵심 가치 제안 (한 줄씩)

### 외부 사용자에게
```
"살아있는 팀을 만나보세요"
→ 최근 활동 증거로 신뢰 구축
```

### 관리자에게
```
"운영은 AI에게 맡기고, 팀은 운동만 하세요"
→ 자동화로 시간 절약
```

### Pro 전환 시
```
"이미 효과를 본 뒤, 멈추지 않으려면 Pro"
→ 자연스러운 전환
```

## 📈 성공 지표 (KPI)

### 무료 플랜
- 블로그 생성률: 목표 30%
- 첫 글 생성률: 목표 80%
- 공개 페이지 방문: 목표 20명/주

### Pro 전환
- 전환율: 목표 15-20%
- 첫 글 생성 직후: 목표 20-25%
- 조회수 증가 시: 목표 10-20%
- 두 번째 글 시도: 목표 25-30%

### Pro 유지
- 해지율: 목표 10% 이하
- 평균 구독 기간: 목표 6개월 이상

## 🎨 UI/UX 핵심 원칙

### 1. 신뢰 우선
- 최근 활동 증거로 신뢰 구축
- 사진/기록으로 "살아있는 팀" 연출

### 2. 전환 최소화
- 클릭 2번으로 가입 의사 전달
- 로그인 → 가입 요청 자동 생성

### 3. 자동화 강조
- "AI가 관리합니다" 메시지
- 관리자 개입 최소화

### 4. 자연스러운 전환
- 강요 없음
- 이미 효과를 본 뒤 결제
