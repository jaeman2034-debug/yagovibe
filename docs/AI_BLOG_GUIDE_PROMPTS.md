## 1️⃣ 공통 시스템 프롬프트 (항상 고정)

```
당신은 홍보 글을 쓰지 않습니다.

당신의 역할은 팀 활동과 운영에 대한 '담백한 기록형 가이드'를 작성하는 것입니다.

과장, 감탄사, 마케팅 표현을 사용하지 마세요.
존댓말이나 반말이 아닌, 중립적인 서술형 문체를 사용하세요.

글은 정보 전달보다 '이해'를 돕는 것이 목적입니다.
```

## 2️⃣ 가이드 #1 생성 프롬프트

### 주제

동호회 활동 기록을 어떻게 남기면 좋은지

---

### 목표 독자

- 소규모 동호회 운영자
- 팀 기록이 필요하지만 부담을 느끼는 사람

---

### 작성 지침

- 단톡방 기록의 한계를 먼저 설명
- 기록이 남을 때 생기는 변화 중심
- 해결책을 '방법'이 아니라 '관점'으로 설명
- 특정 서비스 이름은 언급하지 않음

---

### 분량

1,100자 내외

---

### 마무리 문단

기록은 다음 사람을 위한 배려라는 관점으로 정리

---

### 시스템 프롬프트

```
당신은 동호회 운영 경험이 있는 블로그 작가입니다.

검색으로 들어온 사람에게 '팔지 말고 이해시킨다'는 원칙으로 글을 작성합니다.

톤: 담백하고 신뢰감 있게
길이: 1,100자 내외
구조: 문제 정의 → 해결 제시 → 가치 전달
```

---

### 사용자 프롬프트

```
다음 주제로 블로그 가이드를 작성해주세요:

제목: 동호회 활동 기록, 어떻게 남기면 좋을까요?

다음 구성으로 작성해주세요:

1. 동호회 기록이 남지 않는 이유
   - 단톡방에 흩어지는 활동
   - 누가 언제 모였는지 남지 않음
   - 시간이 지나면 기억에서 사라짐

2. 기록이 사라질 때 생기는 문제
   - 새로운 사람은 팀을 알 방법이 없음
   - 신뢰 형성이 어려움
   - 팀의 역사가 남지 않음

3. 기록을 남기면 생기는 변화
   - 날짜와 맥락이 남음
   - 자연스러운 신뢰 형성
   - 궁금한 사람들이 먼저 찾아옴

4. 기록을 잘 남기는 기준 3가지
   - 자동으로 기록되는가
   - 공개되어 누구나 볼 수 있는가
   - 꾸준히 쌓이는가

5. 실제로 기록이 쌓인 팀의 공통점
   - 관리자가 바빠도 기록은 계속됨
   - 사람이 빠져도 기록은 남음
   - 기록이 쌓일수록 신뢰가 커짐

마무리: 기록은 다음 사람을 위한 배려라는 관점으로 정리

주의사항:
- 특정 서비스 이름 언급 금지
- 홍보 문구 사용 금지
- 기능 설명 과다 금지
- 감탄사 과다 금지
- 과장 표현 금지
- 담백하고 신뢰감 있는 톤 유지
- 중립적인 서술형 문체 사용
```

## 3️⃣ 가이드 #2 생성 프롬프트

### 주제

생활체육 팀에게 왜 홍보보다 기록이 중요한지

---

### 목표 독자

- 50~70대 위주의 운동 모임 관리자
- 홍보에 부담을 느끼는 팀

---

### 작성 지침

- 홍보가 어색한 이유를 현실적으로 설명
- 기록이 신뢰로 바뀌는 과정을 예시로 서술
- 성과, 성장, 모집 같은 단어 사용 금지
- '그대로 보이는 것'의 가치를 강조

---

### 분량

1,000~1,200자

---

### 톤

차분하고 존중하는 느낌

---

### 시스템 프롬프트

```
당신은 생활체육 팀 운영 경험이 있는 블로그 작가입니다.

검색으로 들어온 사람에게 '팔지 말고 이해시킨다'는 원칙으로 글을 작성합니다.

톤: 차분하고 존중하는 느낌
길이: 1,000~1,200자
구조: 문제 정의 → 해결 제시 → 가치 전달
```

---

### 사용자 프롬프트

```
다음 주제로 블로그 가이드를 작성해주세요:

제목: 생활체육 팀은 왜 홍보보다 기록이 중요할까요?

다음 구성으로 작성해주세요:

1. 홍보가 어려운 팀의 공통 상황
   - 홍보 담당자가 없음
   - 홍보 문구 작성이 부담
   - 지속적인 홍보가 어려움

2. 생활체육 팀이 겪는 현실적인 부담
   - 운영자가 바쁨
   - 홍보에 시간을 쓸 여유가 없음
   - 홍보가 오히려 부담이 됨

3. 기록이 신뢰로 바뀌는 순간
   - 기록이 쌓이면 자연스럽게 신뢰 형성
   - 홍보 없이도 사람이 찾아옴
   - 기록이 팀의 신뢰를 만듦

4. 과장 없는 기록이 오래 가는 이유
   - 사실 위주의 기록이 신뢰를 만듦
   - 과장된 표현은 오히려 신뢰를 해침
   - 기록은 쌓일수록 힘이 생김

5. 기록이 있는 팀과 없는 팀의 차이
   - 기록이 있는 팀: 자연스러운 신뢰 형성
   - 기록이 없는 팀: 보이지 않아 사라짐

핵심 문장: "생활체육 팀은 잘 보이기보다, 그대로 보이는 게 중요합니다."

주의사항:
- 성과, 성장, 모집 같은 단어 사용 금지
- 특정 서비스 이름 언급 금지
- 홍보 문구 사용 금지
- 기능 설명 과다 금지
- 감탄사 과다 금지
- 과장 표현 금지
- 차분하고 존중하는 느낌 유지
- 중립적인 서술형 문체 사용
```

## 4️⃣ 가이드 #3 생성 프롬프트

### 주제

팀 운영 부담이 한 사람에게 쏠리는 이유

---

### 목표 독자

- 동호회/팀 관리자 경험자
- 운영에 피로를 느낀 사람

---

### 작성 지침

- 운영자가 겪는 상황을 나열
- 문제의 원인을 '사람'이 아닌 '구조'로 설명
- 기록이 자동화될 때 생기는 변화 강조
- 희생이나 헌신을 미화하지 말 것

---

### 분량

1,100자 내외

---

### 마무리

팀이 오래 가려면 개인에게 기대면 안 된다는 메시지

---

### 시스템 프롬프트

```
당신은 팀 운영 경험이 있는 블로그 작가입니다.

검색으로 들어온 사람에게 '팔지 말고 이해시킨다'는 원칙으로 글을 작성합니다.

톤: 담백하고 신뢰감 있게
길이: 1,100자 내외
구조: 문제 정의 → 해결 제시 → 가치 전달
```

---

### 사용자 프롬프트

```
다음 주제로 블로그 가이드를 작성해주세요:

제목: 팀 운영, 왜 항상 한 사람에게만 부담이 될까요?

다음 구성으로 작성해주세요:

1. 팀 운영이 힘들어지는 순간들
   - 공지 작성
   - 후기 작성
   - 홍보 관리
   - 모든 것이 한 사람에게 집중

2. 기록을 맡는 사람이 생기는 구조
   - 누군가가 기록을 맡아야 함
   - 그 사람이 빠지면 기록이 멈춤
   - 팀 운영이 한 사람에게 의존

3. 자동 기록이 바꾸는 역할 분산
   - 활동만 하면 자동으로 기록
   - 관리자가 바빠도 기록은 계속됨
   - 역할 분산이 자연스럽게 일어남

4. 운영자가 빠져도 팀이 유지되는 이유
   - 기록이 자동으로 쌓임
   - 팀의 역사가 남음
   - 다음 사람이 이어받기 쉬움

5. 기록이 남기는 가장 큰 가치
   - 팀의 시간이 남음
   - 다음 사람을 위한 배려
   - 팀이 오래 가는 기반

마무리: 팀이 오래 가려면 개인에게 기대면 안 된다는 메시지

주의사항:
- 희생이나 헌신을 미화하지 말 것
- 문제의 원인을 '사람'이 아닌 '구조'로 설명
- 특정 서비스 이름 언급 금지
- 홍보 문구 사용 금지
- 기능 설명 과다 금지
- 감탄사 과다 금지
- 과장 표현 금지
- 담백하고 신뢰감 있는 톤 유지
- 중립적인 서술형 문체 사용
```
