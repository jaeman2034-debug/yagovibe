# 🏁 공지-대회 연동 실전 사용 가이드 (최종 정리)

## ✅ 현재 상태 요약 (정상 동작 기준)

### 공지 내용이 자동으로 안 들어오는 이유

**지금은 직접 `/tournaments/new`로 진입한 상태**
- `fromNotice` 파라미터가 없으므로 빈 폼이 정상
- **의도된 설계**

**자동 주입은 공지에서 생성했을 때만**
- 운영 사고/덮어쓰기 방지 목적

---

## ✅ 실전 사용 "정답 루트"

### 1️⃣ 공지 상세 화면

**버튼**: `[이 공지로 대회 생성]`

**위치**: 공지 본문 하단

**표시 조건**: 관리자만, `published` 상태 공지만

### 2️⃣ 이동 URL

```
/tournaments/new?fromNotice=NOTICE_ID
```

### 3️⃣ 결과

- ✅ **제목** → 공지 제목 자동 입력
- ✅ **본문** → 공지 본문 자동 입력
- ✅ **참가비 정책** → `feePolicy` 자동 입력
- ✅ **장소** → 본문에서 자동 추출 (패턴 매칭)
- ✅ **기간** → 본문에서 자동 추출 (날짜 패턴)
- ✅ **배지** → "공지에서 생성된 대회" 표시

**👉 이 루트로 들어오면 빈 화면은 절대 나오지 않습니다.**

## ✅ 운영 안전장치 (이미 설계 반영됨)

### 공지 수정 시 → 대회 자동 변경 ❌

**이유**:
- 공지는 "안내"
- 대회는 "운영 기록"
- 자동 동기화하면 사고 발생

### 변경 제안 알림 → 관리자 선택 반영 ⭕

**원칙**:
- 공지 수정 시 연결된 대회에 변경 제안 알림
- 관리자가 승인해야만 반영
- 모든 변경은 사람이 승인

**향후 확장**: `docs/NOTICE_TOURNAMENT_INTEGRATION_FINAL.md` 참고

### 대회 문서에 sourceNoticeId 저장 → 감사/분쟁 추적 가능

**저장 위치**: `tournaments/{tid}.sourceNoticeId`

**용도**:
- 이 대회는 어떤 공지에서 생성되었는지 추적
- 감사/행정/분쟁 대응 시 계보 증명

## 📋 빠른 참조

### 정상 동작 확인 방법

**테스트 1: 공지에서 버튼 클릭**
1. 공지 상세 페이지로 이동
2. "[이 공지로 대회 생성]" 버튼 클릭
3. URL에 `?fromNotice=NOTICE_ID` 포함 확인
4. 대회 등록 페이지에서 제목/본문 자동 입력 확인

**테스트 2: 직접 URL 입력**
```
/association/{associationId}/admin/tournaments?fromNotice=실제공지ID
```
→ 제목/본문이 자동으로 채워지면 연결 정상

### 문제 해결

**Q: 공지 내용이 자동으로 안 들어와요**
- A: URL에 `?fromNotice=NOTICE_ID`가 있는지 확인
- A: 공지 상세 페이지에서 버튼을 통해 진입했는지 확인

**Q: 빈 폼이 보여요**
- A: 정상 동작입니다. 직접 `/tournaments/new`로 진입한 경우
- A: 공지 상세 페이지에서 버튼을 통해 진입하세요
