# 🎯 온보딩 마지막 화면 카피

## 0️⃣ 이 화면의 역할

**시작을 알리는 화면이 아니라, 끝났다는 걸 알려주는 화면**

- ❌ 축하 과다
- ❌ 다음 액션 강요
- ⭕ 안심 + 종료

---

## 📋 목적
가입 후 "아, 끝났네"라는 느낌을 주는 문구

**사용자가 "이제 진짜 할 일 없네"라고 느끼게 만드는 엔딩**

---

## 🎯 원칙

### UX 목표

- 안심감
- 완료감
- 다음 행동 명확
- 부담 없음

---

### 톤 가이드

- 담백함
- 과장 없음
- 신뢰감
- 편안함

---

## 1️⃣ 완료 화면 구조

### 화면 순서

1. 완료 메시지
2. 자동화 상태 요약
3. 다음 단계 안내
4. 마무리 문구

---

## 1️⃣ 메인 메시지 (중앙)

### 헤드라인

```
설정이 모두 끝났습니다
```

### 서브 문장

```
이제 팀 활동이 있으면
기록장이 자동으로 남겨줍니다.
```

---

## 2️⃣ 상태 요약 (짧게)

```
🟢 자동 기록: 켜짐

🟢 자동 공유: 켜짐

🗓 다음 기록 예상: 이번 주 중

별도로 하실 일은 없습니다.
```

---

## 3️⃣ 안심 문구 (중요)

```
필요할 때만 확인하세요.
기록은 알아서 쌓입니다.

언제든지 끄거나 다시 켤 수 있습니다.
```

**📌 컨트롤은 사용자에게 있다는 느낌**

---

## 4️⃣ 선택형 버튼 (1개만 강조)

### Primary

```
[ 팀 페이지 보러 가기 ]
```

### Secondary (작게)

```
[ 설정 다시 보기 ]
```

**❌ "글 쓰러 가기" ❌**
**❌ "공유하기" ❌**

---

## 5️⃣ 하단 마이크로 카피 (브랜드 각인)

```
기록장은 팀을 바꾸지 않습니다.
팀의 시간을 남길 뿐입니다.
```

---

## 6️⃣ 자동 전환 규칙 (UX)

- 5초 후 자동으로 팀 페이지 이동
- 사용자가 클릭하면 즉시 이동
- 뒤로 가기 ❌ (의도적 종료)

---

## 7️⃣ 개발용 컴포넌트 요약

```typescript
interface OnboardingComplete {
  statusSummary: AutomationSettings;
  primaryAction: "go_to_team_page";
  secondaryAction: "view_settings";
  autoRedirect: true;
}
```

---

## 8️⃣ 실패 신호 (이 화면에서 나오면 안 됨)

- "이제 뭘 해야 하지?"
- "이거 제대로 된 건가?"
- "설정 더 해야 하나?"

**👉 이런 생각 들면 카피 수정**

---

## 4️⃣ 다음 단계 안내 (하단)

### Free 사용자

```
지금은 무료로 사용 중입니다.

Pro로 전환하시면
더 많은 기능을 사용하실 수 있습니다.
```

**CTA:**
```
[ 지금은 이대로 사용하기 ]
[ Pro 알아보기 ]
```

---

### Pro 사용자

```
Pro 사용 중입니다.

모든 기능이 활성화되어 있습니다.
```

**CTA:**
```
[ 팀 대시보드로 이동 ]
```

---

## 5️⃣ 마무리 문구 (최하단)

### 기본 문구

```
궁금한 점이 있으시면
언제든지 문의해 주세요.
```

**또는**

```
이제 활동만 하시면 됩니다.
```

---

## 6️⃣ 화면별 카피 세트

### 세트 A: "끝났다" 느낌

**메인:**
```
설정이 완료되었습니다.
```

**서브:**
```
이제 활동만 하시면 됩니다.
```

**상태:**
```
🟢 자동 기록 중
다음 활동이 있으면 자동으로 기록됩니다.
```

**마무리:**
```
별도로 할 일은 없습니다.
```

---

### 세트 B: "안심" 느낌

**메인:**
```
모든 것이 준비되었습니다.
```

**서브:**
```
기록은 자동으로 쌓입니다.
```

**상태:**
```
🟢 자동 기록 중
활동이 있으면 자동으로 기록됩니다.
```

**마무리:**
```
언제든지 설정을 변경하실 수 있습니다.
```

---

### 세트 C: "간단함" 느낌

```

```

