## 🔍 검증 체크 포인트 5개

### 1️⃣ 조 추첨 실행 시 seed 저장되는지
**확인 위치**: `test_drawLogs` / `drawLogs`
```json
{
  "algorithm": {
    "seed": "SHA256_HASH",
    "seedString": "tournamentId_timestamp_adminId",
    "timestamp": 1678176000000
  }
}
```
**검증 방법**: 조 추첨 실행 후 `test_drawLogs` 컬렉션에서 `algorithm.seed` 확인

### 2️⃣ 팀 수 바뀌어도 조 분배가 균등한지
**확인 위치**: `test_groups.groups` 배열
```json
{
  "groups": [
    { "groupId": "A", "teams": [...] }, // 팀 수
    { "groupId": "B", "teams": [...] }, // 팀 수
    ...
  ]
}
```
**검증 방법**: 
- 각 조의 `teams.length` 확인
- 최대 차이 1팀 이내인지 확인
- 예: 16팀 4조 → 각 조 4팀씩 균등 분배

### 3️⃣ 대진표 생성 시 경기 수 계산이 맞는지
**확인 위치**: `PromoteTestToOfficialButton` 상태 표시
```
경기 수 검증: 정상 (예상: 24경기, 실제: 24경기)
```
**검증 방법**:
- 조별 리그: `n*(n-1)/2` (n = 조 내 팀 수)
- 예: 4팀 조 → 4*3/2 = 6경기
- 총 경기 수 = 각 조 경기 수 합계

### 4️⃣ 테스트 화면에 "테스트" 배너가 확실히 뜨는지
**확인 위치**: 
- OPS 페이지 헤더: "(테스트 모드)" 배지
- 조 추첨 섹션: "🧪 테스트 모드" 표시
- 대진표 섹션: "⚠️ 테스트 대진표" 배너
**검증 방법**: URL에 `?mode=test` 추가 후 모든 화면에서 배너 확인

### 5️⃣ 승격 후 공식 화면에서 동일 결과가 보이는지
**확인 위치**: 
- `tournament.drawDivisions` (조 추첨 결과)
- `matches` 컬렉션 (경기 목록)
**검증 방법**:
1. 테스트 모드에서 조 추첨 실행
2. 테스트 모드에서 경기 생성
3. 승격 실행
4. 일반 모드에서 동일한 조/경기 결과 확인

## 🔁 공식 승격 UX 동작 (확정)

### 승격 버튼 활성 조건
- ✅ `test_groups` 존재
- ✅ `test_matches` 존재
- ✅ 관리자 체크박스 체크

### 클릭 시 2차 모달
- "되돌릴 수 없음" 경고
- "공식 기록 반영" 고지
- "로그 생성" 안내
- [취소] / [확정] 버튼

### 확정 시 서버 처리
1. `test_groups` → `tournament.drawDivisions` 복사
2. `test_matches` → `matches` 복사
3. `test_drawLogs` → `drawLogs` 복사 (최신 1개)
4. `auditLogs`: `PROMOTE_TEST_RESULT` 1줄 저장
5. `opsLogs`: 승격 로그 저장

### 승격 후 UI 변화
- ✅ 테스트 토글/버튼 비활성
- ✅ "공식 확정 완료" 배지 표시
- ✅ 테스트 모드 URL → 일반 모드로 자동 리다이렉트
- ✅ 조 추첨 관리 영역 → 읽기 전용
