# 🌍 공개 팀 블로그 랜딩 – 실전 문구 세트

## 📋 목적
디자이너/개발자/기획자가 바로 사용할 수 있는 실제 서비스용 문구 (복붙 가능)

---

## 1️⃣ 히어로 영역 실제 문구

### 케이스 A: 최근 활동 있음 (14일 이내)

**상태 뱃지:**
```
🟢 활발한 팀
```

**최근 활동 텍스트:**
```
최근 경기: 7월 21일 (일) - 정기 경기
이번 주 훈련 완료 (15명 참여)
```

**또는 (출석만 있는 경우):**
```
이번 주 훈련 완료 (15명 참여)
```

**또는 (블로그 포스트만 있는 경우):**
```
최근 소식: "소흘 60대 FC, 이렇게 활동하고 있습니다"
```

---

### 케이스 B: 활동 없음 (14일 초과)

**상태 뱃지:**
```
🟡 모집 중
```

**최근 활동 텍스트:**
```
최근 활동 정보를 준비 중입니다.
```

---

### 신뢰 배지 (항상 표시)
```
✔ 주 1회 활동
✔ 실제 팀 운영 중
```

**Pro 플랜인 경우 추가:**
```
✔ AI 관리 블로그
```

---

### 메인 CTA 버튼

**초기 상태 (비로그인/미가입):**
```

```

**요청 완료 상태:**

**이미 팀원 상태:**

**관리자 상태:**

## 2️⃣ 팀 한 줄 캐치프레이즈

### 기본 템플릿
```
⚽ {팀명}
{연령대}에도 즐겁게 뛰고, 오래 함께하는 {종목}팀
```

### 실제 예시
```
⚽ 소흘 60대 FC
60대에도 즐겁게 뛰고, 오래 함께하는 축구팀
```

### AI 자동 생성 가능 문구
- 팀 데이터 기반으로 자동 생성
- 관리자가 수정 가능

---

## 3️⃣ 팀 핵심 정보 카드

### 제목
```
팀 핵심 정보
```

### 정보 항목 (4개)

**1. 활동 지역**
```
📍 활동 지역
{지역명}
예: 포천
```

**2. 평균 연령**
```
👥 평균 연령
{연령대}
예: 60대
```

**3. 활동 빈도**
```
📅 활동 빈도
주 1회
```

**4. 분위기**
```
🎯 분위기
즐겁게 / 안전하게
```

## 4️⃣ "이런 분을 기다립니다" 모집 카드

### 제목
```
🙋 이런 분을 기다립니다
```

### 추천 대상 리스트 (3개)

**1. 연령대 기반 (팀 데이터 기반)**
```
✔ {연령대} 이상 {종목} 좋아하시는 분
예: ✔ 60대 이상 축구 좋아하시는 분
```

**2. 경험 무관**
```
✔ 운동 경험 무관
```

**3. 가치 중심**
```
✔ 건강하고 즐겁게 운동하고 싶은 분
```

**또는:**
```
✔ 승부보다 건강과 팀워크 중시
```

---

### CTA 버튼

**Primary (메인):**
```
이 팀에 가입하기
```

**Secondary (서브):**
```
문의하기
```

## 5️⃣ "이 팀에 가입하기" 신청 화면 문구

### 모달/페이지 제목
```
⚽ {팀명}에 가입 신청하기
```

### 팀 정보 요약
```
• 연령대: {연령대}
• 활동 지역: {지역}
• 활동 빈도: 주 1회
```

### 메시지 입력 (선택)
```
간단한 자기소개나 참여 의사를 남겨주세요
(선택 사항)
```

**입력 필드 플레이스홀더:**
```
예: "60대 남성입니다. 건강하게 즐겁게 운동하고 싶어요."
```

### 제출 버튼
```
가입 신청하기
```

---

### 신청 완료 메시지

**성공:**
```
✅ 가입 신청이 완료되었습니다

관리자가 확인 후
참여 안내를 드릴게요!
```

**중복 요청:**
```
ℹ️ 이미 가입 요청을 보내셨습니다.

관리자의 승인을 기다려주세요.
```

**에러:**
```
❌ 가입 요청 전송에 실패했습니다.

잠시 후 다시 시도해주세요.
```

### 신청 완료 후 액션 버튼

## 6️⃣ 관리자에게만 보이는 Pro 전환 문구

```

```


### 트리거 ②: 블로그 조회수 증가

## ✅ 사용 가이드

### 디자이너용
- 모든 문구는 실제 서비스에 사용되는 문장
- 복붙해서 바로 사용 가능
- 팀 데이터 기반으로 `{변수}` 부분만 치환

### 개발자용
- 각 문구는 컴포넌트 props 또는 상수로 정의
- 다국어 지원 시 이 문서를 번역 기준으로 사용

### 기획자용
- A/B 테스트 후보 문구 포함
- 각 문구의 사용 위치와 조건 명시
