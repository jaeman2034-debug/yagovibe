# 🔥 조 추첨 알고리즘 고도화 완료

## ✅ 완료된 기능

### 알고리즘 레벨 0 → 1 → 2 진화

#### 레벨 0: 완전 랜덤
- ✅ 완전 랜덤 배정
- ✅ 테스트/데모용
- ✅ 알고리즘 버전: `v0.0`

#### 레벨 1: 시드 분산 + 랜덤 (권장)
- ✅ 시드팀은 서로 다른 조로 자동 분산
- ✅ 나머지는 랜덤 배치
- ✅ 알고리즘 버전: `v1.0`
- ✅ 대부분의 생활체육 대회에서 충분히 합리적

#### 레벨 2: 시드 + 회피 규칙 + 균형 점수 (고급)
- ✅ 시드 분산
- ✅ 동일 클럽/아카데미 회피
- ✅ 동일 지역 팀 분산
- ✅ 조별 평균 시드 편차 최소화
- ✅ 점수 기반 최적화 (100번 시도 후 최적 조합 선택)
- ✅ 알고리즘 버전: `v2.1`

### 점수 계산 공식

```
score = seedVariance * 0.6 + sameClubPenalty * 0.3 + regionPenalty * 0.1
```

- `seedVariance`: 조별 평균 시드 표준편차
- `sameClubPenalty`: 같은 클럽이 같은 조에 있는 경우 페널티
- `regionPenalty`: 같은 지역이 3팀 이상 같은 조에 있는 경우 페널티

## 📦 알고리즘 버전 관리

### 저장 위치
- `tournament.algorithmVersion`: `v0.0`, `v1.0`, `v2.1`
- `drawLog.algorithm.version`: 알고리즘 버전
- `drawLog.algorithm.level`: 알고리즘 레벨 (0, 1, 2)
- `drawLog.algorithm.rules`: 적용된 규칙 배열

### 버전 이력 관리
```json
{
  "algorithmVersion": "v2.1",
  "algorithm": {
    "version": "v2.1",
    "level": 2,
    "rules": ["seed", "clubAvoid", "balance"],
    "metadata": {
      "score": 0.45,
      "attempts": 100
    }
  }
}
```

**👉 나중에 정책 변경 시:**
- "작년엔 v1.0"
- "올해는 v2.1"
이렇게 정책 변경 이력 관리 가능

## 🎨 UX에 반영

### 조 추첨 결과 화면 상단
```
조 추첨 방식
- 시드 분산 + 랜덤 배정
- 동일 클럽 회피 적용
- 알고리즘 버전: v2.1
```

**👉 이 한 줄이 민원을 막는다.**

### 조 추첨 실행 모달
- 알고리즘 레벨 선택 드롭다운
  - 레벨 0: 완전 랜덤 (테스트용)
  - 레벨 1: 시드 분산 + 랜덤 (권장)
  - 레벨 2: 시드 + 회피 규칙 + 균형 (고급)
- 레벨 2 선택 시 "동일 클럽/지역 회피" 체크박스 표시
