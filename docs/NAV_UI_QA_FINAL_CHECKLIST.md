# 🏁 지도 페이지 QA 최종 체크리스트 (출시 직전용)

> 이거 통과하면 **지도 페이지는 끝**이다.
> 하나라도 ❌면 출시 불가

---

## 📋 체크리스트 사용법

각 항목을 **실제로 테스트**하고 결과를 기록:
- ✅ 통과
- ❌ 실패 (수정 필요)
- ⚠️ 부분 통과 (개선 권장)

---

## Ⅰ. 상태 머신 완성도

### 1.1 상태 정의
- [ ] `SEARCH / SELECTED / PRE_NAV / NAVIGATING` 4상태만 존재
- [ ] 다른 상태(예: `LOADING`, `ERROR`)는 서브 상태로 처리
- [ ] 상태 타입이 TypeScript로 명확히 정의됨

### 1.2 상태 전환
- [ ] 각 상태 전환이 **명확한 트리거**로만 발생
- [ ] 상태 전환 시 **UI 세트가 통째로 교체**됨
- [ ] 상태 전환이 **역전 불가능**하지 않음 (뒤로 가기 가능)

### 1.3 디버그 도구
- [ ] 개발 모드에서 현재 상태가 화면에 표시됨
- [ ] 상태 전환 시 콘솔 로그로 추적 가능
- [ ] 상태 머신이 **단일 소스 오브 트루스**로 동작

---

## Ⅱ. 출발 UX (네비 성립 여부)

### 2.1 출발 버튼 가시성
- [ ] 출발 버튼은 **PRE_NAV 상태에서만** 보임
- [ ] 다른 상태에서는 출발 버튼이 **절대 보이지 않음**
- [ ] 출발 버튼이 하단 카드 내부에 위치

### 2.2 출발 버튼 피드백
- [ ] 클릭 즉시 (0-150ms) 버튼 상태 변경
- [ ] "경로 계산 중…" 텍스트 표시
- [ ] 버튼이 비활성화됨 (중복 클릭 방지)

### 2.3 출발 후 UI 변화
- [ ] 300ms 이내에 상단 UI 교체 (검색바 → 턴바이턴)
- [ ] 300ms 이내에 하단 UI 교체 (출발 카드 → 안내 카드)
- [ ] 지도 카메라가 500ms 이내에 변화
- [ ] **최소 2개 이상의 UI 요소가 동시에 변함**

### 2.4 출발 후 지도 반응
- [ ] 지도가 내 위치 중심으로 이동
- [ ] 경로선이 표시됨 (또는 표시 예정)
- [ ] 지도 줌 레벨이 NAVIGATING 모드로 변경

---

## Ⅲ. 지도 레이어 UX

### 3.1 상태별 지도 반응

#### SEARCH 상태
- [ ] 지도 중심이 내 위치
- [ ] 내 위치 마커 표시
- [ ] 경로선 없음
- [ ] 줌 레벨 적절 (동네 단위)

#### SELECTED 상태
- [ ] 지도가 출발지 + 목적지 한 화면에 fit
- [ ] 내 위치 마커 + 목적지 마커 모두 표시
- [ ] 경로선 없음 (또는 연한 색 미리보기)

#### PRE_NAV 상태
- [ ] 지도가 경로 전체를 포함하도록 fit
- [ ] 경로선 표시 (연한 색)
- [ ] 출발지 + 목적지 마커 표시

#### NAVIGATING 상태
- [ ] 지도가 내 위치 중심으로 추적
- [ ] 경로선 표시 (진한 색)
- [ ] 지도 줌 레벨이 차량 시점 (가까이)
- [ ] 지도 드래그/줌 조작 차단

### 3.2 지도 카메라 전환
- [ ] 상태 전환 시 지도가 **부드럽게** 이동 (`panTo` / `fitBounds`)
- [ ] 갑작스러운 점프 없음
- [ ] 카메라 전환이 500ms 이내 완료

### 3.3 지도 인터랙션
- [ ] SEARCH/SELECTED/PRE_NAV: 드래그/줌 가능
- [ ] NAVIGATING: 드래그/줌 차단 (또는 추적 해제)
- [ ] 내 위치 버튼 클릭 시 지도가 내 위치로 이동

---

## Ⅳ. 상단 UI 규칙

### 4.1 상태별 상단 UI

#### SEARCH / SELECTED
- [ ] 검색바만 표시
- [ ] 상태바/턴바이턴 **절대 없음**
- [ ] 검색바가 상단 중앙에 위치

#### PRE_NAV
- [ ] 상태 텍스트만 표시 ("🚗 출발 준비됨")
- [ ] 검색바 **완전히 숨김**
- [ ] 턴바이턴 없음

#### NAVIGATING
- [ ] 턴바이턴만 표시 ("⬆️ 300m 직진")
- [ ] 검색바 **완전히 숨김**
- [ ] 상태바 없음

### 4.2 UI 겹침 방지
- [ ] 검색바 + 상태바 동시 표시 안 됨
- [ ] 검색바 + 턴바이턴 동시 표시 안 됨
- [ ] 상태바 + 턴바이턴 동시 표시 안 됨

---

## Ⅴ. 하단 UI 규칙

### 5.1 상태별 하단 UI

#### SEARCH
- [ ] 검색 결과 리스트만 표시
- [ ] 출발 버튼 없음
- [ ] 확인 카드 없음

#### SELECTED
- [ ] 확인 카드만 표시 ("여기에 갈게요" 버튼)
- [ ] 검색 결과 리스트 없음
- [ ] 출발 버튼 없음

#### PRE_NAV
- [ ] 출발 대기 카드만 표시 (경로 정보 + 출발 버튼)
- [ ] 확인 카드 없음
- [ ] 안내 카드 없음

#### NAVIGATING
- [ ] 안내 카드만 표시 (남은 시간 + 종료 버튼)
- [ ] 출발 버튼 없음
- [ ] 확인 카드 없음

### 5.2 하단 카드 역할 분리
- [ ] 한 상태당 **하나의 역할**만
- [ ] 정보 + 액션이 섞이지 않음
- [ ] 출발 버튼은 PRE_NAV에만 존재

---

## Ⅵ. 뒤로 가기 / 종료 UX

### 6.1 뒤로 가기 동작

#### NAVIGATING → PRE_NAV
- [ ] 뒤로 가기 시 PRE_NAV 상태로 복귀
- [ ] 상단 UI가 상태바로 복귀
- [ ] 하단 UI가 출발 대기 카드로 복귀
- [ ] 지도 카메라가 경로 전체 보기로 복귀

#### PRE_NAV → SELECTED
- [ ] 뒤로 가기 시 SELECTED 상태로 복귀
- [ ] 상단 UI가 검색바로 복귀
- [ ] 하단 UI가 확인 카드로 복귀
- [ ] 지도 카메라가 출발지 + 목적지 fit으로 복귀

#### SELECTED → SEARCH
- [ ] 뒤로 가기 시 SEARCH 상태로 복귀
- [ ] 상단 UI가 검색바로 복귀
- [ ] 하단 UI가 검색 결과 리스트로 복귀
- [ ] 지도 카메라가 내 위치 중심으로 복귀

### 6.2 종료 버튼
- [ ] NAVIGATING 상태에서 "안내 종료" 버튼 동작
- [ ] 종료 시 PRE_NAV 상태로 복귀
- [ ] 경로선이 사라짐 (또는 연한 색으로 변경)

---

## Ⅶ. 실패 상황 UX

### 7.1 경로 계산 실패
- [ ] 경로 계산 실패 시 에러 메시지 표시
- [ ] "다시 시도" 버튼 제공
- [ ] 상태는 SELECTED 유지 (PRE_NAV로 가지 않음)
- [ ] 지도는 그대로 유지

### 7.2 위치 권한 없음
- [ ] 위치 권한 없을 시 안내 메시지 표시
- [ ] "설정으로 이동" 버튼 제공
- [ ] 지도는 기본 위치(예: 서울)로 표시

### 7.3 네트워크 오류
- [ ] 네트워크 오류 시 안내 메시지 표시
- [ ] "다시 시도" 버튼 제공
- [ ] 지도는 그대로 유지

---

## Ⅷ. 애니메이션 및 전환

### 8.1 출발 전환 애니메이션
- [ ] 버튼 클릭 즉시 피드백 (0-150ms)
- [ ] 지도 카메라 전환 애니메이션 (300-600ms)
- [ ] 상단/하단 UI 교체 애니메이션 (250ms)
- [ ] 전체 전환이 500ms 이내 완료

### 8.2 상태 전환 애니메이션
- [ ] 모든 상태 전환에 애니메이션 적용
- [ ] 갑작스러운 UI 변화 없음
- [ ] 애니메이션이 너무 길지 않음 (최대 500ms)

---

## Ⅸ. 모바일 UX

### 9.1 터치 인터랙션
- [ ] 모든 버튼이 터치하기 쉬운 크기 (최소 44x44px)
- [ ] 버튼 간 간격이 충분함 (최소 8px)
- [ ] 하단 카드가 Safe Area 고려

### 9.2 화면 크기 대응
- [ ] 작은 화면에서도 모든 UI 요소 보임
- [ ] 큰 화면에서도 UI가 적절히 배치됨
- [ ] 가로 모드에서도 동작 (또는 차단)

### 9.3 성능
- [ ] 상태 전환 시 버벅임 없음
- [ ] 지도 이동 시 부드러움
- [ ] 메모리 누수 없음

## Ⅹ. 최종 판정 질문

아래 질문에 **모두 Yes**여야 출시 가능:

### 10.1 상태 인식
- [ ] **지금 무슨 상태인지 설명 없이 알 수 있는가?**
  - 테스트: 화면만 보고 상태 설명 가능해야 함

### 10.2 버튼 가시성
- [ ] **출발 버튼은 "눌러야 할 때만" 보이는가?**
  - 테스트: PRE_NAV 상태에서만 보이고, 다른 상태에서는 안 보여야 함

### 10.3 즉각 반응
- [ ] **출발 후 0.5초 안에 변화가 보이는가?**
  - 테스트: 출발 버튼 클릭 후 0.5초 이내에 최소 2개 UI 요소 변화

### 10.4 동시 변화
- [ ] **지도, 상단, 하단 중 최소 2개가 동시에 변하는가?**
  - 테스트: 상태 전환 시 여러 UI 요소가 함께 변화

### 10.5 맥락 유지
- [ ] **실패 상황에서도 맥락이 유지되는가?**
  - 테스트: 경로 계산 실패 시에도 지도는 그대로, 메시지만 표시

### 10.6 뒤로 가기
- [ ] **뒤로 가기로 이전 상태로 복귀 가능한가?**
  - 테스트: 각 상태에서 뒤로 가기 시 이전 상태로 복귀

## 🎯 최종 판정 기준

### ✅ 출시 가능
- 모든 체크리스트 항목 ✅
- 최종 판정 질문 모두 Yes
- 모바일에서도 동일하게 동작

### ⚠️ 수정 후 출시
- 일부 항목 ❌
- 최종 판정 질문 일부 No
- 모바일에서 일부 문제

### ❌ 출시 불가
- 핵심 항목 다수 ❌
- 최종 판정 질문 대부분 No
- 모바일에서 심각한 문제
