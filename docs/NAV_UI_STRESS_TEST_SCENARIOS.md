# 🧪 지도 페이지 실전 테스트 시나리오 10개

> **의도적 UX 파괴 테스트**
> 이거 통과 못 하면 연동 금지

---

## 🎯 테스트 목적

지도 페이지 상태 머신 구조가 **실전 압박에서도 절대 흔들리지 않는지** 확인.

**통과 기준**: 모든 시나리오에서 UI 상태가 명확히 유지되어야 함.

---

## 시나리오 1️⃣: 상태 빠르게 왕복

### 테스트 방법

```
1. SEARCH → SELECTED (장소 선택)
2. SELECTED → SEARCH (뒤로 가기)
3. SEARCH → SELECTED (다시 선택)
4. SELECTED → PRE_NAV (여기에 갈게요)
5. PRE_NAV → SELECTED (뒤로 가기)
6. SELECTED → PRE_NAV (다시)
7. PRE_NAV → NAVIGATING (출발)
8. NAVIGATING → PRE_NAV (안내 종료)
9. PRE_NAV → NAVIGATING (재출발)
```

**속도**: 각 전환을 **1초 이내**로 빠르게 진행

### 통과 기준

- [ ] 모든 상태 전환 시 UI가 즉시 교체됨
- [ ] 중간 상태(예: SEARCH UI가 SELECTED에 섞임)가 절대 발생하지 않음
- [ ] 디버그 배지가 항상 정확한 상태를 표시
- [ ] 지도 카메라가 각 상태에 맞게 반응

### 실패 시나리오

- ❌ 상태 전환이 느려서 중간에 UI가 섞임
- ❌ 빠른 전환 시 일부 UI가 업데이트 안 됨
- ❌ 지도 카메라가 상태와 맞지 않음

---

## 시나리오 2️⃣: 출발 → 종료 → 재출발 연타

### 테스트 방법

```
1. PRE_NAV 상태에서 출발 버튼 클릭
2. NAVIGATING 상태 진입 확인
3. 즉시 "안내 종료" 버튼 클릭
4. PRE_NAV 상태 복귀 확인
5. 즉시 다시 출발 버튼 클릭
6. NAVIGATING 상태 재진입 확인
```

**반복**: 10회 연속

### 통과 기준

- [ ] 출발 버튼 클릭 즉시 (0-150ms) 피드백
- [ ] 상태 전환이 300ms 이내 완료
- [ ] 종료 시 이전 상태로 정확히 복귀
- [ ] 재출발 시 동일한 UX 유지
- [ ] 10회 반복해도 UI가 흔들리지 않음

### 실패 시나리오

- ❌ 빠른 연타 시 상태가 꼬임
- ❌ 종료 후 UI가 이상한 상태로 남음
- ❌ 재출발 시 이전 상태가 섞임

---

## 시나리오 3️⃣: 실패/로딩/뒤로가기 연타

### 테스트 방법

```
1. SELECTED → PRE_NAV (여기에 갈게요)
2. 경로 계산 중 (로딩 상태)
3. 즉시 뒤로 가기 클릭
4. SELECTED 복귀 확인
5. 다시 PRE_NAV 진입
6. 경로 계산 실패 시뮬레이션
7. 에러 메시지 표시 확인
8. 뒤로 가기 클릭
9. SELECTED 복귀 확인
```

### 통과 기준

- [ ] 로딩 중에도 뒤로 가기 가능
- [ ] 로딩 중 뒤로 가기 시 이전 상태로 정확히 복귀
- [ ] 경로 계산 실패 시 UI 상태는 SELECTED 유지
- [ ] 에러 메시지는 오버레이로만 표시
- [ ] 에러 후 뒤로 가기 시 정상 복귀

### 실패 시나리오

- ❌ 로딩 중 뒤로 가기가 안 됨
- ❌ 경로 실패 시 UI 상태가 이상하게 변경됨
- ❌ 에러 메시지가 UI 상태를 덮어씀

---

## 시나리오 4️⃣: 네트워크 끊김 시뮬레이션

### 테스트 방법

```
1. NAVIGATING 상태 진입
2. 네트워크 끊김 시뮬레이션
3. 위치 업데이트 실패
4. 경로 재계산 실패
5. 10초 대기
6. 네트워크 복구
```

### 통과 기준

- [ ] 네트워크 끊김 시에도 UI 상태는 NAVIGATING 유지
- [ ] 위치 업데이트 실패 시 오버레이만 표시
- [ ] 경로 재계산 실패 시 오버레이만 표시
- [ ] UI 상태가 자동으로 변경되지 않음
- [ ] 네트워크 복구 시 자동으로 정상 동작 재개

### 실패 시나리오

- ❌ 네트워크 끊김 시 UI 상태가 SEARCH로 튕김
- ❌ 위치 업데이트 실패 시 안내가 자동 종료됨
- ❌ 네트워크 복구 후 상태가 이상하게 남음

---

## 시나리오 5️⃣: GPS 신호 불안정

### 테스트 방법

```
1. NAVIGATING 상태 진입
2. GPS 신호를 의도적으로 불안정하게 설정
3. 위치 업데이트가 1-2초씩 지연
4. 위치가 튀는 상황 시뮬레이션
5. 30초 동안 지속
```

### 통과 기준

- [ ] GPS 신호 불안정해도 UI 상태는 NAVIGATING 유지
- [ ] 위치 업데이트 지연 시 오버레이만 표시
- [ ] 위치가 튀어도 UI 상태는 안정적
- [ ] 지도 카메라는 위치 업데이트에 따라 부드럽게 이동
- [ ] UI가 깜빡이거나 점프하지 않음

### 실패 시나리오

- ❌ GPS 불안정 시 UI 상태가 변경됨
- ❌ 위치 튐 시 UI가 깜빡임
- ❌ 지도 카메라가 이상하게 움직임

---

## 시나리오 6️⃣: 앱 백그라운드/포그라운드 전환

### 테스트 방법

```
1. NAVIGATING 상태 진입
2. 앱을 백그라운드로 전환
3. 10초 대기
4. 앱을 포그라운드로 복귀
5. 상태 확인
```

**반복**: 5회

### 통과 기준

- [ ] 백그라운드 전환 시 상태 유지
- [ ] 포그라운드 복귀 시 이전 상태 그대로
- [ ] 지도 카메라가 정확히 복구됨
- [ ] UI가 정상적으로 표시됨
- [ ] 5회 반복해도 문제 없음

### 실패 시나리오

- ❌ 포그라운드 복귀 시 SEARCH 상태로 리셋됨
- ❌ 지도 카메라가 이상한 위치로 이동
- ❌ UI가 깨져서 표시됨

---

## 시나리오 7️⃣: 경로 재계산 연타

### 테스트 방법

```
1. NAVIGATING 상태 진입
2. 경로 이탈 시뮬레이션
3. 경로 재계산 트리거
4. 재계산 중 다시 이탈
5. 재계산 재트리거
6. 5회 반복
```

### 통과 기준

- [ ] 경로 재계산 중에도 UI 상태는 NAVIGATING 유지
- [ ] 재계산 중 메시지만 표시 (UI 상태 변경 안 함)
- [ ] 지도는 부드럽게 경로만 업데이트
- [ ] 연속 재계산해도 UI가 흔들리지 않음
- [ ] 재계산 완료 후 정상 동작

### 실패 시나리오

- ❌ 재계산 시 UI 상태가 PRE_NAV로 변경됨
- ❌ 연속 재계산 시 UI가 깜빡임
- ❌ 지도가 이상하게 업데이트됨

---

## 시나리오 8️⃣: 검색 중 상태 전환

### 테스트 방법

```
1. SEARCH 상태에서 검색어 입력
2. 검색 결과 로딩 중
3. 즉시 다른 장소 선택
4. SELECTED 상태 진입
5. 검색 결과가 아직 로딩 중이어도
6. SELECTED UI만 표시
```

### 통과 기준

- [ ] 검색 로딩 중에도 장소 선택 가능
- [ ] 장소 선택 시 즉시 SELECTED 상태로 전환
- [ ] 검색 결과가 로딩 중이어도 SELECTED UI만 표시
- [ ] 이전 검색 결과가 SELECTED UI에 섞이지 않음
- [ ] 상태 전환이 명확함

### 실패 시나리오

- ❌ 검색 로딩 중 장소 선택이 안 됨
- ❌ SELECTED 상태인데 검색 결과가 보임
- ❌ 상태 전환이 애매함

---

## 시나리오 9️⃣: 동시 액션 충돌

### 테스트 방법

```
1. PRE_NAV 상태 진입
2. 출발 버튼 클릭 (300ms 지연)
3. 동시에 뒤로 가기 클릭
4. 두 액션이 충돌하는 상황
```

### 통과 기준

- [ ] 마지막 액션이 우선순위를 가짐
- [ ] 상태가 명확하게 결정됨
- [ ] UI가 중간 상태로 남지 않음
- [ ] 디버그 배지가 정확한 상태 표시

### 실패 시나리오

- ❌ 두 액션이 충돌하여 상태가 꼬임
- ❌ UI가 중간 상태로 남음
- ❌ 어떤 상태인지 알 수 없음

---

## 시나리오 🔟: 장시간 사용 스트레스

### 테스트 방법

```
1. 전체 플로우를 30분 동안 반복
   - SEARCH → SELECTED → PRE_NAV → NAVIGATING → 종료
2. 각 상태에서 1-2분씩 머무름
3. 다양한 액션 시도
4. 메모리 사용량 모니터링
```

### 통과 기준

- [ ] 30분 사용해도 UI 상태가 정확함
- [ ] 메모리 누수 없음
- [ ] 성능 저하 없음
- [ ] UI가 깜빡이지 않음
- [ ] 모든 상태 전환이 정상 동작

### 실패 시나리오

- ❌ 시간이 지나면 상태가 이상해짐
- ❌ 메모리 사용량이 계속 증가
- ❌ UI가 느려지거나 깜빡임
