# 🔥 Admin 대시보드 가이드

## 📋 개요

관리자만 접근 가능한 운영 대시보드로, 유저 현황, 온보딩 퍼널, SMS 로그를 한 화면에 표시합니다.

**목표:**
- 관리자만 접근 (`/admin`)
- 유저 현황 / 온보딩 퍼널 / SMS 로그
- Firestore read-only 안전 조회
- 코드 최소 · 확장 쉬움

**원칙:**
- 운영이 '감'이 아니라 '데이터'가 되도록
- 실시간 업데이트 (onSnapshot)
- Firestore Rules로 보안 보장

---

## ✅ 구현된 컴포넌트

### 1️⃣ `AdminDashboard` 페이지

**파일:** `src/pages/admin/AdminDashboard.tsx`

**역할:**
- 유저 현황 통계
- 온보딩 퍼널 (단계별 이탈률)
- SMS 인증 로그 최근 목록
- 실시간 업데이트

**표시 정보:**
- 전체 유저 수
- 온보딩 완료/미완료 수
- 전환율
- 단계별 유저 수 (Step 0~4)
- SMS 요청/성공/실패 수
- SMS 성공률
- 최근 SMS 인증 로그 (최대 100건)

## 🔐 접근 제어

### 라우트 보호
- `/admin` → `AdminRoute`로 보호
- 관리자가 아닌 사용자는 `/sports-hub`로 리다이렉트

### Firestore Rules
- `isGlobalAdmin()` 함수로 관리자 권한 확인
- `role == "ADMIN"` (대문자)로 체크
- 클라이언트에서는 절대 role 변경 불가

---

## 📊 데이터 소스

### `users` 컬렉션
- `onboardingCompleted`: 온보딩 완료 여부
- `onboardingStep`: 현재 온보딩 단계 (0~4)
- `createdAt`: 가입일

### `auth_logs` 컬렉션
- `type`: 이벤트 타입 (`sms_request`, `sms_success`, `sms_error`, `verify_success`, `verify_error`)
- `phoneNumber`: 전화번호 (마스킹 표시)
- `errorCode`: 에러 코드 (실패 시)
- `errorMessage`: 에러 메시지 (실패 시)
- `createdAt`: 발생 시간

---

## 🧠 대시보드 구조

### 📊 유저 현황
- 전체 유저 수
- 온보딩 완료 수
- 온보딩 미완료 수
- 전환율 (온보딩 완료 / 전체)

### 📈 온보딩 퍼널
- Step 0 (이름 입력): 현재 이 단계에 있는 유저 수
- Step 1 (목적 선택): 현재 이 단계에 있는 유저 수
- Step 2 (종목 선택): 현재 이 단계에 있는 유저 수
- Step 3 (지역 선택): 현재 이 단계에 있는 유저 수
- Step 4 (완료): 현재 이 단계에 있는 유저 수
- 단계 정보 없음: `onboardingStep`이 없는 유저 수

### 📱 SMS 인증 통계
- 요청 수: `sms_request` 이벤트 수
- 성공 수: `sms_success` 이벤트 수
- 실패 수: `sms_error` 이벤트 수
- 성공률: (성공 / 요청) × 100

### 📋 최근 SMS 인증 로그
- 최근 100건의 SMS 인증 로그 표시
- 이벤트 타입별 색상 구분
- 전화번호 마스킹 표시
- 에러 코드/메시지 표시 (실패 시)
- 발생 시간 표시

---

## 🔄 실시간 업데이트

### onSnapshot 사용
- `users` 컬렉션 실시간 구독
- `auth_logs` 컬렉션 실시간 구독 (최근 100건)
- 데이터 변경 시 자동 업데이트

### 성능 최적화
- `auth_logs`는 `limit(100)`으로 제한
- `orderBy("createdAt", "desc")`로 최신순 정렬

## 📞 문제 해결

### 문제: 대시보드가 로드되지 않음

**원인:**
- Firestore Rules에서 읽기 권한 없음
- `role`이 `"ADMIN"`이 아님

**해결:**
1. Firestore Rules 확인
2. Firebase Console에서 `users/{uid}.role` 확인
3. `role`을 `"ADMIN"`으로 설정

---

### 문제: 실시간 업데이트가 안 됨

**원인:**
- `onSnapshot` 구독 실패
- 네트워크 오류

**해결:**
1. 브라우저 콘솔에서 에러 확인
2. 네트워크 탭에서 Firestore 요청 확인
3. Firestore Rules 확인
