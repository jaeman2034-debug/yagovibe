# 🔥 테스트 완료 → 공식 승격 UX 완성

## ✅ 완료된 기능

### STEP 3: 테스트 완료 기준 정의
- ✅ 테스트 조 추첨 완료 확인 (`test_groups` 존재)
- ✅ 테스트 대진표 생성 완료 확인 (`test_matches` 존재)
- ✅ 경기 수 정상 검증 (예상 경기 수와 일치)
- ✅ 관리자 수동 확인 체크 (필수)

### STEP 4: 공식 승격 UX
- ✅ 상태 표시 박스 (조 추첨/대진표/운영 기록 반영 상태)
- ✅ 관리자 확인 체크박스 (필수)
- ✅ 공식 승격 버튼 (조건 만족 시 활성화)
- ✅ 2차 확인 모달 (되돌릴 수 없음 경고)

### STEP 5: 승격 시 실제 내부 동작
- ✅ `test_groups` → `drawDivisions` 복사
- ✅ `test_matches` → `matches` 복사
- ✅ `test_drawLogs` → `drawLogs` 복사
- ✅ 감사 로그 기록 (`auditLogs`)
- ✅ 운영 로그 기록 (`opsLogs`)

- 이후 수정 시 기록이 남습니다

계속 진행하시겠습니까?

[취소] [확정]

## 🔁 데이터 승격 로직

### 컬렉션 이동 구조
| 테스트 | 공식 |
|--------|------|
| `test_groups` | `tournament.drawDivisions` |
| `test_matches` | `matches` |
| `test_drawLogs` | `drawLogs` |

### 승격 프로세스
1. `test_groups` → `tournament.drawDivisions` 복사
2. `test_drawLogs` → `drawLogs` 복사 (최신 1개)
3. `test_matches` → `matches` 복사 (모든 경기)
4. `tournament.drawExecuted = true` 설정
5. 감사 로그 기록 (`auditLogs`)
6. 운영 로그 기록 (`opsLogs`)

## ✅ 테스트 완료 조건

### 활성화 조건 (모두 만족)
1. ✅ `test_groups` 존재
2. ✅ `test_matches` 존재
3. ✅ 경기 수 정상 (예상 = 실제)
4. ✅ 관리자 확인 체크

### 비활성화 조건 (하나라도 만족)
- ❌ `test_groups` 없음
- ❌ `test_matches` 없음
- ❌ 경기 수 불일치
- ❌ 관리자 확인 미체크

## 🎯 핵심 원칙

1. **완전 분리**: 테스트/운영 데이터 완전 분리
2. **안전 보장**: 2차 확인 모달 + 체크박스 필수
3. **감사 대응**: 모든 승격 과정 로그 기록
4. **되돌릴 수 없음**: 명확한 경고 및 확인 절차

## 🚀 사용 방법

1. 테스트 모드로 조 추첨 실행
2. 테스트 모드로 경기 자동 생성
3. 테스트 결과 검증 (조/경기 수 확인)
4. 관리자 확인 체크
5. "공식 조 추첨 및 대진표 확정" 버튼 클릭
6. 2차 확인 모달에서 "확정" 클릭
7. 승격 완료 → 공식 기록 반영
