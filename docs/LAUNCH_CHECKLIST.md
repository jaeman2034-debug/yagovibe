## 🔐 A. Firebase / 인프라

### Auth (인증)

- [ ] **Phone Provider 활성화**
  - Firebase Console → Authentication → Sign-in method
  - Phone 번호 인증 **ON** 확인
  - 테스트 모드 비활성화 (PROD 환경)

- [ ] **테스트 번호 관리**
  - DEV 환경: 테스트 번호 등록 OK
  - PROD 환경: 테스트 번호 **전부 삭제**
  - 확인: `VITE_AUTH_MODE=prod`일 때 테스트 번호 차단 확인

- [ ] **Authorized domains 등록**
  - Firebase Console → Authentication → Settings → Authorized domains
  - PROD 도메인 추가 (예: `yagovibe.com`, `www.yagovibe.com`)
  - `localhost`는 DEV 전용으로만 사용

- [ ] **reCAPTCHA 설정**
  - Invisible reCAPTCHA 정상 동작 확인
  - `VITE_RECAPTCHA_SITE_KEY` 환경 변수 설정 확인
  - SMS 전송 시 reCAPTCHA 렌더링 확인

- [ ] **App Check (선택)**
  - `VITE_USE_APP_CHECK` 환경 변수 확인
  - PROD에서 App Check 사용 시 Secret Key 설정 확인

---

### Firestore (데이터베이스)

- [ ] **Rules 배포 완료**
  - `firestore.rules` 최신 버전 확인
  - `firebase deploy --only firestore:rules` 실행
  - 배포 후 콘솔에서 Rules 확인

- [ ] **보안 Rules 확인**
  - `auth_logs` read 차단 / admin만 허용 확인
  - `users.role` 클라이언트 수정 불가 확인
  - `deleted` 유저 접근 차단 확인

- [ ] **인덱스 에러 없음**
  - Firebase Console → Firestore → Indexes
  - 에러 표시 ❌ 확인
  - 필요한 인덱스 자동 생성 대기 완료

- [ ] **컬렉션 구조 확인**
  - `users/{uid}` 구조 확인
  - `auth_logs` 컬렉션 존재 확인
  - `eventLogs` 컬렉션 존재 확인

---

### Functions (서버리스)

- [ ] **Slack Webhook 환경 변수**
  - `firebase functions:config:get` 실행
  - `slack.webhook` 값 확인
  - 테스트 알림 전송 확인

- [ ] **Functions 리전 통일**
  - 모든 Functions 리전 확인 (예: `asia-northeast3`)
  - 리전 불일치 시 수정

- [ ] **로그 폭주 방지**
  - `alertOnPhoneAuthError` 함수 로그 확인
  - 불필요한 로그 제거
  - 에러 로그만 남기기

- [ ] **Functions 배포 확인**
  - `firebase deploy --only functions` 실행
  - 배포 성공 확인
  - Functions 콘솔에서 함수 목록 확인

## 🚦 F. 배포 직전 최종 테스트 (강추)

### 시나리오 1: 신규 번호 → 가입 → 온보딩 완료

1. **신규 전화번호로 가입**
   - 전화번호 입력
   - SMS 인증번호 수신
   - 인증번호 입력 및 확인

2. **자동 유저 생성 확인**
   - Firestore에 `users/{uid}` 문서 생성 확인
   - `onboardingCompleted: false` 확인
   - `status: "active"` 확인

3. **온보딩 완료**
   - 온보딩 단계별 진행
   - 완료 버튼 클릭
   - 홈 페이지로 이동 확인

**✅ 성공 기준:**
- SMS 수신 정상
- 유저 문서 생성 정상
- 온보딩 완료 후 홈 이동 정상

---

### 시나리오 2: 온보딩 중 이탈 → 재접속

1. **온보딩 시작**
   - 로그인 후 온보딩 시작
   - 2단계까지 진행

2. **이탈**
   - 브라우저 닫기 또는 다른 페이지 이동

3. **재접속**
   - 다시 로그인
   - 온보딩 페이지 접근
   - 2단계에서 이어지는지 확인

**✅ 성공 기준:**
- `onboardingStep: 2` 유지 확인
- 이전 입력 데이터 유지 확인
- 2단계에서 이어짐

---

### 시나리오 3: 잘못된 인증번호

1. **SMS 인증번호 요청**
   - 전화번호 입력
   - SMS 인증번호 수신

2. **잘못된 인증번호 입력**
   - 잘못된 번호 입력
   - 확인 버튼 클릭

3. **에러 메시지 확인**
   - 사용자 친화적 에러 메시지 표시
   - 재시도 가능 확인

**✅ 성공 기준:**
- 에러 메시지가 기술적 코드가 아님
- 재시도 가능
- 최대 재시도 제한 확인

---

### 시나리오 4: SMS 연속 요청 → 제한 메시지

1. **SMS 연속 요청**
   - 같은 번호로 SMS 여러 번 요청
   - 쿨다운 타이머 확인

2. **제한 메시지 확인**
   - 일일 제한 도달 시 안내 메시지
   - 다음 날 재시도 안내

**✅ 성공 기준:**
- 쿨다운 타이머 정상 작동
- 제한 메시지 명확
- 다음 날 재시도 안내 확인

---

### 시나리오 5: 탈퇴 → 재가입

1. **탈퇴**
   - 마이페이지에서 탈퇴 버튼 클릭
   - 확인 다이얼로그 확인
   - 탈퇴 완료

2. **재로그인 시도**
   - 동일 번호로 로그인 시도
   - 로그인 불가 확인

3. **재가입**
   - 동일 번호로 새로 가입
   - 새 UID로 새 문서 생성 확인
   - 기존 데이터는 `status: "deleted"` 유지 확인

**✅ 성공 기준:**
- 탈퇴 후 로그인 불가
- 재가입 가능
- 새 문서 생성 확인
