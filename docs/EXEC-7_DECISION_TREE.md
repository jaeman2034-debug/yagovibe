# 🔒 100명 이후 의사결정 트리 (GO / FIX / KILL)

## 목적

**감(感)·회의·토론 없이 숫자 5개로 다음 액션을 자동 결정한다.**

---

## 0️⃣ 전제 (아주 중요)

- 기능 추가 ❌
- 피드백 반영 ❌
- "느낌상 별로" ❌

👉 **오직 KPI로만 판단**

---

## 1️⃣ 판단에 쓰는 지표 (EXEC-5에서 고정한 5개)

다시 적는다. 다른 건 보지 않는다.

1. **QR Entry** - QR 유입량
2. **Guest Avg Views** - 게스트 평균 조회
3. **Auth Required Rate** - Lazy Signup 트리거율
4. **Signup Conversion** - 가입 전환율
5. **Post-signup Resume Rate** - 복귀 성공률

---

## 2️⃣ 최상위 분기 (제일 먼저 본다)

### ❓ 질문 1

**QR Entry가 하루 10 이상인가?**

#### NO →

- ❌ 제품 문제 아님
- ❌ UX 문제 아님
- ⭕ **유통 문제**

**액션:**
- QR 위치 변경
- 상품/채팅 QR 비중 ↑
- 오프라인 밀도 ↑
- 제품 수정 금지

#### YES → 다음 질문

---

## 3️⃣ 가치 전달 분기

### ❓ 질문 2

**Guest Avg Views ≥ 3 인가?**

#### NO →

- ❌ 가입 UX 문제 아님
- ⭕ **마켓/상품 매력 문제**

**액션:**
- 상품 썸네일 교체
- 가격/사진 가시성 개선
- ❌ 가입·QR 건드리지 말 것

#### YES → 다음 질문

---

## 4️⃣ 행동 욕구 분기 (Lazy Signup 전)

### ❓ 질문 3

**Auth Required / QR Entry ≥ 15% 인가?**

#### NO →

- ⭕ **"보고는 보는데 하고 싶진 않음"**

**액션:**
- 채팅 버튼 위치 ↑
- 판매 버튼 강조
- ❌ 가입 UX 수정 금지

#### YES → 다음 질문

---

## 5️⃣ 가입 마찰 분기

### ❓ 질문 4

**Signup Completed / Auth Required ≥ 40% 인가?**

#### NO →

- ⭕ **가입 UX 문제**

**액션:**
- 가입 수단 줄이기
- 문구 더 짧게
- 전화번호 OTP UX 개선
- ❌ QR/게스트 구조 건드리지 말 것

#### YES → 다음 질문

---

## 6️⃣ 제품 완성도 분기 (가장 중요)

### ❓ 질문 5

**Post-signup Resumed ≥ 80% 인가?**

#### NO →

- 🔥 **구조 실패**

**액션:**
- postSignupAction 저장/복구 전수 점검
- 자동 실행 여부 확인
- ❌ 기능 추가 전면 금지

#### YES → **GO**

---

## 7️⃣ GO 판정 시에만 허용되는 다음 단계

### ⭕ 이제서야 가능한 것

- 추천 노출 실험
- 인기 상품 정렬
- 알림
- 결제 고도화

### ❌ GO 전엔 절대 금지

- "이제 마케팅해보자"
- "AI 붙이자"
- "프로필 예쁘게"
- "온보딩 추가"

---

## 8️⃣ KILL 판정 기준 (냉정)

### 아래 중 2개 이상이면 중단 고려

- QR Entry 안 나옴 (<5/day)
- Guest Avg Views < 2
- Auth Required < 10%
- Signup Conversion < 30%
- Resume < 60%

👉 **이건 UX 문제가 아니라 '시장/맥락 문제'**

### KILL 판정 시 액션

- 현재 구조 유지
- 시장/맥락 재검토
- 피봇 고려

---

## 📊 의사결정 트리 전체 구조

```
질문 1: QR Entry ≥ 10?
  ├─ NO → 유통 문제 (QR 위치 변경)
  └─ YES → 질문 2

질문 2: Guest Avg Views ≥ 3?
  ├─ NO → 마켓/상품 매력 문제 (썸네일/가격 개선)
  └─ YES → 질문 3

질문 3: Auth Required Rate ≥ 15%?
  ├─ NO → 행동 욕구 문제 (버튼 위치/강조)
  └─ YES → 질문 4

질문 4: Signup Conversion ≥ 40%?
  ├─ NO → 가입 UX 문제 (문구/단계 개선)
  └─ YES → 질문 5

질문 5: Resume Rate ≥ 80%?
  ├─ NO → 구조 실패 (복귀 로직 점검)
  └─ YES → GO
```

---

## 🎯 판정 결과별 액션 정리

### GO (모든 질문 YES)

- ✅ 구조 검증 완료
- ✅ 다음 단계 진행 가능 (추천, 알림, 결제 등)
- ✅ 스케일 준비 시작

### FIX (특정 질문 NO)

- ⚠️ 해당 지표만 수정
- ⚠️ 구조는 절대 건드리지 않음
- ⚠️ 문구/위치/가시성만 미세 조정

### KILL (2개 이상 NO)

- 🔴 UX 문제가 아님
- 🔴 시장/맥락 재검토 필요
- 🔴 피봇 고려
