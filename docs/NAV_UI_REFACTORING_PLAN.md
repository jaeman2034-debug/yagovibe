# 🔥 네비 UI 리팩토링 계획

## 🧨 현재 UI의 가장 큰 결함

1. ❌ **출발 대기 (Pre-Navigation) 상태가 존재하지 않음**
2. ❌ 출발 버튼 클릭 시 UI 변화가 없음
3. ❌ "내 위치에서 시작한다"는 감각이 없음

---

## 📋 리팩토링 우선순위

### 🔴 **PRIORITY 1: 출발 대기 상태 추가 (가장 시급)**

#### 현재 문제
- CONFIRMED 상태에서 바로 "출발" 버튼이 보임
- 출발 버튼 클릭 시 즉시 NAVIGATING으로 전환되어 사용자가 준비할 시간이 없음
- 경로가 계산되기 전에 출발 버튼이 노출됨

#### 해결 방안
1. **새로운 상태 추가: `PRE_NAVIGATING`**
   - CONFIRMED → PRE_NAVIGATING → NAVIGATING
   - CONFIRMED에서 "여기에 갈게요" 버튼 클릭 시 PRE_NAVIGATING 진입
   - PRE_NAVIGATING에서 경로 계산 완료 후 "출발" 버튼 노출

2. **UI 변화**
   - CONFIRMED: "여기에 갈게요" 버튼 (경로 계산 전)
   - PRE_NAVIGATING: 경로 계산 중 로딩 + "출발" 버튼 (경로 계산 완료 후)
   - NAVIGATING: 상단 상태바 + 경로선 + "이동 중..." 카드

#### 구현 순서
1. `MapPageContainer.tsx`에 `PRE_NAVIGATING` phase 추가
2. `NavigationCard.tsx`에 PRE_NAVIGATING 모드 추가
3. CONFIRMED → PRE_NAVIGATING 전환 로직 추가
4. PRE_NAVIGATING에서 경로 계산 완료 대기
5. 경로 계산 완료 시 "출발" 버튼 활성화
6. "출발" 버튼 클릭 시 NAVIGATING 전환

---

### 🟠 **PRIORITY 2: 출발 버튼 클릭 시 즉각적인 UI 변화**

#### 현재 문제
- 출발 버튼 클릭 후 아무 변화가 없어서 "버튼이 먹혔는지" 모름
- 경로 계산이 완료될 때까지 대기 상태가 불명확

#### 해결 방안
1. **출발 버튼 클릭 즉시**
   - 버튼 비활성화 (로딩 상태)
   - 상단 상태바 즉시 표시 ("경로 계산 중...")
   - 하단 카드 업데이트 ("경로 계산 중...")

2. **경로 계산 완료 시**
   - 상단 상태바 업데이트 (거리/시간 표시)
   - 파란 경로선 표시
   - 하단 카드 업데이트 ("이동 중..." + 거리/시간)

#### 구현 순서
1. `handleStartNavigation`에서 즉시 UI 상태 업데이트
2. 상단 상태바 즉시 표시 (로딩 상태)
3. 하단 카드 즉시 업데이트
4. 경로 계산 완료 시 UI 업데이트

---

### 🟡 **PRIORITY 3: "내 위치에서 시작" 감각 강화**

#### 현재 문제
- 출발지가 UI에 명확히 표시되지 않음
- "내 위치에서 출발한다"는 개념이 없음

#### 해결 방안
1. **CONFIRMED 상태**
   - 지도에 출발지 마커 추가 (파란 점 + "내 위치" 라벨)
   - 출발지 → 목적지 방향 화살표 표시

2. **PRE_NAVIGATING 상태**
   - 출발지 마커 강조 (pulse 애니메이션)
   - "내 위치에서 출발합니다" 텍스트 표시

3. **NAVIGATING 상태**
   - 출발지 마커 유지 (파란 점)
   - 경로선이 출발지에서 시작하는 것을 명확히 표시

#### 구현 순서
1. `WebMapRenderer.tsx`에 출발지 마커 추가
2. CONFIRMED/PRE_NAVIGATING 상태에서 출발지 마커 표시
3. 출발지 → 목적지 방향 화살표 추가
4. 출발지 마커 pulse 애니메이션 추가

---

### 🟢 **PRIORITY 4: 상태별 검색바 처리 개선**

#### 현재 문제
- NAVIGATING 상태에서 검색바가 숨겨지지만, 다른 상태에서의 검색바 처리가 불명확

#### 해결 방안
1. **IDLE/SEARCHING**: 검색바 표시 (활성)
2. **CONFIRMED**: 검색바 유지 (입력값 고정, ✕ 아이콘 추가)
3. **PRE_NAVIGATING**: 검색바 숨김
4. **NAVIGATING**: 검색바 숨김
5. **ARRIVED**: 검색바 숨김

#### 구현 순서
1. 각 phase별 검색바 표시 조건 명확화
2. CONFIRMED 상태에서 검색바 입력값 고정
3. CONFIRMED 상태에서 ✕ 아이콘 추가 (선택 취소)

## 🔧 구체적인 코드 수정 계획

### 1. Phase 타입 확장

```typescript
// MapPageContainer.tsx
type Phase = 'IDLE' | 'SEARCHING' | 'CONFIRMED' | 'PRE_NAVIGATING' | 'NAVIGATING' | 'ARRIVED';
```

### 2. NavigationCard 모드 확장

```typescript
// NavigationCard.tsx
type NavigationMode = 'CONFIRMED' | 'PRE_NAVIGATING' | 'NAVIGATING' | 'ARRIVED';
```

### 3. 상태 전환 로직 수정

```typescript
// CONFIRMED → PRE_NAVIGATING
const handleConfirmDestination = () => {
  setPhase('PRE_NAVIGATING');
  // 경로 계산 시작
  calculateRoute();
};

// PRE_NAVIGATING → NAVIGATING
const handleStartNavigation = () => {
  if (routeInfo) {
    setPhase('NAVIGATING');
  }
};
```

### 4. UI 조건부 렌더링 수정

```typescript
// 검색바 표시 조건
const shouldShowSearchBar = phase === 'IDLE' || phase === 'SEARCHING' || phase === 'CONFIRMED';

// 상단 상태바 표시 조건
const shouldShowStatusBar = phase === 'PRE_NAVIGATING' || phase === 'NAVIGATING';

// 하단 카드 모드
const cardMode = phase === 'CONFIRMED' ? 'CONFIRMED' :
                 phase === 'PRE_NAVIGATING' ? 'PRE_NAVIGATING' :
                 phase === 'NAVIGATING' ? 'NAVIGATING' :
                 phase === 'ARRIVED' ? 'ARRIVED' : null;
```
