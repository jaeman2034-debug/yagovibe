## 1️⃣ 가이드 #1 (핵심 롱테일)

### 제목

```
동호회 활동 기록, 어떻게 남기면 좋을까요?
```

**검색 의도:**
- "동호회 기록"
- "팀 활동 기록 남기기"

---

### 개요

단톡방에 흩어지는 활동 기록의 한계와  
왜 '공개 기록'이 팀에 도움이 되는지 설명

---

### 구성

**1. 동호회 기록이 남지 않는 이유**
- 단톡방에 흩어지는 활동
- 누가 언제 모였는지 남지 않음
- 시간이 지나면 기억에서 사라짐

**2. 기록이 사라질 때 생기는 문제**
- 새로운 사람은 팀을 알 방법이 없음
- 신뢰 형성이 어려움
- 팀의 역사가 남지 않음

**3. 기록을 남기면 생기는 변화**
- 날짜와 맥락이 남음
- 자연스러운 신뢰 형성
- 궁금한 사람들이 먼저 찾아옴

**4. 기록을 잘 남기는 기준 3가지**
- 자동으로 기록되는가
- 공개되어 누구나 볼 수 있는가
- 꾸준히 쌓이는가

**5. 실제로 기록이 쌓인 팀의 공통점**
- 관리자가 바빠도 기록은 계속됨
- 사람이 빠져도 기록은 남음
- 기록이 쌓일수록 신뢰가 커짐

---

### 핵심 문장

**"기록은 관리가 아니라, 다음 사람을 위한 배려입니다."**

---

### CTA (하단)

```
👉 팀 활동 기록 예시 보기
(기록장 팀 페이지 링크)
```

## 2️⃣ 가이드 #2 (노년·생활체육 특화)

### 제목

```
생활체육 팀은 왜 홍보보다 기록이 중요할까요?
```

**검색 의도:**
- "생활체육 팀 홍보"
- "노년 동호회 운영"

---

### 개요

홍보가 오히려 부담이 되는 이유와  
기록이 신뢰를 만드는 과정 설명

---

### 구성

**1. 홍보가 어려운 팀의 공통 상황**
- 홍보 담당자가 없음
- 홍보 문구 작성이 부담
- 지속적인 홍보가 어려움

**2. 생활체육 팀이 겪는 현실적인 부담**
- 운영자가 바쁨
- 홍보에 시간을 쓸 여유가 없음
- 홍보가 오히려 부담이 됨

**3. 기록이 신뢰로 바뀌는 순간**
- 기록이 쌓이면 자연스럽게 신뢰 형성
- 홍보 없이도 사람이 찾아옴
- 기록이 팀의 신뢰를 만듦

**4. 과장 없는 기록이 오래 가는 이유**
- 사실 위주의 기록이 신뢰를 만듦
- 과장된 표현은 오히려 신뢰를 해침
- 기록은 쌓일수록 힘이 생김

**5. 기록이 있는 팀과 없는 팀의 차이**
- 기록이 있는 팀: 자연스러운 신뢰 형성
- 기록이 없는 팀: 보이지 않아 사라짐

---

### 핵심 문장

**"생활체육 팀은 잘 보이기보다, 그대로 보이는 게 중요합니다."**

---

### CTA

```
👉 기록으로 운영되는 팀 사례 보기
```

## 3️⃣ 가이드 #3 (관리자 공감형)

### 제목

```
팀 운영, 왜 항상 한 사람에게만 부담이 될까요?
```

**검색 의도:**
- "동호회 운영 힘들다"
- "팀 관리자 부담"

---

### 개요

운영 피로의 원인을 짚고  
'기록 자동화'가 왜 해결책이 되는지 설명

---

### 구성

**1. 팀 운영이 힘들어지는 순간들**
- 공지 작성
- 후기 작성
- 홍보 관리
- 모든 것이 한 사람에게 집중

**2. 기록을 맡는 사람이 생기는 구조**
- 누군가가 기록을 맡아야 함
- 그 사람이 빠지면 기록이 멈춤
- 팀 운영이 한 사람에게 의존

**3. 자동 기록이 바꾸는 역할 분산**
- 활동만 하면 자동으로 기록
- 관리자가 바빠도 기록은 계속됨
- 역할 분산이 자연스럽게 일어남

**4. 운영자가 빠져도 팀이 유지되는 이유**
- 기록이 자동으로 쌓임
- 팀의 역사가 남음
- 다음 사람이 이어받기 쉬움

**5. 기록이 남기는 가장 큰 가치**
- 팀의 시간이 남음
- 다음 사람을 위한 배려
- 팀이 오래 가는 기반

---

### 핵심 문장

**"팀이 오래 가려면 누군가의 희생 위에 서면 안 됩니다."**

---

### CTA

```
👉 기록 자동화가 어떻게 작동하는지 보기
```

## 4️⃣ 공통 본문 템플릿

### 서론 (1문단)

```
[문제 상황]을 겪고 계신가요?

[구체적 어려움] 때문에 [결과]가 발생합니다.

기록장은 이런 문제를 해결하기 위해 만들어졌습니다.
```

---

### 본론 (3~4문단)

**1. 기록장 소개 (1문단)**
```
기록장은 [핵심 기능]을 자동으로 [결과]하는 서비스입니다.

[구체적 설명]이 [방식]으로 [결과]됩니다.
```

**2. 작동 방식 (1~2문단)**
```
[단계 1 설명]

[단계 2 설명]

[단계 3 설명]
```

**3. 차별점 (1문단)**
```
[기존 방식]과 달리 [차별점]을 제공합니다.

[구체적 차별점 설명]
```

---

### 결론 (1문단)

```
[가치 제안]

[자연스러운 CTA 연결]
```

## 5️⃣ SEO 최적화

### 제목 최적화

- 검색 키워드 포함
- 60자 이내
- 명확한 가치 제안

---

### 본문 최적화

- 키워드 자연스럽게 포함
- H2/H3 태그 활용
- 내부 링크 구조

---

### 메타 태그

**Title:**
```
[제목] | 기록장
```

**Description:**
```
[요약 2줄] 기록장으로 [핵심 가치]를 경험해 보세요.
```
