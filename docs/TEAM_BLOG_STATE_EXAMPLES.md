## 5️⃣ 상태별 데이터 구조

### TeamPageState

```typescript
interface TeamPageState {
  hasRecentActivity: boolean;  // 14일 이내 활동 있음
  hasBlogs: boolean;           // 블로그 포스트 있음
  userRole: 'guest' | 'member' | 'pending' | 'admin';
  showProNotice: boolean;      // 관리자 + 조건 충족
  plan: 'free' | 'pro';        // 팀 플랜
}
```

### 예시 데이터

```typescript
// Guest 상태
const guestState: TeamPageState = {
  hasRecentActivity: true,
  hasBlogs: true,
  userRole: 'guest',
  showProNotice: false,
  plan: 'free'
};

// Member 상태
const memberState: TeamPageState = {
  hasRecentActivity: true,
  hasBlogs: true,
  userRole: 'member',
  showProNotice: false,
  plan: 'free'
};

// Admin 상태
const adminState: TeamPageState = {
  hasRecentActivity: true,
  hasBlogs: true,
  userRole: 'admin',
  showProNotice: true,  // 조건 충족 시
  plan: 'pro'
};

// Pending 상태
const pendingState: TeamPageState = {
  hasRecentActivity: true,
  hasBlogs: true,
  userRole: 'pending',
  showProNotice: false,
  plan: 'free'
};
```

## 6️⃣ 상태별 CTA 동작 규칙

| 상태 | Primary CTA | Secondary CTA | 동작 |
|------|------------|---------------|------|
| guest | 함께 운동하기 | 팀 소개 보기 | 로그인/가입 요청 |
| member | 일정 보기 | - | 팀 대시보드 이동 |
| pending | 가입 승인 대기중 (비활성) | 팀 소개 보기 | - |
| admin | 팀 관리하기 | 팀 소개 보기 | 팀 대시보드 이동 |
