# 💬 채팅 자연어 기능 설계

## 핵심 철학

**"거래 성사의 마지막 병목을 제거한다"**

검색 잘 됨 ✅  
알림 잘 옴 ✅  
근데 채팅에서 죽는다 ❌

## 목표

상품 상세페이지에서 "사람이 보통 하는 말"을 버튼 + 자동 문장으로 만들어준다.

❌ 챗봇 아님  
❌ 자유 대화 아님  
👉 **거래용 자연어 자동화**

## 채팅 자연어의 정체

채팅은 사실 이 4가지뿐이다:

1. 재고/상태 확인
2. 가격 흥정
3. 거래 방식/위치
4. 시간 조율

👉 이걸 100% 커버하면 끝

## 구현 항목

### 1️⃣ 상품 상세 → "자동 질문" 생성

**구매자용 퀵 메시지:**
- "아직 판매 중인가요?"
- "실사용감 어느 정도인가요?"
- "직거래 위치가 어디쯤인가요?"
- "오늘 거래 가능할까요?"
- "가격 조정 가능할까요?"

**UI:**
```
[💬 아직 판매 중인가요?]
[📍 직거래 위치가 어디쯤인가요?]
[💸 가격 조정 가능할까요?]
```

👉 타이핑 0, 진입 장벽 0

### 2️⃣ 판매자 자동 응답

**규칙:**
"판매자가 쓸 말을 미리 만들어준다"

**자동 응답 템플릿:**
- 재고: "네 아직 판매 중입니다!"
- 위치: "직거래는 {locationText} 근처에서 가능합니다."
- 상태: "실사용은 거의 없고 {condition}입니다."
- 가격: "{price}원 생각하고 있습니다."

**판매자 UI:**
```
[자동 답장 보내기]
```

👉 귀찮음 제거 = 응답 속도 ↑

### 3️⃣ 채팅 자연어 "의도 분류" (룰 기반)

**구매자 메시지 예시:**
- "아직 있어요?" → availability
- "얼마까지 가능해요?" → price
- "어디서 거래해요?" → location

**룰 파서 (AI ❌):**
```typescript
function parseChatIntent(text: string) {
  if (text.includes("있")) return "availability";
  if (text.includes("얼마") || text.includes("가격")) return "price";
  if (text.includes("어디") || text.includes("위치")) return "location";
  // ...
}
```

✔ 빠름  
✔ 예측 가능  
✔ 실패해도 문제 없음

### 4️⃣ 거래 성사 가속 버튼

**구매자:**
- [📍 위치 제안]
- [🕒 시간 제안]
- [💰 가격 제안]

**판매자:**
- [✅ 거래 확정]
- [📍 위치 보내기]
- [🕒 시간 확정]

👉 대화 → 행동으로 전환

### 5️⃣ (선택) AI는 여기만 쓴다

**AI는 "문장 다듬기"에만**

AI 입력: "홍대역 근처에서 저녁에 거래 가능"  
AI 출력: "홍대입구역 근처에서 오늘 저녁 거래 가능합니다!"

❌ 의도 판단 ❌  
❌ 가격 결정 ❌  
✔ 말투만 자연스럽게
