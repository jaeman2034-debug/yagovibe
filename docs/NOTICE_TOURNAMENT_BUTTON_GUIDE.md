# 🏆 "이 공지로 대회 생성" 버튼 사용 가이드

## ✅ 여기서 반드시 클릭해야 할 것 (정답)

### 🏆 「🏆 이 공지로 대회 생성」 버튼

**위치**: 공지 상세 페이지 하단 중앙

**표시 조건**:
- 관리자만 보임 (`isAdmin`)
- `published` 상태 공지만 표시

**버튼 모양**:
- 트로피 아이콘 🏆
- "이 공지로 대회 생성" 텍스트
- 파란색 배경 (기본 버튼 스타일)

---

## 🔁 클릭 후 실제로 일어나는 일

### 1️⃣ URL 이동

**이동 경로**:
```
/association/{associationId}/admin/tournaments?fromNotice={noticeId}
```

**예시**:
```
/association/abc123/admin/tournaments?fromNotice=PP2QyFUvCD5HQXpCbh9q
```

### 2️⃣ 대회 등록 페이지 자동 열림

**자동으로 채워지는 항목**:
- ✅ **제목** → 공지 제목
- ✅ **본문** → 공지 본문
- ✅ **참가비 정책** → 공지 기준 (`feePolicy`)
- ✅ **공식 기준 여부** → 자동 체크
- ✅ **장소** → 본문에서 자동 추출 (패턴 매칭)
- ✅ **기간** → 본문에서 자동 추출 (날짜 패턴)

### 3️⃣ 화면 표시

**헤더 배지**:
- "공지에서 생성된 대회" 뱃지 표시

**본문 안내**:
- 파란색 배경 안내 박스
- "본 대회는 선택한 공지 내용을 기준으로 생성되었습니다"

## ❌ 여기서는 클릭해도 "업로드" 안 되는 것들

| 위치 | 이유 |
|------|------|
| 상단 수정 | 공지 편집 (대회 생성과 무관) |
| 이 공지로 문의하기 | 문의 작성 (대회 생성과 무관) |
| 상단 배지(공식 기준) | 상태 표시 (클릭 불가) |
| 스크롤 위쪽 텍스트 | 단순 안내 (클릭 불가) |

👉 **오직 '이 공지로 대회 생성'만 연결됨**

---

## 🧠 왜 이렇게 설계했나 (중요)

### "자동 업로드"는 의도적 행동일 때만

**이유**:
1. **실수로 공지 내용이 대회에 들어가는 사고 방지**
2. **감사/행정 대응 시 명확성**
   - "이 대회는 이 공지에서 생성됨" 명확
   - `sourceNoticeId` 필드로 추적 가능
3. **운영 안전성**
   - 공지 = 안내 기준
   - 대회 = 운영 기록
   - 자동 동기화 방지

**그래서 버튼을 아래에, 명확하게 둔 것임**

## 📋 정상 동작 확인 체크리스트

### 버튼 표시 확인
- [ ] 관리자로 로그인했는가?
- [ ] 공지 상태가 `published`인가?
- [ ] 버튼이 화면 하단 중앙에 보이는가?

### 클릭 후 동작 확인
- [ ] URL에 `?fromNotice={noticeId}`가 붙는가?
- [ ] 대회 등록 페이지가 열리는가?
- [ ] 제목이 자동으로 채워지는가?
- [ ] 본문이 자동으로 채워지는가?
- [ ] 헤더에 "공지에서 생성된 대회" 배지가 보이는가?

### 저장 후 확인
- [ ] 대회가 정상적으로 생성되는가?
- [ ] `sourceNoticeId` 필드가 저장되는가?

## 🚨 문제 해결

### Q: 버튼이 안 보여요
- A: 관리자 권한 확인
- A: 공지 상태가 `published`인지 확인

### Q: 버튼을 눌렀는데 빈 폼이 나와요
- A: URL에 `?fromNotice={noticeId}`가 있는지 확인
- A: 공지 ID가 올바른지 확인

### Q: 자동 입력이 안 돼요
- A: 브라우저 콘솔에서 에러 확인
- A: 공지 데이터가 올바르게 로드되는지 확인
