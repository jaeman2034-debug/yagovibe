## 🚨 사고 유형별 대응

### 1. SMS 전송 실패 급증

**징후:**
- Slack 알림 급증
- `sms_error` 로그 급증
- 사용자 문의 급증

**즉시 대응 (5분 내):**

1. **상황 파악**
   ```bash
   # Firebase Console 확인
   - Authentication → Usage → Phone Auth
   - SMS 전송량 확인
   - 쿼터 초과 여부 확인
   ```

2. **로그 분석**
   ```javascript
   // auth_logs 컬렉션 확인
   const errors = await getDocs(
     query(
       collection(db, "auth_logs"),
       where("type", "==", "sms_error"),
       where("createdAt", ">=", oneHourAgo),
       orderBy("createdAt", "desc")
     )
   );
   ```

3. **원인 파악**
   - 쿼터 초과: Firebase Console에서 쿼터 증가 요청
   - Firebase 서비스 장애: Firebase Status 확인
   - 특정 번호/지역 문제: `auth_logs` 패턴 분석

**대응 조치:**

- **쿼터 초과:**
  - Firebase Console → Billing → Quotas
  - 쿼터 증가 요청
  - 사용자 안내 메시지 준비

- **Firebase 서비스 장애:**
  - Firebase Status 페이지 확인
  - 사용자 공지 준비
  - 대체 인증 방법 검토

- **특정 번호/지역 문제:**
  - `auth_logs` 분석
  - 통신사별 문제 확인
  - 제한 완화 검토

**복구 확인:**
- [ ] SMS 전송 성공률 95% 이상 회복
- [ ] 사용자 문의 감소
- [ ] Slack 알림 정상화

### 2. 서비스 접속 불가

**징후:**
- 사이트 접속 불가
- 404/500 에러 발생
- Functions 에러 발생

**즉시 대응 (5분 내):**

1. **상황 파악**
   ```bash
   # Firebase Hosting 상태 확인
   firebase hosting:channel:list
   
   # Functions 상태 확인
   firebase functions:log --limit 50
   ```

2. **원인 파악**
   - Hosting 배포 실패: 재배포 시도
   - Functions 에러: 로그 확인
   - 도메인 DNS 문제: DNS 확인

**대응 조치:**

- **Hosting 배포 실패:**
  ```bash
  # 재배포
  firebase deploy --only hosting
  ```

- **Functions 에러:**
  ```bash
  # Functions 로그 확인
  firebase functions:log --limit 100
  
  # 에러 수정 후 재배포
  firebase deploy --only functions
  ```

- **도메인 DNS 문제:**
  - DNS 설정 확인
  - 도메인 제공업체 확인
  - 필요 시 긴급 복구

**복구 확인:**
- [ ] 사이트 접속 정상
- [ ] 모든 기능 정상 작동
- [ ] 에러 로그 없음
