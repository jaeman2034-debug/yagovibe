## ✅ 완성된 핵심 기능

### 🧠 구조 (Structure)

#### 1. 상태 머신 (State Machine)
- **파일**: `src/utils/userState.ts`
- **상태**: `P1-A` (팀 없음) | `P1-W` (대기 중) | `P2` (팀원) | `P3` (팀장)
- **함수**: `getUserState()` - 단일 진실 소스
- **효과**: 조건 분기 혼란 끝, 이후 기능 추가 속도 2배 이상

#### 2. `/me` 페이지 - 단일 진실의 허브
- **파일**: `src/pages/me/MePage.tsx`
- **역할**: 모든 상태의 중심점
- **기능**:
  - 상태별 UI 분기
  - 실시간 알림 처리
  - CTA 레일 명확

#### 3. 팀 관리 시스템
- **대시보드**: `src/pages/team/TeamManageDashboard.tsx`
  - 팀 상태 요약
  - 해야 할 일 (To-Do)
  - 팀 관리 메뉴
- **탭 구조**:
  - `JoinRequestsTab`: 가입 요청 관리
  - `MembersTab`: 팀원 관리 (위임/추방)
  - `SettingsTab`: 팀 설정

### 🔐 질서 (Order)

#### 1. 권한 가드
- **파일**: `src/components/guard/CaptainOnlyRoute.tsx`
- **역할**: 팀장(P3)만 접근 가능한 라우트 보호
- **체크**: 상태 머신 우선 → team_members → teams 컬렉션 이중 체크

#### 2. 서버 권한 체크
- **Cloud Functions**:
  - `transferOwner`: 팀장 위임
  - `removeMember`: 팀원 추방
- **원칙**: 프론트는 UX, 서버는 질서

#### 3. 안전한 트랜잭션
- 가입 승인/거절
- 팀장 위임
- 팀원 추방
- 가입 요청 취소

### 🧭 UX (Experience)

#### 1. CTA 레일 명확
- **P1-A**: 팀 만들기 / 팀 찾기 (동등한 무게)
- **P1-W**: 가입 요청 대기 중 (취소 가능)
- **P2**: 팀 정보 보기
- **P3**: 팀 관리하기

#### 2. 대기 상태 공백 제거
- **P1-W UX**: `PendingRequestCard` 컴포넌트
- 실시간 상태 반영
- 취소 기능 제공

#### 3. 상태 변화 알림
- **알림 타입**:
  - `TEAM_JOIN_APPROVED`: 팀 가입 승인
  - `TEAM_CAPTAIN_DELEGATED`: 팀장 위임
  - `TEAM_MEMBER_REMOVED`: 팀원 추방
- **표시**: `/me` 진입 시 자동 토스트
- **중복 방지**: 처리된 알림 ID 추적

## 🔄 상태 전이 다이어그램

```

 ├─ 팀 만들기 ─────────▶ P3 (팀장)

                             │
                             ├─ 승인 ─▶ P2 (팀원)
                             └─ 거절/취소 ─▶ P1-A

P2 (팀원)
 ├─ 팀장 위임 받음 ─────▶ P3 (팀장)
 └─ 추방 ─────────────▶ P1-A (팀 없음)

P3 (팀장)
 └─ 팀장 위임 ─────────▶ P2 (팀원)
```

## 🎯 핵심 원칙 (영구 봉인)

### 1. 상태 머신 = 단일 진실 소스
- 모든 UI/라우트/CTA는 `getUserState()` 결과에만 의존
- 조건 분기 흩어놓지 말고 한 곳에 모음

### 2. 권한 = 서버가 최종 심판
- 프론트는 UX (버튼 가시성, 라우트 가드)
- 서버는 질서 (Cloud Functions에서 최종 검증)

### 3. 상태 변화 = 사건으로 인식
- 알림으로 즉시 전달
- 토스트로 즉각 피드백
- 중복 표시 방지

### 4. 대기 상태 = 공백 없음
- P1-W 상태 명확히 표시
- 취소 기능 제공
- 실시간 상태 반영
