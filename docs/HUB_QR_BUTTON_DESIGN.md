# 🔒 HUB 상단 헤더 우측 QR 아이콘 설계 (FINAL LOCK)

## 0️⃣ 전제 재확인

- 위치: 상단 헤더 우측
- 형태: 아이콘 단독
- 역할: 전파/데모 보조 스위치
- 절대: 메인 CTA ❌ / 유입 버튼 ❌

```
YAGO SPORTS                          ⬜ (QR 아이콘)
```

---

## 1️⃣ UI 설계 (보는 순간 헷갈림 0)

### 아이콘 규칙

- 아이콘: QR 심볼 단독 (QrCode from lucide-react)
- 크기: 알림/프로필 아이콘과 동일 (20px)
- 색상: 헤더 서브 컬러 (강조 ❌)
- 텍스트: 절대 없음

**목표:** "아는 사람만 누른다"

---

## 2️⃣ 탭 시 동작 (가장 중요)

### ❌ 하면 안 되는 것

- 다른 페이지 이동
- 모달에서 설명 길게
- 로그인/가입 언급

### ⭕ 정답 동작 (1초 규칙)

1. 탭
2. → 즉시 QR 오버레이 표시
3. → 배경은 현재 허브 화면 그대로

### QR 내용

- 연결 URL: `/qr?market=home`
- 의미: "지금 이 화면을 그대로 보여주는 입구"

---

## 3️⃣ QR 오버레이 구성 (최소)

```
[ QR 이미지 ]

(작게)
이 화면을 바로 보여줄 수 있어요

닫기: X / 바깥 터치
```

- 공유 버튼 ❌ (현장용)
- 저장 버튼 ❌ (복잡도 ↑)

**QR은 '보여주는 도구'지 '관리 대상'이 아니다**

---

## 4️⃣ 스캔 결과 동작 (B 사용자 기준)

2. → `/qr?market=home`
3. → 로그인 ❌
4. → 가입 ❌
5. → 게스트 마켓 홈
6. → 자유 탐색
7. → 행동 시 Lazy Signup

- ✔ 친구 초대 ❌
- ✔ 설명 ❌
- ✔ 강요 ❌

## 9️⃣ QR 오버레이 디자인 옵션 (스캔 성공률 고려)

### 옵션 A: 현재 화면 위에 얹기 (현재 구현)

**구현:**
- 반투명 배경 (rgba(0,0,0,0.5))
- 중앙 모달 (흰색 카드)
- 배경은 허브 화면 그대로

**장점:**
- 맥락 유지 (어디서 QR을 봤는지 명확)
- 빠른 닫기 (바깥 터치)
- 시각적 부담 적음

**단점:**
- 배경이 어두워서 QR 대비가 낮을 수 있음
- 밝은 환경에서 스캔 어려울 수 있음

---

### 옵션 B: 자동 밝기/풀스크린 (향후 개선 가능)

**구현:**
- 흰색 풀스크린 배경
- QR만 중앙에 크게
- 자동 밝기 증가 (CSS filter: brightness)

**장점:**
- 최대 대비 (검은 QR + 흰 배경)
- 어두운 환경에서도 스캔 용이
- 스캔 성공률 ↑

**단점:**
- 맥락 단절 (어디서 QR을 봤는지 모호)
- 닫기 버튼 찾기 어려울 수 있음
- 시각적 부담 큼

---

### 추천 (현재 단계)

**옵션 A 유지** (현재 화면 위에 얹기)

**이유:**
1. 허브의 QR은 "보여주는 도구"
2. 맥락 유지가 스캔 성공률보다 중요
3. 실제 사용은 밝은 환경 (데모/전파)
4. 필요하면 나중에 옵션 B로 개선 가능
