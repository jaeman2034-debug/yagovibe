## ✅ 완료된 개선 사항

### 1. 팀장 활동 추적

**파일**: `src/pages/team/TeamManagePage.tsx`

**추가된 기능**:
- 팀 관리 페이지 접근 시 `lastActiveAt` 업데이트
- `teams/{teamId}/members/{uid}/lastActiveAt` 업데이트
- `users/{uid}/lastActiveAt` 업데이트 (전역 활동 추적)

**활동 추적 기준**:
- 로그인
- 팀 관리 페이지 접근
- 승인/관리 액션 (audit_logs에서 확인)

### 2. 스케줄러 함수 생성

**파일**: `functions/src/autoTransferInactiveCaptain.ts`

**기능**:
- 매일 오전 9시 실행 (Asia/Seoul)
- 휴면 팀장 체크
- 사전 경고 알림 (D-7, D-1)
- 자동 위임 실행

---

### 3. 사전 경고 알림

**D-7 경고 (23일째)**:
```
⚠️ 팀장 활동이 23일째 없어요. 7일 후 자동으로 팀장이 위임될 수 있어요.
```

**D-1 경고 (29일째)**:
```
🚨 내일 팀장 권한이 자동 위임될 예정이에요. 활동을 재개하면 위임되지 않아요.
```

**클릭 시 이동**: `/me`

---

### 4. 자동 위임 실행 로직

**위임 대상 우선순위**:
1. **부팀장** (있다면)
   - `role === "vice"` 또는 `role === "부팀장"`
   - `accessLevel === "ADMIN"`
2. **팀원 중 가입 오래된 순**
3. **최근 활동 있음**
4. ❌ 아무도 없으면 → 위임 안 함

**트랜잭션 처리**:
```ts
transaction {
  teams.ownerUid = nextCaptain.uid
  members[oldCaptain].role = 'member'
  members[newCaptain].role = 'owner'
}
```

---

### 5. 자동 위임 후 알림

**새 팀장**:
```
🛡️ {팀명}의 팀장으로 자동 위임되었어요. 팀 활동을 계속 이어주세요.
```
→ `/me/team/${teamId}/manage`

**이전 팀장**:
```
ℹ️ 장기간 활동이 없어 {팀명}의 팀장 권한이 위임되었어요. 현재는 팀원으로 활동 중이에요.
```
→ `/me`

**팀원 전체**:
```
📢 {팀명}의 팀장이 변경되었어요. 새 팀장: {이름}
```
→ `/teams/${teamId}`

## 🎯 핵심 원칙

### 1. 자동 위임은 "최후의 수단"
- 30일간 활동 없음
- 사전 경고 2회 (D-7, D-1)
- 활동 재개 시 위임 취소

### 2. 사전 고지 필수
- D-7 경고 알림
- D-1 최종 경고 알림
- 모든 과정은 기록 + 알림

### 3. 위임 대상은 명확한 우선순위
- 부팀장 > 가입 오래된 순 > 최근 활동 있음
- 위임 대상 없으면 위임 안 함

### 4. 모든 과정은 기록 + 알림
- 트랜잭션으로 원자적 처리
- 알림 발송 (새 팀장, 이전 팀장, 팀원 전체)
- audit_logs 기록

---

## ⏱️ 휴면 기준

**기준**: 30일간 아래 활동 없음
- 로그인
- 팀 관리 페이지 접근
- 승인/관리 액션

**활동 추적 위치**:
- `users/{uid}/lastActiveAt` (로그인/앱 접근)
- `teams/{teamId}/members/{uid}/lastActiveAt` (팀 관리 페이지 접근)
- `audit_logs` (최근 활동 확인)

## 🛡️ 안전장치

### 1. 되돌리기 불가
- 자동 위임은 되돌릴 수 없음
- 하지만 이전 팀장은 다시 위임 요청 가능
- 새 팀장이 승인하면 복귀 가능 (선택)

### 2. 위임 대상 없음
- 위임 대상이 없으면 위임 안 함
- 팀 잠금 (선택사항, 현재 미구현)

### 3. 활동 재개 시 위임 취소
- D-7, D-1 경고 후 활동 재개하면 위임되지 않음
- `lastActiveAt` 업데이트 시 자동으로 위임 취소
