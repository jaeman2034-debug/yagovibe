# 🎯 팀 블로그 MVP 범위 고정 + 출시 체크리스트

## 📋 목적
지금 당장 만들어야 할 것 / 버려야 할 것 명확히 구분

---

## 0️⃣ MVP 정의 (한 문장)

**"팀의 활동 기록을 자동으로 보여주고, 그 기록이 자연스럽게 사람을 데려오게 만든다."**

---

## 🎯 MVP 핵심 원칙

### 내부 의사결정 기준 (4가지)

1. **과장되는가?** → ❌
2. **기록처럼 보이는가?** → ⭕
3. **자동화가 신뢰를 해치지 않는가?** → ⭕
4. **사람이 없어도 굴러가는가?** → ⭕

**이 4가지만 통과하면 출시 OK**

---

### 내부 합의용 한 문장

**"이건 MVP인가, 아니면 불안을 덜기 위한 장식인가?"**

**장식이면 지금은 안 만든다.**

## 5️⃣ 출시 직후 7일 검증 지표 (핵심만)

| 지표 | 기준 |
|------|------|
| 공유 유입 이탈률 | ≤ 65% |
| CTA 클릭률 | ≥ 20% |
| 가입 클릭률 | ≥ 8% |
| 관리자 Pro 클릭 | ≥ 10% |

**이 중 2개 이상 통과 → 유지**  
**1개 이하 → UX 미세 조정**
