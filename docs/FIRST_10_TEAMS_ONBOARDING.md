# 🚀 첫 10개 팀 온보딩 시나리오 (실전)

## 📋 목적
이 MVP로 바로 실험 가능한 '첫 10개 팀 온보딩 시나리오' 설계

**어디서, 어떻게, 무슨 말로 데려올지**

---

## 0️⃣ 원칙 (절대 깨지면 안 됨)

- ❌ "써보세요" 금지
- ❌ 기능 설명 먼저 금지
- ⭕ "기록해 드릴게요"로 시작

---

## 🎯 목표

### 단기 (2주)

- 10개 팀 확보
- 각 팀 첫 블로그 글 생성
- 공유 1회 이상

---

### 중기 (1개월)

- 10개 팀 중 2~3개 Pro 전환
- 공유 유입 → 가입 전환 1건 이상
- 피드백 수집 및 개선

---

## 1️⃣ 타깃 팀 정의 (초기 성공 확률 ↑)

### 최우선 (1–6번)

- 50~70대 축구/족구/게이트볼
- 주 1회 이상 정기 모임
- 이미 단톡방/밴드 존재
- 홍보 담당자 없음 (중요)

---

### 차순위 (7–10번)

- 동네 배드민턴/산책 모임
- 지역 체육회 산하 소모임

---

## 2️⃣ 어디서 데려올 것인가 (채널)

### A. 지역 커뮤니티 (최우선)

**타겟:**
- 네이버 카페 / 다음 카페
- 지역 체육회 게시판
- "동호회 운영자 모임"

**예시 채널:**
- 네이버 카페
- 지역 맘카페의 남편/부모 운동 글
- 동호회 밴드 댓글

**접근 방법:**
- 게시글 작성 (공지 게시판)
- 댓글 작성
- 운영자 1:1 DM

---

### B. 오프라인 연결

**예시 채널:**
- 체육공원 게시판
- 구장 관리자 소개
- 기존 팀의 지인 추천 (1팀 → 2팀)

**접근 방법:**
- 게시판에 안내문 부착
- 구장 관리자에게 소개 요청
- 기존 팀에게 추천 요청

---

## 3️⃣ 첫 접촉 문구 (복붙용)

### 댓글/DM 첫 문장

```
안녕하세요.

팀 활동 기록을 정리해 드리는 서비스를 만들고 있습니다.

모집이나 홍보가 아니라,
'그날그날 활동을 정리해서 남겨드리는' 쪽에 가깝습니다.
```

**📌 기능/가격 언급 금지**

---

### 반응 왔을 때 2차 문구

```
혹시 팀 이름이랑
최근 활동 날짜 하나만 알려주실 수 있을까요?

저희가 예시 페이지를 하나 만들어 드릴게요.
```

**👉 이 시점까지 회원가입 없음**

---

## 4️⃣ 어떻게 데려올 것인가 (접근 방법)

**장점:**
- 한 번에 여러 운영자에게 노출
- 신뢰도 높음 (게시판 공개)

**단점:**
- 스팸으로 오해 가능
- 관리자 삭제 가능

**템플릿:**
```
[제목] 동호회 공지/홍보, AI가 대신해줍니다 (무료 체험)

안녕하세요 운영자님들.

동호회 공지나 활동 후기 작성하시느라 번거로우시지 않나요?

저희는 팀 활동만 하면 AI가 자동으로
- 팀 소개 글
- 활동 후기
- 주간 요약
까지 만들어주는 서비스를 만들었습니다.

초기 팀 대상으로 무료 체험 열어드리고 있어요.
관심 있으시면 댓글이나 쪽지 주세요 🙂

[링크: 팀 블로그 예시 페이지]
```

---

### 방법 B: 1:1 DM (전환율 최고)

**장점:**
- 개인화 가능
- 전환율 높음

**단점:**
- 시간 소모
- 스팸으로 오해 가능

**템플릿:**
```
안녕하세요 운영자님.

[팀명] 운영하시느라 고생 많으십니다.

혹시 공지나 후기 작성하시는 게 번거로우신가요?

저희는 팀 활동만 하면 AI가 자동으로 블로그 글을 만들어주는 서비스를 만들었습니다.

초기 팀 대상으로 무료 체험 열어드리고 있어요.
10분만 같이 보시면 바로 써보실 수 있습니다.

관심 있으시면 링크 드릴게요 🙂
```

---

### 방법 C: 기존 사용자 제안 (가장 자연스러움)

**장점:**
- 이미 신뢰 형성
- 전환율 매우 높음

**단점:**
- 기존 사용자 수에 의존

**템플릿:**
```
안녕하세요 [관리자명]님.

[팀명]의 AI 리포트 잘 활용하고 계신가요?

이제 이 리포트를 공개 블로그로 만들어서
외부에도 공유할 수 있게 되었습니다.

팀 활동만 하면 자동으로 블로그 글이 생성되고,
공개 페이지로 신뢰 있는 팀을 만듭니다.

한 번 써보시겠어요?
```

## 5️⃣ 내부 실행 루트 (핵심)

### Step 1

- 팀명 + 지역 + 활동 1건 수동 입력
- 공개 팀 블로그 랜딩 즉시 생성

---

### Step 2

- 자동 블로그 1건 생성
- 공유 카드 1장 생성

---

### Step 3

- 링크 전달
- "이런 식으로 팀 활동이 기록됩니다. 불편한 점 있으면 편하게 말씀 주세요."

---

## 6️⃣ '아, 이거 좋네요'가 나오는 순간

**❌ 가격 설명**  
**⭕ 니즈 확인**

## 1️⃣3️⃣ 온보딩 시퀀스 (D-0 ~ D-14)

### D-0: 첫 접촉

**목표:** 관심 확인

**액션:**
1. 게시글/DM 전달
2. 간단한 응답 확인
3. "10분만 같이 보시면 됩니다" 안내

**성공 기준:**
- 응답 받음
- "한 번 봐볼게요" 또는 "관심 있어요"

---

### D-1: 팀 생성 & 첫 글 생성

**목표:** 첫 블로그 글 생성

**액션:**
1. 팀 생성 안내
2. 팀 정보 입력 (이름, 종목, 지역, 연령대)
3. "AI 블로그 생성하기" 클릭
4. 첫 소개 글 생성 확인

**성공 기준:**
- 첫 글 생성 완료
- "이거 누가 쓴 거예요?" 또는 "우리 팀 정보 맞네요?"

**절대 하지 말 것:**
- 프롬프트 설명 ❌
- 기술 설명 ❌

---

### D-2: 공개 페이지 확인

**목표:** 공개 페이지 가치 인식

**액션:**
1. 공개 URL 공유
2. "이제 검색이나 공유로 누구나 볼 수 있어요" 멘트

**성공 기준:**
- "이거 외부에도 보이나요?" 또는 "공유해도 되나요?"

---

### D-3: 활동 후기 자동 생성

**목표:** 자동화 체감

**액션:**
1. 일정/출석 1회만 있어도 OK
2. "이제부터는 활동만 하시면 후기는 자동으로 쌓입니다" 멘트

**성공 기준:**
- "이거 자동으로 되네요?"

---

### D-7: 통계 한 줄 노출

**목표:** Pro 전환 씨앗

**액션:**
1. "이번 주 블로그 조회수 ○○회" 노출
2. "운영 안 하셨는데 이 정도 반응이 나왔어요" 멘트

**성공 기준:**
- "이거 계속 돌아가네요?"

---

### D-10: 주간 요약 (Pro 체험)

**목표:** Pro 차이점 명확화

**액션:**
1. 주간 요약 자동 생성
2. "이 기능은 Pro에서 계속 자동으로 돌아갑니다" 멘트

**성공 기준:**
- "Pro가 뭐예요?" 또는 "얼마예요?"

---

### D-12: 전환 예고 (부담 없이)

**목표:** 선택권 강조

**액션:**
1. 운영자에게만 노출
2. "다음 주부터도 이대로 자동 운영하려면 Pro로 전환하시면 됩니다"
3. "안 맞으면 그냥 지금 상태로 써도 괜찮아요"

**성공 기준:**
- "한 번 더 써볼게요" 또는 "Pro로 전환할게요"

---

### D-14: 전환 or 예약

**목표:** Pro 전환 또는 관계 유지

**액션:**
1. "Pro 계속 사용하기 – 월 19,000원" CTA
2. 결과 케이스 분기:
   - 즉결 결제 ✅
   - 다음 달 결제 예약 ⭕
   - "조금 더 써볼게요" → 성공 (관계 유지)

**성공 기준:**
- "이거 계속 돌아가네"
- "우리가 따로 할 게 없네"
- "이 정도면 돈 낼 만하죠"

## 5️⃣ 실전 스크립트 (복붙 가능)

### 첫 접촉 DM (A안: 운영자 공감형)

```
안녕하세요 운영자님.

팀 공지/후기 관리 때문에 항상 손이 많이 가시죠?

저희는 팀 활동만 하면 AI가 자동으로
- 팀 소개 글
- 활동 후기
- 주간 요약
까지 대신 만들어주는 서비스를 운영 중입니다.

초기 팀 대상으로 카드 등록 없는 무료 체험 열어드려요.
필요하시면 바로 링크 드릴게요 🙂
```

---

### 첫 접촉 DM (B안: 결과 제시형)

```
운영자님 안녕하세요.

팀 블로그/공지, 요즘은 AI가 대신 관리합니다.

활동만 하면
- 팀 소개 글 자동 생성
- 후기 자동 포스팅
- 공개 홍보 페이지 운영

초기 체험 팀을 찾고 있어요.
관심 있으시면 바로 써보실 수 있게 도와드릴게요.
```

---

### 첫 글 생성 후 (와우 포인트)

```
이거 누가 쓴 거예요?
→ AI가 자동으로 만들어드렸어요.

우리 팀 정보 맞네요?
→ 네, 입력하신 정보 기반으로 만들었어요.

이거 계속 돌아가나요?
→ 네, 활동만 하시면 자동으로 글 쌓입니다.
```

---

### 공개 페이지 확인 후

```
이거 외부에도 보이나요?
→ 네, 이제 검색이나 공유로 누구나 볼 수 있어요.

공유해도 되나요?
→ 네, 카카오톡이나 밴드로 공유하시면 됩니다.
```

---

### Pro 전환 제안

```
이 기능은 Pro에서 계속 자동으로 돌아갑니다.

다음 주부터도 이대로 자동 운영하려면
Pro로 전환하시면 됩니다.

안 맞으면 그냥 지금 상태로 써도 괜찮아요.
```

## 7️⃣ 실패 시 대응 (즉시 수정)

### 케이스 A: 설명 부족

**증상:**
- "이거 뭐 하는 거예요?"
- "어떻게 쓰는 거예요?"

**대응:**
- 간단한 가이드 영상 제작
- 스크린샷 + 화살표로 설명

---

### 케이스 B: UX 복잡

**증상:**
- "복잡하네요"
- "어디서 뭘 해야 하지?"

**대응:**
- 단계별 가이드 추가
- "다음 단계" 버튼 명확화

---

### 케이스 C: 가치 전달 실패

**증상:**
- "필요 없어요"
- "우리는 이렇게 안 해요"

**대응:**
- 문제 공감부터 다시
- 실제 사례 보여주기
