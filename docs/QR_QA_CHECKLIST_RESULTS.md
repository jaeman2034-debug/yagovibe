# 🔒 QR 직행 회원가입 — QA 체크리스트 검증 결과

## ✅ 합격 항목 (즉시 배포 가능)

### 1️⃣ QR 스캔 직후
- ✅ **중간 Preview 화면 없이 바로 회원가입 화면이 뜬다**
  - `QRPage.tsx`에서 `need_auth` 상태 시 즉시 `/qr/signup`으로 리다이렉트
- ✅ **상단 문구 정확히 구현됨**
  - "초대를 통해 참여 중입니다" + "회원가입 후 바로 참여됩니다"
  - 팀명 있으면: "{teamName}팀에 참여 중입니다" + "회원가입 후 바로 참여됩니다"

### 2️⃣ 회원가입 화면
- ✅ **CTA 버튼 문구 정확**
  - "회원가입하고 참여하기" ✅
- ✅ **이메일 입력 하단 안내**
  - "이 이메일로 팀 계정이 생성됩니다" ✅

### 3️⃣ 이미 계정 있는 경우
- ✅ **에러 처리 완벽**
  - "이미 가입된 계정이에요\n로그인하고 바로 참여할 수 있어요" ✅
  - "로그인하고 참여하기" 버튼 인라인 표시 ✅

### 4️⃣ 가입 성공 후
- ✅ **완료 화면 문구 정확**
  - "참여가 완료되었습니다" ✅
  - "곧 팀 화면으로 이동합니다" ✅

### 5️⃣ 음성 위젯
- ✅ **보조 수단으로만 존재**
  - 강조 없음, 방해 없음 ✅

## ⚠️ 개선 필요 항목 (2개만 고치면 전부 YES)

### 🔴 문제 1: 입력 필드 구성
**현재 상태:**
- 이름 (필수)
- 이메일
- 비밀번호

**체크리스트 요구사항:**
- 이메일
- 비밀번호
- 비밀번호 확인

**판단:**
- WeChat Mode에서는 **이름을 제거**하는 게 맞음
- 또는 이름을 선택(optional)으로 변경
- **비밀번호 확인 필드 추가** 필요

**우선순위:** 중간 (이름은 나중에 프로필에서 입력 가능)

---

### 🔴 문제 2: 하단 "이미 계정 있으신가요?" 링크
**현재 상태:**
```tsx
<div className="mt-4 text-center">
  <button onClick={() => navigate("/qr/login")}>
    이미 계정이 있으신가요? 로그인
  </button>
</div>
```

**문제점:**
- WeChat Mode 핵심 원칙: **처음부터 로그인 선택지 노출 ❌**
- 이미 계정 있으면 에러 시에만 로그인 옵션 표시해야 함

**해결:**
- 이 링크 **완전 제거** 필요

**우선순위:** 높음 (전환율에 직접 영향)
