# 🔥 선수 명단 연령 자동 검증 시스템 (최종 설계)

## 🎯 핵심 원칙

**"엑셀을 입력으로 쓰고, 엑셀을 검증 결과로 다시 돌려준다"**

## 📋 전체 흐름

```
1. 팀 입력
   - 엑셀/구글시트/메모장에서 복붙
   - 형식: 이름 | 생년월일 | 포지션 | 비고
   
2. 시스템 자동 처리
   - 출생연도 자동 계산
   - ageRule 기준 연령 적합 여부 판단
   - 연령대 그룹 자동 분류
   
3. 검증 결과 제공
   - 웹 UI: 필터링된 테이블 (즉시 확인)
   - 엑셀: 연령대별 시트 자동 생성 (사무국 제출용)
```

## 🧩 데이터 구조

### 입력 형식 (팀이 입력)
```
이름 | 생년월일 | 포지션 | 비고
김민수 | 2011-03-12 | FW |
박철우 | 2008-07-01 | DF |
이성훈 | 2013-11-22 | MF |
```

### 검증 결과 (시스템이 생성)
```typescript
interface PlayerVerification {
  name: string;
  birthDate: string; // YYYY-MM-DD
  birthYear: number; // 자동 계산
  position?: string;
  notes?: string;
  
  // 🔥 자동 판별 결과
  eligible: boolean;
  reason: string; // "연령 OK" | "연령 초과" | "생년월일 오류"
  ageGroup?: string; // "U-12" | "OVER-40" 등
}
```

## 🔧 구현 방법 (2가지 옵션)

### 방법 A: 복붙 UI + 웹 검증 (추천 · 빠름) ⭐
**장점:**
- 즉시 결과 확인
- 엑셀 없이도 가능
- 실시간 필터링
- 빠른 구현

**구현:**
1. `PlayerListPasteForm.tsx` - 텍스트 영역에 복붙
2. `checkAgeEligibility()` 유틸 함수 ✅ (완료)
3. `PlayerVerificationTable.tsx` - 필터링된 결과 표시
4. "엑셀로 다운로드" 버튼 (선택)

### 방법 B: 엑셀 업로드 + 결과 엑셀 생성 (전문적)
**장점:**
- 사무국 익숙한 방식
- 결과물도 엑셀로 제공
- 대량 처리 용이

**구현:**
1. `PlayerListUploadForm.tsx` - 엑셀 파일 업로드
2. Cloud Function: 엑셀 파싱 + 연령 판별
3. 결과 엑셀 자동 생성 (연령대별 시트)
4. 다운로드 제공

## 🧠 천재 포인트

### 왜 이 설계가 좋은가?

1. **사람이 판단 안 함**
   - 연령 계산 ❌
   - U-12 / U-15 나누기 ❌
   - 엑셀 색칠 ❌
   - 시스템이 자동으로 처리 ✅

2. **사무국 입장**
   - 익숙한 엑셀 방식 유지
   - 검증 결과도 엑셀로 받음
   - 체크 → 승인만 하면 됨

3. **팀 관리자 입장**
   - 선수 명단 복붙
   - 바로 "누가 안 되는지" 보임
   - 전화할 필요 없음

4. **민원/실수 방지**
   - 시스템 기준으로 대응
   - JoinKFA 연동과 완전 분리
   - 모든 판별 기록 남음
