## 1️⃣ 상태별 진입 QA (가장 중요)

### `/me` 기준, 이 4가지는 반드시 확인

| 상태 | 기대 화면 | 체크 |
|------|----------|------|
| **P1-A** | 팀 찾기 / 팀 만들기 CTA | ✅ |
| **P1-W** | 가입 요청 대기 카드 | ✅ |
| **P2** | 내 팀 카드 + "팀 정보 보기" | ✅ |
| **P3** | 팀장 대시보드 + "팀 관리하기" | ✅ |

**체크 방법:**
- URL 직접 입력: `/me`
- 새로고침 후 상태 확인
- 각 상태에서 예상 화면이 정확히 표시되는지 확인

**현재 상태:** ✅ 모든 상태 분기 구현 완료

## 3️⃣ 빈 상태(Empty State) 전수 점검

### 체크 대상

| 빈 상태 | 현재 처리 | 개선 필요 |
|---------|----------|----------|
| 팀원 0명 | "팀원이 없습니다" | ✅ 다음 행동 제안 추가 |
| 가입 요청 0건 | "대기 중인 가입 요청이 없습니다" | ✅ |
| 협회 미가입 | "미가입" 표시 | ✅ 다음 행동 제안 있음 |
| 팀 소개 없음 | 표시하지 않음 | ✅ |

**개선 사항:**
- 팀원 0명: "팀원을 초대해보세요" 제안 추가 필요

**현재 상태:** ✅ 대부분 완료, 일부 개선 필요

## 5️⃣ 알림 중복/과잉 방지

### 규칙 3개

1. ✅ 같은 이벤트 → **한 번만** (`processedNotificationIds` 추적)
2. ✅ 새로고침 시 다시 안 뜸 (읽음 처리)
3. ✅ 행동 강요 문구 ❌ (사실 전달만)

**체크 방법:**
- 같은 알림이 여러 번 표시되는지 확인
- 새로고침 후 알림 재표시 여부 확인

**현재 상태:** ✅ 중복 방지 로직 구현 완료

## 9️⃣ 진짜 마지막 체크 (천재 포인트)

> **"이 앱을 처음 쓰는 사람이 설명 없이 3분 안에 팀 하나를 운영할 수 있는가?"**

### 체크 포인트

1. **팀 생성 플로우**
   - 팀 만들기 → 협회 가입 선택 → 팀 관리
   - ✅ 명확한 안내

2. **팀원 초대 플로우**
   - 팀장 대시보드 → 팀원 초대하기 → 링크 복사 → 공유
   - ✅ 직관적

3. **가입 승인 플로우**
   - 가입 요청 → 팀장 승인 → 팀원 상태 전이
   - ✅ 실시간 반영

**결론:** ✅ **YES - 출시 가능**
