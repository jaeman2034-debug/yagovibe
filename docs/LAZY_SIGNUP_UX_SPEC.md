# 🔒 가입 UX & Lazy Signup 모달 스펙 (STEP 4 LOCK)

## 목표

**행동을 막되, 흐름은 끊지 않는다.**
사용자는 "방해받았다"가 아니라 "자연스럽게 이어졌다"고 느껴야 한다.

---

## 1️⃣ Lazy Signup이 뜨는 정확한 조건 (고정)

### 트리거 원칙

- 화면 진입 ❌
- 버튼 클릭(행동 시도) ⭕

### 트리거 대상 행동 (초기 LOCK)

- 채팅 보내기
- 판매 등록
- 찜
- 거래 요청
- 결제

👉 **"보는 행위"에는 절대 모달 띄우지 않는다.**

---

## 2️⃣ Lazy Signup 모달의 형태 (UX 고정)

### 형태

- 바텀시트(모바일) / 센터 모달(웹)
- 화면 완전 차단 ❌
- 배경 dim ⭕ (약하게)

### 닫기

- X 버튼 ⭕
- 바깥 터치 ⭕
- ESC ⭕

👉 닫아도 원래 화면 유지

---

## 3️⃣ Lazy Signup 모달 카피 (절대 수정 금지)

### 제목
```
이 기능을 사용하려면 회원가입이 필요해요
```

### 보조 문구
```
10초 만에 가입하고
하던 일을 계속할 수 있어요
```

### CTA (단 하나)
```
[ 가입하고 계속 ]
```

❌ "로그인"
❌ "권한 없음"
❌ "필수입니다"

---

## 4️⃣ 가입 화면 UX (초경량 규칙)

### 가입 수단 우선순위

1. 전화번호 + OTP (default)
2. Google / Apple
3. 비밀번호 가입 (옵션)

### UI 원칙

- 입력 필드 최소
- 이유 설명 ❌
- "계속" 느낌 유지

### CTA 문구
```
가입하고 계속하기
```

---

## 5️⃣ 가입 중 이탈 방지 규칙 (중요)

가입 화면 상단에 반드시 표시:

```
지금 하던 작업을
가입 후 바로 이어서 할 수 있어요
```

👉 사용자의 맥락 불안 제거

---

## 6️⃣ 가입 완료 후 복귀 로직 (절대 규칙)

### 반드시 지켜야 할 순서

1. 인증 성공
2. postSignupAction 존재 확인
3. 해당 액션 자동 실행
4. 모달/가입 화면 완전 종료

### 사용자가 느끼는 흐름

**"아까 누른 게 지금 실행됐네"**

---

## 7️⃣ 실패/취소 시 UX (침착해야 합격)

### 가입 취소

- 에러 ❌
- 알림 ❌
- 그냥 원래 화면

### 인증 실패

- 기술 메시지 ❌
- 사용자 언어로 한 줄

```
다시 한 번만 시도해 주세요
```

---

## 8️⃣ Lazy Signup에서 절대 하지 말아야 할 것

- ❌ 혜택 설명
- ❌ 서비스 설명
- ❌ 왜 가입해야 하는지 장황한 이유
- ❌ 프로필 설정 유도

👉 **Lazy Signup은 문턱이지 온보딩이 아니다.**

---

## ✅ QA 합격 기준

이 질문에 YES면 완료:

1. **"가입을 강요당했다는 느낌이 없다"**
   → ✅ 자연스러운 흐름

2. **"하던 걸 계속할 수 있었다"**
   → ✅ 가입 후 액션 복귀

3. **"가입이 생각보다 빨랐다"**
   → ✅ 초경량 가입 (전화번호 + OTP)

4. **"왜 가입했는지 명확했다"**
   → ✅ 행동 시도 → 가입 요구
