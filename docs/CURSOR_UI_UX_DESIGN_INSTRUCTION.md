# 🧠 Cursor 지시문 — YAGO VIBE UI/UX DESIGN (천재모드)

너는 지금부터 AI 기반 스포츠 조직 운영 플랫폼 'YAGO VIBE'의 UI/UX 디자이너 + 제품 설계자 역할을 수행한다.

이번 작업의 목표는 '우리 팀 관리' 기능을 단순 팀 관리 화면이 아니라 "협회를 중심으로 한 축구 조직 운영 UI"로 실제 구현 가능한 디자인으로 만드는 것이다.

---

## [핵심 전제 — 반드시 지켜야 할 구조]

### 1. 협회가 항상 상위 컨텍스트다
- 협회는 메뉴가 아니라 상위 조직 컨텍스트(그룹)로 표현
- 모든 화면에서 협회가 최상위 컨텍스트임을 명확히 표현

### 2. 팀은 협회에 소속될 수도 있고, 비소속일 수도 있다
- 협회 소속 팀: 회원팀, 비회원팀, 아카데미/유소년 클럽
- 비소속 팀: 내 팀 (협회와 독립적으로 존재 가능)

### 3. 팀 유형은 3가지다

**회원팀**
- 연 240만원 납부
- 운동장 우선 대관
- 협회 우선 대관 시설 선대관 가능

**비회원팀**
- 회비 없음
- 잔여 대관만 가능
- 협회 우선 대관 시설 사용 불가

**아카데미/유소년 클럽**
- 비회원이지만 발전기금 100~200만원 납부
- 협회를 통해 대관 (선대관 일정 내 배정)
- 협회 우선 대관 시설 일정 내 사용 가능

### 4. 협회 우선 대관 시설
협회는 아래 시설에 대해 '우선 대관권'을 가진다:

- 육군사관학교 축구장
- 경기기계공업고등학교 축구장
- 서울과학기술대학교 운동장

이 시설들은 "협회 명의 선대관 자산"이며, 개별 팀이 직접 대관할 수 없다.

## [우리 팀 관리 정보 구조 — 확정]

```
우리 팀 관리
 ├─ 협회 (노원구축구협회)
 │    ├─ 회원팀
 │    ├─ 비회원팀
 │    └─ 아카데미(파트너)
 │
 └─ 내 팀 (비소속 가능)
```

**중요**: '협회'는 메뉴가 아니라 상위 조직 컨텍스트(그룹)로 표현할 것.

---

## [UI/UX 설계 목표]

- **기능 설명 금지**: 버튼에 "이 기능은..." 같은 설명 금지
- **강요 금지**: "회원이 되세요" 같은 강요 문구 금지
- **차이를 '상태'로 보여줄 것**: 색상, 아이콘, 메시지로 권한 차이 표현
- **운동장 대관 UX가 협회 가치의 핵심**: 대관 화면이 가장 중요
- **사용자가 스스로 "왜 회원이 되는지" 이해하게 만들 것**: 설명 없이 상태로 이해하게

## [요청 작업 1 — 화면 목록 정의]

아래 화면들을 Figma 기준으로 설계하라:

### 1. 우리 팀 관리 메인 화면

**구성 요소:**
- 조직 컨텍스트 바 (상단)
  - 협회 / 내 팀 전환 탭
  - 현재 선택된 조직 표시
- 핵심 기능 카드 (최상단)
  - 운동장 대관 카드 (가장 크게, 상단)
  - 팀 목록 카드
  - 회원 현황 카드
- 협회 소속 팀 목록
  - 회원팀 섹션
  - 비회원팀 섹션
  - 아카데미 섹션
- 내 팀 목록 (협회 비소속)

**디자인 원칙:**
- 운동장 대관 카드가 가장 크고 눈에 띄게
- 조직 컨텍스트 바는 항상 상단 고정
- 카드 중심 레이아웃

---

### 2. 운동장 대관 메인 화면

**구성 요소:**
- 이번 주 대관 요약 카드 (상단)
  - 회원팀 대관 수
  - 아카데미 대관 수
  - 비회원팀 대관 수
  - 총 대관 수
- 협회 우선 대관 시설 섹션
  - 섹션 제목: "협회 우선 대관 시설"
  - 육군사관학교 축구장 카드
  - 경기기계공업고등학교 축구장 카드
  - 서울과학기술대학교 운동장 카드
  - 각 카드에 "협회 선대관" 뱃지
- 일반 시설 섹션
  - 섹션 제목: "일반 대관 시설"
  - 기타 시설 카드들

**디자인 원칙:**
- 협회 우선 대관 시설이 상단, 시각적으로 구분
- 요약 카드로 한눈에 파악 가능
- 카드 클릭 시 시설 상세 화면으로 이동

---

### 3. 시설 상세 화면

**구성 요소:**
- 시설 정보 (상단)
  - 시설명
  - 위치
  - 시설 사진/지도
- 팀 유형별 다른 상태 메시지와 CTA
  - 회원팀: "우선 배정 대상입니다" + [대관 신청] 버튼
  - 아카데미: "협회 선대관 일정 내 배정됩니다" + [대관 요청] 버튼
  - 비회원팀: "협회 우선 대관 대상이 아닙니다" + [대기 신청] 버튼 + [회원팀 전환 안내] 링크
- 대관 캘린더 (하단)
  - 권한별로 다른 표시
  - 클릭 시 권한별 메시지 분기

**디자인 원칙:**
- 상태 메시지는 담담하게, 강요 없이
- 버튼은 작게, 눈에 띄지 않게
- 권한 차이가 색상과 메시지로 명확히 구분

---

### 4. 대관 캘린더 화면

**구성 요소:**
- 캘린더 뷰
  - 🟢 회원팀 배정 슬롯 (녹색)
  - 🔵 아카데미 배정 슬롯 (파랑)
  - ⚪ 잔여 슬롯 (회색/흰색)
- 슬롯 클릭 시 권한별 메시지 분기
  - 회원팀: 대관 신청 가능
  - 아카데미: 협회 선대관 일정 내 신청 가능
  - 비회원팀: 대기 신청 또는 회원팀 전환 안내

**디자인 원칙:**
- 색상으로 권한 구분 (녹색/파랑/회색)
- 클릭 시 모달/바텀시트로 메시지 표시
- 설명 없이 색상만으로 이해 가능하게

## [요청 작업 2 — UX 분기 규칙]

다음 조건에 따라 UX가 달라지게 설계하라:

### 회원팀
```
상태 메시지: "우선 배정 대상입니다"
CTA: [대관 신청] 버튼 (녹색, 작게)
동작: 즉시 대관 신청 가능
```

### 아카데미
```
상태 메시지: "협회 선대관 일정 내 배정됩니다"
CTA: [대관 요청] 버튼 (파랑, 작게)
동작: 협회 선대관 일정 확인 후 요청
```

### 비회원팀
```
상태 메시지: "협회 우선 대관 대상이 아닙니다"
CTA: [대기 신청] 버튼 (회색, 작게)
추가: [회원팀 전환 안내] 링크 (작게, 부드럽게)
  → 링크 클릭 시: "연회비 240만원으로 회원팀 전환 시 우선 대관 가능"
```

**디자인 원칙:**
- 버튼은 작게, 눈에 띄지 않게
- 문구는 담담하게, 강요 없이
- 설명은 최소화, 상태로 이해하게

## [요청 작업 3 — 디자인 가이드]

### 스타일
- **행정 UI ❌**: 딱딱한 공문 느낌 금지
- **서비스 UI ⭕**: 모던하고 친근한 느낌
- **카드 중심 레이아웃**: 모든 정보를 카드로 표현
- **여백 충분**: 답답하지 않게

### 색상 시스템
색상으로 권한 표현:

- **회원팀: 녹색** (#059669)
  - 우선 배정, 선대관 가능
  - 버튼, 뱃지, 캘린더 슬롯

- **아카데미: 파랑** (#2563EB)
  - 협회 선대관 일정 내 배정
  - 버튼, 뱃지, 캘린더 슬롯

- **비회원: 회색** (#6B7280)
  - 대기 신청, 우선 대관 불가
  - 버튼, 텍스트, 캘린더 슬롯

### 타이포그래피
- **제목**: 18px, Bold
- **본문**: 14px, Regular
- **보조 텍스트**: 12px, Regular, 회색
- **버튼**: 14px, Medium

### 간격 시스템
- **카드 간격**: 16px
- **섹션 간격**: 24px
- **내부 패딩**: 16px
