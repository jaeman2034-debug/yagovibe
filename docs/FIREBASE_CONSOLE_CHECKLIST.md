# ✅ Firebase Console Authentication 설정 체크리스트

## 📋 현재 화면에서 확인할 사항

### 1️⃣ "로그인 방법" 탭 확인 (가장 중요!)

**경로:**
```
현재 화면: Authentication → 사용자 (Users)
→ 상단 탭에서 "로그인 방법" (Sign-in methods) 클릭
```

**확인 사항:**
- [ ] Phone (전화번호) 인증이 **활성화**되어 있는지
- [ ] Phone 인증 옆에 **"사용 설정됨"** 또는 **"Enabled"** 표시 확인
- [ ] 비활성화되어 있으면 **활성화** 버튼 클릭

---

### 2️⃣ Phone Auth 설정 상세 확인

**"로그인 방법" 탭에서 Phone 클릭 후:**

#### A. 테스트 전화번호 확인 (가장 중요!)
- [ ] "테스트 전화번호" 섹션 확인
- [ ] **테스트 번호가 하나라도 있으면 전부 삭제**
- [ ] 테스트 코드도 제거
- [ ] 저장 버튼 클릭

**⚠️ 중요:**
- 테스트 번호가 있으면 실제 SMS가 발송되지 않습니다
- 코드에서 `PROD MODE`로 보여도 Firebase Console 설정이 우선합니다

#### B. Phone Auth 활성화 상태
- [ ] Phone 인증이 **활성화**되어 있는지 확인
- [ ] 비활성화되어 있으면 **활성화** 버튼 클릭

---

### 3️⃣ "사용량" (Usage) 탭 확인

**경로:**
```
Authentication → 상단 탭 → "사용량" (Usage) 클릭
```

**확인 사항:**
- [ ] Phone Auth → SMS sent 개수 확인
- [ ] Quota 초과 여부 확인
- [ ] 잠금 상태 확인

---

### 4️⃣ 현재 화면에서 보이는 것

**사용자 목록:**
- 이메일 로그인 사용자들
- Google 로그인 사용자들
- 익명 사용자들

**⚠️ 주의:**
- Phone Auth로 로그인한 사용자는 **전화번호가 표시**되어야 합니다
- 현재 화면에는 Phone Auth 사용자가 보이지 않음
- 이는 Phone Auth가 아직 성공적으로 작동하지 않았다는 의미일 수 있습니다

---

## 🎯 즉시 확인해야 할 것

### ✅ STEP 1: "로그인 방법" 탭으로 이동

1. 현재 화면 상단의 **"로그인 방법"** 탭 클릭
2. Phone (전화번호) 항목 찾기
3. 활성화 상태 확인

### ✅ STEP 2: 테스트 번호 삭제

1. Phone 항목 클릭
2. "테스트 전화번호" 섹션 확인
3. 모든 테스트 번호 삭제
4. 저장

### ✅ STEP 3: Phone Auth 활성화 확인

1. Phone 인증이 비활성화되어 있으면 활성화
2. 저장

---

## 🔍 예상되는 문제

### 문제 1: Phone Auth가 비활성화되어 있음

**증상:**
- "로그인 방법" 탭에서 Phone이 비활성화 상태

**해결:**
- Phone 항목 클릭
- "사용 설정" 또는 "Enable" 버튼 클릭
- 저장

---

### 문제 2: 테스트 번호가 등록되어 있음

**증상:**
- "로그인 방법" → Phone → "테스트 전화번호" 섹션에 번호가 있음

**해결:**
- 모든 테스트 번호 삭제
- 테스트 코드도 제거
- 저장

---

### 문제 3: Phone Auth 사용자가 없음

**증상:**
- "사용자" 탭에서 Phone Auth로 로그인한 사용자가 없음
- (현재 화면 상태)

**의미:**
- Phone Auth가 아직 성공적으로 작동하지 않았을 가능성
- 또는 테스트 번호 모드로만 작동했을 가능성

**해결:**
- 테스트 번호 삭제 후 실제 번호로 테스트
- 성공 시 Phone Auth 사용자가 생성되어야 함
