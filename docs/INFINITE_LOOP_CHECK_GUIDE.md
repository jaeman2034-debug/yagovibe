# 🔍 무한 루프 확인 가이드

## ✅ 수정 완료 사항

1. **필드명 수정**: `uid` → `userId` (team_members 컬렉션)
2. **useEffect 의존성 수정**: `fetchMyTeams` 제거 (무한 루프 방지)

---

## 🔍 확인 방법

### 1️⃣ 브라우저 콘솔 로그 확인

**브라우저 개발자 도구 열기** (F12)

**Console 탭에서 확인:**

#### 정상 작동 시:
```
🔄 [TeamList] 팀 목록 조회 시작: { uid: "..." }
🔍 [TeamList] team_members 조회 결과: { count: 1 }
✅ [TeamList] 팀 목록 조회 완료: { count: 1, teams: [...] }
```
→ **한 번만 출력됨** ✅

#### 무한 루프 발생 시:
```
🔄 [TeamList] 팀 목록 조회 시작: { uid: "..." }
🔄 [TeamList] 팀 목록 조회 시작: { uid: "..." }
🔄 [TeamList] 팀 목록 조회 시작: { uid: "..." }
🔄 [TeamList] 팀 목록 조회 시작: { uid: "..." }
...
```
→ **계속 반복됨** ❌

---

### 2️⃣ Node 프로세스 CPU 사용량 확인

**PowerShell에서 실행:**

```powershell
Get-Process | Where-Object {$_.ProcessName -like "*node*"} | Select-Object ProcessName, Id, CPU, @{Name="Memory(MB)";Expression={[math]::Round($_.WorkingSet64/1MB,2)}}
```

#### 정상 작동 시:
- CPU 사용량이 **안정적** (0-5% 범위)
- 프로세스가 **1-2개**만 실행 중

#### 무한 루프 발생 시:
- CPU 사용량이 **계속 증가** (50% 이상)
- 프로세스가 **여러 개** 실행 중

---

### 3️⃣ 브라우저 네트워크 탭 확인

**개발자 도구 → Network 탭**

**Firestore 요청 확인:**

#### 정상 작동 시:
- `team_members` 쿼리가 **1-2번**만 실행됨
- 요청이 **완료**되고 더 이상 요청하지 않음

#### 무한 루프 발생 시:
- `team_members` 쿼리가 **계속 반복**됨
- 요청이 **끊임없이** 발생함

---

### 4️⃣ Firestore 읽기 횟수 확인

**Firebase Console → Firestore → Usage**

**확인 사항:**
- 읽기 횟수가 **급격히 증가**하는지 확인
- 정상: 초당 1-5회
- 무한 루프: 초당 10회 이상

---

### 5️⃣ 브라우저 성능 모니터링

**개발자 도구 → Performance 탭**

1. **Record 버튼 클릭**
2. **10초간 기록**
3. **Stop 클릭**

**확인 사항:**
- **Flame Chart**에서 반복되는 패턴 확인
- `useEffect`가 계속 실행되는지 확인

---

## 🎯 빠른 확인 방법 (권장)

### 단계 1: 브라우저 새로고침
```
Ctrl + Shift + R (강력 새로고침)
```

### 단계 2: 콘솔 로그 확인
```
F12 → Console 탭
```

### 단계 3: 로그 패턴 확인
- ✅ **정상**: 로그가 1-2번만 출력되고 멈춤
- ❌ **무한 루프**: 로그가 계속 반복됨

---

## 🐛 문제가 계속되면

### 추가 확인 사항:

1. **다른 페이지에서도 발생하는지 확인**
   - `/team` 페이지만 문제인지
   - 다른 팀 관련 페이지도 확인

2. **브라우저 캐시 클리어**
   ```
   Ctrl + Shift + Delete
   → 캐시된 이미지 및 파일 삭제
   ```

3. **시크릿 모드에서 테스트**
   - 확장 프로그램 영향 제거

4. **다른 브라우저에서 테스트**
   - Chrome, Edge, Firefox 등
