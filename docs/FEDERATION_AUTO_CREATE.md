# 협회 AI 자동 생성 (구현 요약)

## AI Brain (Cloud Function)

- **`generateFederationAIContent`** (Callable)
  - OpenAI `gpt-4o-mini`로 소개·인사말·연혁·임원 구성 자동 생성
  - 입력: `{ name, sport?, audience? }`
  - 출력: `{ description, introMessage, history, presidentName, executives, themeColor, templateType, region }`
  - 빈 필드·파싱 오류 시 fallback 템플릿 적용
  - 배포: `OPENAI_API_KEY` 환경 변수 필요

## 데이터 (Firestore)

- **`federations/{slug}`** (문서 ID = `slug`)
  - `name`, `slug`, `sport`, `audience`, `region`, `introMessage`, `description`, `history`
  - `presidentName`, `themeColor`, `templateType`, `autoGenerated`, `leagueCount` (초기 0)
  - `ownerId`, `adminIds`, `status`, `createdAt`, `updatedAt`
- **`federations/{slug}/executives/{autoId}`** — `name`, `role`, `photo?`, `createdAt`
- **`federations/{slug}/gallery/{autoId}`** — (생성 시 비움, 추후 이미지 업로드)

## 클라이언트 서비스

- `src/services/federationService.ts`
  - `generateFederationContent()` — Cloud Function `generateFederationAIContent` 호출. 실패 시 로컬 더미 fallback.
  - `autoCreateFederation(ownerId, input)` — AI 콘텐츠 생성 후 Firestore 배치 쓰기.

## 향후 HTTP API (스펙)

```http
POST /federations/auto-create
Content-Type: application/json

{ "name": "노원구 축구협회", "sport": "soccer", "audience": "youth" }
```

→ 서버에서 동일 필드로 `federations` + `executives` 저장 후 `{ slug, id }` 반환.
