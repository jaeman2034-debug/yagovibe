## 🎯 매일 10분 루틴 (고정)

### ⏱ 0–3분 | 대시보드 위젯 확인

**API 호출:**
```bash
GET /api/admin/widgets/all?region=seoul&date=오늘
```

**체크리스트:**
- [ ] **위젯 1 (Story Health)**: CTR ≥ 2.5%, FillRate = 100%, FreshRate ≥ 40%
- [ ] **위젯 2 (Booking Funnel)**: View→Slot ≥ 35%, Reserve CR ≥ 22%, Pay CR ≥ 95%
- [ ] **위젯 6 (System Guard)**: PayFail ≤ 5건, OfflineRate ≤ 12%

**✅ 모든 지표 정상이면 다음 단계로**

---

### ⏱ 3–7분 | 위험 신호 대응

**위험 신호 발견 시 즉시 액션:**

#### 1. CTR < 2% (위험)
```
→ 자동 액션: 스토리 1개 교체 (오토파일럿이 이미 실행됨)
→ 확인: 새 스토리가 생성되었는지 확인
```

#### 2. Reserve CR < 15% (위험)
```
→ 자동 액션: 할인 캠페인 생성 (오토파일럿이 이미 실행됨)
→ 확인: 할인 캠페인이 활성화되었는지 확인
```

#### 3. PayFail > 10건 (위험)
```
→ 자동 액션: 결제 보호 모드 활성화 (오토파일럿이 이미 실행됨)
→ 확인: PG 재시도 로그 확인
```

#### 4. FillRate < 100% (위험)
```
→ 수동 액션: 운영 픽 즉시 투입
→ 시간: 5분
```

**✅ 위험 신호 없으면 다음 단계로**

---

### ⏱ 7–10분 | 일일 체크리스트 확인

**API 호출:**
```bash
GET /api/admin/daily/check?region=seoul&date=오늘
```

**체크리스트:**
- [ ] **스토리 슬롯 5/5** 유지 (빈 슬롯 절대 금지)
- [ ] **AB 실험 상태** 확인 (표본 ≥ 3,000, Uplift ≥ 10%)
- [ ] **커뮤니티 엔진** 확인 (팀 ≥ 20, 일 가입 ≥ 5)
- [ ] **정산 상태** 확인 (정산 대기 건수)

**✅ 모든 체크리스트 통과하면 완료**

## 🚨 위기 대응 플레이북

### 1. CTR 급락 (< 1.5%)

**즉시 액션:**
1. 오토파일럿이 자동으로 스토리 교체 실행
2. 운영자가 수동으로 스토리 1개 추가 생성
3. 24시간 모니터링

**체크리스트:**
- [ ] 새 스토리 생성 완료
- [ ] 24시간 후 CTR 재확인

---

### 2. PayFail 급등 (> 15건)

**즉시 액션:**
1. 오토파일럿이 자동으로 결제 보호 모드 활성화
2. PG 웹훅 로그 확인
3. PG 담당자 연락 (필요 시)

**체크리스트:**
- [ ] 결제 보호 모드 활성화 확인
- [ ] PG 웹훅 로그 확인
- [ ] 1시간 후 PayFail 재확인

---

### 3. FillRate < 80%

**즉시 액션:**
1. 운영 픽 즉시 투입 (5분)
2. 협회 동기화 확인
3. 사용자 업로드 승인 (대기 중인 것)

**체크리스트:**
- [ ] 운영 픽 생성 완료
- [ ] FillRate = 100% 확인

---

### 4. 커뮤니티 엔진 저조 (팀 < 10)

**즉시 액션:**
1. 오토파일럿이 자동으로 모집 스토리 부스팅
2. 운영팀 3개 시드 생성 (수동)
3. 모집 스토리 2개 추가 생성

**체크리스트:**
- [ ] 모집 스토리 부스팅 확인
- [ ] 운영팀 3개 생성 완료
- [ ] 1주일 후 팀 수 재확인
