# 🎯 팀 만들기 UX 플로우 체크리스트

**생성일**: 2025-01-27  
**목적**: 사용자 관점 + 화면 전환 기준으로 정확한 UX 플로우 검증  
**상태**: ✅ 체크 완료 및 수정 완료

---

## ✅ 전체 플로우 검증

### 0️⃣ 출발 지점 (현재 화면의 의미)

**현재 상태:**
- 사용자 상태: **P1-A (팀 없음)**
- 화면 메시지: "아직 소속된 팀이 없어요. 아래 중 하나를 선택하세요."

**상태:** ✅ 정상

---

### 1️⃣ [팀 만들기] 버튼 클릭

**사용자의 심리:**
- "그럼 내가 하나 만들어볼까?"

**시스템 액션:**
```tsx
// MePage.tsx - handleCreateTeam
navigate("/sports/football/team/create?mode=non-member")
```

**수정 전 문제:**
- ❌ `mode` 파라미터 없음 → TeamCreateChoice 화면으로 이동 (선택 화면)

**수정 후:**
- ✅ `mode=non-member` 파라미터 추가 → 바로 TeamCreateForm으로 이동

**상태:** ✅ **수정 완료**

---

### 2️⃣ 팀 만들기 페이지 (입력 단계)

**화면 목적:**
- "팀을 만들기 위해 최소한의 정보만 받는다"

**입력 요소:**
- ✅ 팀 이름
- ✅ 활동 지역 (고정값: "서울시 노원구")

**상태:** ✅ 정상

---

### 3️⃣ [팀 생성] 버튼 클릭

**이 순간 일어나는 일:**

**서버:**
- ✅ 팀 생성
- ✅ 생성자 → CAPTAIN(P3)으로 저장

**프론트:**
- ✅ 팀 생성 성공 응답 수신
- ✅ 성공 랜딩 화면으로 이동

**코드:**
```tsx
// TeamCreateForm.tsx
navigate(`/sports/${sportType}/team/create?step=2&teamId=${teamId}`, { replace: true });
```

**상태:** ✅ 정상

---

### 4️⃣ 팀 생성 완료 화면 (결정 화면)

**이 화면의 정체성:**
- ❌ 정보 화면
- ✅ **"다음 행동을 고르는 화면"**

**사용자에게 보이는 선택지:**

1. **협회에 가입하고 활동 시작하기**
   - 의미: 외부 활동 바로 시작
   - 다음: → 협회 가입 UX

2. **지금은 팀부터 준비할게요**
   - 의미: 내부 정비 먼저
   - 다음: → `/me` (팀장 대시보드)

**상태:** ✅ 정상

---

### 5️⃣ (추천 루트) "지금은 팀부터 준비할게요" 클릭 시

**시스템 액션:**
```tsx
// TeamCreateStep2.tsx - handleSkip
navigate("/me", { replace: true });
```

**수정 전 문제:**
- ❌ `navigate(`/me/team/${teamId}/manage`)` → 팀 관리 페이지로 이동
- ❌ 사용자 요구사항과 불일치: `/me`로 이동해야 함

**수정 후:**
- ✅ `/me`로 이동
- ✅ `/me`에서 `useMyTeams` 실시간 구독으로 팀 상태 자동 판단

**/me에서의 상태 변화:**

**상태:** P3 (팀장)

**기존 CTA 사라짐** ❌

**대신:**
```
[ 팀장 대시보드 ]

👋 팀을 만들었어요!
먼저 팀원을 초대해보세요.

[ 팀원 초대하기 ]
```

**상태:** ✅ **수정 완료**

---

## 🔍 발견된 문제점 및 수정 사항

### ❌ 문제 1: MePage handleCreateTeam에 mode 파라미터 누락

**문제:**
```tsx
// 수정 전
navigate("/sports/football/team/create");
```

**결과:**
- TeamCreateChoice 화면으로 이동 (선택 화면)
- 사용자가 다시 "비회원팀으로 시작하기" 버튼을 눌러야 함

**수정:**
```tsx
// 수정 후
navigate("/sports/football/team/create?mode=non-member");
```

**상태:** ✅ **수정 완료**

---

### ❌ 문제 2: TeamCreateStep2 handleSkip이 팀 관리 페이지로 이동

**문제:**
```tsx
// 수정 전
if (teamId) {
  navigate(`/me/team/${teamId}/manage`, { replace: true });
} else {
  navigate("/me", { replace: true });
}
```

**결과:**
- 팀 관리 페이지로 바로 이동
- 사용자 요구사항과 불일치: `/me`로 이동해야 함

**수정:**
```tsx
// 수정 후
navigate("/me", { replace: true });
```

**상태:** ✅ **수정 완료**

## ✅ 최종 플로우 검증

### 전체 흐름 다이어그램

```
/me (P1-A, 팀 없음)
   ↓
[팀 만들기] 클릭
   ↓
/sports/football/team/create?mode=non-member
   ↓
TeamCreateForm (팀 정보 입력)
   ↓
[팀 생성] 클릭
   ↓
팀 생성 성공 (서버)
   ↓
/sports/football/team/create?step=2&teamId={teamId}
   ↓
TeamCreateStep2 (결정 화면)
   ↓
[지금은 팀부터 준비할게요] 클릭
   ↓
/me (P3, 팀장 대시보드)
   ↓
✅ "팀을 만들었어요!" + "팀원 초대하기" CTA 표시
```

## 🎯 핵심 원칙 검증

### ✅ 중간에 팀 검증 안 함
- TeamCreateStep2는 완전 Stateless
- teamId가 없어도 화면 표시
- 버튼 클릭 시에만 체크

**상태:** ✅ 정상

---

### ✅ 에러 팝업 안 띄움
- `alert()` → `toast`로 변경 완료
- teamId 없을 때 조용히 `/me`로 이동

**상태:** ✅ 정상

---

### ✅ 상태 변화는 /me에서만 판단
- `useMyTeams` 실시간 구독 (`onSnapshot`)
- 팀 생성 후 `/me` 진입 시 자동으로 P3 상태로 전환

**상태:** ✅ 정상

---

### ✅ 사용자는 항상 "다음 버튼"만 누름
- 각 단계마다 명확한 CTA 제공
- 막히는 구간 없음

**상태:** ✅ 정상

---

## 🔚 최종 판정

> **팀 만들기 UX는 '생성'이 아니라 '팀장이 되는 과정'이다.**

**현재 구조:**
- ✅ 막히는 구간 없음
- ✅ 불안 구간 없음
- ✅ 선택이 명확함
- ✅ 실서비스 기준으로도 좋은 UX

**결론:** ✅ **UX 플로우 정상 작동**

## 📝 추가 개선 가능 사항 (선택)

1. **팀 만들기 실패/중복 케이스 UX**
   - 현재: `toast.error()` 표시
   - 개선 가능: 더 친화적인 에러 메시지

3. **플로우 문구 한 줄 더 다듬기**
   - 현재 문구는 이미 충분히 명확함
