# 🧠 YAGO VIBE 지도 페이지 — 완벽한 UI 리모델링 설계

**목적**: 실서비스 기준 완성형 UI 설계  
**대상**: 개발자 / 디자이너  
**상태**: 리모델링 설계 완료

---

## 🎯 리모델링 목표 (한 문장)

> **지도는 조용히, 행동은 명확히, 상태는 하나만**

---

## 0️⃣ 현재 UI의 핵심 문제 요약

| 문제 | 현재 상태 | 개선 방향 |
|------|----------|----------|
| 상단 정보 과다 | 정보·상태·기능이 섞여 있음 | 역할 분리 |
| 지도 위 오버레이 | 떠다님 | 정리 |
| Debug/상태 UI | 동시에 노출 | 단일화 |
| 다음 행동 | 눈에 안 들어옴 | 하단 고정 |

**해결책**: 역할 분리 + 위계 재설계

---

# 1️⃣ 전체 레이아웃 재설계 (정답 구조)

```
┌──────────────────────────┐
│ Header (고정, 48px)      │  ← 브랜드 / 계정
├──────────────────────────┤
│ Transport Tabs (고정)    │  ← 기능 (도보/차/대중교통)
└──────────────────────────┘

        [ 상태 메시지 ]
        (중앙, 1개만, pill 형태)

┌──────────────────────────┐
│                          │
│          MAP             │  ← 주인공 (깨끗하게)
│                          │
└──────────────────────────┘

┌──────────────────────────┐
│ Bottom Action Sheet      │  ← 행동 (엄지 UX)
└──────────────────────────┘
```

**레이어 구조**:
- 상단 = 기능 (Header + Transport Tabs)
- 중앙 = 상태 (상태 메시지 1개)
- 지도 = 주인공 (깨끗하게)
- 하단 = 행동 (Bottom Action Sheet)

---

# 2️⃣ 상단 영역 리모델링 (극도로 절제)

## ✅ Header (존재감 최소)

**스펙**:
- 높이: **48px** (기존 56px → 48px)
- 위치: `fixed top-0`
- z-index: `1000`

**요소**:
- 좌: 로고 (텍스트만, "YAGO VIBE")
- 우: 알림 / 마이

**제거**:
- ❌ 상태 메시지
- ❌ Debug
- ❌ 설명 문구

**원칙**: 상단은 **"항상 같은 모습"**이어야 신뢰감이 생긴다.

---

## ✅ 교통수단 바 (기능만 존재)

**스펙**:
- 위치: Header 바로 아래
- 높이: **44px**
- 위치: `fixed top: calc(48px + env(safe-area-inset-top))`
- z-index: `999`

**UI**:
```
[ 🚶 도보 ] [ 🚗 자동차 ] [ 🚇 대중교통 ]
```

**규칙**:
- ❌ 상태 메시지 없음
- ❌ 설명 문구 없음
- 선택 시 지도만 바뀜 (조용히)

---

# 3️⃣ 상태 메시지 시스템 (천재 포인트)

## 🎯 핵심 규칙

> **한 번에 하나, 중앙에, 자동으로 사라진다**

---

## 상태 메시지 위치

- 지도 **상단 중앙**
- pill 형태 (rounded-full)
- 지도 컨트롤과 겹치지 않음
- 위치: `fixed top: calc(48px + 44px + env(safe-area-inset-top) + 16px)`
- z-index: `50`

---

## 상태 타입 (딱 3개만)

### ① 로딩

**UI**:
```
🔍 찾는 중…
```

**스타일**:
- 배경: `bg-neutral-100`
- 텍스트: `text-neutral-700`
- 애니메이션: `pulse` (부드럽게)

---

### ② 음성 대기

**UI**:
```
🎙 지금 말해도 돼요
```

**스타일**:
- 배경: `bg-green-100`
- 텍스트: `text-green-700`
- 애니메이션: `pulse` (부드럽게)

---

### ③ 에러

**UI**:
```
📡 연결이 불안정해요
[다시 시도]
```

**스타일**:
- 배경: `bg-red-100`
- 텍스트: `text-red-700`
- 버튼: `bg-red-600 text-white`

---

## ❌ 제거할 상태 메시지

- "축구장 근처에서…" 같은 설명성 문구
  → **하단 결과 영역으로 이동**

---

# 4️⃣ 지도 영역 (주인공 대우)

## 지도 위에 **절대 두지 말 것**

- ❌ Debug
- ❌ 설명 문구
- ❌ 여러 pill
- ❌ 상태 메시지 여러 개

---

## 지도 위에 **있는 것**

- ✅ 상태 메시지 1개 (상단 중앙)
- ✅ Google 기본 컨트롤 (줌, 로고 등)

**원칙**: 지도는 **배경이 아니라 주인공**

---

# 5️⃣ 하단 액션 영역 (엄지 UX의 핵심)

## 위치

- `fixed bottom-0`
- `z-index: 100`
- Safe Area 적용: `padding-bottom: env(safe-area-inset-bottom)`

---

## 상태별 하단 UI

### 🟢 결과 없음 (대기)

**UI**: 없음 (하단 숨김)

---

### 🔍 검색 완료 (장소 선택됨)

**UI**:
```
┌──────────────────────────┐
│ 📍 연희동 주민센터        │
│ ──────────────────────── │
│ [ 여기로 안내할게요 ]     │
└──────────────────────────┘
```

**스타일**:
- 배경: `bg-white`
- 그림자: `shadow-xl`
- 버튼: `bg-blue-600 text-white`

---

### 🧭 안내 중

**UI**:
```
┌──────────────────────────┐
│ 🧭 연희동 주민센터까지    │
│    안내 중                │
│ ──────────────────────── │
│ [ 다시 길찾기 ] [ 취소 ]  │
└──────────────────────────┘
```

**스타일**:
- 배경: `bg-white`
- 그림자: `shadow-xl`
- 버튼: `bg-blue-600 text-white` / `bg-neutral-100 text-neutral-700`

---

### ❌ 에러

**UI**:
```
┌──────────────────────────┐
│ 지금은 길찾기를 열 수     │
│ 없어요                    │
│ ──────────────────────── │
│ [ 다시 시도 ]             │
└──────────────────────────┘
```

**스타일**:
- 배경: `bg-white`
- 그림자: `shadow-xl`
- 버튼: `bg-red-600 text-white`

---

**원칙**: **행동은 항상 하단**, 사용자는 생각 안 하고 누른다

---

# 6️⃣ Debug UI 완전 제거 (중요)

## 원칙

> **운영 화면에 Debug는 범죄다**

---

## 처리 방법

### 1️⃣ 환경 분기 렌더링

```tsx
{process.env.NODE_ENV === 'development' && <DebugUI />}
```

### 2️⃣ URL 파라미터 (선택)

```tsx
{searchParams.get('debug') === 'true' && <DebugUI />}
```

### 3️⃣ 콘솔 로그

- Debug 정보는 `console.log`로만
- 운영 화면에는 표시 안 함

---

## 제거 대상

- ❌ Debug 배지
- ❌ 내부 상태 값 표시
- ❌ 테스트 버튼
- ❌ 개발자만 이해하는 문구

---

# 7️⃣ 시각적 톤 & 감성 (천재 느낌)

## 컬러 시스템

| 용도 | 컬러 | 사용처 |
|------|------|--------|
| Primary CTA | Blue 600 | 하단 액션 버튼 |
| 로딩 | Neutral 100/700 | 상태 pill |
| 음성 | Green 100/700 | 상태 pill |
| 에러 | Red 100/700 | 상태 pill |

---

## 애니메이션

### 등장

```css
animation: fadeIn 0.2s ease-out, slideDown 0.2s ease-out;
@keyframes slideDown {
  from { transform: translateY(-4px); opacity: 0; }
  to { transform: translateY(0); opacity: 1; }
}
```

### 사라짐

```css
animation: fadeOut 0.2s ease-in;
```

### 금지

- ❌ 튀는 애니메이션
- ❌ 과도한 bounce
- ❌ 깜빡임

# 9️⃣ 컴포넌트 구조 (리팩터링 기준)

## 새로운 컴포넌트 구조

```
GeneralMapPage
├── Header (48px, fixed)
├── TransportTabs (44px, fixed)
├── StatusPill (상태 메시지, fixed, 중앙)
├── GoogleMapCanvas (지도, 주인공)
└── BottomActionSheet (하단 액션, fixed)
```

---

## 컴포넌트 목록

### 1. `MapHeader`
- 높이: 48px
- 위치: `fixed top-0`
- 역할: 브랜드 / 계정

### 2. `TransportTabs`
- 높이: 44px
- 위치: `fixed top: calc(48px + env(safe-area-inset-top))`
- 역할: 교통수단 선택

### 3. `StatusPill`
- 위치: `fixed top: calc(48px + 44px + env(safe-area-inset-top) + 16px)`
- 역할: 상태 메시지 (로딩/음성/에러)

### 4. `GoogleMapCanvas`
- 위치: `relative`
- 역할: 지도 (주인공)

### 5. `BottomActionSheet`
- 위치: `fixed bottom-0`
- 역할: 하단 액션 (엄지 UX)
