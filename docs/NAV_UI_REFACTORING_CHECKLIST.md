# 🔥 네비 UI 리팩토링 체크리스트 (현재 화면 기준)

## 🥇 1순위: 상태 표시 UI 추가

### 현재 상태
- ❌ 화면 어디에도 현재 상태가 텍스트로 표시되지 않음
- ❌ 사용자가 "지금 뭘 하고 있는지" 알 수 없음

### 추가할 UI 요소

#### 위치: 상단 (헤더 아래, 검색바 위)

```
┌─────────────────────────────────────────┐
│  [헤더: "지도"]                          │
├─────────────────────────────────────────┤
│  [상태 표시] ← 여기에 추가              │
│  ┌─────────────────────────────────┐   │
│  │ 🔍 장소를 검색하세요...         │   │
│  └─────────────────────────────────┘   │
```

#### 각 Phase별 상태 텍스트

| Phase | 상태 텍스트 | 스타일 |
|-------|-----------|--------|
| IDLE | "검색 중" | 회색, 작은 글씨 |
| SEARCHING | "검색 중" | 회색, 작은 글씨 |
| CONFIRMED | "목적지 선택됨" | 파란색, 중간 글씨 |
| NAVIGATING | "안내 중" | 초록색, 굵은 글씨 |
| ARRIVED | "도착했습니다" | 초록색, 굵은 글씨 |

### 구현 위치
- `MapPageContainer.tsx`에서 phase에 따라 조건부 렌더링
- `NavigationStatusBar.tsx` 위에 배치 (또는 통합)

## 🥈 2순위: 출발 버튼 클릭 시 즉각적인 UI 변화

### 현재 문제
- ❌ 출발 버튼 클릭 후 아무 변화 없음
- ❌ 사용자가 "버튼이 먹혔는지" 모름

### 해결: 출발 버튼 클릭 즉시 3가지 중 1개 이상 발생

#### Option 1: 상단 상태바 즉시 표시
```typescript
// handleStartNavigation에서
setIsRouteCalculating(true); // 즉시
// 상단 상태바 표시 ("경로 계산 중...")
```

#### Option 2: 하단 카드 즉시 업데이트
```typescript
// NavigationCard에서
if (isRouteCalculating) {
  return "경로 계산 중...";
}
```

#### Option 3: 지도 줌 즉시 변경
```typescript
// 출발 버튼 클릭 시
map.fitBounds([origin, destination]);
```

### 구현 위치
- `MapPageContainer.tsx`의 `handleStartNavigation` 함수
- `NavigationCard.tsx`의 CONFIRMED 모드

## 🥉 3순위: 하단 카드 "의미 부여"

### 현재 문제
- ❌ 하단 카드가 모든 상태에서 비슷하게 보임
- ❌ 상태별 역할 구분이 없음

### 해결: 상태별 카드 역할 명확화

#### CONFIRMED 상태
```
┌─────────────────────────────────┐
│ 여기로 갈까요?                  │
│                                 │
│ 🎯 서울월드컵경기장             │
│    약 1.2km                     │
│                                 │
│  [ 여기에 갈게요 ]               │ ← 출발 버튼 아님
└─────────────────────────────────┘
```
**역할**: 목적지 확인 + 다음 단계 유도

#### NAVIGATING 상태 (경로 계산 중)
```
┌─────────────────────────────────┐
│ 🎯 서울월드컵경기장             │
│    ⏳ 경로 계산 중...            │ ← 즉시 표시
└─────────────────────────────────┘
```
**역할**: 진행 상태 표시

#### NAVIGATING 상태 (경로 계산 완료)
```
┌─────────────────────────────────┐
│ 🎯 서울월드컵경기장             │
│    🚗 이동 중...                │
│    1.2km · 15분                 │
│                                 │
│  [ 중지 ]                       │
└─────────────────────────────────┘
```
**역할**: 네비게이션 정보 + 제어

### 구현 위치
- `NavigationCard.tsx`의 각 mode별 렌더링
- `isRouteCalculating` 상태에 따른 조건부 렌더링

## 4️⃣ 상단 검색바 처리

### 현재 문제
- ❌ NAVIGATING 상태에서도 검색바가 보일 수 있음
- ❌ 상태와 맞지 않는 UI 표시

### 해결: Phase별 검색바 표시 규칙

| Phase | 검색바 표시 | 대체 UI |
|-------|-----------|---------|
| IDLE | ✅ 표시 | - |
| SEARCHING | ✅ 표시 | - |
| CONFIRMED | ✅ 표시 (입력값 고정) | - |
| NAVIGATING | ❌ 숨김 | 상단 상태바 |
| ARRIVED | ❌ 숨김 | - |

### 구현 위치
- `MapPageContainer.tsx`의 검색바 조건부 렌더링
- 현재: `phase === 'IDLE' || phase === 'SEARCHING'`
- 수정: `phase !== 'NAVIGATING' && phase !== 'ARRIVED'`

---

## 5️⃣ 지도 연출 (후순위)

### 현재 상태
- ✅ 기본 지도 렌더링은 정상
- ❌ 출발 시 줌/중앙 조정이 불명확

### 개선 사항
1. CONFIRMED → 출발지 + 목적지 fitBounds
2. NAVIGATING → 경로 전체 fitBounds
3. 출발지 마커 추가 (파란 점)

### 구현 위치
- `WebMapRenderer.tsx`의 카메라 제어 로직
- `MapController.tsx`의 경로 계산 완료 시 fitBounds
