# 🔥 Phase 29: 실전 테스트 & 성공 기준 정의

## 🎯 목표

> **설계가 아니라 행동으로 "이 지도 UX가 된다"는 걸 확인한다.**

---

## 1️⃣ 실전 테스트 환경 세팅

### 테스트 조건 (절대 기준)

- ❌ 개발자 설명 없음
- ❌ 사용법 안내 없음
- ⭕ 지도 화면만 보여줌
- ⭕ **타이틀: "지금 말하면 찾아드립니다" 노출**

👉 사용자는 **스스로 행동해야 한다**

### 테스트 환경 체크리스트

- [ ] 지도 화면이 정상적으로 로드됨
- [ ] "지금 말하면 찾아드립니다" 타이틀 노출 확인
- [ ] 마이크 권한 허용됨
- [ ] 개발자 설명/안내 없음

## 2️⃣ 테스트 대상 (최소 3명)

### 구성 권장

- **테스터 A**: 지도 앱 많이 쓰는 사람
- **테스터 B**: 지도 앱 보통 사용자
- **테스터 C**: 음성 인터페이스 어색해하는 사람

### 테스터 정보 기록

```
테스터: [이름/코드]
연령: [대략적]
지도 앱 사용 빈도: [자주/보통/드물게]
음성 인터페이스 경험: [많음/보통/없음]
```

---

## 3️⃣ 테스트 시나리오

### 🧪 시나리오 A — 첫 진입 (10초)

**관찰 포인트**

- 사용자가 **말을 하는가?**
- "이거 말하면 되는 건가?"를 묻는가?
- 화면을 보며 무엇을 하는지 고민하는 시간

**✅ 성공 기준**

- 10초 안에 **자발적 발화 1회 이상**
- 설명 없이 행동 시작

**❌ 실패 신호**

- "이거 어떻게 쓰는 거야?"
- "마이크 눌러야 돼?"
- 10초 이상 멍하니 화면만 봄

---

### 🧪 시나리오 B — 추천 수용

**사용자 발화 예시**

```
"근처 카페"
"조용한 곳"
"여기서 가까운 곳"
```

**관찰 포인트**

- 추천 1곳을 **이해했는가**
- "여기로 갈게요" 같은 후속 발화가 나오는가
- 추천 배너를 읽고 반응하는가

**✅ 성공 기준**

- 설명 없이 **후속 의도 발화 발생**
- 추천을 자연스럽게 수용

**❌ 실패 신호**

- "이게 뭐야?"
- "왜 이걸 추천해?"
- 추천을 무시하고 다른 행동

---

### 🧪 시나리오 C — 실행 & 중단

**관찰 포인트**

- 방향 힌트 이해 여부
- "잠깐만요" / "그만할게요" 자연 사용 여부
- 중단 시 불안/거부 반응

**✅ 성공 기준**

- 중단 UX에 **불안/거부 반응 없음**
- "안내 그만할게요" 버튼을 자연스럽게 사용

**❌ 실패 신호**

- "갑자기 왜 길 안내야?"
- "이거 끌 수 있어?"
- 중단 버튼을 못 찾음

---

### 🧪 시나리오 D — 기억 질문

**관찰 포인트**

- "기억해줘요"를 **의식적으로 누르는가**
- 거절 시 불만 없는가
- 기억 설명 토스트를 이해하는가

**✅ 성공 기준**

- 기억 UX를 **통제 가능하다고 인식**
- 선택권이 있다고 느낌

**❌ 실패 신호**

- "이거 저장된 거야?"
- "왜 저장하라고 해?"
- 불안/감시 느낌

---

## 4️⃣ 실패로 간주하는 신호 (중요)

아래 중 하나라도 나오면 **리팩 대상**:

- ❌ "이거 어떻게 쓰는 거야?"
- ❌ "마이크 눌러야 돼?"
- ❌ "갑자기 왜 길 안내야?"
- ❌ "이거 저장된 거야?"
- ❌ "이게 뭐야?" (추천/배너에 대해)

👉 이 질문이 나오면 **UX가 설명을 요구한 것**

---

## 5️⃣ 성공 판정표 (체크리스트)

| 항목 | 성공 기준 | 테스터 A | 테스터 B | 테스터 C |
|------|----------|----------|----------|----------|
| 첫 발화 | 10초 이내 | ☐ | ☐ | ☐ |
| 추천 이해 | 설명 없이 수용 | ☐ | ☐ | ☐ |
| 실행 | 강요 느낌 없음 | ☐ | ☐ | ☐ |
| 중단 | 불안 없음 | ☐ | ☐ | ☐ |
| 기억 | 선택권 인식 | ☐ | ☐ | ☐ |

👉 **5개 중 4개 이상 통과 = 성공**
