# 🔒 초기 KPI 대시보드 (딱 5개 지표, 바로 쓰는 기준)

## 목적

**"지금 이 제품이 살아 있는가?"를 하루 1분 안에 판단**

---

## KPI 설계 원칙

- 지표는 5개를 넘기지 않는다
- 퍼널 전체를 한 줄로 읽을 수 있어야 한다
- "좋아 보이는 숫자" ❌ → "결정에 쓰는 숫자" ⭕

---

## 📊 KPI 1 — QR 유입량 (바이럴 생존 여부)

### 지표
**QR Entry / Day**

### 이벤트
`qr_entered`

### 계산
```
QR Entry / Day = 하루 동안 qr_entered 이벤트 수
```

### 판단 기준

| 수치 | 의미 | 대응 |
|------|------|------|
| 0~5 | 아직 전파 안 됨 | QR 배포 위치 확인 |
| 10+ | 바이럴 씨앗 있음 | 현재 유지 |
| 30+ | 구조 검증 시작 | 바이럴 전략 확대 검토 |

### 👉 QR 전략이 먹히는지 바로 판단

---

## 📊 KPI 2 — 게스트 체류 (가치 전달 여부)

### 지표
**Guest Avg Views**

### 이벤트
`guest_view`

### 계산
```
Guest Avg Views = guest_view 이벤트 수 / qr_entered 이벤트 수
```

### 판단 기준

| 수치 | 의미 | 대응 |
|------|------|------|
| <2 | 가치 전달 실패 | 첫 화면 UX 점검 |
| 3~5 | 탐색 시작 | 현재 유지 |
| 5+ | "일단 보게 만든다" 성공 | 게스트 전략 성공 |

### 👉 무가입 전략의 핵심 지표

---

## 📊 KPI 3 — Lazy Signup 트리거율 (전환 잠재력)

### 지표
**Lazy Signup Trigger Rate**

### 계산
```
Lazy Signup Trigger Rate = auth_required / qr_entered × 100%
```

### 판단 기준

| 수치 | 의미 | 대응 |
|------|------|------|
| <10% | 행동 욕구 약함 | 상품/채팅 품질 점검 |
| 15~30% | 정상 | 현재 유지 |
| 40%+ | UX 유혹 강함 | 전환율 최적화 필요 |

### 👉 "보고만 하고 끝나는지" 확인

---

## 📊 KPI 4 — 가입 전환율 (마찰 체크)

### 지표
**Signup Conversion Rate**

### 계산
```
Signup Conversion Rate = signup_completed / auth_required × 100%
```

### 판단 기준

| 수치 | 의미 | 대응 |
|------|------|------|
| <30% | 가입 UX 문제 | 가입 화면 문구/단계 점검 |
| 40~60% | 정상 | 현재 유지 |
| 70%+ | 매우 좋음 | 가입 UX 우수 |

### 👉 가입이 '문턱'인지 '다리'인지 판단

---

## 📊 KPI 5 — 복귀 성공률 (제품 완성도)

### 지표
**Resume Success Rate**

### 계산
```
Resume Success Rate = post_signup_resumed / signup_completed × 100%
```

### 판단 기준

| 수치 | 의미 | 대응 |
|------|------|------|
| <70% | 구조 실패 (가장 위험) | 복귀 로직 즉시 점검 |
| 80~90% | 정상 | 현재 유지 |
| 95%+ | 제품 감각 우수 | 복귀 UX 완벽 |

### 👉 이 지표가 제품의 생사
(여기서 떨어지면 전부 무너진다)

---

## 🧮 하루 1분 판단 공식

### 아침에 이것만 본다 👇

```
QR 들어오나? (KPI 1)
  ↓
  → 보고 있나? (KPI 2)
    ↓
    → 하려다 막히나? (KPI 3)
      ↓
      → 가입하나? (KPI 4)
        ↓
        → 다시 돌아오나? (KPI 5)
```

### 판단 규칙

- ✅ **YES가 이어지면** → 계속 간다
- ⚠️ **중간에 끊기면** → 그 지점만 고친다

---

## 📊 전체 퍼널 한 눈에 보기

```
QR Entry (KPI 1)
  ↓
  → Guest Views (KPI 2)
    ↓
    → Auth Required (KPI 3)
      ↓
      → Signup Completed (KPI 4)
        ↓
        → Post Signup Resumed (KPI 5)
```

### 예시

```
Day 1:
- QR Entry: 15
- Guest Avg Views: 4
- Lazy Signup Trigger: 20% (3명)
- Signup Conversion: 66% (2명)
- Resume Success: 100% (2명)
→ 판단: 정상, 계속 진행
```

## 📈 KPI 대시보드 구성 (간단)

### 필수 표시 항목

1. **QR Entry / Day** (숫자)
2. **Guest Avg Views** (숫자)
3. **Lazy Signup Trigger Rate** (퍼센트)
4. **Signup Conversion Rate** (퍼센트)
5. **Resume Success Rate** (퍼센트)

### 선택 표시 (선택)

- 전체 퍼널 시각화 (선형 차트)
- 일별 트렌드 (7일)

---

## 🎯 일일 체크리스트

### 매일 아침 (1분)

- [ ] QR Entry 확인 (10+?)
- [ ] Guest Avg Views 확인 (3+?)
- [ ] Lazy Signup Trigger 확인 (15%+?)
- [ ] Signup Conversion 확인 (40%+?)
- [ ] Resume Success 확인 (80%+?)

### 문제 발견 시

- ⚠️ **한 지표만 떨어짐** → 해당 지표만 점검
- 🔴 **복귀 성공률(KPI 5) 떨어짐** → 즉시 긴급 점검

---

## 🔥 천재 모드 결론

**이 5개 지표만 보면:**
- QR 전략이 먹히는지 알 수 있음
- 게스트 전략이 먹히는지 알 수 있음
- 가입 UX가 문제인지 알 수 있음
- 복귀 로직이 작동하는지 알 수 있음

**더 복잡해지면:**
- 판단 못 함
- 데이터만 쌓임
- 결정 못 함

**5개로 고정한다.**
