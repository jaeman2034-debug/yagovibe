# 🧲 "함께 운동하기" 버튼 클릭 UX 설계

## 📋 목적
외부인이 공개 블로그에서 "이 팀에 가입하고 싶다"는 욕구를 실제 가입으로 전환시키는 완전한 플로우

---

## 1️⃣ 전체 플로우 다이어그램

```
공개 블로그 페이지
  ↓
[함께 운동하기] 클릭
  ↓
┌─────────────────┐
│ 로그인 여부 체크 │
└─────────────────┘
  ↓
  ├─ 비로그인 → 로그인 페이지 → 로그인 완료 → 원래 블로그로 복귀 → 가입 요청 자동 생성
  │
  └─ 로그인됨 → 가입 요청 생성 → 성공 메시지 → 관리자 승인 대기
```

---

## 2️⃣ 케이스별 상세 플로우

### 케이스 A: 비로그인 사용자

**1단계: 버튼 클릭**
- 위치: 공개 블로그 히어로 영역 또는 모집 카드
- 버튼: `함께 운동하기` / `이 팀에 가입하기`

**2단계: 로그인 페이지로 이동**
- URL: `/start` 또는 `/login?redirect=/teams/{teamSlug}/blog`
- 메시지: `이 팀에 가입하려면 로그인이 필요합니다.`

**3단계: 로그인 완료**
- 원래 블로그 페이지로 자동 복귀
- URL 파라미터에 `?join=true` 추가
- 페이지 로드 시 자동으로 가입 요청 생성

**4단계: 가입 요청 생성**
- `team_requests` 컬렉션에 문서 생성
- 성공 메시지 표시: `가입 요청이 전송되었습니다! 관리자의 승인을 기다려주세요.`

---

### 케이스 B: 로그인된 사용자 (일반)

**1단계: 버튼 클릭**
- 위치: 공개 블로그 히어로 영역 또는 모집 카드
- 버튼: `함께 운동하기` / `이 팀에 가입하기`

**2단계: 즉시 가입 요청 생성**
- `team_requests` 컬렉션에 문서 생성
- 중복 요청 체크:
  - 이미 요청 있음 → `이미 가입 요청을 보내셨습니다. 관리자의 승인을 기다려주세요.`
  - 새 요청 → 성공 메시지 표시

**3단계: 성공 메시지**
- 토스트 또는 모달: `가입 요청이 전송되었습니다! 관리자의 승인을 기다려주세요.`
- 버튼 비활성화 또는 "요청 완료" 상태로 변경

---

### 케이스 C: 이미 팀 멤버

**1단계: 버튼 클릭**
- 위치: 공개 블로그 히어로 영역 또는 모집 카드

**2단계: 멤버 확인**
- `team_members` 컬렉션에서 현재 사용자 확인
- 이미 멤버면 → 버튼 텍스트 변경: `이미 팀원입니다` (비활성화)
- 또는 팀 대시보드로 이동 버튼 표시

---

### 케이스 D: 관리자 (소유자)

**1단계: 버튼 클릭**
- 위치: 공개 블로그 히어로 영역 또는 모집 카드

**2단계: 관리자 확인**
- 자신의 팀이면 → 가입 요청 버튼 숨김
- 대신 "팀 대시보드로 이동" 버튼 표시

---

## 3️⃣ 가입 요청 데이터 구조

### Firestore: `team_requests/{teamId}_{uid}`

```typescript
{
  teamId: string;
  uid: string;
  status: "pending" | "approved" | "rejected";
  message?: string; // 사용자가 입력한 가입 메시지 (선택)
  createdAt: Timestamp;
  reviewedAt?: Timestamp;
  reviewedBy?: string; // 관리자 uid
  rejectionReason?: string; // 거절 사유
}
```

### 중복 요청 방지
- 문서 ID: `${teamId}_${uid}` (고유)
- 이미 존재하면 `setDoc` 대신 업데이트 또는 에러 처리

## 4️⃣ UI 상태 변화

### 버튼 상태

**초기 상태 (비로그인)**
```
[ 함께 운동하기 ] (파란색, 활성)
```

**초기 상태 (로그인, 미요청)**
```
[ 함께 운동하기 ] (파란색, 활성)
```

**요청 완료 상태**
```
[ ✓ 가입 요청 완료 ] (회색, 비활성)
또는
[ 가입 요청 보냄 · 승인 대기 중 ] (회색, 비활성)
```

**이미 멤버 상태**
```
[ 이미 팀원입니다 ] (회색, 비활성)
또는
[ 팀 대시보드로 이동 ] (파란색, 활성) → 클릭 시 대시보드로 이동
```

**관리자 상태**
```
[ 팀 대시보드로 이동 ] (파란색, 활성)
```

## 5️⃣ 성공/에러 메시지

### 성공 메시지
```
✅ 가입 요청이 전송되었습니다!
관리자의 승인을 기다려주세요.
승인되면 알림을 드리겠습니다.
```

### 중복 요청 메시지
```
ℹ️ 이미 가입 요청을 보내셨습니다.
관리자의 승인을 기다려주세요.
```

### 에러 메시지
```
❌ 가입 요청 전송에 실패했습니다.
잠시 후 다시 시도해주세요.
```

### 이미 멤버 메시지
```
✅ 이미 이 팀의 멤버입니다.
[ 팀 대시보드로 이동 ] 버튼으로 팀 페이지로 가실 수 있습니다.
```

## 6️⃣ 관리자 승인 플로우 (참고)

### 관리자 대시보드에서 보이는 것

**가입 요청 알림**
```
📩 새로운 가입 요청 3건
[ 승인하기 ] [ 거절하기 ]
```

**가입 요청 목록**
- 요청자 이름/이메일
- 가입 메시지 (있는 경우)
- 요청 일시
- 승인/거절 버튼

### 승인 시
1. `team_requests` 문서 업데이트: `status: "approved"`
2. `team_members` 컬렉션에 멤버 추가
3. 요청자에게 알림 전송 (선택)
4. 요청자 화면에서 자동으로 "이미 팀원입니다" 상태로 변경

### 거절 시
1. `team_requests` 문서 업데이트: `status: "rejected"`, `rejectionReason`
2. 요청자에게 알림 전송 (선택)
3. 요청자 화면에서 버튼 다시 활성화 (재요청 가능)

## 7️⃣ Analytics 이벤트

### 추적 이벤트

1. **team_blog_join_click**
   - 위치: 공개 블로그 페이지
   - 파라미터: `teamId`, `teamSlug`, `source: "hero" | "recruitment_card"`

2. **team_join_request_sent**
   - 위치: 가입 요청 생성 성공
   - 파라미터: `teamId`, `teamSlug`, `hasMessage: boolean`

3. **team_join_request_duplicate**
   - 위치: 중복 요청 시도
   - 파라미터: `teamId`, `teamSlug`

4. **team_join_login_required**
   - 위치: 비로그인 사용자 클릭
   - 파라미터: `teamId`, `teamSlug`

## 9️⃣ 실제 사용 예시

### 시나리오 1: 비로그인 사용자

1. 공개 블로그 페이지 방문
2. `함께 운동하기` 클릭
3. 로그인 페이지로 이동
4. 로그인 완료
5. 원래 블로그 페이지로 복귀 (`?join=true`)
6. 자동으로 가입 요청 생성
7. 성공 메시지: `가입 요청이 전송되었습니다!`

### 시나리오 2: 로그인된 사용자

1. 공개 블로그 페이지 방문
2. `이 팀에 가입하기` 클릭
3. 즉시 가입 요청 생성
4. 성공 메시지: `가입 요청이 전송되었습니다!`
5. 버튼 비활성화: `가입 요청 보냄 · 승인 대기 중`

### 시나리오 3: 이미 멤버

1. 공개 블로그 페이지 방문
2. `함께 운동하기` 버튼이 `이미 팀원입니다`로 표시됨
3. 또는 `팀 대시보드로 이동` 버튼 표시
