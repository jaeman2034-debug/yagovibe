# 🎯 QR 로그인 알림 기준선 튜닝 가이드

## 📊 목적

실제 운영 데이터를 기반으로 알림 임계치를 조정하여:
- ✅ 진짜 문제만 감지
- ❌ 오탐(false positive) 최소화
- ⚡ 빠른 대응 가능

---

## 📈 튜닝 프로세스 (1~2일 후)

### 1단계: 데이터 수집 (24~48시간)

#### 수집할 데이터

1. **성공률 분포**
   - 시간대별 성공률
   - 피크 시간대 vs 오프피크 시간대
   - 최소/최대/평균 성공률

2. **SMS 실패율 분포**
   - 시간대별 SMS 실패율
   - 실패 원인별 분류

3. **로그인 시간 분포**
   - 평균 로그인 시간
   - P50, P95, P99 로그인 시간
   - 시간대별 차이

4. **세션 만료율 분포**
   - 시간대별 만료율
   - 만료 원인별 분류

#### 데이터 수집 방법

**대시보드에서 확인:**
```
/admin/qr-login
```

**또는 Firestore 쿼리:**
```javascript
// 최근 48시간 성공률
const stats = await getQRLoginStats(2); // 2일
console.log('성공률:', stats.successRate);
console.log('평균 시간:', stats.avgTimeToLogin);
```

### 2단계: 기준선 설정

#### 성공률 기준선

**현재 임계치:** 90%

**튜닝 방법:**
1. 최근 48시간 평균 성공률 확인
2. 최소 성공률 확인 (피크 시간대 제외)
3. 기준선 = 평균 성공률 - 2σ (표준편차)

**예시:**
```
평균 성공률: 95%
최소 성공률: 92%
표준편차: 2%

→ 기준선: 95% - (2 × 2%) = 91%
→ 임계치: 91% (현재 90%보다 높게)
```

#### SMS 실패율 기준선

**현재 임계치:** 5%

**튜닝 방법:**
1. 최근 48시간 평균 SMS 실패율 확인
2. 최대 SMS 실패율 확인
3. 기준선 = 평균 실패율 + 2σ

**예시:**
```
평균 SMS 실패율: 2%
최대 SMS 실패율: 4%
표준편차: 1%

→ 기준선: 2% + (2 × 1%) = 4%
→ 임계치: 4% (현재 5%보다 낮게, 더 민감하게)
```

#### 평균 로그인 시간 기준선

**현재 임계치:** 60초

**튜닝 방법:**
1. P95 로그인 시간 확인 (95%의 사용자가 이 시간 내에 로그인)
2. P99 로그인 시간 확인
3. 기준선 = P95 또는 P99

**예시:**
```
평균: 35초
P95: 55초
P99: 75초

→ 기준선: 60초 (P99 근처, 현재 유지)
또는
→ 기준선: 55초 (P95, 더 민감하게)
```

#### 세션 만료율 기준선

**현재 임계치:** 10%

**튜닝 방법:**
1. 최근 48시간 평균 만료율 확인
2. 최대 만료율 확인
3. 기준선 = 평균 만료율 + 2σ

**예시:**
```
평균 만료율: 5%
최대 만료율: 8%
표준편차: 1.5%

→ 기준선: 5% + (2 × 1.5%) = 8%
→ 임계치: 8% (현재 10%보다 낮게, 더 민감하게)
```

### 3단계: 시간대별 분리 (고급)

#### 피크 시간대 vs 오프피크 시간대

**피크 시간대 (예: 오전 9시~12시, 오후 6시~9시):**
- 트래픽이 많아 성공률이 약간 낮을 수 있음
- 임계치를 약간 낮춤 (예: 성공률 88%)

**오프피크 시간대 (예: 새벽 2시~6시):**
- 트래픽이 적어 성공률이 높을 수 있음
- 임계치를 높게 설정 (예: 성공률 95%)

#### 구현 방법

`functions/src/monitoring/qrLoginAlert.ts` 수정:

```typescript
function getThresholdsForTime(hour: number): typeof THRESHOLDS {
  // 피크 시간대 (9-12시, 18-21시)
  const isPeak = (hour >= 9 && hour < 12) || (hour >= 18 && hour < 21);
  
  if (isPeak) {
    return {
      successRate: 88, // 피크 시간대는 약간 낮게
      smsFailureRate: 6, // 피크 시간대는 약간 높게
      avgLoginTime: 70, // 피크 시간대는 약간 길게
      expirationRate: 12, // 피크 시간대는 약간 높게
    };
  }
  
  // 오프피크 시간대
  return {
    successRate: 95, // 오프피크는 높게
    smsFailureRate: 3, // 오프피크는 낮게
    avgLoginTime: 50, // 오프피크는 짧게
    expirationRate: 8, // 오프피크는 낮게
  };
}
```

### 4단계: 임계치 적용

#### 코드 수정

`functions/src/monitoring/qrLoginAlert.ts`:

```typescript
/**
 * 임계치 설정 (실제 데이터 기반 튜닝)
 * 
 * 튜닝 날짜: 2024-01-XX
 * 데이터 기간: 최근 48시간
 * 평균 성공률: 95%
 * 평균 SMS 실패율: 2%
 * P95 로그인 시간: 55초
 * 평균 만료율: 5%
 */
const THRESHOLDS = {
  successRate: 91, // 평균 95% - 2σ
  smsFailureRate: 4, // 평균 2% + 2σ
  avgLoginTime: 55, // P95
  expirationRate: 8, // 평균 5% + 2σ
};
```

### 5단계: 검증 (24시간)

#### 모니터링

1. **알림 빈도 확인**
   - 하루 알림 횟수: 목표 0~2회
   - 오탐 발생 여부

2. **실제 문제 감지 확인**
   - 알림 발생 시 실제 문제가 있었는지 확인
   - 문제가 없었는데 알림이 온 경우 → 임계치 조정

3. **누락 확인**
   - 문제가 있었는데 알림이 오지 않은 경우 → 임계치 조정

## 📊 튜닝 결과 예시

### Before (초기 기준선)

```
성공률 < 90% → 알림
SMS 실패율 > 5% → 알림
평균 로그인 시간 > 60초 → 알림
세션 만료율 > 10% → 알림
```

**결과:**
- 하루 알림: 15회
- 오탐: 12회 (80%)
- 실제 문제 감지: 3회

### After (튜닝 후)

```
성공률 < 91% → 알림
SMS 실패율 > 4% → 알림
평균 로그인 시간 > 55초 → 알림
세션 만료율 > 8% → 알림
```

**결과:**
- 하루 알림: 2회
- 오탐: 0회 (0%)
- 실제 문제 감지: 2회
