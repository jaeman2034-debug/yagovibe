# 협회 도입 미팅 마무리 멘트 & 계약 전환 스크립트

## 🎯 사용 가이드

**설명 끝난 뒤, 질문 몇 개 오간 뒤, 공기가 살짝 애매해질 때 사용**

**목표:** 결정 유도 (질문이 아닌 제안으로)

**원칙:** ❌ "어떻게 생각하세요?" ❌ "검토해 보시죠" ⭕ 제안으로 결정 만들기

---

## 1️⃣ 가장 안전한 클로징 (기본형)

```
그럼 이렇게 해보는 건 어떨까요.
```

**[2초 멈춤]**

```
노원구 축구협회에서

시범 운영으로 한 번만 써보시고,

운영이 맞는지 직접 판단하시는 겁니다.
```

**👉 효과:**
- 부담 없음
- 퇴로 열어둠
- 결정 부담 최소화

---

## 2️⃣ 시범 운영 조건 제시 (안심 장치)

```
시범 운영 기간에는

협회 방식은 그대로 두고,

기존 운영을 방해하지 않습니다.
```

**[1초 멈춤]**

```
문제 생기면 언제든 중단하셔도 됩니다.
```

**👉 효과:**
- 협회가 가장 듣고 싶어 하는 말
- 리스크 최소화
- 안심감 제공

---

## 3️⃣ 결정 버튼을 '작게' 만든다 (천재 포인트)

### ❌ 하지 말 것
```
"도입하시겠습니까?"
```

### ⭕ 이렇게 말해라

```
그럼 관리자 계정 한 분만 지정해 주시면

저희가 나머지는 준비하겠습니다.
```

**👉 효과:**
- 결정의 크기를 줄이면 결정이 나온다
- "도입"이 아니라 "계정 지정"
- 부담감 최소화

---

## 4️⃣ 협회장이 망설일 때 쓰는 한 문장

```
이건 새로운 걸 도입하는 게 아니라,

지금 하시던 일을 더 편하게 하는 겁니다.
```

**[3초 멈춤 - 반응 관찰]**

**👉 효과:**
- 변화에 대한 두려움 완화
- 기존 방식의 개선이라는 인식
- 대부분 이 지점에서 고개 끄덕임

---

## 5️⃣ 실무자가 걱정할 때 쓰는 문장

```
실무자분들께는

전화 받는 일이 줄어드는 구조입니다.
```

**[1초 멈춤]**

```
업무가 늘어나는 방향은 아닙니다.
```

**👉 효과:**
- 실무자 반대 차단
- 업무 부담 감소 메시지
- 안심감 제공

---

## 6️⃣ 가격/비용 질문이 나올 때 (중요)

### ❌ 바로 가격 말하지 마라

### ⭕ 이렇게 말해라

```
시범 운영에서는

운영이 맞는지부터 보시죠.
```

**[1초 멈춤]**

```
그 다음에 협회 상황에 맞춰

협의해도 늦지 않습니다.
```

**👉 효과:**
- 가치 → 사용 → 비용 순서 고정
- 시범 운영 중 비용 부담 없음
- 협회 상황에 맞춰 협의 가능

---

## 7️⃣ 미팅 종료 직전, 반드시 이 말은 남겨라

```
노원구 축구협회가

이 모델의 첫 기준이 될 수 있습니다.
```

**[2초 멈춤]**

```
다른 구에서 참고하게 될 사례입니다.
```

**👉 효과:**
- 결정자의 '위치'를 올려주는 말
- 자존감 충족
- 결정 동기 제공

## 🧠 이 클로징의 구조적 이유

### 협회가 싫어하는 것
- 위험 싫어함
- 책임 싫어함

### 그래서 주는 것
1. **작은 결정** ("계정 지정" vs "도입")
2. **중단 가능** ("언제든 중단 가능")
3. **기존 방식 유지** ("방식은 그대로")

**👉 이 3개를 동시에 주면 결정이 나온다**

## 📋 상황별 클로징

### 상황 1: 긍정적 반응

```
좋으시면 이렇게 진행하겠습니다.

관리자 계정 한 분만 지정해 주시면

저희가 나머지는 준비하겠습니다.

1주일 내에 준비 완료되면

다시 만나서 시스템을 보여드리겠습니다.
```

---

### 상황 2: 망설임

```
이건 새로운 걸 도입하는 게 아니라,

지금 하시던 일을 더 편하게 하는 겁니다.

시범 운영으로 한 번만 써보시고

판단하시면 됩니다.

문제 생기면 언제든 중단하셔도 되니까

부담 갖지 마세요.
```

---

### 상황 3: 부정적 반응

```
네, 충분히 이해합니다.

지금 방식이 편하시다면

굳이 바꿀 필요는 없습니다.

다만, 혹시 나중에 필요하실 수도 있으니

자료는 남겨두시면 좋을 것 같습니다.

언제든 연락 주시면

도와드리겠습니다.
```

---

### 상황 4: 가격 질문

```
시범 운영에서는

운영이 맞는지부터 보시죠.

비용은 그 다음에 협회 상황에 맞춰

협의해도 늦지 않습니다.

효과가 검증되면

그때 가격을 논의해도 됩니다.
```

---

### 상황 5: 실무자 걱정

```
실무자분들께는

전화 받는 일이 줄어드는 구조입니다.

업무가 늘어나는 방향은 아닙니다.

오히려 작업량이 줄어들 겁니다.

시범 운영 기간 동안

저희가 교육도 해드리고

지원도 해드리니까

걱정하지 마세요.
```

## ✅ 체크리스트

### 클로징 전
- [ ] 설명과 질문 완료 확인
- [ ] 핵심 멘트 준비
- [ ] 다음 단계 명확화

### 클로징 중
- [ ] "그럼 이렇게 해보는 건 어떨까요"로 시작
- [ ] 결정 버튼 작게 만들기 ("계정 지정")
- [ ] 안심 장치 제시 (중단 가능)
- [ ] 마지막 한 문장 남기기 ("노원구가 기준")

### 클로징 후
- [ ] 다음 단계 확인 (일정, 자료 등)
- [ ] 감사 표현
- [ ] 후속 조치

## 🔑 핵심 원칙

### DO (해야 할 것)
- ✅ 제안으로 결정 만들기
- ✅ 결정 버튼 작게 만들기
- ✅ 안심 장치 제시
- ✅ 결정자의 위치 올려주기

### DON'T (하지 말 것)
- ❌ "어떻게 생각하세요?" (질문)
- ❌ "검토해 보시죠" (미루기)
- ❌ "도입하시겠습니까?" (큰 결정)
- ❌ 바로 가격 말하기

## 💡 클로징 타이밍

### 적절한 타이밍
- 설명과 질문이 충분히 끝난 후
- 공기가 살짝 애매해질 때
- 긍정적 반응이 있을 때
- 시간이 5분 정도 남았을 때

### 부적절한 타이밍
- 설명 중간
- 질문이 계속 나올 때
- 부정적 반응이 강할 때
- 시간이 부족할 때
