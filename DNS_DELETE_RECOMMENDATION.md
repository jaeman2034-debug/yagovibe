# 🗑️ DNS 레코드 삭제 추천

## ✅ 유지해야 할 레코드

1. **CNAME 레코드**
   - 호스트: `www`
   - 이유: 이미 연결되어 있음

2. **TXT 레코드**
   - 호스트: `@`
   - 값: `"hosting-site-yago-vibe-spt"`
   - 이유: Firebase 사이트 인증용

---

## ⚠️ 삭제해야 할 A 레코드 (임시 추천)

Firebase Hosting은 보통 **2개의 A 레코드**만 필요합니다.

### **현재 A 레코드 4개:**
1. `@` → `199.36.158.100`
2. `@` → `199.36.158.101`
3. `@` → `199.36.158.102`
4. `@` → `199.36.158.103`

### **추천 삭제 방법:**

**옵션 1: 마지막 2개 삭제 (일반적)**
- 삭제: `199.36.158.102`, `199.36.158.103`
- 유지: `199.36.158.100`, `199.36.158.101`

**옵션 2: 모든 A 레코드 삭제 후 재설정 (가장 안전)**
- 모든 A 레코드 4개 삭제
- Firebase 콘솔에서 요청한 정확한 IP 주소 2개로 새로 추가

---

## 🎯 최종 추천

### **방법 1: Firebase 콘솔 확인 후 삭제 (권장)**

1. Firebase 콘솔 → Hosting → 커스텀 도메인
2. `yagovibe.com` 클릭
3. 요청하는 정확한 IP 주소 2개 확인
4. 요청하지 않은 IP의 A 레코드 삭제

### **방법 2: 임시로 마지막 2개 삭제**

Firebase 콘솔 확인이 어렵다면:
- **삭제할 호스트:**
  - `@` → `199.36.158.102` (A 레코드)
  - `@` → `199.36.158.103` (A 레코드)

- **유지할 호스트:**
  - `@` → `199.36.158.100` (A 레코드)
  - `@` → `199.36.158.101` (A 레코드)
  - `www` → `yago-vibe-spt.web.app.` (CNAME 레코드)
  - `@` → `"hosting-site-yago-vibe-spt"` (TXT 레코드)

---

## ⚠️ 주의사항

1. **CNAME (`www`)는 절대 삭제하지 마세요!**
2. **TXT 레코드는 절대 삭제하지 마세요!**
3. **A 레코드 삭제 후 Firebase 콘솔에서 "검증" 클릭**
4. **문제가 계속되면 모든 A 레코드를 삭제하고 Firebase가 요청한 IP로 재설정**
