# Phase 4 로드맵 초안 (모듈 우선순위만)

**목적**: Phase 4 확장 시 구현할 모듈의 우선순위를 정의 (아직 구현 ❌, 순서만 ⭕)

---

## 1️⃣ Phase 4 확장의 의미

### Phase 4 = 자동화(정산/기록) 단계

**핵심 원칙:**
- Phase 3-4-1: "보여주고 잠그는 것" (정보 확인 중심)
- Phase 4: "자동화(정산/기록)" (반복 업무 자동화)
- Phase 5: "승인/관리 시스템" (워크플로우)

---

## 2️⃣ Phase 4 모듈 우선순위 (순서만)

### 모듈 1: 회계/정산 (Phase 4-1, 최우선)

#### 목적
- 사무국의 손을 가장 많이 묶는 일을 클릭 몇 번으로 끝내기
- 풀칠·스캔·폴더 관리 사라지게 만들기

#### 범위
- 대관료 수납/정산 보드
- 영수증 디지털화
- 보고서 자동 생성

#### 우선순위: **1순위 (Phase 4-1, 최우선)**

---

### 모듈 2: 알림 자동화 (Phase 4-2)

#### 목적
- 상태 변경 시 자동 알림 발송
- 전화/KakaoTalk 문의 최소화

#### 범위
- FacilitySection: 슬롯 상태 변경 시 알림
- TournamentSection: 대회 상태 변경 시 알림
- NoticeSection: 공지 등록 시 알림 (선택적)

#### 구현 내용
- Kakao Alimtalk API 연동 (또는 Stub)
- 상태 변경 이벤트 감지
- 알림 템플릿 정의
- 수신자 목록 관리

#### 우선순위: **1순위 (최우선)**

---

### 모듈 3: 기록/이력 (Phase 4-3, 3순위)

#### 목적
- 상태 변경 이력 기록
- 운영 기록 자동 생성
- 구청 보고용 데이터 생성

#### 범위
- FacilitySection: 슬롯 상태 변경 이력
- TournamentSection: 대회 상태 변경 이력
- NoticeSection: 공지 등록/수정 이력

#### 구현 내용
- Firestore에 변경 이력 저장
- 변경 일시/작성자 기록
- 이력 조회 기능 (Admin만)
- 월간/연간 리포트 생성 (선택적)

#### 우선순위: **2순위**

---


---

## 3️⃣ 모듈 간 의존성

### 의존성 관계
```
회계/정산 (Phase 4-1, 1순위)
  ↓
알림 자동화 (Phase 4-2, 2순위)
  ↓
기록/이력 (Phase 4-3, 3순위)
```

### 구현 순서
1. **회계/정산 (Phase 4-1)** → 먼저 구현 (사무국 시간 절감)
2. **알림 자동화 (Phase 4-2)** → 회계 이후 구현 (즉시 효과)
3. **기록/이력 (Phase 4-3)** → 알림 이후 구현 (데이터 축적)

---

## 4️⃣ 각 모듈의 구현 범위 (Phase 4 기준)

### 회계/정산 (Phase 4-1, 1순위)
- ✅ 대관료 수납/정산 보드
- ✅ 영수증 디지털화
- ✅ 보고서 자동 생성
- ❌ 온라인 결제 (Phase 5 이후)
- ❌ 자동 송금 (Phase 5 이후)

### 알림 자동화 (Phase 4-2, 2순위)
- ✅ 상태 변경 감지
- ✅ 알림 발송
- ❌ 알림 설정/관리 UI (Phase 5 이후)
- ❌ 알림 이력 조회 (Phase 5 이후)

### 기록/이력 (Phase 4-3, 3순위)
- ✅ 변경 이력 저장
- ✅ 변경 일시/작성자 기록
- ✅ Admin 조회 기능 (선택적)
- ❌ 리포트 UI (Phase 5 이후)
- ❌ 엑셀 내보내기 (Phase 5 이후)


---

## 5️⃣ Phase 4 구현 순서 (타임라인)

### Step 1: 회계/정산 (Phase 4-1, 2-3주)
- 대관료 수납/정산 보드
- 영수증 디지털화 (OCR/AI 분류)
- 보고서 자동 생성 (PDF/엑셀)

### Step 2: 알림 자동화 (Phase 4-2, 1-2주)
- FacilitySection 슬롯 상태 변경 알림
- Kakao Alimtalk API 연동 (또는 Stub)
- 알림 템플릿 정의

### Step 3: 기록/이력 (Phase 4-3, 1주)
- 변경 이력 저장 로직
- Firestore 스키마 정의
- Admin 조회 기능 (선택적)

- ❌ 승인 워크플로우 (Phase 5 이후)
- ❌ 엑셀 업로드 (Phase 5 이후)
- ❌ 회계/정산 UI (Phase 5 이후, 기록만 Phase 4)

## 7️⃣ 성공 기준

### Phase 4-1 완료 기준
1. ✅ 대관료 수납/정산 보드 구현 완료
   - 수납 상태 변경 가능
   - 협회 선지급 금액 표시

2. ✅ 영수증 디지털화 구현 완료
   - 사진 업로드 가능
   - 자동 분류 성공률 80% 이상

3. ✅ 보고서 자동 생성 구현 완료
   - PDF/엑셀 생성 가능
   - 수입/지출/잔액 정확히 계산

### Phase 4-2 완료 기준
1. ✅ 알림 자동화 구현 완료
   - 상태 변경 시 알림 정상 발송
   - 전화 문의 감소 확인

### Phase 4-3 완료 기준
1. ✅ 기록/이력 구현 완료
   - 변경 이력 정상 저장
