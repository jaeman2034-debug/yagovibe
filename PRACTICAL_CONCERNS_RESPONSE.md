# 실무자 걱정 포인트 대응 스크립트

## 🎯 사용 가이드

**협회 실무자(사무국, 총무)의 실제 걱정을 다루는 대응 멘트**

**원칙:** 공감 → 해결책 제시 → 안심

---

## 📋 걱정 포인트 3가지

### 1️⃣ 민원 대응

#### 걱정하는 점
- "시스템 도입 후 민원이 더 늘어나면 어쩌죠?"
- "팀이 시스템 사용을 거부하면 어떻게 하나요?"
- "전화로 신청하는 팀은 어떻게 처리하나요?"

#### 대응 멘트

**A. 민원 증가 걱정**
```
민원은 오히려 줄어듭니다.

지금 민원이 생기는 이유는 '왜 안 되냐'는 질문 때문인데,

시스템에서는 기준이 명확하니까

'왜 안 되냐'는 질문 자체가 줄어들어요.

D-7일 전부터 예약할 수 있다는 게 화면에 나와 있으니까

'왜 나는 안 되냐'고 물어볼 이유가 없어요.
```

**B. 시스템 거부 걱정**
```
팀 사용은 자율적입니다.

하지만 회원 팀의 경우,

우선 대관 혜택을 받으려면 시스템을 통해서 예약해야 합니다.

점진적으로 자연스럽게 전환될 거예요.

강제가 아니라 선택의 문제입니다.
```

**C. 전화 신청 처리**
```
전화 신청도 가능합니다.

협회 관리자가 시스템에 직접 입력해 주시면 됩니다.

온라인 신청과 동일한 승인 프로세스를 거치게 되고,

모든 기록이 시스템에 남아 관리가 편해집니다.
```

---

### 2️⃣ 책임 문제

#### 걱정하는 점
- "승인 잘못하면 누가 책임지나요?"
- "시스템 오류로 문제 생기면 어쩌죠?"
- "데이터 손실은 안 되죠?"

#### 대응 멘트

**A. 승인 책임**
```
승인은 협회가 하시는 겁니다.

시스템은 도구일 뿐이고,

최종 결정은 협회 관리자가 하시는 거예요.

다만 시스템의 장점은

모든 승인 기록이 남는다는 점입니다.

누가, 언제, 어떤 이유로 승인했는지

모두 기록되어 있어서 나중에 문제가 생겨도

추적이 가능합니다.
```

**B. 시스템 오류**
```
Firebase(Google Cloud) 기반의 안정적인 시스템을 사용합니다.

데이터는 자동으로 백업되고,

장애 시에도 데이터 손실 없이 복구 가능합니다.

기술 지원도 제공되니까

문제 생기면 바로 해결할 수 있습니다.
```

**C. 데이터 보안**
```
데이터 보안은 Google Cloud의 보안 인증을 받았습니다.

개인정보 보호법을 준수하며,

정기적인 보안 점검을 실시합니다.

협회 데이터는 협회만 볼 수 있고,

다른 협회와 섞이지 않습니다.
```

---

### 3️⃣ 통제권 문제

#### 걱정하는 점
- "협회가 모든 것을 통제할 수 있나요?"
- "시스템 업체가 데이터를 가져갈 수 있나요?"
- "정책 변경은 협회가 할 수 있나요?"

#### 대응 멘트

**A. 통제권**
```
협회가 모든 것을 통제합니다.

승인, 거절, 정책 변경, 모든 결정은 협회가 하시는 겁니다.

시스템은 그 결정을 실행하는 도구일 뿐이에요.

협회가 원하시면 언제든 시스템을 중단할 수 있습니다.

데이터도 협회 소유입니다.
```

**B. 데이터 소유권**
```
데이터는 협회 소유입니다.

시스템 업체는 기술 지원만 제공하고,

데이터에 접근하거나 사용할 수 없습니다.

협회가 원하시면 언제든 데이터를 내보낼 수 있고,

시스템을 중단할 수도 있습니다.

협회가 완전한 통제권을 갖고 있습니다.
```

**C. 정책 변경**
```
정책은 협회가 결정합니다.

기본 정책(D-7 vs D-3)은 표준을 따르지만,

협회 특별 규정이 있으시면

반영할 수 있습니다.

다만 표준 정책을 유지하는 게

운영 효율성과 공정성 측면에서 유리합니다.
```

## 💬 실제 대화 시나리오

### 시나리오 1: 민원 걱정

**실무자:** "시스템 도입 후 민원이 더 늘어나면 어쩌죠?"

**나:**
```
좋은 질문입니다. 오히려 민원은 줄어들 겁니다.

지금 민원이 생기는 이유는

'왜 나는 안 되냐', '왜 그 팀은 되는데 우리는 안 되냐'는

애매한 기준 때문이에요.

하지만 시스템에서는 기준이 명확합니다.

회원 팀은 D-7일 전부터, 비회원 팀은 D-3일 전부터

이게 화면에 그대로 나와 있어요.

그래서 '왜 안 되냐'고 물어볼 이유가 없습니다.

화면이 대신 설명하니까요.
```

---

## 🎯 공감 → 해결책 → 안심 패턴

### 1단계: 공감
```
"그런 걱정이 당연하시죠. 많은 협회에서 같은 걱정을 하시더라고요."
```

### 2단계: 해결책 제시
```
"하지만 시스템은 오히려 그런 걱정을 줄여드립니다."
```

### 3단계: 구체적 설명
```
"예를 들어..."
```

### 4단계: 안심
```
"그래서 걱정하실 필요가 없습니다. 저희가 다 처리해 드립니다."
```

## 💡 효과적인 답변 팁

### 1. "괜찮습니다"보다는 구체적 해결책
❌ "괜찮습니다, 문제 없어요"
⭕ "그런 걱정을 해소하기 위해 이런 방식으로 처리합니다..."

### 2. 추상적 설명보다는 구체적 예시
❌ "시스템이 안전합니다"
⭕ "Google Cloud의 보안 인증을 받았고, 데이터는 자동으로 백업됩니다"

### 3. 부정보다는 긍정
❌ "민원이 늘어나지 않습니다"
⭕ "오히려 민원이 줄어들 겁니다. 기준이 명확하니까요"
