# 🔒 AI 분석 섹션 최종 설계 잠금 문서 (FINAL LOCK)

**목적**: AI 분석 섹션의 최종 문구, UX, 배지 설계를 명확히 정의하고 동결  
**원칙**: 사용자 성공률 > AI 완성도, 장애 리스크 최소화  
**대상**: 개발자 / 기획자 / 모든 이해관계자

## 0️⃣ 최종 한 줄 결론

**AI는 똑똑해 보일수록 더 조심스럽게 숨겨야 한다.**

---

## 1️⃣ AI 버튼 문구 최종안 (고정)

### 1-1. 기본 상태 (접힘)

```
▸ AI로 상품 설명 추천받기 (선택)
```

**이 문구가 정답인 이유:**
- ✅ "AI" 과장 없음
- ✅ "선택" 명시 → 심리적 압박 0
- ✅ "추천" → 자동 적용 아님을 암시

### 1-2. 금지 문구

❌ "AI 분석 시작" (너무 진지함, 압박감)  
❌ "AI 필수" (필수로 만들면 안 됨)  
❌ "자동 생성" (자동 적용 오해 가능)

### 1-3. 열렸을 때 (설명 + 버튼)

```
AI가 이미지와 입력 정보를 바탕으로
상품 설명 작성을 도와드려요.

[ AI 분석 시작 ]
```

**버튼은 단일**
- "분석"이라는 단어 유지 (마법처럼 보이지 않게)
- 명확하고 직관적인 액션

---

## 2️⃣ AI 결과 적용 UX 미세 조정 최종안

### 2-1. 성공 시 구조 (자동 적용 ❌)

```
추천 상품 설명
────────────────
(미리보기 텍스트…)

[ 설명에 적용하기 ]
```

### 2-2. 핵심 규칙

#### ❌ 금지 사항
- 자동으로 input에 써버리지 않음
- 성공 즉시 필드 자동 채우기

#### ✅ 허용 사항
- "적용하기"를 눌러야 반영
- 기존 입력 내용은 덮어쓰기 전에 확인 가능
- 👉 통제권 100% 사용자

### 2-3. 실패 시 UX (아주 중요)

#### ❌ 금지 사항
- alert 사용
- 전체 에러 UI
- 붉은 경고 박스

#### ✅ 허용 사항
- toast 1줄만 표시

**실패 메시지**:
```
AI 분석에 실패했어요. 잠시 후 다시 시도해 주세요.
```

**그리고:**
- 페이지 유지
- 입력값 유지
- 아무 일도 없었던 것처럼 돌아감

---

## 3️⃣ "AI 베타" 배지 — 붙인다 (하지만 조용히)

### 3-1. 위치

```
AI 분석  [BETA]
```

AI 섹션 제목 옆

### 3-2. 스타일

- 회색 (#9ca3af 또는 유사)
- 작은 텍스트 (10-11px)
- 시선 끌지 않음 (강조 없음)

### 3-3. 붙이는 이유 (천재 모드 논리)

- **기대치 관리**: 완벽한 AI가 아님을 암시
- **실패 허용에 대한 면책**: 베타이므로 실패해도 자연스러움
- **"완성품" 착각 방지**: 실험적 기능임을 명시

👉 안 붙이면 AI 실패 시 UX 신뢰도가 더 크게 떨어진다.

---

## 4️⃣ AI 섹션 최종 잠금 규칙

### 🔒 규칙 1: 항상 접힌 상태로 시작
- 페이지 진입 시 AI 섹션은 기본적으로 접힘
- "▸ AI로 상품 설명 추천받기 (선택)" 문구로 시작

### 🔒 규칙 2: AI 실행은 버튼 클릭으로만
- 자동 실행 금지
- 명시적 "AI 분석 시작" 버튼 클릭 필요

### 🔒 규칙 3: 결과는 미리보기 → 적용
- 성공 시 자동 적용 금지
- 미리보기 표시 → "설명에 적용하기" 버튼 → 사용자 승인 후 적용

### 🔒 규칙 4: 실패는 toast + 무시 가능
- alert 사용 금지
- toast 1줄만 표시
- 실패해도 페이지/입력값 유지, 아무 일 없었던 것처럼

### 🔒 규칙 5: AI는 보조 도구
- 상품 등록의 전제 조건 ❌
- "있으면 좋은 버튼"이지 "있어야만 하는 단계" 아님

## 7️⃣ 이 설계의 효과 (천재 모드 논리)

### 7-1. 사용자 성공률 ↑
- AI가 등록 흐름을 방해하지 않음
- 실패해도 등록 진행 가능
- 심리적 압박 없음 ("선택" 명시)

### 7-2. 장애 리스크 ↓
- AI 실패가 등록 실패로 이어지지 않음
- 베타 배지로 기대치 관리
- 실패 시 무시 가능 (toast만)

### 7-3. 제품 신뢰도 ↑
- AI 실패가 제품 전체 신뢰도에 영향 없음
- 사용자 통제권 명확 (적용하기 버튼)
- 베타 배지로 솔직한 커뮤니케이션

## 8️⃣ 절대 변경 금지 사항

### ❌ 문구 변경 금지
- "선택" 단어 제거 금지
- "AI 분석 시작"을 다른 문구로 변경 금지
- "필수", "자동" 같은 단어 사용 금지

### ❌ UX 변경 금지
- 자동 실행 추가 금지
- 자동 적용 추가 금지
- 실패 시 차단 로직 추가 금지
- alert 사용 금지

### ❌ 배지 제거 금지
- [BETA] 배지 제거 금지
- 배지 스타일을 강조로 변경 금지

---

## 9️⃣ 변경 허용 범위 (미세 조정만)

### ✅ 허용되는 미세 조정
- toast 메시지 문구 미세 조정 (의미 유지)
- 로딩 애니메이션 개선
- 미리보기 텍스트 스타일링
- 버튼 색상/크기 미세 조정

### ❌ 허용되지 않는 변경
- 문구 의미 변경 (예: "선택" 제거)
- 자동 실행/적용 추가
- 실패 시 차단 로직 추가
- 베타 배지 제거 또는 강조
