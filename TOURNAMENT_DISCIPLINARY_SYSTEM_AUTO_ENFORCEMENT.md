# 「대회 징계 기준」을 시스템에 박제하는 설계

**규정의 지위를 한 단계 올린다**

**규정 문장 → 시스템 구조 → 현장 집행 멘트**
**이 3개를 완전히 일치시킨다.**

---

## 1️⃣ 규정의 지위를 한 단계 올린다 (핵심 전환)

### 지금 상태

- 징계 기준 = 문서 📄

### 천재 모드 전환

- **징계 기준 = 시스템 규칙 + 문서 증거**

즉,

- ❌ "규정에 이렇게 돼 있습니다"
- ⭕ **"시스템이 이 규정을 집행했습니다"**

**이게 분쟁 제로 상태다.**

---

## 2️⃣ 징계 기준 → 시스템 자동 트리거 매핑

아래는 1page 징계 기준을 그대로 시스템 이벤트로 바꾼 것이다.

---

### 🔹 (A) 부정선수 트리거

#### 입력

- 선수 명단 제출
- 선수 정보 입력 (이름 / 생년 / 조인KFA 체크)

#### 시스템 처리

```
조인KFA 미등록 체크 = true
→ 상태: ⚠️ 자격 위반 가능
```

#### 결과

- 관리자 화면: 경고 배지
- 대진표 확정 버튼: ❌ 비활성

---

### 📌 포인트

시스템이 "의심"만 표시
→ 판단과 책임은 클럽 회장에게 남김

---

### 🔹 (B) 회비·참가비 위반 트리거

#### 입력

- 참가 신청

#### 시스템 처리

```
회비 미납 OR 참가비 미결제
→ 참가 상태: ❌ 참가 불가
```

#### 결과

- 참가 버튼 자동 차단
- 명단 미포함
- 대진표 자동 제외

---

### 📌 여기서 중요한 말

**"시스템상 참가자가 아닙니다."**

---

### 🔹 (C) 징계 확정 트리거

#### 입력

- 협회 관리자 → **[징계 확정]** 클릭

#### 시스템 처리

해당 팀 / 선수 상태:

- 몰수패
- 출전정지
- 차기대회 제한

#### 결과

- 대진표 자동 수정
- 결과 고정
- 이의 제기 버튼 ❌ (기간 경과 시)

---

## 3️⃣ 사무국·임원이 쓰는 단 한 문장 매뉴얼

이게 진짜 천재 포인트다.
누가 와도 이 말만 하면 된다.

---

### 🎯 항의 시 공식 멘트 (복붙)

```
"해당 사항은
대회 징계 기준에 따라
시스템에서 자동 처리된 결과입니다.

규정과 시스템 외의 판단은 하지 않습니다."
```

- 설명 ❌
- 설득 ❌
- 감정 ❌

**👉 책임이 '사람'에서 '기준'으로 이동**

---

## 4️⃣ 규정·시스템·책자의 관계 최종 정리

```
[대회 규정]  ← 법적 기준
      ↓
[징계 기준 1page] ← 해석 기준
      ↓
[시스템 자동 집행] ← 실행 기준
```

이 3개가 일치하면:

- 규정 바꿔라 ❌
- 예외 봐달라 ❌
- 전임 때는 ❌

---

## 5️⃣ 천재 모드의 진짜 결과

이제 이 시스템은:

- 징계를 "결정"하지 않는다
- 징계를 **"집행"**한다

그리고 사람은:

- 판단자 ❌
- 설득자 ❌
- 감정 쓰레기통 ❌

**👉 기준 관리자만 된다.**

---

## 🏁 최종 선언

**규정 문장 → 시스템 구조 → 현장 집행 멘트**
**이 3개가 완전히 일치합니다.**

**이제 징계는 "결정"이 아니라 "집행"입니다.**
