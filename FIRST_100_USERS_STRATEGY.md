# ✅ 첫 100명 확보 전략 (Firebase·Phone Auth 서비스 기준)

> 목표는 "크게"가 아니라
> **✔ 진짜 유저 100명 + 데이터 + 감각**이다.

---

## 🎯 원칙 3가지 (이거 어기면 실패)

### 1. 광고부터 ❌

**왜?**
- 아직 전환율/메시지 모른다
- 광고비 낭비
- 가짜 유저 유입

**대신:**
- 무료 채널부터
- 직접 대화
- 피드백 수집

### 2. 한 채널만 판다

**왜?**
- 여기저기 흩어지면 0명
- 집중력 분산
- 결과 측정 불가

**대신:**
- 1개 채널 정복
- 완전히 끝내고 다음으로

### 3. 사람 말로 설명 가능해야 함

**왜?**
- "이거 뭐임?"에 10초 답변 가능해야 함
- 복잡하면 공유 안 됨
- 입소문 안 남

**대신:**
- 한 문장 설명 준비
- 친구에게 설명하는 톤

# 🔥 루트 1 — 지인·지인2 (가장 빠르고 정확)

## 방법

* DM / 카톡 / 메신저
* **링크 + 한 줄 설명**

## 예시 문구 (그대로 써도 됨)

```
요즘 [문제 X] 때문에 이거 만들고 있어요.
전화번호로 바로 쓰는 서비스인데
지금 써보고 피드백 주면 진짜 도움돼요 🙏

[링크]
```

### 커스터마이징 예시

**운동 서비스:**
```
요즘 운동 팀 찾기 어려워서 이거 만들고 있어요.
전화번호로 바로 가입하는 서비스인데
지금 써보고 피드백 주면 진짜 도움돼요 🙏
```

**학습 서비스:**
```
요즘 스터디 그룹 찾기 어려워서 이거 만들고 있어요.
전화번호로 바로 가입하는 서비스인데
지금 써보고 피드백 주면 진짜 도움돼요 🙏
```

## 실행 체크리스트

- [ ] 지인 리스트 작성 (최소 20명)
- [ ] 한 줄 설명 준비
- [ ] 초대 링크 생성
- [ ] DM/카톡 발송
- [ ] 피드백 받을 준비

## 🎯 목표

* **Day 1에 10~20명**
* 이 사람들 = **초기 QA + UX 디버거**

# 🔥 루트 2 — 커뮤니티 1곳만 정복

## 고르는 기준

* 문제를 **이미 겪고 있는 사람들**
* 글이 살아 있음 (댓글 있음)
* 규칙이 너무 엄격하지 않음

## 예시 플랫폼

* 디스코드 (운동/취미 서버)
* 슬랙 커뮤니티
* 네이버 카페
* 레딧 / 디스콰이엇
* 브런치 댓글
* 페이스북 그룹

## 올리는 글 공식

### ❌ 하지 말 것

```
"서비스 출시했습니다!"
"새로운 앱이 나왔어요!"
"회원가입 해주세요!"
```

### ✅ 올바른 방식

```
제목: [문제] 때문에 삽질하다가 직접 만들어봤는데...

안녕하세요!

요즘 [구체적인 문제] 때문에 고생하고 있었는데,
기존 해결책들이 [한계점]이라서
직접 간단하게 만들어봤어요.

아직 부족한 점이 많지만,
혹시 써보고 욕해주실 분 계실까요? 😅

[링크]

피드백 주시면 정말 감사하겠습니다!
```

## 실행 체크리스트

- [ ] 타겟 커뮤니티 1곳 선정
- [ ] 커뮤니티 규칙 확인
- [ ] 글 작성 (피드백 요청 톤)
- [ ] 게시
- [ ] 댓글/피드백 모니터링

## 🎯 목표

* 게시글 1개
* **10~30명 유입**

# 🔥 루트 3 — '문제 설명 글' 하나

## 플랫폼

* 브런치
* 노션 (Public)
* 미디엄
* 벨로그
* 티스토리

## 구조 (5단)

### 1. 내가 겪은 문제

```
요즘 운동 팀을 찾기 어려워서 고생하고 있었어요.
온라인으로 찾으면 신뢰가 안 가고,
오프라인으로 찾으면 시간이 너무 걸리고...
```

### 2. 기존 해결법의 한계

```
기존 앱들은 회원가입이 복잡하고,
이메일 인증하고 비밀번호 만들고...
전화번호만 있으면 바로 쓸 수 있으면 좋겠다고 생각했어요.
```

### 3. 그래서 만든 방법

```
그래서 전화번호로 바로 가입하는 서비스를 만들었어요.
SMS 인증만 하면 바로 시작할 수 있어요.
```

### 4. 아직 부족한 점

```
아직 기능이 부족하고,
UX도 개선할 점이 많아요.
하지만 핵심 기능은 동작합니다.
```

### 5. 써보고 말해달라 (링크)

```
혹시 비슷한 문제를 겪고 계신 분들,
써보시고 뭐가 불편한지 알려주시면
정말 감사하겠습니다!

[링크]
```

## 📌 CTA는 "가입하세요" ❌

→ **"써보고 뭐가 불편한지 알려달라"**

## 실행 체크리스트

- [ ] 글 작성 (5단 구조)

# 🧠 첫 100명에게 꼭 해야 할 질문 3개

## 1. "왜 써봤어요?"

**목적:**
- 마케팅 메시지 발견
- 타겟 고객 이해

**예시 답변:**
- "운동 팀 찾기 어려워서"
- "친구가 추천해서"
- "궁금해서"

**활용:**
- 랜딩 페이지 문구
- 광고 카피
- SEO 키워드

## 2. "어디서 막혔어요?"

**목적:**
- UX 문제 발견
- 개선 포인트 파악

**예시 답변:**
- "온보딩이 너무 길어요"
- "버튼이 안 보여요"
- "설명이 부족해요"

**활용:**
- 즉시 수정
- 다음 실험 주제

## 3. "이거 누구한테 추천할 것 같아요?"

**목적:**
- 타겟 고객 명확화
- 마케팅 채널 발견

**예시 답변:**
- "운동하는 친구들"
- "대학생들"
- "취미 모임 찾는 사람들"

**활용:**
- 타겟 설정
- 채널 선택

**왜?**
- 핵심 가치 희석
- 복잡도 증가

**대신:**
- 피드백 기반 최소 수정
- 핵심 기능 강화

- 표본 부족

- 피드백 우선
- 정성적 인사이트

**왜?**
- 개선 기회 상실

**대신:**
- 적극적 피드백 요청
