# 협회 미팅 질문 TOP 10 + 바로 써먹는 모범 답변

## 🎯 사용 가이드

**현장에서 그대로 읽어도 되는 답변**

**원칙:** 기술 방어 ❌ / 운영 안심 ⭕ / 책임 명확 ⭕

---

## ❓ Q1. "이거 쓰면 협회가 더 바빠지는 거 아닌가요?"

### A.

```
아닙니다.

지금은 '관리'를 하고 계시고,

이 시스템에서는 **'결정'만 하시면 됩니다.**
```

```
신청된 팀만 보고

승인 여부만 누르시면 됩니다.
```

**👉 핵심:**
- 관리 ↓
- 판단 ↓
- 책임 명확

---

## ❓ Q2. "회원 기준은 누가 정합니까?"

### A.

```
기준은 협회가 그대로 유지합니다.

시스템은 기준을 실행할 뿐입니다.
```

```
받을 팀, 안 받을 팀은

지금처럼 협회가 결정합니다.
```

**👉 핵심:**
- 권한은 협회에 그대로

---

## ❓ Q3. "비회원 팀도 운동장을 쓸 수 있나요?"

### A.

```
네, 쓸 수 있습니다.

다만 시점이 다를 뿐입니다.
```

```
회원 팀은 7일 전,

비회원 팀은 3일 전부터 예약합니다.
```

**👉 핵심:**
- 차별 ❌
- 질서 ⭕

---

## ❓ Q4. "그럼 민원이 더 생기지 않나요?"

### A.

```
오히려 줄어듭니다.
```

```
사람이 판단하면 민원이 생기고,

기준이 판단하면 민원이 줄어듭니다.
```

```
화면에 기준이 그대로 보입니다.
```

**👉 핵심:**
- 설명할 필요 없음

---

## ❓ Q5. "우리가 승인 안 하면 팀이 불만 갖지 않나요?"

### A.

```
불만은 '기준이 없을 때' 생깁니다.
```

```
이 시스템에서는

승인 여부와 상태가 명확히 기록됩니다.
```

```
개인 판단이 아니라

협회 결정으로 남습니다.
```

**👉 핵심:**
- 책임 분산이 아니라 책임 보호

---

## ❓ Q6. "우리 협회 방식이랑 안 맞으면요?"

### A.

```
그래서 시범 운영을 제안드립니다.
```

```
1~2개월 써보시고

안 맞으면 중단하셔도 됩니다.
```

**👉 핵심:**
- 도입 리스크 최소화

---

## ❓ Q7. "컴퓨터 잘 못하는 분들도 쓸 수 있나요?"

### A.

```
전화 받는 것보다 쉽습니다.
```

```
신청 목록 보고

승인 / 반려 버튼만 누르시면 됩니다.
```

**👉 핵심:**
- 실무자 안심용 답변

---

## ❓ Q8. "나중에 다른 구랑도 같이 써야 할 수도 있는데요?"

### A.

```
그걸 전제로 만든 구조입니다.
```

```
구가 늘어나도

방식은 바뀌지 않습니다.
```

**👉 핵심:**
- 노원 = 기준 모델

---

## ❓ Q9. "우리 협회 통제권이 줄어드는 건 아닌가요?"

### A.

```
오히려 더 명확해집니다.
```

```
누가, 언제, 어떤 팀을 승인했는지가

전부 기록으로 남습니다.
```

## ❓ Q10. "그래서 지금 당장 뭘 하면 되나요?"

### A.

```
딱 두 가지만 하시면 됩니다.
```

```
첫째, 시범 운영 동의

둘째, 관리자 계정 지정
```

```
나머지는 저희가 준비합니다.
```

**👉 핵심:**
- 결정 부담 최소화

---

## 🧠 이 Q&A 세트의 역할

### 1. 기술 질문 → 운영 질문으로 전환
- 기술적 설명 대신 운영 관점에서 답변
- "시스템이 어떻게"보다 "협회가 어떻게"에 집중

### 2. 불안 → 통제감
- 모든 답변이 "협회가 통제한다"는 메시지
- 시스템이 협회를 대체하는 게 아니라 협회를 돕는다는 인상

### 3. 거절 명분 제거
- 모든 우려에 대해 명확한 해결책 제시
- "그래서 시범 운영을 제안합니다"로 연결

### 4. 미팅이 '설명회'가 아니라 '결정의 자리'가 된다
- Q10에서 명확한 다음 단계 제시
- 결정 부담을 최소화 ("딱 두 가지만")

## 💡 답변 전략

### DO (해야 할 것)
- ✅ "협회가 결정한다"를 반복
- ✅ 구체적 예시 제공
- ✅ 시범 운영 제안으로 연결
- ✅ 간단하고 명확하게

### DON'T (하지 말 것)
- ❌ 기술 용어 사용
- ❌ 장황한 설명
- ❌ "나중에 됩니다" 같은 모호한 답변
- ❌ 다른 협회와 비교 비하
