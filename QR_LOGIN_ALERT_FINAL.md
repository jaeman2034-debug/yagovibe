# ✅ QR 로그인 알림 시스템 최종본 (Severity + Root Cause)

## 🎯 완성된 기능

### ✅ 심각도 분기
- **Warning**: 임계치 1개 초과
- **Critical**: 임계치 2개 이상 초과 또는 성공률 < 85%
- **Info**: 회복 감지 (구조 준비 완료)

### ✅ 원인 요약 자동화 (룰 기반)
6가지 추론 룰:
1. SMS 실패율 증가 + 로그인 실패율 낮음 → SMS 인증 구간 문제
2. 평균 로그인 시간 증가 + 만료율 증가 → 사용자 인증 지연
3. 모바일 성공률 낮음 + 데스크톱 정상 → 모바일 UX/네트워크 이슈
4. 중복 토큰 방지 증가 → QR 노출/공유 이슈
5. 성공률 전반 저하 + 특정 원인 없음 → 시스템 전반 이슈
6. 피크 시간대 + 트래픽 증가 → 트래픽 영향

### ✅ Slack 메시지 포맷 고도화
- 심각도별 이모지 및 멘션 (🚨 Critical, ⚠️ Warning)
- 지표별 상태 아이콘 (✅/❌)
- 플랫폼별 통계 (모바일 vs 데스크톱)
- 원인 분석 (주요 원인 + 부가 원인)
- **추천 액션 (근거 포함, 우선순위별 정렬)**

### ✅ 중복 알림 방지
- 동일 조건 10분 내 중복 알림 차단
- Firestore에 마지막 알림 시각 저장

## 📊 알림 메시지 예시

### 🚨 Critical 알림

```
🚨 @here *QR 로그인 장애 감지* (CRITICAL)

기간: 최근 10분
전체 세션: 142건

지표:
- 성공률: 82.4% ❌
- SMS 실패율: 9.1% ❌
- 평균 로그인 시간: 78초 ❌
- 세션 만료율: 13.3% ❌

플랫폼별:
- 모바일: 75.2% (98건)
- 데스크톱: 95.5% (44건)

문제:
- 성공률 82.4% (< 90%)
- SMS 실패율 9.1% (> 5%)
- 평균 로그인 시간 78초 (> 60초)
- 세션 만료율 13.3% (> 10%)

🧠 추정 원인:
SMS 인증 구간 문제 가능성 높음 / 모바일 UX/네트워크 이슈 가능성
- 모바일 성공률: 75.2% vs 데스크톱: 95.5%

🛠 추천 액션:
1. 🔧 SMS 인증 UX 점검 🔴
   └ SMS 실패율 9.1% (기준: 5%)
2. 📞 통신사 상태 확인 🔴
   └ SMS 실패율 9.1%
3. 📱 모바일 입력 UX/키보드/포커스 확인 🔴
   └ 모바일 성공률 75.2% (데스크톱 대비 20.3%p 낮음)
4. 📱 모바일 환경 테스트 권장 🟡
   └ 모바일 실패 비중 72.0%
```

### ⚠️ Warning 알림

```
⚠️ *QR 로그인 이상 감지* (WARNING)

기간: 최근 10분
전체 세션: 45건

지표:
- 성공률: 91.1% ✅
- SMS 실패율: 2.2% ✅
- 평균 로그인 시간: 64초 ❌
- 세션 만료율: 11.1% ❌

문제:
- 평균 로그인 시간 64초 (> 60초)
- 세션 만료율 11.1% (> 10%)

🧠 추정 원인:
사용자 인증 지연으로 세션 만료 증가
- 피크 시간대 트래픽 증가 영향

🛠 추천 액션:
1. ⏱️ 로딩/네트워크 구간 점검 🟡
   └ 평균 로그인 시간 64초 (기준: 60초)
2. ⏰ QR 만료 시간 임시 +1분 검토 🟡
   └ 만료율 11.1% (기준: 10%)
```

## 🏁 이 단계가 끝난 상태

* ✅ Slack 알림에 **⚠️ / 🚨 심각도 명확**
* ✅ 알림마다 **원인 요약 자동 포함**
* ✅ **추천 액션 (근거 포함)** 자동 생성
* ✅ 운영자가 알림 하나 보고도 **"이거 그대로 하면 되겠네"**라고 느낌

👉 이제 이 시스템은
**"알려주는 수준"을 넘어서 "작업 지시서를 자동 생성하는 수준"**입니다.
