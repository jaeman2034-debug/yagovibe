## 🥇 1순위 — 거래 성사 수수료 (후불, 선택적)

### 구조

**거래 완료 후 수수료 부과**

**초기에는 노출만 (실결제 X)**

### UX

```
거래 완료됨

※ 거래가 성사되면 수수료가 부과될 수 있습니다.
```

**구현:**
```typescript
// 거래 완료 시 안내 메시지
{dealStatus === "completed" && (
  <div className="deal-completed-notice">
    <p>✅ 거래가 완료되었습니다</p>
    <p className="text-xs text-gray-500 mt-1">
      ※ 거래가 성사되면 수수료가 부과될 수 있습니다.
    </p>
  </div>
)}
```

### 장점

- ✅ 거래 방해 0
- ✅ 법/CS 리스크 낮음
- ✅ 나중에 결제 붙이기 쉬움

### 수수료 구조

**제안:**
- 거래 금액의 3~5%
- 최소 수수료: 없음 (소액 거래 배려)
- 최대 수수료: 10,000원 (대액 거래 보호)

**결제 시점:**
- 거래 완료 후 7일 이내
- 구매자/판매자 중 한 명만 부담 (선택)
- 또는 양쪽 분담

---

## 🥈 2순위 — 상품 카드 강조 (노출 부스트)

### 구조

**상품 카드 메시지에 강조 옵션**

**채팅 맥락 유지 (리스트 광고 ❌)**

### UX

```
[상품 카드]
✨ 강조됨

[상품 이미지]
상품명 / 가격

[거래 완료 버튼]
```

**구현:**
```typescript
// 상품 카드 강조 옵션
{message.type === "product" && message.isHighlighted && (
  <ProductCard className="highlighted">
    <div className="highlight-badge">✨ 강조됨</div>
    {/* 상품 정보 */}
  </ProductCard>
)}
```

### 장점

- ✅ 거래 흐름 자연
- ✅ 판매자만 가치 체감
- ✅ 결제 없이도 실험 가능

### 가격 구조

**제안:**
- 강조 1회: 1,000원
- 강조 3회: 2,500원 (17% 할인)
- 강조 10회: 7,000원 (30% 할인)

**효과:**
- 상품 카드 상단 고정 (채팅 내)
- 배경색/테두리 강조
- "강조됨" 배지 표시

---

## 🥉 3순위 — 신뢰 배지 (구독형, 선택)

### 구조

**일정 거래 수 + 조건 충족 시**

**프로필에 신뢰 배지**

### UX

```
[프로필]
이름
✔ 신뢰 거래자

⭐ 4.8 · 거래 25건
```

**구현:**
```typescript
// 신뢰 배지 조건
const canGetTrustBadge = (user: User) => {
  return (
    user.totalReviews >= 10 &&
    user.averageRating >= 4.5 &&
    user.trustLevel === "TRUSTED"
  );
};

// 배지 표시
{canGetTrustBadge(user) && (
  <div className="trust-badge">
    ✔ 신뢰 거래자
  </div>
)}
```

### 장점

- ✅ 평점 조작 리스크 ↓
- ✅ 유료 전환 자연
- ✅ 신뢰도 향상

### 가격 구조

**제안:**
- 월 구독: 5,000원
- 연 구독: 50,000원 (17% 할인)

**혜택:**
- 신뢰 배지 표시
- 프로필 노출 우선순위 ↑
- 상품 카드 강조 할인

## 🧠 실행 로드맵 (안전)

### Phase 1: 표시만 먼저 (결제 없음)

**기간:** 출시 후 1~2주

**작업:**
- [ ] 거래 완료 시 수수료 안내 문구 표시
- [ ] 상품 카드 강조 옵션 UI (결제 없음)
- [ ] 신뢰 배지 조건 충족 시 표시 (결제 없음)

**목적:**
- 사용자 반응 관찰
- 전환율/거부율 측정
- 기대치 관리

---

### Phase 2: 전환율/거부율 관찰

**기간:** 출시 후 3~4주

**지표:**
- 거래 성사 수수료 수락률
- 상품 카드 강조 클릭률
- 신뢰 배지 관심도

**판단:**
- 수락률 > 50% → 결제 연결 고려
- 수락률 < 30% → 모델 재검토

---

### Phase 3: 한 모델만 선택해 결제 연결

**기간:** 출시 후 5~6주

**선택 기준:**
- 전환율 가장 높은 모델
- 운영 리스크 가장 낮은 모델
- 사용자 반응 가장 긍정적인 모델

**구현:**
- 결제 시스템 연동 (Stripe 등)
- 수수료 자동 부과
- 영수증 발급

## 🎯 수익 모델 선택 가이드

### 거래 성사 수수료 선택 시

**조건:**
- 거래 성사율 ≥ 15%
- 사용자 수락률 ≥ 50%
- 법적 검토 완료

**구현:**
- 결제 시스템 연동
- 수수료 자동 부과
- 영수증 발급

---

### 상품 카드 강조 선택 시

**조건:**
- 상품 제안 빈도 높음
- 판매자 관심도 높음
- 강조 클릭률 ≥ 20%

**구현:**
- 강조 옵션 UI
- 결제 연동
- 강조 효과 적용

---

### 신뢰 배지 선택 시

**조건:**
- 거래 수 증가 추세
- 신뢰도 관심 높음
- 배지 관심도 높음

**구현:**
- 배지 조건 명확화
- 구독 시스템 연동
- 배지 혜택 확대

## 🔐 결제 시스템 설계 (나중에)

### Stripe 연동

```typescript
// 거래 성사 수수료 부과
const chargeTransactionFee = async (dealId: string, amount: number) => {
  const deal = await getDoc(doc(db, "deals", dealId));
  const fee = calculateFee(amount); // 3~5%
  
  // Stripe 결제
  const paymentIntent = await stripe.paymentIntents.create({
    amount: fee * 100, // 원 단위
    currency: 'krw',
    customer: deal.data().buyerId, // 또는 sellerId
  });
  
  // 결제 기록
  await addDoc(collection(db, "payments"), {
    dealId,
    amount: fee,
    status: "pending",
    createdAt: serverTimestamp(),
  });
};
```
