# 📱 모바일 캐시 완전 초기화 가이드

## 🎯 목적

구글 로그인 튕김 현상을 최종적으로 해결하기 위해 모바일 브라우저의 모든 캐시, 쿠키, 사이트 데이터를 완전히 초기화합니다.

## ✅ 이미 완료된 작업

1. ✅ Service Worker 강제 제거 스크립트 추가 및 배포 완료
2. ✅ 코드 재빌드 및 재배포 완료
3. ✅ 최신 코드가 Firebase Hosting에 배포됨

## 📱 모바일 캐시 초기화 방법

### 안드로이드 (Chrome)

1. **설정 앱 열기**
   - 기기 설정 → 앱 → Chrome

2. **저장공간 메뉴 접근**
   - Chrome 앱 정보 화면에서 "저장공간" 또는 "저장 공간 및 캐시" 선택

3. **데이터 삭제**
   - **[데이터 삭제]** 또는 **[모든 데이터 지우기]** 버튼 클릭
   - 확인 대화상자에서 **[삭제]** 선택

4. **기기 재부팅**
   - 전원 버튼 길게 누르기 → 재시작

5. **시크릿 모드로 테스트**
   - Chrome 앱 열기
   - 메뉴 (⋮) → **새 시크릿 탭** 선택
   - `https://www.yagovibe.com/login` 접속

### iOS (Safari)

1. **설정 앱 열기**
   - 설정 → Safari

2. **웹사이트 데이터 지우기**
   - **"웹사이트 데이터 지우기"** 선택
   - 확인 대화상자에서 **[모두 지우기]** 선택

3. **기기 재부팅**
   - 전원 버튼 + 볼륨 다운 버튼 동시에 길게 누르기 → 슬라이드하여 끄기
   - 전원 버튼으로 다시 켜기

4. **시크릿 모드로 테스트**
   - Safari 앱 열기
   - 탭 아이콘 → **개인 보호 모드** 선택
   - `https://www.yagovibe.com/login` 접속

### 삼성 인터넷 (안드로이드)

1. **설정 앱 열기**
   - 기기 설정 → 앱 → 삼성 인터넷

2. **저장공간 메뉴 접근**
   - 삼성 인터넷 앱 정보 화면에서 "저장공간" 선택

3. **데이터 삭제**
   - **[데이터 삭제]** 또는 **[모든 데이터 지우기]** 버튼 클릭
   - 확인 대화상자에서 **[삭제]** 선택

4. **기기 재부팅**

5. **시크릿 모드로 테스트**
   - 삼성 인터넷 앱 열기
   - 메뉴 (☰) → **시크릿 모드** 선택
   - `https://www.yagovibe.com/login` 접속

## 🔍 확인 방법

### 브라우저 콘솔에서 확인할 로그

모바일 브라우저에서 개발자 도구를 열고 콘솔을 확인하세요:

```
🔍 [Service Worker 제거] 등록된 Service Worker 개수: X
✅ [Service Worker 제거] 이전 Service Worker 등록 해제 성공: [scope]
🎯 [Service Worker 제거] 등록 해제 완료
🗑️ [Service Worker 제거] 캐시 삭제 시작...
✅ [Service Worker 제거] 캐시 삭제 성공: [cache name]
🎯 [Service Worker 제거] 캐시 삭제 완료
✅ [Service Worker 제거] Service Worker 제거 후 캐시 새로고침 완료.
```

### 모바일 개발자 도구 열기

**안드로이드 Chrome**:
1. PC에서 Chrome 열기
2. 주소창에 `chrome://inspect` 입력
3. USB로 모바일 기기 연결
4. "원격 디바이스"에서 기기 선택
5. "inspect" 클릭

**iOS Safari**:
1. Mac에서 Safari 열기
2. Safari → 환경설정 → 고급 → "메뉴 막대에서 개발자용 메뉴 보기" 체크
3. USB로 iPhone 연결
4. Safari → 개발자 → [기기 이름] → [웹페이지] 선택

## ⚠️ 주의사항

- **데이터 삭제 시**: 저장된 비밀번호, 자동완성 데이터, 쿠키 등이 모두 삭제됩니다.
- **재부팅**: 캐시가 완전히 초기화되도록 반드시 재부팅하세요.
- **시크릿 모드**: 최신 코드가 로드되었는지 확인하기 위해 시크릿 모드로 테스트하세요.

## 🎯 예상 결과

캐시 초기화 후:

1. ✅ 이전 Service Worker가 자동으로 등록 해제됨
2. ✅ 모든 캐시가 삭제됨
3. ✅ 최신 코드가 로드됨
4. ✅ `detectInAppBrowser is not defined` 오류 해결
5. ✅ `Minified React error #300` 오류 해결
6. ✅ 구글 로그인 정상 작동 (튕김 현상 해결)

## 📞 문제가 지속되는 경우

만약 위 단계를 모두 거쳤는데도 문제가 지속된다면:

1. **다른 브라우저로 테스트**
   - Chrome → Firefox 또는 Edge로 변경
   - Safari → Chrome으로 변경

2. **네트워크 확인**
   - Wi-Fi와 모바일 데이터 모두 테스트
   - VPN 사용 중이면 비활성화

3. **Firebase Hosting 캐시 확인**
   - Firebase Console → Hosting → 캐시 무효화 실행
