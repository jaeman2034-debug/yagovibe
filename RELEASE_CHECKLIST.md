# ✅ 출시 체크리스트 (ONE-PAGER)

> **프린트해서 벽에 붙이는 버전** | 버전 1.0.0

---

## A. 점 3개 메뉴 UX

- [ ] 클릭/연타 안정 (5회 연타해도 상태 꼬임 없음)
- [ ] 외부 클릭/ESC 닫힘 (정상 작동)
- [ ] 스크롤 잠금/복귀 (메뉴 열릴 때 잠금, 닫힐 때 복귀)
- [ ] z-index 충돌 없음 (모달/메뉴 겹침 없음)
- [ ] 모바일 터치 44px+ (최소 터치 영역 확보)

---

## B. 신고 / 차단

- [ ] 신고 → 확인 → 토스트 ("필요한 조치를 검토 중입니다.")
- [ ] 차단 → 이유 선택(선택사항) → 채팅 종료
- [ ] 중복 요청 가드 (`isReporting`, `isBlocking` 플래그)
- [ ] 실패 시 명확한 안내 ("요청에 실패했습니다. 잠시 후 다시 시도해주세요.")
- [ ] 차단 후 재진입 불가 (서버+클라 가드 모두 작동)

---

## C. 채팅 상단 UX

- [ ] 긴 이름 말줄임 (`truncate`, `max-width: 180px`)
- [ ] 상태 텍스트 자연 ("온라인" / "마지막 접속 5분 전")
- [ ] 스크롤 쉐도우 (스크롤 시 헤더 그림자 표시)
- [ ] 버튼 히트박스 충분 (뒤로가기, 점 3개 모두 44px+)

---

## D. UX Polish

- [ ] 즉각 반응 (<50ms, 버튼 클릭 즉시 피드백)
- [ ] 로딩 300ms 규칙 (<300ms는 스피너 안 보임)
- [ ] 토스트 톤 중립/과거형 ("차단되었습니다.", "필요한 조치를 검토 중입니다.")
- [ ] 메뉴 닫힘 딜레이 (≈80ms, 부드러운 닫힘 애니메이션)

---

## E. QA & 안정성

- [ ] 연타/경계값 OK (메뉴 연타, 신고 연타, 차단 연타 모두 안정)
- [ ] 네트워크 실패 처리 (오프라인 토스트, 재시도 가능)
- [ ] 에러 바운더리 (`<ErrorBoundary>` 적용)
- [ ] 모바일 실기기 테스트 (iPhone SE, Android 최소 1대)

---

## F. 운영 대비

- [ ] 핵심 이벤트 로그 (`track("report_clicked")`, `track("block_confirmed")`)
- [ ] 기능 플래그 (차단/신고 플로우 on/off 가능)
- [ ] 롤백 플랜 (이전 빌드 보관, 5분 이내 롤백 가능)
- [ ] 사고 등급 (P0/P1/P2/P3 분류 및 대응 프로세스)

## 🎯 출시 판정 질문 (YES면 배포)

### 질문 1: "안 눌린 느낌" 0인가?
- [ ] 모든 버튼이 즉각 반응 (<50ms)
- [ ] 터치 미스 없음
- [ ] 로딩 상태 명확

### 질문 2: 행동 결과가 즉시 보이는가?
- [ ] 신고 후 토스트 표시
- [ ] 차단 후 채팅 종료
- [ ] 시스템 메시지 표시

### 질문 3: 실수해도 복구 가능한가?
- [ ] 에러 바운더리 작동
- [ ] 네트워크 실패 시 재시도 가능
- [ ] 명확한 에러 메시지

### 질문 4: 운영자가 상황을 재현할 수 있는가?
- [ ] 로그 추적 가능
- [ ] 에러 컨텍스트 포함
- [ ] 사용자 행동 재현 가능
