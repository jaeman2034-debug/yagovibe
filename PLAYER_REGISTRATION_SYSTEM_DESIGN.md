# 선수 명단 제출 시스템 설계 (KFA 검증 레이어)

**검증 레이어 완성 - 현실적·깔끔한 방식**

---

## 시스템 설계 (권장 방식)

### ❌ 하지 말 것

- 조인 KFA API 직접 연동 (복잡, 정책 리스크)
- 실시간 검증 시도

---

### ✅ 권장 방식 (표준 운영)

#### 선수 등록 시 필수 입력

- 성명
- 생년월일
- 조인 KFA 등록 여부 체크
- (선택) KFA 등록번호

---

### 검증 방식

- ✔ 선수 본인 책임 확인
- ✔ 사무국은 사후 검증 권한만 가짐
- ✔ 문제 발생 시 조인 KFA 기준으로 판단

**👉 이게 행정·법적 리스크가 가장 낮습니다.**

---

## 화면/UI 처리 방식

### 선수 명단 제출 화면에 들어갈 문구 (고정)

```
선수 등록 안내

본 대회에 참가하는 선수는
대한축구협회 '조인 KFA' 시스템에 등록된 회원만 가능합니다.
미등록 선수 또는 허위 등록 시
해당 선수는 출전이 제한될 수 있습니다.
```

이 문구 하나로:

- 사무국 보호 ✔
- 이의 제기 차단 ✔
- 책임 전가 ❌

---

## 대회 운영 구조에서의 위치

```
공지
 → 대회
   → 참가 신청 (회비·참가비)
     → 선수 명단 제출
       → ✔ 조인 KFA 등록 기준
         → 대진표 확정
```

**👉 이게 대회 운영의 최종 완성 구조입니다.**

---

## 효과

이제 사무국은 이렇게 말할 수 있습니다.

**"저희가 판단하지 않습니다.**
**조인 KFA 기준입니다."**

이 말이 나오는 순간:

- 감정 싸움 ❌
- 개인 청탁 ❌
- 책임 전가 ❌
