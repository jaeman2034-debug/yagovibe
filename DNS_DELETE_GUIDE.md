# 🗑️ DNS 레코드 삭제 가이드

## 📊 현재 DNS 설정 분석

### ✅ **유지해야 할 레코드:**

1. **CNAME 레코드** (www) - 유지
   - 호스트: `www`
   - 값: `yago-vibe-spt.web.app.`
   - 이유: 이미 연결되어 있음

2. **TXT 레코드** - 유지
   - 호스트: `@`
   - 값: `"hosting-site-yago-vibe-spt"`
   - 이유: Firebase 사이트 인증용

### ⚠️ **삭제해야 할 레코드:**

Firebase Hosting은 보통 **2개의 A 레코드**만 필요합니다.

현재 4개의 A 레코드:
- A 레코드 1: `@` → `199.36.158.100`
- A 레코드 2: `@` → `199.36.158.101`
- A 레코드 3: `@` → `199.36.158.102`
- A 레코드 4: `@` → `199.36.158.103`

---

## 🔍 Firebase 콘솔 확인 필요

**먼저 Firebase 콘솔에서 확인하세요:**
1. Firebase 콘솔 → Hosting → 커스텀 도메인
2. `yagovibe.com` 클릭
3. "설정 필요" 섹션에서 **정확히 어떤 IP 주소 2개를 요청하는지** 확인

---

## 💡 일반적인 삭제 방법

### **방법 1: 가장 안전한 방법 (권장)**

1. **Firebase 콘솔에서 요청하는 IP 주소 2개 확인**
2. **요청하지 않은 IP 주소의 A 레코드만 삭제**

예를 들어:
- Firebase가 `199.36.158.100`과 `199.36.158.101`을 요청한다면
- `199.36.158.102`와 `199.36.158.103`의 A 레코드 삭제

### **방법 2: 일시적으로 모든 A 레코드 삭제 후 재설정**

1. **모든 A 레코드 4개 삭제**
2. **Firebase 콘솔에서 요청한 IP 주소 2개로 새로 추가**

---

## 🎯 추천 삭제 순서

Firebase 콘솔 확인 없이 진행한다면:

### **1단계: Firebase 콘솔 확인**
- Firebase 콘솔에서 요청하는 정확한 IP 주소 2개 확인

### **2단계: 불일치하는 A 레코드 삭제**
- Firebase가 요청하지 않은 IP 주소의 A 레코드 삭제

### **3단계: 최종 확인**
- A 레코드는 정확히 2개만 남기기
- CNAME (`www`)와 TXT 레코드는 유지

---

## ⚠️ 주의사항

1. **CNAME 레코드 (`www`)는 절대 삭제하지 마세요!**
   - 이미 연결되어 있습니다

2. **TXT 레코드는 절대 삭제하지 마세요!**
   - Firebase 사이트 인증용입니다

3. **A 레코드 삭제 전 Firebase 콘솔 확인 필수!**
   - 잘못된 IP를 삭제하면 연결이 더 안 될 수 있습니다

---

## ✅ 최종 체크리스트

삭제 후 최종 상태:
- [ ] CNAME 레코드 (`www`) - 유지
- [ ] TXT 레코드 (`@`) - 유지
- [ ] A 레코드 (`@`) - 정확히 2개만 유지 (Firebase가 요청한 IP)
