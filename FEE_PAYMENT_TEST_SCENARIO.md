# 🧪 실전 테스트 시나리오

**"홍길동 20,000원 납부" — 처음부터 끝까지**

## 🎯 테스트 목표

관리자인 내가  
👉 어디 들어가서  
👉 무엇을 누르면  
👉 홍길동이 '완납'이 되는지  
눈으로 확인한다.

---

## STEP 0️⃣ 사전 조건 (이거 맞아야 테스트 성립)

✅ 나는 관리자 (OWNER 권한)  
✅ 팀 plan = PRO  
✅ 이번 달 = 2025년 7월  
✅ 홍길동 = 미납 상태  
✅ 회비 금액 = 20,000원

---

## STEP 1️⃣ 관리자 동선 (사용자 기준)

### 1. 메뉴 이동

```
우리 팀 관리
 → 회비 / 회계
```

### 2. 월 선택

```
[2025년 7월]
```

### 3. 회원 목록 확인

**화면에 이렇게 보인다** 👇

```
┌─────────────────────────────────────┐
│ 홍길동                              │
│ 월회비: 20,000원                    │
│ 미납 1개월                          │
│ [납부 완료] ← 이 버튼이 보여야 성공 │
└─────────────────────────────────────┘
```

**체크리스트**:
- [ ] 회원 목록이 표시됨
- [ ] 홍길동이 미납 상태로 표시됨
- [ ] "납부 완료" 버튼이 활성화되어 있음

👉 **여기서 멈추면 실패**  
👉 **이 버튼이 보여야 성공**

---

## STEP 2️⃣ 핵심 액션 (딱 1번 클릭)

### 4. [납부 완료] 버튼 클릭

**확인창이 뜬다** 👇

```
┌─────────────────────────────────────┐
│ 회비 납부 확인                      │
│                                     │
│ 회원: 홍길동                        │
│ 월: 2025년 7월                      │
│ 금액: 20,000원                      │
│                                     │
│ 이 회원의 회비를                    │
│ '납부 완료'로 처리할까요?           │
│                                     │
│ [취소]   [확인]                    │
└─────────────────────────────────────┘
```

**체크리스트**:
- [ ] 확인 다이얼로그가 표시됨
- [ ] 회원명, 월, 금액이 정확히 표시됨
- [ ] "확인" 버튼 클릭 가능

### 5. [확인] 클릭

**체크리스트**:
- [ ] 확인 버튼 클릭
- [ ] 다이얼로그가 닫힘
- [ ] 처리 중 표시 (선택사항)

---

## STEP 3️⃣ 시스템이 자동으로 하는 일 (관리자는 모름)

**서버에서 내부적으로** 👇

### fee 기록 생성/수정

```typescript
// teams/{teamId}/fees/{memberId}_{month}
{
  teamId: "team1",
  memberId: "hong",
  month: "2025-07",
  amount: 20000,
  paid: true,
  paidAt: serverTimestamp(),
  processedBy: "owner-uid",
}
```

### 성공 응답 반환

```json
{
  "success": true
}
```

**체크리스트**:
- [ ] 서버에서 fee 기록 저장됨
- [ ] unpaidMonths 재계산됨
- [ ] 상태 자동 전환됨

---

## STEP 4️⃣ 즉시 화면 변화 (눈으로 확인)

### 회비 / 회계 화면

**Before**:
```
홍길동 | 미납 | 20,000원 | [납부 완료]
```

**After**:
```
홍길동 | 완납 | ✔ 완료
```

**체크리스트**:
- [ ] 버튼이 사라짐
- [ ] "✔ 완료" 표시로 변경됨
- [ ] 배경색이 초록색으로 변경됨
- [ ] 상태가 즉시 바뀜

---

## STEP 5️⃣ 다른 화면에서도 자동 반영 (중요)

### 회원 관리 페이지

**Before**:
```
홍길동 | 재원 | 미납 1개월
```

**After**:
```
홍길동 | 재원 | 미납 0
```

### 팀 관리 보드(요약)

**Before**:
```
이번 달 미납 인원: 1명
```

**After**:
```
이번 달 미납 인원: 0명
```

**체크리스트**:
- [ ] 회원 관리 화면에 미납 0 표시
- [ ] 관리보드 요약에 미납 인원 감소
- [ ] 어디도 따로 수정 안 함
- [ ] 전부 자동

---

## STEP 6️⃣ 실패 케이스 테스트 (꼭 해봐야 함)

### ❌ FREE 팀일 때

**같은 화면**

**버튼은 🔒 비활성**

```
┌─────────────────────────────────────┐
│ 홍길동                              │
│ 월회비: 20,000원                    │
│ 미납 1개월                          │
│ [🔒 납부 완료] ← 비활성화           │
└─────────────────────────────────────┘
```

**클릭 시**:

```
┌─────────────────────────────────────┐
│ Pro 플랜 필요                       │
│                                     │
│ 회비 납부 처리는 Pro 플랜에서만     │
│ 사용할 수 있습니다.                 │
│                                     │
│ [닫기]  [업그레이드]                │
└─────────────────────────────────────┘
```

**체크리스트**:
- [ ] 버튼이 "🔒 납부 완료"로 표시됨
- [ ] 버튼이 비활성화됨 (회색)
- [ ] 클릭 시 업그레이드 모달 표시
- [ ] "Pro 플랜 필요" 메시지 표시
- [ ] 이게 떠야 정상
