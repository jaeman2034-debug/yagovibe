# Tournament 운영·재정 결합 구현 체크리스트

**목표: 대회 = "참가·결제·대진·정산의 기준 엔진"**

---

## ✅ 필수 구현 항목

### 1️⃣ 참가비 통합 처리

#### 설계 원칙
- [ ] ❌ 참가 신청 따로
- [ ] ❌ 입금 확인 따로
- [ ] ❌ 엑셀 대조
- [ ] ⭕ 참가 신청 = 참가비 처리 전제조건

#### 참가 흐름
- [ ] 회원 로그인
- [ ] **[대회 참가 신청]**
- [ ] 참가비 표시
- [ ] 결제 / 납부 확인
- [ ] 상태 변경:
  - [ ] 🟡 신청 (미결제)
  - [ ] 🟢 **참가 확정 (결제 완료)**

**목표 문장: "참가자는 시스템 기준입니다."**

---

### 2️⃣ 미참가자 자동 배제

#### 자동 배제 기준
아래 중 하나라도 해당되면 자동 미참가:

- [ ] 참가 신청 ❌
- [ ] 참가비 미결제
- [ ] 승인 대기 상태

#### 결과
- [ ] 대진표 생성 대상 = 참가 확정자만
- [ ] 명단 혼선 ❌
- [ ] "나는 신청했는데요?" 논쟁 ❌

---

### 3️⃣ 대진표와 재정의 결합

#### 구조
- [ ] 대진표 = 참가 확정자 기준 자동 생성 / 업로드

#### 대진표 상태
- [ ] 🟡 준비 중
- [ ] 🟢 **확정 (공식)**

#### 공식 문장
- [ ] **"대진표는 참가비 납부 완료자 기준으로 확정됩니다."**

이 문장으로 차단:
- [ ] 예외 요청 ❌
- [ ] 추가 참가 요구 ❌
- [ ] 뒷말 ❌

---

### 4️⃣ 정산 자동 연결

이미 Phase 4에서 만든 구조 활용:

- [ ] 참가비 수입 → 대회 수입 자동 반영
- [ ] 참가자 수 → 정산 자동 계산
- [ ] 보고서 출력 → 클릭 1번

**사무국 작업 제거:**
- [ ] 돈 계산 ❌
- [ ] 명단 정리 ❌
- [ ] 엑셀 ❌

---

### 5️⃣ 권한 기준 재확정

#### 비회원
- [ ] 대회 일정
- [ ] 대진표 열람

#### 회원
- [ ] 참가 신청
- [ ] 결제 상태 확인
- [ ] 본인 참가 여부

#### 협회 관계자
- [ ] 참가 승인
- [ ] 참가비 확인
- [ ] 대진표 확정 버튼

---

## 📋 구현 순서

1. [ ] 참가 신청 상태에 결제 상태 추가
2. [ ] 참가 확정 조건 = 결제 완료
3. [ ] 대진표 생성 대상 필터링
4. [ ] 대회 정산 자동 연동
5. [ ] 공식 기준 문구 삽입
