# 🛡️ MVP 개발 중 질문 필터링 체크리스트

> 개발 중 나오는 "이거 넣어야 하지 않나?" 질문들을 걸러내는 기준

---

## 🎯 핵심 원칙 (이거 하나만 기억)

> **"말로 주변 장소 찾기"에
> 안 들어가는 건 전부 나중이다.**

---

# ❓ 자주 나오는 질문 + 답변

---

## Q1. "이건 지금 넣어야 하지 않나?"

### 체크리스트

**이 질문에 YES면 → 넣어도 됨**

1. [ ] **말로 주변 장소 찾기**에 직접 필요함?
2. [ ] 없으면 **음성 검색이 작동 안 함**?
3. [ ] 사용자가 **"뭘 말해야 하지?"** 질문하게 만듦?

**3개 모두 YES → 넣기**
**하나라도 NO → 나중**

---

### 예시

#### ✅ 넣어야 하는 것

* 음성 인식 버튼
* 지도 핀 표시
* 결과 카드 1개
* 길찾기 연결

#### ❌ 나중인 것

* 즐겨찾기
* 검색 기록
* 장소 비교
* 리뷰 보기
* 상세 정보 (P1)

---

## Q2. "이 기능 빠진 거 아니야?"

### 체크리스트

**이 질문에 YES면 → 빠진 게 맞음**

1. [ ] **말로 주변 장소 찾기**가 작동 안 함?
2. [ ] 사용자가 **"어떻게 하면 되죠?"** 질문하게 만듦?
3. [ ] **기존 지도보다 불편함**?

**3개 모두 YES → 빠진 게 맞음**
**하나라도 NO → 나중**

---

### 예시

#### ✅ 빠진 게 맞는 것

* 음성 인식 실패 처리
* 결과 없을 때 대안 제시
* 지도 자동 줌

#### ❌ 빠진 게 아닌 것

* 로그인
* 프로필
* 설정 화면
* 알림
* 푸시

---

## Q3. "이거 AI로 하면 더 멋질 텐데?"

### 체크리스트

**이 질문에 YES면 → 나중**

1. [ ] **말로 주변 장소 찾기**에 필수 아님?
2. [ ] AI 없어도 **음성 검색 작동함**?
3. [ ] AI 넣으면 **복잡해짐**?

**3개 모두 YES → 나중 (Phase 2 이후)**

---

### 예시

#### ❌ 지금 하면 안 되는 것

* 자연어 완벽 이해 (키워드 매칭으로 충분)
* 자동 추천
* 상황 추론
* 이동 예측
* AI 멘트

#### ✅ 지금 해야 하는 것

* 키워드 → 카테고리 매핑 (하드코딩 OK)
* 거리순 정렬
* 조건 필터 (간단)

---

## Q4. "사용자가 이걸 원할 것 같은데?"

### 체크리스트

**이 질문에 YES면 → 테스트 후 결정**

1. [ ] **말로 주변 장소 찾기**에 직접 필요 없음?
2. [ ] 있으면 **좋지만 없어도 됨**?
3. [ ] 사용자 테스트에서 **요청이 나왔음**?

**3개 모두 YES → Phase 1 이후**
**하나라도 NO → 지금은 NO**

---

### 예시

#### ❌ 지금 하면 안 되는 것

* 즐겨찾기
* 검색 기록
* 장소 비교
* 리뷰
* 공유하기

#### ✅ 테스트 후 결정

* 상세 정보 (P1)
* 조건 검색 (P1)
* 음성 재검색 (P1)

---

## Q5. "이거 추가하면 좋을 것 같은데?"

### 체크리스트

**이 질문에 YES면 → 나중**

1. [ ] **말로 주변 장소 찾기**에 필수 아님?
2. [ ] 없어도 **음성 검색 작동함**?
3. [ ] 있으면 **복잡해질 수 있음**?

**3개 모두 YES → 나중**

---

# 🎯 빠른 판단 기준 (3초 체크)

## 넣어도 되는 것

✅ **말로 주변 장소 찾기**에 직접 필요
✅ 없으면 음성 검색 작동 안 함
✅ 사용자가 "뭘 말해야 하지?" 질문하게 만듦

## 나중인 것

❌ 있으면 좋지만 없어도 됨
❌ AI/추천/상태 관련
❌ 사용자 요청 (테스트 후)
❌ "멋져 보이는" 기능
