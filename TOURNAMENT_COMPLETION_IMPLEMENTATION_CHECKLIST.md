# Tournament Section 완결 구현 체크리스트

**목표: "대회 관련 전화는 구조적으로 사라집니다."**

---

## ✅ 필수 구현 항목

### 1️⃣ 대회 참가 신청

#### 위치
- [ ] Tournament 상세 화면 내부
- [ ] 버튼: **[대회 참가 신청]**

#### 흐름
- [ ] 회원 로그인
- [ ] 참가 신청 클릭
- [ ] 참가 상태 표시:
  - [ ] 신청 완료
  - [ ] 승인 대기 / 승인 완료

#### 핵심 원칙
- [ ] ❌ 문자 신청
- [ ] ❌ 카톡 명단
- [ ] ❌ 엑셀 접수
- [ ] ⭕ "신청은 이 화면 기준"

---

### 2️⃣ 참가 신청 화면 필수 정보 (전화 차단용)

참가 신청 버튼 위/아래에 반드시 포함:

- [ ] 대회 날짜
- [ ] 장소
- [ ] 참가 대상
- [ ] 마감 여부
- [ ] 현재 신청 현황 (숫자만)

**목표: "아직 신청 받아요?" 질문 차단**

---

### 3️⃣ 대진표 고정 (공식 기준의 핵심)

#### 설계 원칙
- [ ] 대진표는 첨부 파일 ❌
- [ ] 대진표는 이미지 전달 ❌
- [ ] **대진표는 시스템 화면 ⭕**

#### 상태
- [ ] 🟡 준비 중
- [ ] 🟢 **확정 (이게 중요)**
- [ ] ⚫ 종료

#### 공식 기준 문장
- [ ] **"대진표는 시스템에 표시된 것만 공식입니다."**

이 문장으로 차단:
- [ ] 수정 요구 ❌
- [ ] "단톡에 올라온 게…" ❌
- [ ] 캡처 논쟁 ❌

---

### 4️⃣ 참가 신청 ↔ 대진표 자동 연결

- [ ] 참가 신청 마감
- [ ] → 관리자 확정
- [ ] → 대진표 생성/업로드
- [ ] → 상태 '확정'
- [ ] → 공지 자동 노출 (선택)

**사무국 작업:**
- [ ] 버튼 한 번: **[대진표 확정]**

---

### 5️⃣ 권한 기준 고정

#### 비회원
- [ ] 대회 일정
- [ ] 대진표 (읽기 전용)

#### 회원
- [ ] 참가 신청
- [ ] 본인 참가 여부
- [ ] 대진표 열람

#### 협회 관계자
- [ ] 신청 승인
- [ ] 대진표 확정

---

## 📋 구현 순서
