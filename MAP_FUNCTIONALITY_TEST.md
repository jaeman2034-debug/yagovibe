# 🔥 지도 기능 검증 테스트 가이드

## 📋 테스트 목표

지도 페이지(`/app/map`)의 핵심 기능과 퍼널 연결을 검증합니다.

---

## ✅ 테스트 시나리오

### 시나리오 1: 기본 지도 로드

**단계:**
1. `/app/map` 접속
2. Google Maps 로드 확인
3. 초기 상태 확인

**확인 사항:**
- [ ] 지도가 정상적으로 로드됨
- [ ] 마커가 표시됨 (있는 경우)
- [ ] 검색창이 표시됨
- [ ] 콘솔 에러 없음

**예상 로그:**
```
[MapController] 지도 초기화 완료
[MapPageContainer] 마운트 완료
```

---

### 시나리오 2: 장소 검색 → 선택

**단계:**
1. 검색창에 "마들 스타디움" 입력
2. 검색 결과 확인
3. 장소 선택 (마커 클릭 또는 결과 클릭)

**확인 사항:**
- [ ] 검색 결과가 표시됨
- [ ] 마커가 지도에 표시됨
- [ ] 하단 카드(장소 정보)가 표시됨
- [ ] 콘솔에 검색 로그 출력

**예상 로그:**
```
[MapController] 검색 시작: 마들 스타디움
[MapController] 검색 결과: N개
[MapPageContainer] 장소 선택: { id, name, lat, lng }
```

---

### 시나리오 3: 추천 카드 노출

**단계:**
1. 장소 선택 후
2. 하단 카드 확인
3. 스토리 존 확인 (있는 경우)

**확인 사항:**
- [ ] 장소 상세 정보 카드 표시
- [ ] 길안내 버튼 표시
- [ ] 스토리 존 표시 (있는 경우)
- [ ] `story_impression` 로그 확인

**예상 로그:**
```
[STORY_LOG] { eventName: "story_impression", ... }
[PlaceDetailSheet] 장소 상세 정보 표시
```

---

### 시나리오 4: 스토리 클릭 → 네비게이션

**단계:**
1. 스토리 카드의 CTA 버튼 클릭
2. 페이지 이동 확인
3. 로그 확인

**확인 사항:**
- [ ] `story_click` 로그 전송
- [ ] 해당 페이지로 이동 (팀/마켓 등)
- [ ] 콘솔에 클릭 로그 출력

**예상 로그:**
```
[STORY_LOG] { eventName: "story_click", ... }
[StoryRouter] 네비게이션: /sports/football/team/...
```

---

### 시나리오 5: 길안내 시작

**단계:**
1. 장소 선택 후
2. "길안내" 버튼 클릭
3. 길안내 UI 확인

**확인 사항:**
- [ ] 길안내 모드로 전환
- [ ] 경로가 지도에 표시됨
- [ ] 경로 정보(거리, 시간) 표시
- [ ] 네비게이션 UI 표시

**예상 로그:**
```
[MapController] 길안내 시작
[RouteInfoFloatingBar] 경로 정보 표시
```

---

### 시나리오 6: 지도 → 마켓 연결

**단계:**
1. 장소 선택 후
2. 마켓 관련 버튼/링크 확인
3. 마켓 페이지 이동

**확인 사항:**
- [ ] 마켓 버튼/링크 존재 (있는 경우)
- [ ] 마켓 페이지로 이동
- [ ] `MARKET_VIEW` 로그 전송

**예상 로그:**
```
[ACTIVITY] { event: "MARKET_VIEW", location: "/app/market", ... }
```

## 🔍 이벤트 로그 검증

### 1. story_impression 로그

**확인 위치:** 브라우저 콘솔

**예상 출력:**
```javascript
[STORY_LOG] {
  storyId: "...",
  eventName: "story_impression",
  category: "...",
  source: "...",
  region: "seoul",
  at: "2024-01-27T...",
  ...
}
```

**확인 방법:**
1. StoryZone 컴포넌트가 로드되면 자동 출력
2. 스토리 인덱스 변경 시 새로운 로그 출력

---

### 2. story_click 로그

**확인 위치:** 브라우저 콘솔

**예상 출력:**
```javascript
[STORY_LOG] {
  storyId: "...",
  eventName: "story_click",
  category: "...",
  source: "...",
  region: "seoul",
  at: "2024-01-27T...",
  ...
}
```

**확인 방법:**
1. 스토리 카드의 CTA 버튼 클릭
2. 즉시 로그 출력

---

### 3. API 전송 확인

**확인 위치:** 브라우저 콘솔 + 네트워크 탭

**예상 출력:**
```
[StoryLog] 📤 전송 시도: 1개 이벤트 → /api/logs/story/bulk
[StoryLog] ✅ 전송 성공: 1개 저장됨
```

**네트워크 탭:**
- 요청: `POST /api/logs/story/bulk`
- 상태: `200 OK`
- 응답: `{ count: 1 }`

## 🐛 문제 해결

### 문제 1: 지도가 로드되지 않음

**원인:**
- Google Maps API 키 오류
- 네트워크 오류
- 초기화 실패

**해결:**
1. 콘솔 에러 확인
2. Google Maps API 키 확인
3. 네트워크 탭에서 API 요청 확인

---

### 문제 2: story_impression 로그가 안 보임

**원인:**
- StoryZone 컴포넌트가 렌더링되지 않음
- 스토리 데이터가 없음
- `logStory` 함수 호출 실패

**해결:**
1. StoryZone 컴포넌트 확인
2. 스토리 데이터 로드 확인
3. 콘솔 에러 확인

---

### 문제 3: story_click 로그가 안 보임

**원인:**
- CTA 버튼 클릭 이벤트 미연결
- `handleCta` 함수 호출 실패
- 스토리 데이터에 `cta` 필드 없음

**해결:**
1. CTA 버튼 렌더링 확인
2. `onClick` 핸들러 확인
3. 스토리 데이터 구조 확인

---

### 문제 4: API 전송 실패

**원인:**
- BFF 서버 미실행
- 프록시 설정 오류
- 네트워크 오류

**해결:**
1. BFF 서버 실행 확인 (`npm run dev:bff`)
2. `vite.config.ts` 프록시 설정 확인
3. 네트워크 탭에서 요청 상태 확인
