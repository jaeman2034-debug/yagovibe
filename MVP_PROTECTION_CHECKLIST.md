# 🧱 MVP 보호 체크리스트 (출시 전까지 이거만 본다)

## 🎯 이 체크리스트의 목적

> **"지금 이 기능이
> '말로 장소 찾기'를
> 더 빠르게 만드는가?"**

YES 아니면 **전부 보류**다.

---

## ✅ 추가해도 되는 질문 (통과)

아래 질문에 **YES**면 넣어도 된다.

* 이 기능이 **말하는 횟수를 줄여주나?**
* 이 기능이 **결과를 더 빨리 보여주나?**
* 이 기능이 **키보드를 완전히 대체하나?**
* 이 기능이 **한 화면 안에서 끝나나?**

👉 2개 이상 YES면 **검토 가능**

---

## ❌ 즉시 컷 질문 (이 말 나오면 중단)

이 말이 나오면 **무조건 STOP**이다.

* "AI로 하면 더 멋질 것 같은데요"
* "사용자 상태를 알면 좋지 않을까요?"
* "추천도 같이 보여주면…"
* "리스트도 같이 넣죠"
* "로그인 있으면 좋을 듯"
* "나중에 쓸 기능인데 지금 미리…"
* "이거 추가하면 사용자가 좋아할 것 같은데"
* "기능이 좀 부족한 것 같아요"
* "다른 앱에는 다 있는데…"

👉 전부 **2번 이후**다.

---

## 🧪 기능 추가 전 10초 판단법 (현장용)

기획/개발 중 누가 기능 제안하면
**이 질문 하나만 던져라.**

> ❓ "이거 없으면
> 사용자가
> 말로 장소 찾는 데 실패해?"

* YES → 고려
* NO → **바로 보류**

---

## 🎙️ 음성 관련 절대 규칙 (중요)

* ❌ 대화 기록 남기지 말 것
* ❌ AI 말 길어지게 하지 말 것
* ❌ 실패 원인 설명하지 말 것
* ❌ "다시 말해주세요" 같은 말
* ❌ 음성 파형 표시
* ❌ 음성 인식 중 애니메이션 과다

👉 음성은 **입력 수단**이지
👉 **캐릭터가 아니다**

---

## 🗺️ 지도 관련 절대 규칙

* ❌ 리스트 먼저 보여주기
* ❌ 카드 여러 개
* ❌ 필터 패널
* ❌ 탭 전환
* ❌ 지도 위 UI 오버레이
* ❌ 복잡한 마커 클러스터링

👉 **지도 → 핀 → 카드 1개**
이 순서 깨지면 실패

---

## 📦 MVP 스코프 한 줄 선언 (팀 공지용)

이 문장 그대로 슬랙/노션에 붙여라 👇

> **현재 MVP는
> '음성으로 주변 장소를 빠르게 찾는 지도'입니다.
>
> 이 목적에 직접 기여하지 않는 기능은
> 전부 출시 이후 논의합니다.**

---

## ⏱️ 출시 직전 최종 점검 (5분)

출시 전 마지막으로 이것만 체크:

* [ ] 키보드 없이 찾을 수 있나?
* [ ] 말하면 바로 핀이 찍히나?
* [ ] 카드가 1개만 나오나?
* [ ] "편하다"라는 말이 나왔나?

👉 전부 YES면 **출시**

---

## 🔄 개발 중 매일 체크

### 저녁 체크 (1분)

* [ ] 범위 밖으로 나간 건 없나?
* [ ] 내일도 MVP 범위 내인가?

## 🧠 최종 판단 질문 (3초)

> **"이 기능 없으면
> 사용자가 '어떻게 하면 되죠?' 질문하게 되나?"**

* YES → 넣기
* NO → 나중

---

## 📋 기능 제안 시 체크리스트

누가 기능 제안하면 이 3개만 확인:

1. [ ] **말로 주변 장소 찾기**에 직접 필요함?
2. [ ] 없으면 **음성 검색이 작동 안 함**?
3. [ ] 사용자가 **"뭘 말해야 하지?"** 질문하게 만듦?

**3개 모두 YES → 넣기**
**하나라도 NO → 나중**

---

## 🧠 마지막으로 진짜 중요한 말

> **스타트업이 망하는 이유는
> 기능이 부족해서가 아니라
> '참을 줄 몰라서'다.**

지금 너희는
**참아야 할 타이밍을 정확히 잡았다.**

---

## ✅ MVP 완성 기준 (최종)

### 기능
- [ ] 음성 버튼 클릭 → 인식 시작
- [ ] 음성 텍스트 → 카테고리 매핑
- [ ] 검색 결과 → 지도 핀 표시
- [ ] 가장 가까운 핀 강조
- [ ] 결과 카드 1개 표시
- [ ] "여기로 가기" 버튼 작동

### UX
- [ ] 키보드 없이 말만으로 검색 가능
- [ ] 결과가 바로 지도에 표시
- [ ] 3초 안에 결정 가능
- [ ] "편하다" 피드백 수집
