## 🚀 Phase 4-2 범위 (예상)

### 2-1. FacilitySection 자동화 (우선순위 1)

#### 목표
- 관리자가 슬롯 상태를 `available` ↔ `blocked`로 토글
- 상태 변경 시 자동 알림 발송 (Kakao Alimtalk)
- 전화/KakaoTalk 문의 **0건** 달성

#### 기능 범위
- ✅ 슬롯 상태 토글 (이미 설계 완료)
- ✅ 상태 변경 알림 자동 발송
- ❌ 예약 시스템 (Phase 4-3 이후)
- ❌ 결제 시스템 (Phase 4-4 이후)

#### 성공 기준
- 사무국장이 전화 없이 슬롯 관리 가능
- 상태 변경 시 클럽 대표자에게 자동 알림 전달
- "언제 바뀌었나요?" 전화 문의 0건

---

### 2-2. TournamentSection 연동 (우선순위 2)

#### 목표
- Tournament 데이터와 Hero Section 자동 연동
- "현재 진행 중" 대회가 Hero에 자동 표시
- 대회 상태 변경 시 Hero 자동 업데이트

#### 기능 범위
- ✅ Tournament 데이터 쿼리
- ✅ Hero Section 동적 업데이트
- ✅ Tournament → Facility 자동 연결 (event 슬롯 생성)
- ❌ 대회 신청/접수 (Phase 5 이후)

#### 성공 기준
- 관리자가 Tournament 데이터만 수정하면 Hero 자동 업데이트
- "현재 대회가 뭐예요?" 전화 문의 0건

---

### 2-3. 협회 산하 팀 표시 (우선순위 3)

#### 목표
- 협회 공식 페이지에 산하 팀 목록 표시 (읽기 전용)
- 축구 허브 팀 카드에 배지 표시 (회원/연계)
- "우리 팀은 산하인가요?" 전화 문의 감소

#### 기능 범위
- ✅ 산하 팀 목록 쿼리 및 표시
- ✅ 팀 카드 배지 표시 (회원/연계)
- ❌ 승인 시스템 (Phase 5 이후)
- ❌ 가입 신청 (Phase 5 이후)

#### 성공 기준
- 협회 페이지에서 산하 팀 목록 확인 가능
- 허브에서 팀 관계 상태 즉시 파악 가능

## 🎯 Phase 4-2 성공 기준 (최종 판정)

### 조건 1: 운영 효율성
- 전화/KakaoTalk 문의 **주 1회 이하**로 감소
- 사무국장이 시스템만으로 대관/대회 정보 관리 가능

- 비회원 팀이 "우선순위"를 자연스럽게 인지

### 조건 3: 시스템 안정성
- 상태 변경 알림 정상 발송 (성공률 95% 이상)
- Hero Section 자동 업데이트 정상 작동
- 데이터 일관성 유지 (에러 0건)
