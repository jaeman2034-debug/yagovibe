# Tournament 회원 자격 통제 구현 체크리스트

**목표: 대회 참가 자격 = "회비 납부 회원", 예외·할인·면제 = 없음**

---

## ✅ 필수 구현 항목

### 1️⃣ 참가 신청 버튼 조건

#### 조건
- [ ] ✔ 회원
- [ ] ✔ 회비 납부 완료

#### 하나라도 ❌면
- [ ] 참가 신청 버튼 비활성
- [ ] 안내 문구만 표시

#### 표시 문구 (고정)
- [ ] **"회비 납부 회원만 대회 참가가 가능합니다."**

**목표: 전화 ❌ / 문의 ❌ / 설명 ❌**

---

### 2️⃣ 할인/면제 없음 (시스템 안정성)

#### 효과
- [ ] 규칙이 단순해짐
- [ ] 예외 처리 0
- [ ] 사무국 판단 0
- [ ] "왜 쟤는 되고 나는 안 돼요?" 0

**👉 공정성 = 자동화의 핵심**

---

### 3️⃣ 참가 자격 자동 필터링 구조

#### 시스템 기준 (사람 손 ❌)

| 조건 | 결과 | 구현 |
|------|------|------|
| 회원 + 회비 납부 | 참가 신청 가능 | [ ] |
| 회원 + 회비 미납 | 참가 신청 불가 | [ ] |
| 비회원 | 참가 신청 불가 | [ ] |

#### 대진표 생성 대상
- [ ] 참가 확정자 ∧ 회비 납부자

---

### 4️⃣ 화면에 반드시 들어가야 할 기준 문구

#### 위치
- [ ] Tournament 상세 하단 or 참가 버튼 근처

#### 문구 (고정)
```
대회 참가 자격

본 대회는 회비 납부를 완료한 회원에 한해 참가할 수 있습니다.
할인 및 참가비 면제는 제공되지 않습니다.
```

#### 사용처
- [ ] 공지
- [ ] 규정
- [ ] 회의록

**👉 어디에 복붙해도 동일해야 합니다.**

---

### 5️⃣ 사무국 효과

이제 사무국은:

- [ ] ❌ "이번만 봐주세요" 대응
- [ ] ❌ 할인 요청 판단
- [ ] ❌ 미납자 참가 허용 고민

대신 이 말만 반복:

- [ ] **"시스템 기준입니다."**

---

## 📋 구현 체크 (아주 짧게)

1. [ ] 참가 신청 버튼 조건에 회비 상태 연동
2. [ ] 미납 시 버튼 비활성 + 문구 표시
3. [ ] 참가자 리스트 = 회비 납부 필터
4. [ ] 대진표 = 참가 확정자 기준

**이 4개가 들어가면 Tournament 파트는 완전히 닫힙니다.**
