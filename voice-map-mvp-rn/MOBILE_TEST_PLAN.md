## 1️⃣ 오늘 당장 해야 할 것 (개발자 액션)

### ✅ 모바일 앱 상태 체크

아래 4가지만 되면 **테스트 가능**이다.

- [ ] 앱 설치됨 (Expo Go or Dev Build)
- [ ] 위치 권한 정상
- [ ] 마이크 권한 정상
- [ ] 버튼 누르고 말하면 **Whisper STT 텍스트 로그 찍힘**

### 📌 체크 방법

#### 1. 앱 설치 확인

```bash
# Expo Go 사용
npm start
# 앱에서 Expo Go로 스캔

# 또는 Dev Build
eas build --profile development --platform ios
eas build --profile development --platform android
```

#### 2. 권한 확인

앱 실행 시:
- 위치 권한 팝업 표시 ⭕
- 마이크 권한 팝업 표시 ⭕

#### 3. Whisper STT 확인

버튼 누르고 말한 후:
- 콘솔에 "🎤 Whisper 인식: [텍스트]" 로그 확인
- 서버 로그에 STT 요청 확인

### 📌 중요

- 지도 핀 정확도 ❌
- 디자인 완성도 ❌
- **흐름만 되면 OK**

## 2️⃣ 내부 실전 테스트 (5분짜리)

### 테스트 방법

- 개발자 본인 or 팀원 1명
- 실제 폰 들고 밖에서

### 말해볼 문장 (이 3개만)

1. "근처 축구장"
2. "지금 여는 헬스장"
3. "근처 카페"

### 체크 포인트

- [ ] 말 끝나고 **3초 안에 반응 오는가**
- [ ] 결과가 **지도에 바로 찍히는가**
- [ ] "어? 된다" 느낌 드는가

### ✅ 합격 기준

> 이 단계 통과하면 **외부 테스트로 가도 됨**

---

## 3️⃣ 소규모 외부 테스트 (진짜 중요)

### 대상

- 지인 3명
- **개발/기획 아닌 사람**

### 테스터에게 딱 이 말만

```
지도 앱인데,
설명 없이 한 번 써보고
어땠는지만 말해줘
```

### 📌 주의사항

- ❌ 사용법 설명
- ❌ 기능 설명
- ⭕ **설명 없이 사용해보기**

---

### 테스터 미션 (딱 2개)

1. "근처 축구장 찾아보기"
2. "지금 여는 헬스장 찾아보기"

---

### 반드시 들어야 할 말

- "편하네"
- "말로 하니까 빠르다"
- "키보드 안 쳐서 좋다"

### ✅ 합격 기준

> **이 말 하나라도 나오면 방향 맞음**

---

## 4️⃣ 테스트 중 나올 수 있는 문제 & 대응

### ❌ "음성 잘 안 알아듣네"

**대응:**
- 정상
- **발음 / 억양 문제 아님**
- Whisper는 충분히 정확함
- → 키워드 매핑만 보완하면 됨

**체크:**
- 서버 로그에서 STT 텍스트 확인
- 키워드 매핑 확인 (`CATEGORY_MAP`)

---

### ❌ "결과가 이상해"

**대응:**
- 데이터 문제
- MVP 단계에서는 **UX 성공**

**체크:**
- Mock 데이터 확인
- 실제 장소 API 교체 필요 (나중에)

---

### ❌ "이거 뭐 하는 앱인지 처음엔 헷갈림"

**대응:**
- 상단 문구만 수정
- 구조 ❌ / 기능 ❌

**체크:**
- 첫 화면 문구 확인
- 필요 시 간단한 안내 추가
