# 🧪 TestFlight 테스터 운영법

> 망하지 않는 방식

---

## 🎯 목표

> **"이 앱이 편한지"만 확인한다.
> 기능 피드백은 듣지 않는다.**

---

## 👥 테스터 구성 (최소)

- [ ] 5명 이내
- [ ] 운동 / 외출 자주 하는 사람
- [ ] **개발자 ❌ / 기획자 ❌**

### 📌 테스터 선정 기준

- ⭕ 실제 사용자
- ⭕ 운동/외출 자주 하는 사람
- ❌ 개발자
- ❌ 기획자
- ❌ 친구/가족 (너무 친하면 객관적 피드백 어려움)

---

## 🧭 테스터에게 보낼 메시지 (그대로 복붙)

```
지도 앱 테스트입니다.
설명 없이 한 번 써보시고
어땠는지만 알려주세요.
```

### 📌 주의사항

- ❌ 기능 설명
- ❌ 사용법 설명
- ⭕ **설명 없이 사용해보기**

---

## 🧪 테스터 미션 (딱 2개)

### 미션 1

```
근처 축구장 찾아보기
```

### 미션 2

```
지금 여는 헬스장 찾아보기
```

### 📌 미션 원칙

- ⭕ 실제 사용 시나리오
- ⭕ 간단하고 명확
- ❌ 복잡한 시나리오

## ❓ 테스터에게 묻는 질문 (3개만)

### 질문 1

```
처음에 뭘 해야 할지 바로 알겠어요?
```

### 질문 2

```
말로 찾는 게 편했어요?
```

### 질문 3

```
이 앱 언제 쓸 것 같아요?
```

### 📌 질문 원칙

- ⭕ 사용자 경험 중심
- ⭕ 간단하고 명확
- ❌ 기능 추가 요청
- ❌ 기술적 질문

---

## ✅ 합격 기준

> 5명 중 3명 이상이
> **"편하다" / "간단하다"** 말하면 통과

### 판정 기준

- ⭕ "편하다" / "간단하다" → 통과
- ⭕ "쓰기 쉬워요" → 통과
- ❌ "기능 추가" → 무시
- ❌ "이건 없어요" → 무시

## 🚀 TestFlight 초대 방법

### 1. App Store Connect

1. TestFlight → 테스터 → 외부 테스터 추가
2. 이메일 주소 입력
3. 초대 메시지 작성 (위 템플릿 사용)

### 2. 초대 메시지 예시

```
YAGO 지도 앱 테스트에 초대합니다.

설명 없이 한 번 써보시고
어땠는지만 알려주세요.

테스트 미션:
1. 근처 축구장 찾아보기
2. 지금 여는 헬스장 찾아보기

감사합니다!
```
