## 📅 Week 1 — 안정화 & 신뢰 강화

### 1️⃣ 신고/차단 히스토리 최소화 (운영 + UX)

**우선순위**: 🔴 High  
**예상 소요 시간**: 3일  
**담당자**: TBD

#### Why (왜 필요한가)
사용자는 "내 행동이 기록된다"는 신뢰가 생깁니다. 신고/차단 후 "내가 뭘 했는지" 확인할 수 있는 공간이 있으면:
- 불안감 감소
- 신뢰도 증가
- 재방문 동기 제공

#### Scope (범위)

##### 기능 요구사항
1. **내 설정 → 신고/차단 내역 보기 (읽기 전용)**
   - 경로: `/app/settings/reports` 또는 `/app/settings/blocked`
   - 신고 내역 리스트
   - 차단 내역 리스트
   - 각 항목에 상태 표시

2. **상태 표시**
   - 신고: "접수됨" / "검토 중" / "처리 완료"
   - 차단: "차단됨" (날짜 표시)
   - 상태별 색상 구분 (옵션)

3. **빈 상태 UX**
   - 신고 내역 없음: "아직 신고한 내역이 없습니다."
   - 차단 내역 없음: "차단한 사용자가 없습니다."

##### 기술 요구사항
- Firestore 쿼리: `chatReports` 컬렉션에서 `reporterId`로 필터링
- Firestore 쿼리: `users/{uid}/blockedUsers` 서브컬렉션 조회
- 페이지네이션 (필요시)
- 로딩 상태 처리

### 2️⃣ 네트워크 탄력성 보강 (안정성)

**우선순위**: 🔴 High  
**예상 소요 시간**: 2일  
**담당자**: TBD

#### Why (왜 필요한가)
실사용에서 가장 먼저 터지는 부분입니다. 네트워크 불안정 시:
- 사용자 이탈 증가
- 신뢰도 하락
- 재방문 감소

#### Scope (범위)

##### 기능 요구사항
1. **API 재시도 (1회)**
   - 실패 시 자동으로 1회 재시도
   - 재시도 간격: 500ms ~ 1s
   - 재시도 실패 시 사용자에게 명확한 에러 메시지

2. **타임아웃 처리**
   - API 요청 타임아웃: 10초
   - 타임아웃 시 사용자에게 안내
   - 재시도 가능한 UI 제공

3. **오프라인 토스트**
   - 네트워크 연결 끊김 감지
   - "인터넷 연결이 끊어졌습니다" 토스트 표시
   - 연결 복구 시 "연결이 복구되었습니다" 토스트

##### 기술 요구사항
- 네트워크 상태 감지: `navigator.onLine` 또는 `window.addEventListener('online/offline')`
- 재시도 로직: `retry` 유틸 함수
- 타임아웃: `AbortController` 또는 `Promise.race`

#### Done 기준 (완료 기준)

  - [ ] 재시도 버튼 또는 자동 재연결 안내

- [ ] 재시도 중복 요청 없음
  - [ ] 재시도 중 플래그로 중복 요청 방지
  - [ ] 재시도 실패 시 사용자에게 명확한 에러 메시지
  - [ ] 로딩 상태 정확히 표시

- [ ] 타임아웃 처리
  - [ ] 10초 타임아웃 적용

  - [ ] 재시도 가능한 UI

#### 기술 스택
- 유틸 함수: `retry`, `withTimeout`
- 네트워크 감지: `useNetworkStatus` 커스텀 훅

## 📅 Week 2 — 리텐션 트리거

### 3️⃣ 읽음/미읽음 상태 개선 (가벼운 리텐션)

**우선순위**: 🟡 Medium  
**예상 소요 시간**: 3일  
**담당자**: TBD

#### Why (왜 필요한가)
대화 맥락이 살아남습니다. 사용자가:
- 어디까지 읽었는지 기억
- 새 메시지 위치를 즉시 파악
- 대화 흐름을 자연스럽게 이어감

#### Scope (범위)

##### 기능 요구사항
1. **마지막 읽은 메시지 표시**
   - 채팅방 내에서 마지막 읽은 메시지 위치 표시
   - 시각적 표시: 구분선 또는 배지
   - 클릭 시 해당 위치로 스크롤

2. **새 메시지 배지**
   - 채팅 목록에서 새 메시지 개수 표시
   - 채팅방 내에서 새 메시지가 있을 때 배지 표시
   - 읽으면 배지 제거

##### 기술 요구사항
- Firestore: `lastReadMessageId` 필드 저장/조회
- 실시간 업데이트: `onSnapshot`으로 읽음 상태 동기화
- 스크롤: `scrollIntoView` 또는 `react-virtuoso`의 `scrollToIndex`

### 4️⃣ 입력 중 표시 디바운스 (감정 피로 감소)

**우선순위**: 🟡 Medium  
**예상 소요 시간**: 2일  
**담당자**: TBD

#### Why (왜 필요한가)
- 과도하면 스트레스: "입력 중..."이 너무 자주 깜빡이면 불안감 증가
- 없으면 밋밋: 상대방이 입력 중인지 전혀 모르면 대화 맥락 단절

#### Scope (범위)

##### 기능 요구사항
1. **1.5초 디바운스**
   - 사용자가 입력을 시작한 후 1.5초 후에 "입력 중..." 표시
   - 1.5초 내에 입력이 멈추면 표시하지 않음
   - 디바운스로 불필요한 이벤트 최소화

2. **5초 타임아웃**
   - "입력 중..." 표시 후 5초가 지나면 자동으로 숨김
   - 장시간 입력 중일 때도 사용자에게 부담을 주지 않음

##### 기술 요구사항
- 디바운스: `lodash.debounce` 또는 커스텀 `useDebounce` 훅
- Firestore: `typing` 필드 또는 `users/{uid}/typing/{chatId}` 문서
- 실시간 업데이트: `onSnapshot`으로 상대방 입력 상태 감지

#### Done 기준 (완료 기준)

- [ ] 깜빡임 없음
  - [ ] 1.5초 디바운스 적용
  - [ ] 불필요한 "입력 중..." 표시 없음
  - [ ] 자연스러운 표시/숨김 전환

- [ ] 네트워크 이벤트 최소화
  - [ ] 디바운스로 Firestore 쓰기 최소화
  - [ ] 타임아웃으로 자동 정리
  - [ ] 메모리 누수 없음

#### 기술 스택
- 유틸: `debounce` 함수
- Firestore: `setDoc`으로 `typing` 상태 업데이트
- React: `useEffect`, `useRef`로 타이머 관리

#### 예상 파일 구조
```
src/pages/chat/
  ├── hooks/
  │   └── useTypingIndicator.ts
  └── components/
      └── TypingIndicator.tsx
```

#### 리스크 & 대응
- **리스크**: 디바운스로 인한 반응 지연
- **대응**: 1.5초는 사용자가 느끼지 않는 수준, 필요시 조정 가능
