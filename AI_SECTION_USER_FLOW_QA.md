# 🧭 AI 기능 실사용 QA 시나리오 (사용자 플로우 기준)

**목적**: AI 기능의 실제 사용자 행동 기준 검증  
**핵심 원칙**: "필요하면 AI 써보고, 마음에 들면 적용하고, 아니면 그냥 등록한다"  
**대상**: QA / 사용자 테스트 / 배포 전 최종 검증

---

## 0️⃣ 최종 사용자 관점 정의 (한 줄)

**"필요하면 AI 써보고, 마음에 들면 적용하고, 아니면 그냥 등록한다."**

---

## 1️⃣ 시나리오 1: AI 없이 상품 등록 (Happy Path)

### 목표
**사용자는 AI를 몰라도 상품 등록이 가능해야 함**

### 실행 단계
1. 상품 등록 페이지 진입
2. 이미지 선택
3. 상품명 입력
4. 가격 입력
5. (선택) 설명 입력
6. 상품 등록 버튼 클릭
7. 성공 → 마켓 홈 이동

### 검증 체크리스트
- [ ] 페이지 진입 시 AI 섹션이 접혀 있음 (방해 없음)
- [ ] AI 섹션을 열지 않아도 등록 가능
- [ ] 기본 상품 등록 흐름이 자연스러움
- [ ] 모든 필수 필드 입력 가능
- [ ] 저장 버튼 정상 작동
- [ ] 등록 성공 → 마켓 홈 이동

### 통과 기준
**6개 모두 YES → 시나리오 1 통과**

### 실패 시 영향도
**🔴 높음** - 이게 안 되면 상품 등록 자체가 불가능

---

## 2️⃣ 시나리오 2: AI 펼치기만 하고 안 쓰기

### 목표
**AI를 펼쳐봤지만 사용하지 않고 등록 가능해야 함**

### 실행 단계
1. 상품 등록 페이지 진입
2. 이미지 선택
3. 상품명 입력
4. 가격 입력
5. AI 섹션 클릭 (펼치기)
6. AI 분석 시작 버튼 **안 누름**
7. AI 섹션 닫기 (또는 그냥 두기)
8. 상품 등록 버튼 클릭
9. 성공 → 마켓 홈 이동

### 검증 체크리스트
- [ ] AI 섹션 펼치기 클릭 → 정상적으로 열림
- [ ] "▸ AI로 상품 설명 추천받기 (선택)" → AI 분석 화면 표시
- [ ] AI 분석 시작 버튼을 안 눌러도 문제 없음
- [ ] AI 섹션이 열려 있어도 등록 가능
- [ ] 입력한 내용(상품명, 가격) 유지됨
- [ ] 상품 등록 정상 작동
- [ ] AI 사용 안 해도 불이익 없음

### 통과 기준
**7개 모두 YES → 시나리오 2 통과**

### 실패 시 영향도
**🟡 중간** - AI 사용 의도를 가졌지만 실행하지 않은 경우

---

## 3️⃣ 시나리오 3: AI 분석 실행 후 적용하기

### 목표
**AI 분석 결과를 보고 만족해서 적용**

### 실행 단계
1. 상품 등록 페이지 진입
2. 이미지 선택
3. 상품명 입력 (또는 비워두기)
4. 가격 입력
5. AI 섹션 클릭 (펼치기)
6. "AI 분석 시작" 버튼 클릭
7. 분석 중... (로딩)
8. 결과 미리보기 표시
9. "설명에 적용하기" 버튼 클릭
10. 설명 필드에 AI 결과 적용됨
11. 필요 시 설명 수정
12. 상품 등록 버튼 클릭
13. 성공 → 마켓 홈 이동

### 검증 체크리스트
- [ ] AI 분석 시작 버튼 클릭 → 로딩 상태 표시
- [ ] 분석 중 스피너 + "분석 중..." 표시
- [ ] 분석 완료 후 미리보기 표시
- [ ] **자동으로 필드에 채워지지 않음** (중요!)
- [ ] "설명에 적용하기" 버튼 표시
- [ ] 적용하기 클릭 → 설명 필드에 반영
- [ ] 적용 후에도 수정 가능
- [ ] 적용 후 등록 정상 작동

### 통과 기준
**8개 모두 YES → 시나리오 3 통과**

### 실패 시 영향도
**🟡 중간** - AI의 핵심 기능이지만, 실패해도 등록은 가능해야 함

---

## 4️⃣ 시나리오 4: AI 분석 후 적용 안 하기

### 목표
**AI 분석 결과를 봤지만 적용하지 않고 등록 가능**

### 실행 단계
1. 상품 등록 페이지 진입
2. 이미지 선택
3. 상품명 입력
4. 가격 입력
5. 설명 입력 (기존 내용)
6. AI 섹션 클릭 (펼치기)
7. "AI 분석 시작" 버튼 클릭
8. 분석 완료 → 결과 미리보기
9. **"설명에 적용하기" 버튼 클릭 안 함**
10. AI 섹션 닫기
11. 기존 설명 내용 유지
12. 상품 등록 버튼 클릭
13. 성공 → 마켓 홈 이동 (기존 설명으로 등록)

### 검증 체크리스트
- [ ] AI 분석 결과 미리보기 표시
- [ ] 적용하기 버튼을 안 누르면 필드 변경 없음
- [ ] 기존 입력 내용 유지됨
- [ ] AI 섹션 닫기 가능
- [ ] 닫아도 입력 내용 영향 없음
- [ ] 상품 등록 정상 작동
- [ ] AI 사용 흔적 없음 (불이익 없음)

### 통과 기준
**7개 모두 YES → 시나리오 4 통과**

### 실패 시 영향도
**🟡 중간** - AI 결과를 거부하는 자연스러운 행동

---

## 5️⃣ 시나리오 5: AI 분석 실패 후 등록하기 (가장 중요)

### 목표
**AI가 실패해도 상품 등록은 정상 작동해야 함**

### 실행 단계
1. 상품 등록 페이지 진입
2. 이미지 선택
3. 상품명 입력
4. 가격 입력
5. 설명 입력
6. AI 섹션 클릭 (펼치기)
7. "AI 분석 시작" 버튼 클릭
8. 분석 실패 (네트워크 에러 / 서버 에러 등)
9. Toast 메시지 표시: "AI 분석에 실패했어요. 잠시 후 다시 시도해 주세요."
10. **AI 섹션 닫기 또는 무시**
11. 입력한 내용 유지됨
12. 상품 등록 버튼 클릭
13. 성공 → 마켓 홈 이동

### 검증 체크리스트
- [ ] AI 분석 실패 시 toast 메시지 표시
- [ ] **alert 사용하지 않음** (중요!)
- [ ] **전체 화면 에러 UI 없음** (중요!)
- [ ] 입력한 내용 유지됨
- [ ] 페이지 상태 유지됨
- [ ] AI 섹션 닫기 가능 (또는 무시 가능)
- [ ] 상품 등록 버튼 정상 작동
- [ ] 등록 성공 (AI 실패 영향 없음)
- [ ] AI 실패 후에도 재시도 가능 (있는 경우)

### 통과 기준
**9개 모두 YES → 시나리오 5 통과**

### 실패 시 영향도
**🔴 매우 높음** - AI 실패가 등록 실패로 이어지면 안 됨

## 7️⃣ 엣지 케이스 검증

### 7-1. AI 분석 중 상품 등록 버튼 클릭
- [ ] 분석 중에도 등록 버튼 클릭 가능 (또는 적절히 차단)
- [ ] 분석 중 등록 시 입력 내용 유지
- [ ] 등록 성공 (AI 분석 완료 여부 무관)

### 7-2. AI 적용 후 다시 분석
- [ ] AI 적용 후 다시 분석 시작 가능
- [ ] 새 결과로 덮어쓰기 가능
- [ ] 기존 입력 내용과 충돌 없음

### 7-3. AI 섹션 열고 닫기 반복
- [ ] 여러 번 열고 닫아도 문제 없음
- [ ] 상태가 올바르게 유지됨
- [ ] 입력 내용 영향 없음

### 7-4. 빠른 연속 클릭
- [ ] "AI 분석 시작" 버튼 빠른 연속 클릭 → 중복 실행 방지
- [ ] "설명에 적용하기" 버튼 빠른 연속 클릭 → 중복 적용 방지

## 1️⃣1️⃣ 천재 모드 핵심 원칙 (재확인)

### AI 기능은 "원할 때 펼쳐서, 버튼 눌러 쓰고, 마음에 들면 적용"

1. **필요하면 AI 써보고**
   - 접힌 상태로 시작 (방해 없음)
   - 원할 때만 펼치기

2. **마음에 들면 적용하고**
   - 미리보기 제공
   - 사용자 승인 후 적용

3. **아니면 그냥 등록한다**
   - AI 없이도 등록 가능
   - AI 실패해도 등록 가능
   - AI 결과 거부해도 등록 가능
